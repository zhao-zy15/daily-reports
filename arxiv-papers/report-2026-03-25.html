Pure Text LLMs

Sparser, Faster, Lighter Transformer Language Models
🧠 LLM Efficiency Edoardo Cetin, Stefano Peluchetti, Emilio Castillo, Akira Naruse, Mana Murakami, Llion Jones · Sakana AI arXiv:2603.23198
背景与动机

自回归大语言模型（LLMs）的规模扩展带来了前所未有的能力提升，但计算成本也急剧攀升。前馈网络（FFN）层占据了 Transformer 模型参数量和推理 FLOPs 的大部分（通常 > 60%），却存在大量冗余激活。现有的结构化稀疏方法（如 Mixture-of-Experts）虽然有效，但引入了额外的路由复杂度和负载均衡问题。非结构化稀疏（unstructured sparsity）理论上更灵活，但一直受限于 GPU 硬件对稀疏计算的低效支持。本文的核心问题是：能否在保持模型质量的前提下，实现 >99% 的 FFN 稀疏度，并让 GPU 真正从中获益？

核心方法详解
① L1 正则化诱导极端稀疏

作者在 FFN 层的激活上施加 L1 正则化，训练目标变为：

$$\mathcal{L} = \mathcal{L}_{\text{LM}} + \lambda \sum_{l=1}^{L} \| \mathbf{h}_l \|_1$$

其中 \(\mathbf{h}_l\) 是第 \(l\) 层 FFN 的隐藏激活，\(\lambda\) 是稀疏正则化系数。关键发现是：即使 \(\lambda\) 较小，FFN 激活也会自然收敛到极端稀疏（>99%），因为 FFN 的 ReLU/SwiGLU 激活函数天然倾向于产生零值。训练后对非零激活进行阈值裁剪（threshold pruning），将低于 \(\epsilon\) 的激活置零。

② 稀疏打包格式（Sparse Packing Format）

传统 CSR/CSC 格式在 GPU 上效率低下，因为其不规则的内存访问模式导致 warp divergence。作者设计了一种新的 Block-Aligned Sparse Packing (BASP) 格式：

  1. 将稀疏激活向量按 GPU warp size（32 threads）对齐分块
  2. 每个 block 内部紧凑存储非零元素的 值数组 和 列索引数组
  3. 引入 block-level bitmap 快速跳过全零 block，避免无效计算

存储开销为 \(O(nnz + N/B)\)，其中 \(nnz\) 为非零元素数，\(N\) 为总维度，\(B\) 为 block size。

③ 高效 CUDA 内核实现

为 BASP 格式设计了一组定制 CUDA 内核，针对三个关键操作进行优化：

  • SpMM（稀疏矩阵乘）：利用 block bitmap 跳过零块，每个 warp 处理一个稀疏 block，通过 shared memory 缓存权重矩阵片段
  • SpMV（稀疏矩阵-向量乘）：针对推理时 batch=1 的场景优化，利用 register-level parallelism
  • Sparse Backprop：训练时的梯度计算保持稀疏感知，仅对非零激活对应的权重行计算梯度

内核通过 Nsight Compute 验证，在 99% 稀疏度下实现了接近理论 \(100\times\) 加速的实际 \(40\text{-}60\times\) 吞吐量提升。

④ 稀疏度-质量权衡的量化分析

作者系统地评估了不同稀疏度水平下的 perplexity 变化。定义有效参数量为：

$$P_{\text{eff}} = P_{\text{attn}} + (1 - s) \cdot P_{\text{FFN}}$$

其中 \(s\) 是 FFN 稀疏度。实验发现存在一个 "甜点"（sweet spot）：稀疏度在 95%-99.5% 范围内，perplexity 仅增加 0.1-0.3，而 FLOPs 减少 50-80%。超过 99.8% 稀疏度后质量开始显著下降。

消融实验表明：(1) L1 正则化是实现高稀疏度的关键，移除后稀疏度从 99% 降至 ~70%；(2) BASP 格式相比标准 CSR 带来 3.2× 的额外加速；(3) block size=32 是最优选择，与 GPU warp size 对齐时性能最佳。局限性方面，当前方法仅针对 FFN 层，注意力层的稀疏化尚未探索；此外，99% 稀疏度下的训练需要额外 ~15% 的预训练步数来补偿质量损失。代码和内核已在 GitHub 开源。

EchoKV: Efficient KV Cache Compression via Similarity-Based Reconstruction
🧠 LLM KV Cache Yixuan Wang, Shiyu Ji, Yijun Liu, Qingfu Zhu, Wanxiang Che · HIT arXiv:2603.22910
背景与动机

大语言模型在长上下文场景下，KV Cache 的内存占用随序列长度线性增长，成为部署瓶颈。例如，一个 7B 模型处理 128K 上下文时，KV Cache 占用 ~32GB 显存。现有低秩压缩方法（如将 KV 矩阵投影到低维空间）存在两个核心缺陷：(1) 参数变换不可逆，一旦压缩就无法恢复到全精度推理；(2) 压缩率固定，无法根据运行时内存动态调整。这限制了实际部署中的灵活性。

核心方法详解
① 跨头相似性分析与分组

作者发现注意力头的 KV 表示之间存在显著的层间（inter-layer）和层内（intra-layer）相似性。定义头 \(i\) 和头 \(j\) 之间的 KV 相似度为：

$$\text{sim}(i, j) = \frac{1}{T}\sum_{t=1}^{T} \cos(\mathbf{k}_i^t, \mathbf{k}_j^t) + \cos(\mathbf{v}_i^t, \mathbf{v}_j^t)$$

基于此相似度矩阵，使用层次聚类将所有头分成 \(G\) 个组。每个组内只需要存储一个"锚定头"（anchor head）的完整 KV，其他头的 KV 通过锚定头重构。

② 轻量级残差重构网络

给定锚定头 \(a\) 的 KV 缓存 \((\mathbf{K}_a, \mathbf{V}_a)\)，组内其他头 \(i\) 的 KV 通过轻量级网络重构：

$$\hat{\mathbf{K}}_i = \mathbf{K}_a + f_\theta^K(\mathbf{K}_a, i), \quad \hat{\mathbf{V}}_i = \mathbf{V}_a + f_\theta^V(\mathbf{V}_a, i)$$

其中 \(f_\theta\) 是一个两层 MLP（隐藏维度 = 原维度的 1/4），接收锚定头 KV 和目标头索引的 embedding 作为输入。这确保了残差预测的参数开销极小（每组 < 1M 参数）。

③ 两阶段微调策略

训练分两个阶段进行，实现快速低成本适配：

  1. Stage 1 — 重构预训练：冻结原模型参数，仅训练重构网络 \(f_\theta\)，使用 MSE 损失对齐重构 KV 与原始 KV：\(\mathcal{L}_1 = \|\hat{\mathbf{K}}_i - \mathbf{K}_i\|^2 + \|\hat{\mathbf{V}}_i - \mathbf{V}_i\|^2\)
  2. Stage 2 — 端到端微调：解冻原模型参数，使用语言建模目标联合优化重构网络和原模型，使模型适应压缩后的 KV 表示

总训练时间：7B 模型约 1 个 A100 GPU 小时。

④ 按需压缩切换机制

EchoKV 的独特优势在于可以在运行时动态切换压缩模式。定义压缩率 \(r\) 为保留的锚定头占比：

$$r = \frac{|G|}{|H|}, \quad \text{Memory} = r \cdot M_{\text{full}} + M_\theta$$

其中 \(|H|\) 为总头数，\(|G|\) 为组数（= 锚定头数），\(M_\theta\) 为重构网络参数的内存开销。当内存充足时，系统自动切换到全精度模式（\(r=1\)），无需重新加载模型。

Dynamical Systems Theory Behind a Hierarchical Reasoning Model
🧠 LLM Reasoning Vasiliy A. Es'kin, Mikhail E. Smorkalov arXiv:2603.22871
背景与动机

当前 LLM 依赖线性序列生成和海量参数，但在需要严格逻辑推理的算法任务上（如数独、迷宫求解）表现欠佳。近期的层次推理模型（HRM）和微型递归模型（TRM）证明紧凑的递归网络可以处理此类任务，但其训练动力学缺乏严格的数学保证，容易出现训练不稳定和表征坍缩（representational collapse）。核心挑战是：能否用动力系统理论为递归推理架构提供数学上有保证的训练稳定性？

核心方法详解
① 收缩映射模型（CMM）架构

将离散递归推理重新表述为连续动力系统。给定输入问题编码 \(\mathbf{x}\)，CMM 在潜空间中定义一个连续时间动态：

$$\frac{d\mathbf{z}}{dt} = f_\theta(\mathbf{z}(t), \mathbf{x}), \quad \mathbf{z}(0) = g_\phi(\mathbf{x})$$

其中 \(\mathbf{z}(t)\) 是潜在状态轨迹，\(f_\theta\) 是参数化的向量场（Neural ODE），\(g_\phi\) 是初始化编码器。推理输出通过 \(\hat{\mathbf{y}} = h_\psi(\mathbf{z}(T))\) 解码。

② 收缩约束保证收敛

CMM 的核心创新是在向量场 \(f_\theta\) 上施加 收缩映射约束，确保潜在轨迹收敛到唯一不动点。具体要求 Jacobian 矩阵的谱范数满足：

$$\left\| \frac{\partial f_\theta}{\partial \mathbf{z}} \right\|_2 \leq -\alpha, \quad \alpha > 0$$

这通过 spectral normalization + 负偏移（negative bias）实现：对 \(f_\theta\) 的每一层权重矩阵 \(\mathbf{W}\) 进行谱归一化后乘以缩放因子 \(\gamma < 1\)。由 Banach 不动点定理，这保证了：

$$\|\mathbf{z}(t) - \mathbf{z}^*\| \leq e^{-\alpha t} \|\mathbf{z}(0) - \mathbf{z}^*\|$$

即状态指数收敛到平衡态 \(\mathbf{z}^*\)，且收敛速率由 \(\alpha\) 控制。

③ 超球面排斥损失防止特征坍缩

收缩约束虽然保证了稳定性，但可能导致所有输入映射到同一不动点（特征坍缩）。为此引入超球面排斥损失：

$$\mathcal{L}_{\text{repel}} = -\frac{1}{B^2} \sum_{i \neq j} \log \left(1 - \cos^2(\mathbf{z}_i^*, \mathbf{z}_j^*)\right)$$

其中 \(B\) 是 batch size，\(\mathbf{z}_i^*\) 是第 \(i\) 个样本的平衡态。该损失强制不同输入的不动点在超球面上均匀分布，保持表征多样性。总损失为 \(\mathcal{L} = \mathcal{L}_{\text{task}} + \beta \mathcal{L}_{\text{repel}}\)。

④ 随机微分方程扩展（NSDE）

为进一步增强鲁棒性，CMM 可选地引入随机性：

$$d\mathbf{z} = f_\theta(\mathbf{z}, \mathbf{x})dt + \sigma(\mathbf{z}, \mathbf{x})d\mathbf{W}_t$$

其中 \(\sigma\) 是可学习的扩散系数，\(\mathbf{W}_t\) 是 Wiener 过程。随机性帮助模型在推理时探索潜在空间中的多个解路径，类似于模拟退火。在实验中，NSDE 版本在 Maze benchmark 上比确定性版本高 2.8%。

LLM RL & Alignment

Off-Policy Value-Based Reinforcement Learning for Large Language Models
🎯 RL Off-Policy Peng-Yuan Wang, Ziniu Li, Tian Xu, Bohan Yang, Tian-Shuo Liu, ChenYang Wang et al. arXiv:2603.23355
背景与动机

LLM 的 RL 训练（如 GRPO、PPO）主要是 on-policy 的：每批生成的数据只用一次就丢弃，然后重新采样。这在长序列推理任务（如数学证明、代码生成）中导致严重的样本浪费，因为生成一条 16K token 的轨迹可能需要数十秒。Off-policy 方法理论上可以通过经验回放（experience replay）复用历史数据，但 LLM 领域的 off-policy RL 一直未被充分探索。核心挑战是：如何在 token 级别的自回归生成中有效地定义和学习价值函数？

核心方法详解
① ReVal 框架：Bellman 更新 + 结果验证

ReVal 将 LLM 生成过程建模为 MDP，状态 \(s_t\) 是已生成的前缀 \((x, y_{1:t})\)，动作 \(a_t = y_{t+1}\) 是下一个 token。定义状态-动作价值函数 \(Q(s_t, a_t)\) 并使用 Bellman 一致性进行训练：

$$Q(s_t, a_t) = r(s_t, a_t) + \gamma \max_{a'} Q(s_{t+1}, a')$$

但直接在 token 级别应用 Bellman 更新面临稀疏奖励问题（只有轨迹末尾才有验证信号）。ReVal 结合了两类信号：

  • Step-level signal（逐步信号）：利用相邻步骤的 Q 值差异作为内在奖励 \(r_t^{\text{step}} = Q(s_{t+1}, a_{t+1}) - Q(s_t, a_t)\)
  • Trajectory-level signal（轨迹信号）：来自结果验证器（ORM），为完整解提供二元正确性反馈
② 经验回放缓冲区设计

ReVal 维护一个优先级经验回放缓冲区 \(\mathcal{D}\)，存储历史轨迹及其 Q 值估计。关键设计：

  1. 轨迹级存储：每条轨迹 \(\tau = (x, y_1, \ldots, y_T, r)\) 完整存储，保持推理链的连贯性
  2. 优先级采样：优先采样 TD error 较大的轨迹：\(p(\tau) \propto |\delta_\tau|^\alpha\)，其中 \(\delta_\tau = \frac{1}{T}\sum_t |r_t + \gamma Q(s_{t+1}) - Q(s_t, a_t)|\)
  3. 重要性权重校正：使用 \(\rho_t = \min(c, \frac{\pi(a_t|s_t)}{\mu(a_t|s_t)})\) 校正行为策略 \(\mu\) 和当前策略 \(\pi\) 的分布偏差，截断常数 \(c = 5\) 防止方差爆炸
③ 价值函数参数化与训练

Q 函数通过在 LLM 最后一层隐藏状态上添加一个轻量级 value head（2 层 MLP，隐藏维度 1024）实现：

$$Q(s_t, a_t) = \text{MLP}(\mathbf{h}_t^L), \quad \mathbf{h}_t^L = \text{Transformer}(x, y_{1:t+1})$$

训练损失结合 Bellman 残差和轨迹级监督：

$$\mathcal{L} = \underbrace{\mathbb{E}_{\tau \sim \mathcal{D}} \left[\sum_t \rho_t (Q(s_t,a_t) - y_t^{\text{target}})^2 \right]}_{\text{Bellman loss}} + \lambda \underbrace{\mathbb{E}_\tau \left[(V(s_0) - R(\tau))^2 \right]}_{\text{outcome supervision}}$$

使用 target network（EMA 更新，\(\tau_{\text{ema}} = 0.995\)）稳定训练。

④ 策略提取与推理加速

训练好 Q 函数后，推理时不需要采样多条轨迹进行 majority voting，而是使用 Q 值引导解码：

$$\pi^*(a_t|s_t) \propto \pi_{\text{ref}}(a_t|s_t) \cdot \exp\left(\frac{Q(s_t, a_t)}{\beta}\right)$$

这等价于一种 guided decoding，让模型在每一步偏向于 Q 值更高的 token。\(\beta\) 控制探索程度。在实验中，\(\beta=0.5\) 时只需 1 条采样路径就能达到 GRPO 使用 64 条采样路径的 majority voting 性能。

SortedRL: Accelerating RL Training for LLMs through Online Length-Aware Scheduling
🎯 RL Training Efficiency Yiqi Zhang, Huiqiang Jiang, Xufang Luo et al. · Microsoft Research arXiv:2603.23414
背景与动机

RL 训练 LLM（如 GRPO）的主要瓶颈在于生成阶段：自回归生成速度慢，且生成与策略更新之间的同步开销巨大。当生成长轨迹（例如 16K tokens 的推理链）时，生成阶段可占据总训练时间的 70%。更严重的是，同一 batch 中不同样本的生成长度差异极大（100 到 16K），短样本必须等待最长样本完成才能进入下一步更新，造成大量 "训练气泡"（bubble）。本文的目标是：在不牺牲训练效果的前提下，显著减少这些气泡。

核心方法详解
① 在线长度感知排序（Online Length-Aware Sorting）

SortedRL 的核心思想是对生成样本按输出长度重新排序。在大生成批次中（例如 batch=512），所有样本并行生成，完成后按长度排序并分组：

$$\text{Sort}: \{\tau_1, \ldots, \tau_N\} \rightarrow \{\tau_{\sigma(1)}, \ldots, \tau_{\sigma(N)}\}, \quad \text{s.t. } |\tau_{\sigma(i)}| \leq |\tau_{\sigma(i+1)}|$$

然后将排序后的序列切分为多个更新子批（update mini-batches），短样本优先组成首批更新。这带来了三个好处：(1) 短样本组内 padding 浪费最小化；(2) 短样本可以立即开始更新，无需等待长样本；(3) 自然形成了从短到长的课程学习（curriculum）。

② 缓存机制控制 Off-policy 程度

当生成和更新异步进行时，后续更新批使用的数据可能略微 off-policy（因为策略已经更新了几步）。SortedRL 引入缓存机制来量化和控制这种偏差：

$$\text{staleness}(\tau_i) = k, \quad \text{where } k = \text{number of policy updates since } \tau_i \text{ was generated}$$

定义最大容忍陈旧度 \(K_{\max}\)，当 \(\text{staleness}(\tau_i) > K_{\max}\) 时丢弃该轨迹。实验发现 \(K_{\max} = 3\) 是最优选择，允许适度复用的同时保持近 on-policy 的训练质量。对于仍在容忍范围内的轨迹，使用轻量级重要性加权：

$$w(\tau_i) = \min\left(1, \prod_{t} \frac{\pi_\theta(a_t|s_t)}{\pi_{\text{old}}(a_t|s_t)}\right)^{1/T_i}$$

③ 有状态控制器与生成缓冲区

SortedRL 的基础设施层由两个组件构成：

  1. Stateful Controller：跟踪每条轨迹的生成时间戳、关联的策略版本号和长度，动态决定哪些轨迹进入下一个更新批
  2. Generation Buffer：异步收集完成的轨迹，按长度维护一个有序队列。当累积足够短样本时立即触发更新，无需等待整个生成批完成

这种架构使得生成和训练可以高度重叠（overlap），理想情况下训练气泡趋近于零。

④ 微课程效应（Micro-Curriculum Effect）

SortedRL 的排序策略自然产生了一种从简单到困难的课程：短回答通常对应简单问题，长推理链对应困难问题。形式化地，第 \(k\) 个更新批的平均难度为：

$$\bar{d}_k = \frac{1}{|B_k|} \sum_{\tau \in B_k} \text{difficulty}(\tau) \approx \frac{1}{|B_k|} \sum_{\tau \in B_k} |\tau|$$

其中 \(\bar{d}_1 < \bar{d}_2 < \ldots < \bar{d}_M\)。这种渐进式课程在实验中额外带来 1-2% 的性能提升。

ImplicitRM: Unbiased Reward Modeling from Implicit Preference Data for LLM Alignment
🎯 RL Reward Model Hao Wang, Haocheng Yang, Licheng Pan, Lei Shen, Xiaoxi Li et al. arXiv:2603.23184
背景与动机

RLHF 的核心组件——奖励模型——严重依赖昂贵的人工标注（显式偏好数据，如 "A 好于 B"）。相比之下，用户与 LLM 交互中产生的隐式反馈（如点击、复制、重新生成）天然大量存在且零成本。然而，隐式反馈存在两个本质难题：(1) 缺少明确的负样本（用户不点击可能只是没看到，而非不喜欢）；(2) 存在偏好偏差（propensity bias）——用户更倾向于对某些类型的响应进行交互（如更短的回答更容易被复制）。这使得直接用隐式数据训练奖励模型会产生严重偏差。

核心方法详解
① 四群体分层模型

ImplicitRM 将每个（查询, 响应）对 \((q, r)\) 视为属于四个潜在群体之一：

  • TP（True Positive）：真正好的响应 + 用户给出了隐式正反馈
  • FP（False Positive）：一般的响应 + 用户因偏好偏差给出了正反馈
  • FN（False Negative）：真正好的响应 + 用户未给出反馈（被遗漏）
  • TN（True Negative）：真正差的响应 + 无反馈

定义潜变量 \(z \in \{TP, FP, FN, TN\}\)，偏好偏差函数 \(e(q, r) = P(\text{positive feedback} | q, r)\)，真实质量函数 \(\theta(q, r) = P(\text{good} | q, r)\)。则观测到的隐式正反馈概率为：

$$P(\text{click}=1|q,r) = \theta(q,r) \cdot e(q,r) + (1-\theta(q,r)) \cdot \epsilon(q,r)$$

其中 \(\epsilon\) 是噪声点击率。

② 可辨识性条件与 EM 训练

为从混合分布中分离出真实质量函数 \(\theta\)，作者证明了在以下条件下模型是可辨识的：

$$\text{Assumption: } \exists \text{ anchor set } \mathcal{A} \text{ where } \theta(q,r) = 1 \; \forall (q,r) \in \mathcal{A}$$

实际中，锚定集可通过高一致性（多用户均给正反馈）的样本近似构建。基于此，使用 EM 算法进行训练：

  1. E-step：给定当前参数 \(\hat{\theta}, \hat{e}\)，计算每个样本属于四群体的后验概率
  2. M-step：最大化完整数据似然，分别更新质量网络 \(\hat{\theta}\) 和偏好偏差网络 \(\hat{e}\)
③ 无偏似然目标推导

在四群体模型下，ImplicitRM 推导出一个无偏的学习目标。对于有正反馈的样本集 \(\mathcal{D}^+\) 和无反馈的集 \(\mathcal{D}^-\)：

$$\mathcal{L} = -\sum_{(q,r) \in \mathcal{D}^+} \log\left[\theta(q,r) \cdot e(q,r) + \bar{\theta}(q,r) \cdot \epsilon(q,r)\right] - \sum_{(q,r) \in \mathcal{D}^-} \log\left[\theta(q,r) \cdot \bar{e}(q,r) + \bar{\theta}(q,r) \cdot \bar{\epsilon}(q,r)\right]$$

其中 \(\bar{\theta} = 1 - \theta\)，\(\bar{e} = 1 - e\)。作者证明了在正确指定的模型下，\(\hat{\theta}\) 是 \(\theta^*\) 的一致估计量（consistent estimator），即随着数据量增加收敛到真实值。

④ 奖励分数提取与 RLHF 集成

训练完成后，\(\theta(q, r)\) 直接作为奖励分数使用。为将其集成到 RLHF pipeline：

$$R(q, r) = \log \frac{\theta(q, r)}{1 - \theta(q, r)}$$

即对 \(\theta\) 取 logit 变换，得到一个 unbounded 的奖励信号。该信号可直接用于 PPO/GRPO 的奖励函数，替代传统的显式偏好奖励模型。关键优势：训练数据完全来自用户自然交互日志，无需额外标注。

Code Review Agent Benchmark (c-CRAB)
🤖 Agent Code Review c-CRAB Team arXiv:2603.23448
背景与动机

AI 代码生成 agent（如 Devin、Claude Code、Codex）正在自动化大量代码编写工作，但这些自动生成的代码在集成到大型代码库时，代码审查（code review）成为关键质量保障环节。现有的 agent 评估主要关注代码生成能力（如 SWE-bench），而代码审查能力缺乏系统化评测。核心问题是：AI 代码审查 agent 能否发现人类审查者关注的问题？agent 审查与人类审查的关注维度是否一致？

核心方法详解
① 基于人类审查意见的数据集构建

c-CRAB 数据集的构建流程：

  1. 从开源项目中收集真实的 Pull Request（PR）及其人类审查意见
  2. 对每条人类审查意见，使用 LLM 将其转化为一组可执行测试：\(\text{ReviewComment} \rightarrow \{t_1, t_2, \ldots, t_k\}\)
  3. 每个测试 \(t_i\) 是一个断言，验证审查 agent 的输出是否覆盖了对应的审查关注点
  4. 质量控制：人工验证转化后的测试确实能准确反映原始审查意见的意图

最终数据集包含 来自多个 GitHub 项目的 PR 实例，覆盖代码逻辑错误、性能问题、安全漏洞、API 误用等多种审查维度。

② 评估框架设计

评估遵循"审查即测试"范式。给定一个 PR \(p\) 和其对应的人类审查测试集 \(\{t_1, \ldots, t_k\}\)：

  1. 代码审查 agent 对 PR 生成审查意见 \(\mathcal{R}_{\text{agent}}\)
  2. 自动评估器检查 \(\mathcal{R}_{\text{agent}}\) 是否覆盖了每个测试点 \(t_i\)
  3. 计算覆盖率：\(\text{Coverage}(p) = \frac{|\{t_i : t_i \text{ covered by } \mathcal{R}_{\text{agent}}\}|}{k}\)

关键度量包括：任务通过率（至少覆盖所有测试点）、平均覆盖率、以及误报率（agent 提出的无效/错误审查意见）。

③ 多 Agent 协同测试生成

c-CRAB 的独特之处在于测试本身也是由 AI agent 生成的，形成了三方协作闭环：

  • Code Generation Agent：产生代码（PR）
  • Code Review Agent：审查代码，生成审查意见
  • Test Generation Agent：基于人类审查意见生成测试，作为评估审查 agent 的标准

这种设计意味着测试集本身也是对测试生成 agent 质量的验证，即测试必须足够精确才能有效评估审查 agent。这种"保留测试套件"（holdout test suite）机制为未来三类 agent 的协同优化提供了研究方向。

④ 人机审查维度差异分析

作者对 agent 和人类审查意见的关注维度进行了系统化分类和对比。将审查意见标记为以下类别：

  • 逻辑正确性（Logic）、性能（Performance）、安全（Security）
  • 代码风格（Style）、可维护性（Maintainability）、文档（Documentation）

发现：agent 更倾向于发现显式的逻辑 bug 和安全漏洞，而人类审查者更关注可维护性和代码风格的微妙问题。这种互补性暗示了人机协作审查的巨大潜力。

MemCollab: Cross-Agent Memory Collaboration via Contrastive Trajectory Distillation
🤖 Agent Memory Yurui Chang, Yiran Wu, Qingyun Wu, Lu Lin arXiv:2603.23234
背景与动机

LLM Agent 依赖记忆机制来复用过去的经验知识。但现有记忆系统采用"每个 agent 独立记忆"的方式，记忆与特定模型的推理风格紧密耦合。在异构 agent 部署中（例如同一系统使用 GPT-4、Claude、Llama 的不同 agent），一个自然的问题是：能否构建一个跨 agent 共享的记忆系统？实验发现，直接在 agent 间传递记忆通常会降低性能，因为记忆中混杂了"任务级别的通用知识"和"agent 特有的推理偏差"。

核心方法详解
① 对比轨迹蒸馏（Contrastive Trajectory Distillation）

MemCollab 的核心思想是通过对比不同 agent 在相同任务上的推理轨迹来提取"agent-agnostic"的记忆。给定任务 \(\tau\) 和两个异构 agent \(A_1, A_2\) 的轨迹 \(T_1, T_2\)：

$$\mathbf{m}_\tau = \text{Distill}(T_1, T_2) = \text{LLM}(\text{prompt}_{\text{contrast}}, T_1, T_2)$$

对比提示（contrast prompt）引导 LLM 识别两条轨迹中的共同推理约束（如"必须先检查边界条件"）并过滤掉agent 特有的风格差异（如推理步骤的粒度、格式偏好）。蒸馏出的记忆 \(\mathbf{m}_\tau\) 是抽象的推理规则，而非具体的推理步骤。

② 记忆表示与存储结构

每条蒸馏记忆 \(\mathbf{m}\) 以结构化格式存储：

  • 任务类别标签 \(c\)：自动分类到预定义类别（如"数组操作"、"图算法"、"字符串处理"）
  • 推理约束：一组抽象规则，如"处理递归时需检查栈溢出"
  • 反例模式：从失败轨迹中提取的常见错误模式
  • 置信度分数 \(w\)：基于蒸馏来源的 agent 数量和一致性加权

记忆库采用向量数据库存储，每条记忆同时维护文本表示和 embedding 向量，支持语义检索。

③ 任务感知检索机制

在推理时，MemCollab 根据当前任务检索相关记忆。检索策略结合了语义相似性和类别匹配：

$$\text{score}(\mathbf{m}, \tau_{\text{new}}) = \alpha \cdot \cos(\mathbf{e}_m, \mathbf{e}_\tau) + (1-\alpha) \cdot \mathbb{1}[c_m = c_\tau]$$

其中 \(\mathbf{e}_m, \mathbf{e}_\tau\) 分别是记忆和新任务的 embedding，\(\alpha = 0.7\)。检索 top-K（\(K=3\)）相关记忆后，以结构化格式注入 agent 的系统提示中。关键设计：检索到的记忆是约束条件而非完整解法，因此不会限制 agent 的推理灵活性。

④ 迭代协作记忆增长

记忆库随时间持续增长和优化：

  1. 当新的 agent 加入系统时，在已有任务上生成新轨迹，与已有轨迹对比蒸馏，增强现有记忆的覆盖度
  2. 当记忆被检索并使用后，根据任务成功/失败信号更新置信度分数：\(w \leftarrow w + \eta \cdot (\text{success} - 0.5)\)
  3. 低置信度（\(w < w_{\min}\)）的记忆被自动淘汰，保持记忆库的精炼

实验中，经过 3 轮 agent 加入和记忆优化后，平均任务准确率继续提升 2.1%。

Polaris: A Gödel Agent Framework for Small Language Models through Experience-Abstracted Policy Repair
🤖 Agent Self-Improvement Aditya Kakade, Vivek Srivastava, Shirish Karande arXiv:2603.23129
背景与动机

当前的 LLM Agent 自我改进大多在响应级别（response-level），即修正单次输出。但真正有意义的改进应该在策略级别（policy-level）——即改进 agent 处理某一类问题的通用策略，使改进能够泛化到未见实例。此外，大模型（>70B）天然具有较强的元推理能力（meta-reasoning），但小模型（7B）在这方面严重不足。核心挑战是：如何让小语言模型也具备策略级别的递归自我修正能力？

核心方法详解
① 递归策略修复循环

Polaris 实现了一个"检查 → 分析 → 修补 → 验证"的递归循环：

  1. Execute：Agent 使用当前策略 \(\pi_k\) 在任务集 \(\mathcal{T}\) 上执行，收集轨迹 \(\{(\tau_i, \text{success}_i)\}\)
  2. Diagnose：对失败轨迹进行结构化错误分析，提取错误模式 \(\mathcal{E} = \{e_1, \ldots, e_m\}\)
  3. Repair：基于错误模式生成策略补丁 \(\Delta\pi_k\)，形式为"当遇到[模式]时，应该[新策略]"
  4. Verify：在保留验证集上测试 \(\pi_{k+1} = \pi_k \oplus \Delta\pi_k\)，确保不引入退化

循环持续至策略收敛或达到最大轮次 \(K\)。

② 经验抽象（Experience Abstraction）

为让策略补丁能泛化到未见实例，Polaris 对失败经验进行抽象化处理：

$$\text{Abstract}: (e_{\text{specific}}, \tau_{\text{failed}}) \rightarrow e_{\text{abstract}}$$

例如，从"在第 3 步将 35 × 7 误算为 235"这一具体错误中抽象出"多位数乘法时需要分步验证中间结果"的通用策略。抽象过程使用 LLM 本身完成，并通过提示工程引导 agent 关注错误的结构特征而非具体数值。每条抽象策略保留：

  • 触发条件：描述策略适用的上下文模式
  • 修复动作：具体的推理步骤调整
  • 置信度：基于验证集通过率的加权分数
③ 最小代码路径修复（Minimal Code-Path Repair）

策略在 Polaris 中以结构化的 "策略代码" 形式表示——一组条件规则和对应的行动指令。修复时遵循最小编辑原则：

$$\text{edit\_distance}(\pi_k, \pi_{k+1}) \leq \delta_{\max}$$

这意味着每轮只允许添加/修改少量规则，避免大幅度策略变更导致的灾难性遗忘。实际实现中，\(\delta_{\max} = 3\)（每轮最多修改 3 条规则）。修复候选通过验证集选择最优方案。

④ 元推理引导的错误归因

在诊断阶段，Polaris 要求 agent 进行元推理（meta-reasoning）：

  1. 回溯失败轨迹中的关键决策点（decision points）
  2. 对每个决策点生成反事实推理："如果在此处选择了 [替代方案] 会怎样？"
  3. 基于反事实结果确定根因错误（root cause），而非表层症状

对于小模型，元推理能力通过 few-shot 示例 + 结构化提示模板增强。实验中，使用元推理的错误归因准确率从 48% 提升到 72%。

Can LLM Agents Generate Real-World Evidence? Evaluating Observational Studies in Medical Databases
⚕️ Medical RWE Dubai Li, Yuxiang He, Yan Hu, Yu Tian, Jingsong Li arXiv:2603.22767
背景与动机

观察性研究（observational studies）是产生真实世界证据（Real-World Evidence, RWE）的重要途径，但在真实医疗数据库上执行这些研究是一个高度开放性的任务——需要在队列构建（cohort construction）、统计分析（analysis）、报告撰写（reporting）之间做出连贯的决策链。现有对 LLM Agent 的评估往往只考察孤立步骤或单一问题的回答，忽略了生成完整"证据包"的系统性质量。核心问题是：LLM Agent 能否端到端地在真实医疗数据库上执行观察性研究，并产出可信的证据报告？

核心方法详解
① RWE-bench 基准构建

RWE-bench 基于 MIMIC-IV 数据库构建，参考已发表的同行评审观察性研究，构建了 162 项研究任务。每项任务包含：

  • 研究方案（Protocol）：定义纳排标准、暴露/结局变量、分析方法（如 Cox 回归、Kaplan-Meier 曲线）
  • 参考标准：来自原始论文的关键统计结果（HR、CI、p-value）
  • 评估维度：不仅检查最终数值，还评估证据包的内部一致性（如队列大小与报告是否匹配）

任务涵盖心血管、重症监护、感染性疾病等多个临床领域。

② 树状证据包生成范式

Agent 需要生成一个树状结构的证据包 \(\mathcal{E}\)，层次为：

$$\mathcal{E} = \{\text{Protocol} \rightarrow \text{Cohort} \rightarrow \text{Analysis} \rightarrow \text{Results} \rightarrow \text{Report}\}$$

每一层的输出是下一层的输入，且必须保持内部一致性：

  1. Protocol → SQL 查询：将纳排标准转化为 MIMIC-IV 上的 SQL 查询
  2. SQL → 队列数据：执行查询获取患者队列
  3. 队列 → 统计分析：应用指定的统计方法（生存分析、倾向性评分匹配等）
  4. 分析 → 报告：将结果整理为结构化医学报告
③ 自动化队列评估方法

为快速定位 agent 的错误来源，提出了自动化队列验证：

  • Schema Validator：检查 SQL 查询是否符合 MIMIC-IV 的表结构和命名规范
  • Cohort Size Check：队列大小应在参考论文的 \(\pm 20\%\) 范围内
  • Covariate Balance Check：倾向性评分匹配后各协变量的标准化均值差 \(|SMD| < 0.1\)
  • Result Plausibility：Hazard Ratio 的方向（保护/风险因素）应与参考一致

通过逐层验证，可以精确定位是"SQL 错误"还是"统计方法选择错误"还是"结果解读错误"。

④ 故障模式分类学

对 6 个 LLM × 3 个 Agent 框架的失败案例进行了系统化分类：

  1. Schema Hallucination（28%）：Agent 引用了 MIMIC-IV 中不存在的表或字段
  2. Cohort Leakage（19%）：纳排标准执行不严格，将不符合条件的患者纳入
  3. Method Mismatch（22%）：使用了不适合研究设计的统计方法
  4. Numerical Drift（15%）：中间计算步骤的累积误差导致最终结果偏差
  5. Report Inconsistency（16%）：报告中的数值与实际计算结果不匹配

CLiGNet: Clinical Label-Interaction Graph Network for Medical Specialty Classification
⚕️ Medical Clinical NLP Pronob Kumar Barman, Pronoy Kumar Barman arXiv:2603.22752
背景与动机

将临床文本自动分类到医学专科（如心内科、骨科、精神科）对于诊疗路由和编码至关重要。然而，广泛使用的 MTSamples 基准存在严重的数据泄漏问题——先前研究在训练-测试划分之前应用了 SMOTE 过采样，导致测试集中混入了合成的训练样本，虚高了准确率。此外，40 个医学专科之间存在极端的类别不均衡（最大/最小类比 181:1），以及标签之间的语义关联（如"心脏外科"和"心内科"高度相关）。

核心方法详解
① 无泄漏基准建立

首先修正数据泄漏问题：严格执行"先划分、后过采样"：

  1. 将 4,966 条记录按 80/20 分层随机划分为训练/测试集
  2. 仅在训练集上应用 SMOTE（如需要），测试集保持原始分布
  3. 对比修正前后：先前报告的准确率 ~95% 降至修正后的 ~25%（Macro F1），揭示了任务的真实难度
② BioClinicalBERT + 标签图网络架构

CLiGNet 包含两个平行分支：

  • 文本分支：使用 BioClinicalBERT 编码临床文本，得到文档表示 \(\mathbf{d} \in \mathbb{R}^{768}\)
  • 标签图分支：构建一个 40 节点的医学专科图，节点特征为标签名称的 embedding，边权重基于两种信号：

$$w_{ij} = \alpha \cdot \cos(\mathbf{e}_i, \mathbf{e}_j) + (1-\alpha) \cdot \mathbb{1}[\text{ICD-10 chapter}(i) = \text{ICD-10 chapter}(j)]$$

两层 GCN 传播标签间的关联信息：\(\mathbf{H}^{(l+1)} = \sigma(\hat{\mathbf{A}} \mathbf{H}^{(l)} \mathbf{W}^{(l)})\)，得到标签表示 \(\mathbf{L} \in \mathbb{R}^{40 \times 768}\)。

③ 逐标签注意力门控融合

将文档表示与标签图表示通过注意力门控融合：

$$\mathbf{g}_j = \sigma\left(\mathbf{W}_g [\mathbf{d}; \mathbf{l}_j] + \mathbf{b}_g\right)$$

$$\hat{y}_j = \sigma\left(\mathbf{W}_o (\mathbf{g}_j \odot \mathbf{d} + (1-\mathbf{g}_j) \odot \mathbf{l}_j)\right)$$

其中 \(\mathbf{l}_j\) 是第 \(j\) 个标签的 GCN 输出表示，\(\mathbf{g}_j\) 是门控权重。这允许模型对不同标签动态调整"文本特征"和"标签关系特征"的贡献比例。

④ 焦点损失 + Platt Scaling 校准

使用焦点二元交叉熵损失处理极端类别不均衡：

$$\mathcal{L}_{\text{focal}} = -\frac{1}{C}\sum_{j=1}^{40} \left[(1-\hat{y}_j)^\gamma y_j \log \hat{y}_j + \hat{y}_j^\gamma (1-y_j) \log(1-\hat{y}_j)\right]$$

其中 \(\gamma = 2\) 使模型更关注困难/稀有类别。训练后使用 Platt Scaling 进行概率校准，通过在验证集上学习标量参数 \(a, b\) 将 logits 映射为校准概率：\(P_{\text{cal}} = \sigma(a \cdot z + b)\)。校准后期望校准误差（ECE）仅 0.007。

在线新闻的可信度评估是一个日益紧迫的问题。TREC 2025 DRAGUN Track 针对此需求设立了两个任务：Task 1 生成用于评估新闻可信度的"关键问题"（critical questions），Task 2 基于检索到的证据生成结构化的可信度报告（trust report）。这要求系统同时具备问题生成、大规模检索、重排序和报告综合生成能力。核心挑战是：如何将 LLM 的生成能力与传统 IR 的检索能力有效结合？

核心方法详解
① LLM 驱动的关键问题生成（Task 1）

给定一篇新闻文章 \(d\)，系统生成一组关键问题 \(\{q_1, \ldots, q_n\}\)，帮助用户评估其可信度。流程包括：

  1. 初始生成：使用 LLM 生成 20+ 候选问题，覆盖来源可靠性、事实准确性、偏见、逻辑一致性等维度
  2. 语义过滤：计算问题间的 pairwise cosine similarity，移除冗余问题（\(\text{sim} > 0.85\)）
  3. 多样性聚类：对剩余问题使用 K-Means 聚类（\(K=8\)），每个 cluster 选择最具代表性的问题

最终输出 8-10 个覆盖多维度的关键问题。

② 多策略查询扩展与检索

为从 MS MARCO V2.1 语料库中检索支持可信度判断的证据，使用多种查询扩展策略：

  • Chain-of-Thought (CoT) 扩展：LLM 对每个关键问题进行推理，生成扩展查询 \(q' = \text{LLM}(\text{CoT\_prompt}, q)\)
  • Entity-based 扩展：从文章中提取关键实体，与问题组合形成复合查询
  • Paraphrase 扩展：生成问题的多种表述变体，增加召回覆盖

使用 BM25 + dense retrieval (E5-large) 的 hybrid 检索，初始返回 top-1000 候选文档。

③ monoT5 重排序 + 领域可信度过滤

对检索到的候选文档进行两阶段精排：

  1. monoT5 相关性重排序：使用 fine-tuned monoT5-3B 对 (query, document) pair 打分，选择 top-50
  2. LLM 相关性判断：对 top-50 进一步使用 LLM 评估相关性（0-3 分），阈值过滤保留 ≥2 分的文档
  3. 领域可信度评估：利用预构建的域名可信度数据集，对来源进行可信度加权：\(\text{final\_score}_i = \text{relevance}_i \times \text{domain\_trust}_i\)
④ 检索增强可信度报告生成（Task 2）

将筛选后的证据文档 + 关键问题 + 原始文章输入 LLM，生成结构化可信度报告：

  • 摘要：一句话概括文章可信度判断
  • 逐问题分析：针对每个关键问题，引用具体证据文档给出分析
  • 可信度评分：1-5 分的综合评分及理由
  • 引用：每个分析点标注来源文档 ID，确保可追溯性

报告生成使用 few-shot prompting + 结构化输出约束（JSON schema 引导）。

SIDReasoner: Reasoning over Semantic IDs Enhances Generative Recommendation
🔍 IR Recommendation Yingzhi He, Yan Sun, Junfei Tan, Yuxin Chen, Xiaoyu Kong et al. arXiv:2603.23183
背景与动机

生成式推荐将序列推荐表述为自回归 token 生成——每个物品用一组离散 token（Semantic IDs, SIDs）表示，LLM 在语言 token 和物品 token 的统一空间中做自回归预测。SID 使得 LLM 可以高效解码大规模物品库。然而，LLM 推理能力的爆发式增长虽然带来了新机遇，但对 SID 的有效推理仍未被探索。核心挑战是：物品 token 对 LLM 来说本质上无意义（不像语言 token 有语义），且面向推荐的推理轨迹极难获取标注。

核心方法详解
① SID-语言对应强化（SID-Language Alignment）

为让 LLM 理解物品 token 的语义，SIDReasoner 第一阶段在丰富的以 SID 为中心的语料上进行多任务训练：

  • SID → Description：给定 SID 序列，生成物品描述文本
  • Description → SID：给定物品描述，预测其 SID 序列
  • SID-SID Relation：给定两个物品的 SID，判断其语义关系（互补/替代/无关）

训练语料由更强的教师模型合成。多任务目标为：

$$\mathcal{L}_{\text{align}} = \sum_{k} \lambda_k \mathcal{L}_k = \lambda_1 \mathcal{L}_{\text{S2D}} + \lambda_2 \mathcal{L}_{\text{D2S}} + \lambda_3 \mathcal{L}_{\text{S2S}}$$

经过对齐训练后，LLM 的 SID embedding 空间与语言 embedding 空间对齐。

② 结果驱动的强化优化（Outcome-Driven RL）

第二阶段使用 RL 优化推荐推理能力。关键设计是不需要显式的推理标注——仅使用推荐结果作为奖励信号：

$$R(\tau, y^*) = \begin{cases} 1 & \text{if predicted SID matches } y^* \\ \text{partial}(\hat{y}, y^*) & \text{if first } k \text{ tokens match} \\ 0 & \text{otherwise} \end{cases}$$

其中 \(\text{partial}\) 提供部分匹配奖励（前缀匹配越长，奖励越高），缓解稀疏奖励问题。使用 GRPO 作为策略优化算法，鼓励模型在生成 SID 之前产生推理 token（如"用户之前买了耳机和手机壳 → 可能对数据线感兴趣 → SID:[34,12,7]"）。

③ 推理增强的解码策略

推理时，SIDReasoner 采用"先推理后预测"的模式：

  1. 模型先生成一段自由形式的推理文本（reasoning trace），分析用户历史行为模式
  2. 基于推理文本，预测目标物品的 SID token 序列
  3. 使用 constrained beam search 确保生成的 SID 对应真实物品（通过 SID trie 约束）

推理文本长度通过 length penalty \(\alpha_{\text{len}}\) 控制，实验中最优长度为 30-50 tokens。

④ 跨域泛化能力

SIDReasoner 的一个意外发现是其跨域泛化能力。在域 A（如电子产品）上训练的推理能力可以迁移到域 B（如书籍）：

$$\text{Transfer Performance} = (1 - \delta) \cdot \text{In-domain} + \delta \cdot \text{Random}$$

实验中 \(\delta \approx 0.15\)，即跨域性能仅损失约 15%——远好于无推理的基线（损失 30-40%）。这表明推理能力捕获了领域无关的用户行为模式（如"互补购买"、"品牌忠诚"等抽象规律）。
