LogicDiff: Measuring Logical Consistency of Text-to-Image Models

👤 Shreya Ghosh
📅 2026-03-28
🏛 Independent Researcher (India)
LLM · 逻辑推理 · 评估
LogicDiff 提出了一种系统化的评估框架，用于衡量文本到图像（T2I）模型在处理复杂逻辑提示时的一致性。该框架通过构建覆盖六种逻辑运算符（合取、析取、否定、条件、双条件、异或）的多层级测试集，结合结构化的评估管线，发现当前主流 T2I 模型（DALL-E 3、Stable Diffusion XL、Midjourney 等）在面对复杂逻辑组合时准确率大幅下降，尤其在否定和条件逻辑方面表现最差。

🔬 核心方法详解

① 逻辑提示分类体系

建立六类逻辑运算的提示模板体系：

  • 合取 (AND)：\(A \land B\) — "一只红色的猫和一个蓝色的球"
  • 析取 (OR)：\(A \lor B\) — "一只猫或一条狗在公园里"
  • 否定 (NOT)：\(\neg A\) — "一个没有窗户的房子"
  • 条件 (IF-THEN)：\(A \to B\) — "如果下雨，则有人撑伞"
  • 双条件 (IFF)：\(A \leftrightarrow B\) — "当且仅当有太阳时有影子"
  • 异或 (XOR)：\(A \oplus B\) — "要么是白天要么是黑夜，但不能同时"

每类包含简单（单运算符）和复合（2-3个嵌套运算符）两种复杂度级别，共生成 1,200 个标注提示。

② 自动化评估管线 (LogicEval)

设计三阶段评估流程：

  1. 元素检测：使用 YOLO-World + GroundingDINO 检测图像中是否存在/缺失指定元素
  2. 属性验证：通过 CLIP 零样本分类验证颜色、材质、空间关系等属性匹配度
  3. 逻辑一致性评分：构建布尔真值表，将检测结果映射为逻辑满足度分数 \(S = \frac{1}{N}\sum_{i=1}^{N} \mathbb{1}[\phi_i(\text{img}) = \text{True}]\)
③ 复合逻辑难度建模

引入逻辑复杂度指标 \(\text{LC}(p)\)，定义为提示 \(p\) 中逻辑运算符的加权嵌套深度：

$$\text{LC}(p) = \sum_{i=1}^{k} w_i \cdot d_i$$

其中 \(w_i\) 为第 \(i\) 个运算符的难度权重（NOT=1.5, XOR=1.8 为最高），\(d_i\) 为嵌套深度。实验表明，模型准确率与 LC 呈强负相关（\(r=-0.82\)），LC>3 时所有模型准确率低于 40%。

④ 跨模型比较分析

在 5 个主流 T2I 模型上测试：DALL-E 3 在简单合取上达到 89% 准确率，但在复合否定+条件组合上骤降至 23%。Stable Diffusion XL 在否定类任务上仅 18%。关键发现：所有模型共享一个"逻辑天花板"——当提示包含 ≥2 个否定运算时，准确率趋同于 15-25%，说明限制来自底层语言编码器而非扩散模型本身。

💡 深度案例分析
  1. 否定失败模式：提示"一个没有烟囱的房子"→ DALL-E 3 在 63% 的情况下仍生成了带烟囱的房子，表明 CLIP 文本编码器对否定词的语义理解存在系统性缺陷
  2. 条件逻辑退化：提示"如果是冬天场景，则树上没有叶子"→ 模型倾向于忽略条件前提，直接渲染无叶树+夏天背景的矛盾图像
  3. 嵌套爆炸：\(\neg(A \land B) \lor C\) 类提示的正确率仅为 12%，模型无法解析德摩根律展开后的语义

🔑 启示：当前 T2I 模型的瓶颈不在生成质量，而在逻辑理解。增强文本编码器的逻辑推理能力是下一步突破方向。

daVinci-LLM: 3D Spatial Reasoning with Dynamic Visual Contexts

👤 Zijian Yang et al.
📅 2026-03-28
🏛 (机构信息未获取)
LLM · 3D空间推理 · 动态上下文
daVinci-LLM 提出了一种让纯文本 LLM 进行 3D 空间推理的新范式——动态视觉上下文（DVC）。区别于以往将 3D 场景转为文字描述的方法，DVC 通过在推理过程中动态生成多视角渲染图和标注叠加图来辅助 LLM 的空间理解，实现了"按需查看"的推理模式。在 ScanQA 和 SQA3D 基准上分别取得了 57.3 和 63.8 的 CIDEr 分数，显著超越纯文本基线。

🔬 核心方法详解

① 动态视觉上下文 (DVC) 框架

核心思想是将 3D 推理分解为"观察-推理-再观察"的循环：

  1. 初始观察：给定场景点云，生成 6 个标准视角的渲染图作为初始上下文
  2. 推理触发：LLM 在推理过程中可发出 [VIEW_REQUEST(θ,φ)] 指令请求特定角度的渲染
  3. 动态渲染：系统实时渲染请求视角并叠加物体包围盒、距离标注、方位箭头
  4. 继续推理：LLM 结合新的视觉信息继续推理，最多可进行 5 轮观察-推理循环
② 空间感知标注叠加

在渲染图上叠加四类空间标注：

  • 物体标签：带颜色编码的包围盒 + 物体名称
  • 距离标注：关键物体对之间的欧氏距离 \(d = \|\mathbf{p}_i - \mathbf{p}_j\|_2\)，以米为单位
  • 方位箭头：从查询参考点到目标物体的方向向量投影
  • 空间网格：地面上的 1m×1m 参考网格，帮助建立尺度感
③ 视角选择策略学习

训练一个轻量级视角选择模块 \(f_\theta\)，输入当前推理状态 \(s_t\)（包含已观察视角、当前推理文本、问题类型），输出下一个最优视角 \((\theta^*, \phi^*)\)：

$$(\theta^*, \phi^*) = \arg\max_{(\theta,\phi)} f_\theta(s_t, \theta, \phi)$$

使用 DAgger 算法通过专家策略（穷举搜索最优视角）的监督进行训练。学到的策略在 92% 的情况下能选择信息量最大的视角。

④ 多粒度场景描述生成

对每个视角生成三种粒度的文本描述：全局（房间类型、整体布局）、区域（物体群组、功能分区）、局部（特定物体的精确位置和属性），形成层次化的文本-视觉混合上下文。LLM 可以根据问题复杂度自适应选择所需的描述粒度。

💡 深度案例分析
  1. 多步空间推理：问题"离窗户最近的椅子朝向哪里？" → LLM 先请求俯视图定位所有椅子和窗户，再请求侧视图确认椅子朝向，两步完成

  3. 视角效率：平均仅需 2.7 轮观察即可回答问题，远少于穷举 24 个视角的暴力方法，且准确率更高

GeoBlock: Efficient LLM Inference via Geometric Block Attention

👤 Wei Chen, Jiahao Zhang et al.
📅 2026-03-28
🏛 Xi'an Jiaotong University
LLM · 高效推理 · 注意力机制
GeoBlock 提出了一种基于几何分块的注意力压缩方法，通过将长序列的 KV-cache 组织为自适应几何块结构，在保持生成质量的同时实现了 3.2-4.5× 推理加速。核心创新在于利用注意力分数的空间局部性——观察到注意力热图呈现出可预测的几何模式（对角带、列聚类、稀疏全局峰），并据此设计了分块选择与融合策略。

🔬 核心方法详解

① 注意力几何模式发现

通过分析 Llama-3-70B 在多种任务上的注意力热图，归纳出三种几何原语：

  • 对角带 (Diagonal Band)：相邻 token 间的局部注意力，宽度通常为 64-256
  • 列聚类 (Column Cluster)：少数"锚点"token 吸引全局注意力（如 BOS、分隔符、关键实词）
  • 稀疏全局峰 (Sparse Global Peak)：远距离的零星高注意力分数，通常与语义关联相关

这三种模式共同解释了 95%+ 的注意力权重分布。

② 自适应几何分块算法

将长度为 \(L\) 的 KV-cache 分为大小不等的几何块 \(\{B_1, B_2, ..., B_m\}\)，块大小由局部注意力密度决定：

$$|B_i| = \min\left(\beta \cdot \frac{1}{\bar{a}_i + \epsilon}, B_{\max}\right)$$

其中 \(\bar{a}_i\) 为第 \(i\) 段的平均注意力密度。高密度区域（对角带附近）分块更细以保留精度，低密度区域合并为大块以节省计算。

③ 块内融合与跨块索引

对每个几何块，计算代表性 Key-Value 向量：

$$\mathbf{k}_B = \frac{\sum_{j \in B} a_j \mathbf{k}_j}{\sum_{j \in B} a_j}, \quad \mathbf{v}_B = \frac{\sum_{j \in B} a_j \mathbf{v}_j}{\sum_{j \in B} a_j}$$

建立跨块索引结构（类似 B-tree），支持 \(O(\log m)\) 的快速块定位。对"列聚类"类型的锚点 token 单独保留完整 KV（不参与融合），确保关键信息不丢失。

④ 推理时动态重分块

随着生成持续，新 token 的注意力模式可能改变最优分块策略。GeoBlock 每生成 128 个 token 触发一次轻量级重分块：仅更新受影响的块（增量更新），开销控制在 <1% 的总推理时间。通过维护块级注意力直方图，可以 O(1) 判断是否需要分裂或合并操作。

IF4: Adaptive Block-Scaled Data Types for Inference-Friendly KV-Cache Compression

👤 Sifat Ut Taki, Bhargav Bhusavalli, Xiulong Yang et al.
📅 2026-03-28
🏛 MIT, NVIDIA
LLM · 量化 · KV-Cache 压缩
IF4 提出了一种自适应块级缩放数据类型，专门为 LLM 推理阶段的 KV-Cache 压缩设计。核心创新在于引入可学习的非均匀量化网格，针对 Key 和 Value 张量的不同统计特性分别优化量化策略：Key 使用对称量化（分布近似高斯），Value 使用非对称量化（分布右偏）。在 4-bit 量化下，Llama-3-70B 的困惑度仅增加 0.12，同时 KV-Cache 显存减少 75%。

🔬 核心方法详解

① K/V 分离量化策略

通过统计分析 Llama-3 系列模型各层 KV 张量的分布特征：

  • Key 张量：呈近似对称分布，峰度 \(\kappa_K \approx 3.2\)（接近高斯），使用对称量化 \(q_K = \text{round}(x / s_K)\)，其中 \(s_K = \max(|x|)/2^{b-1}\)
  • Value 张量：呈明显右偏分布（偏度 \(\gamma_V \approx 0.8\)），使用非对称量化 \(q_V = \text{round}((x - z_V) / s_V)\)，额外维护零点 \(z_V\)

这种分离策略在相同 bit 数下比统一量化方案多保留 15% 的信息。

② 自适应块大小选择

不同注意力头和层对量化敏感度不同。IF4 使用校准数据集（128 条 WikiText-2 样本）估计每个块 \(B_{h,l}\)（头 \(h\)，层 \(l\)）的最优块大小 \(g^*_{h,l}\)：

$$g^*_{h,l} = \arg\min_{g \in \{32, 64, 128\}} \mathbb{E}\left[\|Q(x; g) - x\|_2^2\right] + \lambda \cdot \frac{c(g)}{c(32)}$$

其中 \(Q(x; g)\) 为块大小 \(g\) 的量化操作，\(c(g)\) 为存储开销，\(\lambda\) 控制精度-压缩权衡。敏感层（通常为前 4 层和最后 2 层）使用小块（32），中间层使用大块（128）。

③ 非均匀量化网格学习

在标准均匀量化网格之外，IF4 学习每层的最优量化点分布。将 4-bit 的 16 个量化点 \(\{q_0, ..., q_{15}\}\) 作为可学习参数：

$$\mathcal{L}_{\text{grid}} = \sum_{l=1}^{L} \mathbb{E}_{x \sim \mathcal{D}_l}\left[\|x - \text{Dequant}(\text{Quant}(x; \mathbf{q}_l); \mathbf{q}_l)\|_2^2\right]$$

使用直通估计器（STE）进行梯度传播，在校准集上仅需 5 分钟即可收敛。学到的网格在 Value 的高密度区域放置更密集的量化点。

④ 硬件友好的 INT4 解码内核

所有量化/反量化操作均可用 INT4 张量核心实现。在 NVIDIA A100/H100 上利用 \(\text{dp4a}\)（4路 INT4 点积累加）指令，单次 GEMV 操作同时完成反量化和矩阵-向量乘：

$$\text{Attention}(Q, K_q, V_q) = \text{softmax}\left(\frac{Q \cdot \text{Dequant}(K_q)^T}{\sqrt{d}}\right) \cdot \text{Dequant}(V_q)$$

通过算子融合将 dequant 嵌入注意力计算，避免额外的内存读写。端到端推理吞吐量提升 2.1×。

Model Capability Dominates LLM Benchmark Performance

👤 Yuhang Lai
📅 2026-03-28
🏛 Independent Researcher
LLM · 评估 · 基准分析
本文通过大规模实证分析（40+ 模型 × 20+ 基准 × 5 种提示策略）揭示了一个重要发现：LLM 基准测试的性能方差中，模型能力解释了 87-94% 的方差，而提示工程仅贡献 2-6%，评估格式贡献 1-4%。这挑战了"prompt sensitivity"的主流叙事，表明基准排名主要反映了真实的能力差异。作者进一步通过信息论分析表明，当模型能力差距 > 3 ELO 分时，任何提示变体都无法逆转排名。

🔬 核心方法详解

① 多因子方差分解框架

将基准分数 \(Y_{i,j,k,l}\) 建模为四因子 ANOVA：

$$Y_{i,j,k,l} = \mu + \alpha_i^{\text{model}} + \beta_j^{\text{prompt}} + \gamma_k^{\text{format}} + \delta_l^{\text{bench}} + (\alpha\beta)_{ij} + \epsilon_{ijkl}$$

使用 Type III 平方和分解，计算偏 \(\eta^2\) 衡量各因子的效应量。结果：\(\eta^2_{\text{model}} = 0.91\)，\(\eta^2_{\text{prompt}} = 0.03\)，\(\eta^2_{\text{format}} = 0.02\)，交互项 \(\eta^2_{\text{model×prompt}} = 0.01\)。

② 排名稳定性的信息论证明

定义排名逆转概率 \(P_{\text{inv}}(i,j)\) 为通过改变提示使模型 \(i\) 超过模型 \(j\) 的概率。通过贝叶斯推断估计：

$$P_{\text{inv}}(i,j) = \Pr\left[\exists (p_i, p_j): Y_{i,p_i} > Y_{j,p_j} \,|\, \bar{Y}_j > \bar{Y}_i\right]$$

实验表明，当平均分数差 \(|\bar{Y}_i - \bar{Y}_j| > 2\sigma_{\text{prompt}}\) 时（约 3 ELO 分），\(P_{\text{inv}} < 0.01\)。即：提示敏感性虽然存在，但几乎不影响有意义的排名差异。

③ 大规模实验设计

系统化覆盖：

  • 40+ 模型：从 1B 到 405B 参数，包含 Llama、Qwen、Mistral、DeepSeek、Gemma 家族
  • 20+ 基准：MMLU、GSM8K、HumanEval、ARC、HellaSwag、BBH 等
  • 5 种提示：zero-shot、few-shot(3/5/8)、chain-of-thought
  • 3 种格式：multiple-choice、open-ended、scoring

总计 >12,000 个 (模型, 基准, 提示, 格式) 组合的评估结果。

④ 能力维度的因子分析

对 40 个模型在 20 个基准上的分数矩阵进行因子分析，发现仅需 3 个潜在因子即可解释 89% 方差：

  • F1 — 通用推理（解释 52%）：与 MMLU、ARC、BBH 高度相关
  • F2 — 代码/数学（解释 24%）：与 HumanEval、GSM8K、MATH 高度相关

SARL: Sample-Efficient Adversarial Reinforcement Learning for Large Language Models

👤 Arindam Jati, Soumya Suvra Ghosal et al.
📅 2026-03-28
🏛 Purdue University, Texas Instruments
RL · 对抗训练 · 安全对齐
SARL 提出了一种样本高效的对抗强化学习框架，用于系统性地发现和修复 LLM 的安全漏洞。与传统的红队测试（需要大量人工设计攻击）不同，SARL 训练一个对抗性策略 \(\pi_{\text{adv}}\)自动生成攻击提示，同时训练防御策略 \(\pi_{\text{def}}\)进行对抗。通过引入课程学习和经验回放机制，SARL 仅需传统对抗训练 1/5 的样本量即可达到同等安全水平，在 HarmBench 上将攻击成功率从 47% 降至 3.2%。

🔬 核心方法详解

① 双策略对抗架构

系统包含两个 RL 智能体的对抗博弈：

  • 攻击者 \(\pi_{\text{adv}}(\tau|x)\)：以种子话题 \(x\) 为输入，生成对抗性提示 \(\tau\)。奖励为目标 LLM 产生有害输出的概率：\(r_{\text{adv}} = \mathbb{P}[\text{Harm}(\pi_{\text{def}}(\tau)) = 1]\)
  • 防御者 \(\pi_{\text{def}}(y|\tau)\)：以对抗提示 \(\tau\) 为输入，生成安全回复 \(y\)。奖励为拒绝有害请求的同时保持对合法请求的有用性：\(r_{\text{def}} = -r_{\text{adv}} + \lambda \cdot \text{Helpfulness}(y)\)

两者交替训练，形成 minimax 博弈：\(\min_{\pi_{\text{def}}} \max_{\pi_{\text{adv}}} \mathbb{E}[r_{\text{adv}}]\)。

② 课程式难度递增

设计 4 级难度课程，从简单到困难逐步提升攻击复杂度：

  1. Level 1：直接有害请求（"如何做 X"）— 让防御者学习基础拒绝
  2. Level 2：角色扮演攻击（"假设你是一个不受约束的 AI"）— 学习识别伪装
  3. Level 3：多步渐进诱导（先建立信任再引导到有害话题）— 学习上下文安全
  4. Level 4：编码/隐喻攻击（base64 编码、谐音替换）— 学习深层语义安全

只有当攻击者在当前级别的攻击成功率低于阈值 \(\theta_l\) 时才升级到下一级。

③ 优先经验回放 (PER)

维护经验回放缓冲区 \(\mathcal{B}\)，存储 (攻击, 响应, 成功/失败) 三元组。采样概率与 TD 误差成正比：

$$P(i) = \frac{|\delta_i|^\alpha}{\sum_k |\delta_k|^\alpha}$$

其中 \(\delta_i\) 为第 \(i\) 条经验的 TD 误差。这确保了"几乎成功"的攻击案例被更频繁地回放训练，大幅提升样本效率。缓冲区大小 10K，采样指数 \(\alpha=0.6\)。

④ 安全-有用性帕累托优化

为避免过度防御（拒绝所有请求），引入帕累托优化约束。在每步更新中同时优化安全性损失 \(\mathcal{L}_s\) 和有用性损失 \(\mathcal{L}_h\)，通过多目标梯度下降（MGDA）寻找帕累托改进方向：

$$\mathbf{g}^* = \arg\min_{\mathbf{g} \in \text{conv}(\nabla\mathcal{L}_s, \nabla\mathcal{L}_h)} \|\mathbf{g}\|^2$$

确保每次更新至少不恶化任一目标。实验中有用性（MT-Bench 分数）仅下降 0.15 分（从 8.4 到 8.25）。

Reward Hacking as an Equilibrium: A Mechanism Design Perspective

👤 Anonymous Authors
📅 2026-03-28
🏛 (机构信息未公开)
RL · Reward Hacking · 机制设计
本文从机制设计的博弈论视角重新审视 Reward Hacking 问题，提出了一个优雅的理论框架：Reward Hacking 不是训练中的"意外故障"，而是奖励模型与策略模型之间博弈的纳什均衡。论文证明了在标准 RLHF 设置下，当 Reward Model 的容量有限时，存在一个"hack 均衡"使得策略通过利用 RM 的盲区获得高奖励。基于此，提出了 VCG-RLHF 机制，通过改变奖励结构消除 hack 均衡，理论上保证激励相容。

🔬 核心方法详解

① 奖励博弈的形式化

将 RLHF 建模为双玩家博弈 \(\Gamma = \langle \pi, R, \mathcal{A}_\pi, \mathcal{A}_R, u_\pi, u_R \rangle\)：

  • 策略玩家 \(\pi\)：选择生成策略以最大化 RM 给出的奖励 \(u_\pi = \mathbb{E}[R(y|x)]\)
  • 奖励玩家 \(R\)：选择奖励函数以最大化与人类偏好的一致性 \(u_R = \text{Corr}(R, R^*)\)

关键定理：当 RM 的假设空间 \(\mathcal{F}\) 的 VC 维度 \(d_{\text{VC}}(\mathcal{F}) < \infty\) 时，必然存在一个纳什均衡 \((\pi^*, R^*)\) 满足 \(R^*(\pi^*(x)) > R^*(y^*)\)——即策略生成的输出得到的 RM 分数高于真正的最优回答。

② Hack 均衡的存在性证明

证明核心思路：利用 Sauer-Shelah 引理，有限 VC 维的 RM 必然存在"盲区"——一组输出 \(\mathcal{S} \subseteq \mathcal{Y}\) 使得 RM 无法区分其质量差异。策略可以在盲区内找到高 RM 分数但低真实质量的输出：

$$\exists y_h \in \mathcal{S}: R(y_h) \geq R(y^*) \text{ 且 } R^*(y_h) \ll R^*(y^*)$$

直觉：RM 像一个"有限分辨率的相机"，策略学会了在像素边界上放置伪装物体。

③ VCG-RLHF 机制设计

借鉴 Vickrey-Clarke-Groves (VCG) 拍卖机制的核心思想，设计新的奖励结构。不直接使用 RM 分数作为奖励，而是计算"社会福利贡献"：

$$r_{\text{VCG}}(\pi, y) = \underbrace{V(\pi, y)}_{\text{包含}\pi\text{的总效用}} - \underbrace{V(\pi_{-}, y)}_{\text{移除}\pi\text{后的总效用}}$$

其中 \(V\) 包含多个独立 RM 的评分。关键性质：VCG 奖励使得真实报告（生成高质量输出）成为占优策略，即 hack 不再是最优选择。

④ 多 RM 集成与可操纵性界

使用 \(K\) 个独立训练的 RM（不同架构/数据）组成评审团。证明可操纵性界：

$$\Pr[\text{hack成功}] \leq \prod_{k=1}^{K} \Pr[\text{hack } R_k] \leq \left(\frac{d_{\text{VC}}}{N}\right)^K$$

当 \(K=5\) 且每个 RM 在 10K 样本上训练时，hack 成功概率降至 \(< 10^{-6}\)。实际实现中使用 5 个不同大小（1B-7B）的 RM，投票阈值为 3/5。

📅 2026-03-28
🏛 NTU, CMU, Zhejiang University, Sun Yat-sen University, NUS, Fudan, CityU HK, Zhejiang Lab

CoT2-Meta 提出了从"链式思维"到"元推理"的升级路径——不仅让 LLM 展示推理步骤，更让它反思推理策略本身。核心方法是在标准 CoT 之上增加一个元推理层，该层通过强化学习训练 LLM 进行"思维策略选择"（选择何种推理方式）、"置信度监控"（监控推理过程的可靠性）和"策略切换"（在推理受阻时切换方法）。在数学、逻辑和代码推理任务上，CoT2-Meta 比标准 CoT 平均提升 8.3%，在困难样本上提升高达 15.7%。

🔬 核心方法详解

① 元推理三层架构

在基础 LLM 之上构建三层元认知系统：

  • 层 0 — 基础推理：标准 CoT，逐步生成推理链
  • 层 1 — 策略选择器：在推理开始前，分析问题特征并选择最优推理策略（正推 / 反推 / 分治 / 类比）
  • 层 2 — 过程监控器：在推理每一步后评估置信度，决定继续 / 回溯 / 切换策略

层 1 和层 2 共享底层 LLM 参数，通过不同的 LoRA 适配器实现功能分离。

② 策略选择的 RL 训练

将策略选择建模为上下文赌博机（Contextual Bandit）问题。状态 \(s = \text{Encode}(x)\)（问题编码），动作 \(a \in \{\text{forward}, \text{backward}, \text{divide}, \text{analogy}\}\)，奖励为推理结果的正确性：

$$r(s, a) = \mathbb{1}[\text{CoT}_a(x) = y^*] + \alpha \cdot \frac{1}{|c_a|}$$

第二项鼓励选择更简洁的推理路径（\(|c_a|\) 为推理步骤数）。使用 REINFORCE with baseline 训练：

$$\nabla J = \mathbb{E}\left[(r - b) \nabla \log \pi_\theta(a|s)\right]$$

③ 置信度监控与策略切换

在每一推理步 \(t\) 后，监控器计算置信度分数：

$$\text{conf}_t = \sigma\left(\mathbf{w}^T [\mathbf{h}_t; \mathbf{h}_{t-1}; \mathbf{h}_0]\right)$$

其中 \(\mathbf{h}_t\) 为第 \(t\) 步的隐藏状态。设计两个阈值：

  • \(\text{conf}_t < \theta_{\text{low}} = 0.3\)：触发策略切换——放弃当前推理链，由层 1 重新选择策略
  • \(\theta_{\text{low}} \leq \text{conf}_t < \theta_{\text{mid}} = 0.6\)：触发回溯——退回上一步重新推理
  • \(\text{conf}_t \geq \theta_{\text{mid}}\)：继续推理

最多允许 2 次策略切换，防止无限循环。

④ 多策略蒸馏与自举训练

训练数据生成通过自举（bootstrap）流程：

  1. 对每个训练问题用所有 4 种策略生成推理链
  2. 标注每种策略的成功/失败和推理长度
  3. 为策略选择器构建偏好数据：(问题, 最优策略, 次优策略)
  4. 为监控器构建置信度标签：在失败推理链中标注"应该切换"的步骤

使用 DPO 训练策略选择器，使用 BCE loss 训练监控器。每训练 2 轮后重新生成数据（因为模型策略已更新），总共进行 3 次自举迭代。

② 能力感知路由

设计能力匹配分数来分配子任务：

$$\text{Match}(R_k, T_j) = \sum_{c \in \text{caps}(T_j)} w_c \cdot \mathbb{1}[c \in \text{caps}(R_k)] \cdot \text{prof}(R_k, c)$$

  • 并行模式：独立子任务同时执行，共享黑板 (Blackboard) 通信
  • 流水线模式：有依赖关系的子任务按序执行，上游输出作为下游输入
  • 辩论模式：关键决策点触发多 Agent 辩论，每个 Agent 提出方案 + 论据，Orchestrator 裁决

模式切换由 Orchestrator 基于当前进度和阻塞情况自动决策。

① 角色原型空间构建

为每个角色学习一个与外观无关的原型向量 \(\mathbf{z}_c\)：

$$\mathbf{z}_c = \frac{1}{|\mathcal{I}_c|}\sum_{i \in \mathcal{I}_c} f_\theta(\mathbf{x}_i) - \text{Proj}_{\mathcal{A}}(f_\theta(\mathbf{x}_i))$$

其中 \(f_\theta\) 为视觉编码器，\(\text{Proj}_{\mathcal{A}}\) 将特征投影到外观子空间 \(\mathcal{A}\)（由服装、背景、光照等变量张成）并减去，保留纯身份信息。外观子空间通过"同一角色不同外观"和"不同角色相同外观"的对比数据学习。

② 多粒度对比学习

设计三层对比损失实现稳健的角色表示：

  • 实例级：同一角色的不同图片为正对，不同角色为负对：\(\mathcal{L}_{\text{inst}} = -\log\frac{\exp(\text{sim}(\mathbf{z}_i, \mathbf{z}_j^+)/\tau)}{\sum_k \exp(\text{sim}(\mathbf{z}_i, \mathbf{z}_k)/\tau)}\)
  • 原型级：实例特征与角色原型的距离最小化：\(\mathcal{L}_{\text{proto}} = \|\mathbf{z}_i - \mathbf{z}_{c(i)}\|_2^2\)
  • 跨域级：不同画风/域中的同一角色对齐：\(\mathcal{L}_{\text{cross}} = \|\mathbf{z}_{c}^{d_1} - \mathbf{z}_{c}^{d_2}\|_2^2\)

总损失 \(\mathcal{L} = \mathcal{L}_{\text{inst}} + \lambda_1 \mathcal{L}_{\text{proto}} + \lambda_2 \mathcal{L}_{\text{cross}}\)。

  3. 姿态变换：通过 ControlNet 生成同一角色的不同姿态
