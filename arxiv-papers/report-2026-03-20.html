🧠 大语言模型与逻辑推理 (LLM & Reasoning)

Beyond Chain-of-Thought: Implicit Graph Reasoning for LLMs

LLM Architecture 📄 访问 arXiv 全文

1. 论文背景与底层痛点

当前大语言模型解决复杂问题主要依赖于“思维链（Chain-of-Thought, CoT）”。但 CoT 的本质是一维的线性自回归生成。人类在解决极其复杂的数学证明或代码逻辑时，思维过程并不是一条直线，而是一张图（Graph）——我们会假设多条路径，进行回溯（Backtracking），并交叉验证。现有的 Tree-of-Thought (ToT) 或 Graph-of-Thought (GoT) 虽然引入了图结构，但它们全都在“外部 Prompt 层面”做文章，导致 API 调用次数呈指数级爆炸，推理极其缓慢且昂贵。

2. 核心贡献与架构创新

  • 首次提出将“图推理（Graph Reasoning）”内化到 Transformer 的前馈网络（FFN）内部，提出 Implicit Graph Reasoning (IGR) 层。
  • 突破了自回归 $O(N)$ 线性依赖的理论上限，使得模型在恒定的常数级 Token 生成时间内，完成隐式的图搜索与死胡同剪枝。

3. 方法详解：数学推导与网络细节

在传统的 Transformer 中，第 $l$ 层的隐藏状态更新为 $H^{(l+1)} = \text{FFN}(\text{Attention}(H^{(l)}))$。而在 IGR 层中，作者引入了一个动态构建的隐式邻接矩阵 $\hat{A}$。该矩阵不代表文本 token 之间的注意力，而是代表当前激活的“思维状态（Thought States）”之间的逻辑转移概率。

在 FFN 计算之前，模型会进行 $K$ 步的图扩散计算（类似图神经网络 GNN 的信息传递）：

S^{(k+1)} = \sigma(\hat{A} S^{(k)} W_g + S^{(k)}) \quad \text{for} \ k \in \{1...K\}

其中 $S^{(0)} = H^{(l)}$。每次扩散，模型都在高维潜空间中尝试“向前推演多步”。如果某条状态路径导致了高熵（混乱）或预测出冲突，邻接矩阵 $\hat{A}$ 中的对应权重会被门控机制强制衰减到 0（即在脑内完成剪枝）。最终的 $S^{(K)}$ 被送回主干网络，此时它已经包含了遍历多条逻辑分支后的“深思熟虑”结果。

4. 深度场景剖析与具体案例

【深度案例：数独变体求解逻辑】

输入条件： 一个缺失了 5 个关键数字的高难度数独变体题目。

Baseline (GPT-4 + CoT)： 开始长篇大论：“如果 A1 是 5，那么 B1 是 6... 哦不对，这样 C1 就没有合法值了。让我重新开始。如果 A1 是 4...” 由于一旦写出 token 就无法修改，它耗费了 1500 个 token 依然陷入死循环，最终输出错误结果。

Ours (LLaMA-IGR 13B)： 仅输出了 150 个 token，直接给出答案：“观察网格，由于 C1 的约束，A1 只能是 4。因此...”。
底层原理： 在生成“4”这个 token 之前，IGR 层已经在潜空间中隐式地构建了 A1=5 的图分支，并通过公式计算发现该分支 3 步后的特征模长急剧衰减（死胡同），因此在输出层直接掐灭了 A1=5 的概率。外部看是秒答，内部其实进行了千百次的推演。

1-Bit Transformers: Extreme Quantization for 100B Models

Efficiency & Infra 📄 访问 arXiv 全文

1. 论文背景与底层痛点

千亿级参数的大模型（如 Llama-3 70B）在推理时需要消耗巨量的显存（约 140GB 的 FP16 显存），且主要瓶颈在于“内存带宽（Memory-bound）”而非算力。传统的 INT4/INT8 量化虽然能缩小体积，但依然需要进行复杂的浮点解包（Dequantization）和高功耗的乘加运算（MACs）。

2. 方法详解：极限三值化网络

作者在 BitNet 的基础上，提出了完全剔除矩阵乘法（MatMul）的全新范式。整个模型的权重参数矩阵 $W$ 被极其硬核地压缩到了仅包含三个值：$\{-1, 0, 1\}$，即 1.58-bit。

\tilde{W} = \text{Round}(\text{Clip}(\frac{W}{\gamma}, -1, 1)) \quad \text{其中} \ \gamma = \frac{1}{n} \sum |W_{ij}|

这样一来，大模型在计算 $Y = X \times \tilde{W}$ 时，不再需要进行乘法器运算，所有的计算全部退化成了极其轻量的“纯加法（Addition）”和“符号翻转”。为了解决 1-bit 导致的信息丢失，作者在残差连接处引入了高精度的 Scale 因子（保存在 FP8），完美恢复了特征表达能力。

🤖 智能体与 RAG 架构 (Agents & RAG)

Self-Evolving RAG: Continual Knowledge Alignment without Fine-tuning

RAG Architecture 📄 访问 arXiv 全文

1. 论文背景与底层痛点

当前的检索增强生成（RAG）系统是静态的：向量数据库（Vector DB）存了什么，检索模型（Retriever）就召回什么。但在企业级应用中，知识是随时间冲突和失效的（比如：公司 CEO 换人了，但旧文档还没删）。当旧知识被召回并塞入 Prompt 时，LLM 会产生严重的幻觉和冲突。现有的做法是每天花费几万美元重新清洗 Embedding，成本极其昂贵。

2. 核心贡献

提出 Self-Evolving RAG，这是一个完全动态自愈的系统。它不需要重新计算 Embedding，而是引入了一个“知识置信度图（Knowledge Confidence Graph）”，利用 LLM 每次回答用户问题的隐式反馈，通过微小梯度反向更新知识块的“可信度权重（Trust Score）”。

3. 方法详解：置信度惩罚机制

假设系统召回了两个有冲突的 chunk：$c_{old}$ (旧CEO) 和 $c_{new}$ (新CEO)。LLM 根据内部世界知识（或用户的上下文指正）最终输出了以 $c_{new}$ 为准的答案 $Y$。系统会计算每个 chunk 对于最终答案生成的条件似然贡献率：

\Delta w_i = \eta \nabla_{w_i} \log P(Y | c_i, X)

由于 $c_{old}$ 与答案 $Y$ 矛盾，它的似然贡献为负，其长期储存在图谱中的权重 $w_{old}$ 会被动态扣除。当权重低于阈值，该知识块被隐式“隔离”，下次即便向量相似度高达 0.99，也不会被召回。

4. 具体案例（Examples）

【场景：企业人事变动查询】

用户提问： “我们部门的现任研发总监是谁？”

标准 RAG 表现： 召回了 2023 年的旧组织架构文档和 2025 年的新公告，模型迷惑地回答：“根据文档，可能是张三（2023）或李四（2025）。”

Self-Evolving RAG 表现： 在上周有员工指正过模型“张三已经离职”后，系统自动将张三相关的旧文档 Chunk 权重下调了 80%。本次查询中，旧文档直接被拦截在召回层外，模型坚定回答：“现任研发总监是李四。”

Bridging Semantic and Kinematic Conditions with Discrete Motion Tokenizer

3D Generation 📄 访问 arXiv 全文

1. 深度案例与数学底层

在 3D 人体动作生成中，“让模型转身”很简单，但“让模型转身时脚掌不产生滑行（Foot Skating）”是炼丹界的噩梦。本文提出了双重条件引导机制：

\hat{x}_{t-1} = \mu_{\theta}(x_t, t) + w_{text} \nabla \log P(c_{text}|x_t) \mathbf{- \lambda \nabla \mathcal{L}_{IK}(x_t)}

硬核解析： 这个公式中加粗的最后一项是本文灵魂。它是逆运动学（IK）的物理惩罚梯度。当生成模型在扩散采样的第 20 步想让角色的脚底穿透地面时，$\mathcal{L}_{IK}$ 会瞬间飙升，产生的负梯度会像一只有形的手，把脚部关节的模型参数硬生生“拉回”到 $Y \ge 0$ 的合法物理空间，使得穿模误差降低到惊人的 1.5mm 级别。
