MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 100M Tokens

Yu Chen, Runkai Chen, Sheng Yi et al. · arXiv:2603.23516 · cs.CL, cs.AI, cs.IR · Paper Link

背景与动机

长期记忆是人类智能的基石，但当前 LLM 的有效上下文窗口通常限制在 100 万 token 以内。现有扩展方案包括混合线性注意力、固定大小的 RNN 记忆状态、RAG 系统和 Agent 外部存储，但它们在上下文增长时面临精度急剧下降、推理延迟暴涨、无法动态修改记忆内容、且缺乏端到端优化能力等瓶颈。这些局限严重制约了大型语料库摘要、数字孪生、长历史 Agent 推理等场景。

核心方法详解

① 可扩展稀疏注意力 (Scalable Sparse Attention)

MSA 的核心创新在于用稀疏注意力替代全注意力。对于长度为 \(N\) 的序列，传统全注意力的复杂度为 \(O(N^2)\)。MSA 将序列切分为大小为 \(B\) 的文档块，每个块仅与固定数量 \(k\) 的最相关块进行注意力计算：

$$\text{Attn}_{\text{MSA}}(Q_i) = \text{softmax}\left(\frac{Q_i \cdot K_{\text{top-}k}^T}{\sqrt{d}}\right) V_{\text{top-}k}$$

其中 top-\(k\) 块通过高效的块级相似度门控选取。由于每个查询块仅与 \(k \cdot B\) 个 KV 对交互，总复杂度降为 \(O(N \cdot k \cdot B)\)，在 \(k, B\) 固定时实现线性扩展。

② 文档级旋转位置编码 (Document-wise RoPE)

标准 RoPE 的位置 ID 随序列位置单调递增，当序列超过训练长度时外推能力退化。MSA 采用文档级 RoPE：将每个文档块的位置 ID 重置为 0，使得位置编码仅在块内有效：

$$\text{RoPE}_{\text{doc}}(x_j) = R(\text{pos}_j \bmod B) \cdot x_j$$

其中 \(\text{pos}_j \bmod B\) 确保位置编码永远不超出训练范围。跨文档的关系依赖块间稀疏注意力的语义匹配，而非位置外推。实验表明从 16K 扩展到 100M token 性能仅下降不到 9%。

③ KV 缓存压缩与 Memory Parallel

在推理阶段，MSA 对历史 KV 缓存进行压缩。通过学习一个压缩函数 \(C(\cdot)\)，将每个文档块的 KV 对压缩为固定大小的摘要表示：

$$\hat{K}_i, \hat{V}_i = C(K_i, V_i), \quad |\hat{K}_i| \ll |K_i|$$

结合 Memory Parallel 技术，将压缩后的记忆分片到多个 GPU 上并行处理。这使得在仅 2 张 A800 GPU 上即可完成 100M token 的推理，显著降低了硬件门槛。

④ 记忆交错 (Memory Interleaving)

为支持分散记忆段之间的复杂多跳推理，MSA 引入记忆交错机制：在注意力计算中，当前查询块可以同时访问多个不相邻的记忆块，并通过加权聚合实现信息融合：

$$\text{Output}_i = \sum_{j \in \text{selected}} \alpha_{ij} \cdot V_j, \quad \alpha_{ij} = \text{softmax}(Q_i K_j^T / \sqrt{d})$$

这种设计使模型能够在不需要连续上下文的情况下，从散布在百万级 token 中的多个证据片段进行推理。

深度案例

输入：将约 200 本学术教材（合计 ~100M tokens）加载为 MSA 的长期记忆，提问："请对比第 3 章与第 47 章中关于注意力机制的不同阐述。"

对比：GPT-4-turbo（128K 窗口）无法加载该规模文本；RAG 系统需多轮检索且难以处理多跳对比。

Compression Method Matters: Benchmark-Dependent Output Dynamics in LLM Prompt Compression

Warren Johnson · arXiv:2603.23527 · cs.CL · Paper Link

背景与动机

提示压缩（Prompt Compression）通常以输入 token 减少率作为核心评估指标，但实际部署中更关键的问题是：压缩如何改变输出长度和总推理成本？先前的研究得出了相互矛盾的结论——有的报告了极端的输出膨胀效应（高达 56×），有的则认为影响较小。本文通过 5,400 次 API 调用的受控实验，揭示了这些矛盾的根本原因。

核心方法详解

① 指令保留概率 Ψ (Instruction Retention Probability)

作者形式化了一个关键概念——指令保留概率 \(\Psi\)。对于一个压缩后的提示 \(p'\)，其中包含 \(m\) 个任务关键指令段 \(\{s_1, ..., s_m\}\)：

$$\Psi(p', r) = \frac{1}{m} \sum_{i=1}^{m} \mathbb{1}[\text{retained}(s_i, p', r)]$$

其中 \(r\) 是压缩率，\(\mathbb{1}[\cdot]\) 是指示函数。\(\Psi\) 越低，意味着越多关键指令被截断，模型越可能"不知所措"并生成冗余输出。

② 基准依赖性效应分析

在压缩率 \(r=0.3\)（保留 30% 输入）时，不同模型 × 不同基准展现出截然不同的行为：

  • DeepSeek + MBPP：输出膨胀 56×，\(\Psi \approx 0.15\)（MBPP 的函数签名被截断，模型反复尝试猜测任务）
  • DeepSeek + HumanEval：膨胀仅 5×，\(\Psi \approx 0.72\)（HumanEval 的 docstring 结构使关键信息较易保留）
  • GPT-4o-mini：各基准上相对稳定（膨胀 2-8×），因为其更强的指令理解能力可部分补偿信息丢失

这证明了提示结构（而非模型身份）是输出膨胀的主要调节因素。

③ 压缩鲁棒性指数 CRI (Compression Robustness Index)

为实现跨基准的公平评估，作者提出压缩鲁棒性指数：

$$\text{CRI}(M, r) = 1 - \frac{\sigma(\{O_b(M, r)\}_{b \in \mathcal{B}})}{\mu(\{O_b(M, r)\}_{b \in \mathcal{B}})}$$

其中 \(O_b(M, r)\) 是模型 \(M\) 在基准 \(b\) 上压缩率 \(r\) 的输出膨胀倍数，\(\sigma\) 和 \(\mu\) 分别为标准差和均值。CRI 越接近 1 表示模型在不同基准下表现越一致。GPT-4o-mini 的 CRI 为 0.82，而 DeepSeek 仅为 0.34。

④ 能耗实测与 Token-Joule 偏差

通过 RunPod GPU 上的 NVML 直接功耗测量，作者发现 token 节省经常夸大实际焦耳节省。定义能效偏差：

$$\Delta_E = \frac{E_{\text{compressed}}}{E_{\text{original}}} - \frac{T_{\text{compressed}}}{T_{\text{original}}}$$

实验中 \(\Delta_E\) 平均为 +0.18，意味着声称节省 70% token 的压缩实际上仅节省约 52% 的能耗。原因在于输出膨胀导致的解码阶段能耗增加部分抵消了输入压缩的收益。

Mitigating Many-Shot Jailbreaking

Christopher M. Ackerman, Nina Panickssery · arXiv:2504.09604 · cs.LG, cs.AI, cs.CR · Paper Link

背景与动机

Many-Shot Jailbreaking (MSJ) 是一种利用现代 LLM 长上下文窗口的对抗攻击技术。攻击者在提示中插入大量"虚假助手"不当回复的示例，通过 in-context learning 的压倒性效应覆盖模型的安全训练。随着上下文窗口从 8K 扩展到 128K+，MSJ 攻击的成功率从 <5% 飙升至 >60%。现有防御集中在预训练阶段的 RLHF，但对 ICL 驱动的攻击效果有限。

核心方法详解

① 安全微调 (Safety Fine-tuning)

构造包含 MSJ 攻击模式的训练数据，让模型学习在看到大量不当示例后仍然拒绝有害请求。训练目标为条件概率最大化：

$$\mathcal{L}_{\text{safe}} = -\sum_{t} \log P_\theta(y_t^{\text{refuse}} \mid x_{\text{msj}}, y_{其中 \(x_{\text{msj}}\) 包含 50-200 个恶意 in-context 示例，\(y^{\text{refuse}}\) 是安全拒绝响应。关键是让模型在训练中"见过"这种攻击模式。

② 输入清理 (Input Sanitization)

在推理阶段对输入进行结构分析，检测并移除可疑的重复对话模式。定义重复率指标：

$$R_{\text{rep}}(x) = \frac{|\{(i,j): \text{sim}(s_i, s_j) > \tau, i \neq j\}|}{\binom{n}{2}}$$

其中 \(s_i\) 是输入中的第 \(i\) 个对话轮次，\(\tau\) 是相似度阈值。当 \(R_{\text{rep}} > 0.4\) 时触发清理，移除冗余对话轮次至安全数量（<10）。这是一种零开销的防御，不影响正常对话。

③ 组合防御策略

将安全微调和输入清理组合使用，构成两道防线：

  1. 第一道防线（输入层）：输入清理检测并削减 MSJ 攻击的 in-context 示例数量
  2. 第二道防线（模型层）：即使攻击绕过输入清理，安全微调后的模型仍能识别并拒绝

组合效果呈超线性：设微调单独的安全率提升 \(\Delta_F\)，清理单独提升 \(\Delta_S\)，组合后提升 \(\Delta_C > \Delta_F + \Delta_S\)，因为清理降低了攻击强度，使微调更容易发挥作用。

④ 良性 ICL 性能保持

防御不应影响模型的正常 in-context learning 能力。作者验证了在 few-shot 分类、翻译、代码补全等良性 ICL 任务上，防御后的模型性能损失 < 1.5%。这是因为：(1) 微调数据中包含正面 ICL 示例的对比样本；(2) 输入清理的 \(\tau\) 阈值经过调优，仅对高度重复的模式触发。

深度案例

攻击输入：在提示中插入 100 个"用户提问有害问题 → 虚假助手热心回答"的 in-context 对话轮次，然后附加真实有害请求。

🎯 LLM RL & Alignment

Improving Lean4 Autoformalization via Cycle Consistency Fine-tuning

Arsen Shebzukhov · arXiv:2603.24372 · cs.CL · Paper Link

背景与动机

自动形式化（Autoformalization）是将自然语言数学文本自动翻译为 Lean4 等形式证明语言的任务，可以大幅加速 AI 辅助的数学研究与证明验证。然而，这一任务面临的核心挑战是：(1) 高质量的 NL→Lean4 平行数据极度稀缺；(2) 传统监督微调（SFT）依赖于精确的参考翻译，但形式化往往有多种等价表达方式；(3) 课程学习的效果在该任务上存疑。

核心方法详解

① 循环一致性奖励 (Cycle Consistency Reward)

核心创新：不依赖精确的参考翻译，而是利用"循环翻译"的一致性作为奖励信号。对于自然语言输入 \(x\)，模型先将其翻译为 Lean4 代码 \(\hat{y}\)，再将 \(\hat{y}\) 反向翻译回自然语言 \(x'\)。循环一致性奖励定义为：

$$R_{\text{cycle}}(x, \hat{y}) = \cos(\text{Emb}(x), \text{Emb}(x'))$$

其中 \(\text{Emb}(\cdot)\) 是预训练句子嵌入模型的编码函数。奖励值域 \([-1, 1]\)，越接近 1 表示翻译越忠实地保留了原始数学含义。

② GRPO 强化学习训练 (Group Relative Policy Optimization)

使用 GRPO 而非 PPO 来优化策略，避免训练独立的 value function。对于每个输入 \(x\)，采样一组候选翻译 \(\{\hat{y}_1, ..., \hat{y}_G\}\)，计算各自的循环一致性奖励后进行组内归一化：

$$\hat{A}_i = \frac{R_{\text{cycle}}(x, \hat{y}_i) - \mu_G}{\sigma_G}$$

其中 \(\mu_G, \sigma_G\) 是组内奖励的均值和标准差。策略梯度更新为：

$$\nabla_\theta \mathcal{L} = \mathbb{E}\left[\sum_{i=1}^{G} \hat{A}_i \cdot \nabla_\theta \log \pi_\theta(\hat{y}_i | x)\right]$$

GRPO 的关键优势是无需 critic 网络，计算开销仅比 SFT 高约 40%。

③ LoRA 高效微调

基于 Qwen3.5-2B 使用 LoRA 进行参数高效微调，仅更新不到 1% 的参数。LoRA 矩阵分解为：

$$W' = W + \alpha \cdot B \cdot A, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times d}, r \ll d$$

其中 rank \(r = 16\)，缩放因子 \(\alpha = 32\)。在 FineLeanCorpus 数据集上训练，结合循环一致性奖励和 GRPO，使得 2B 参数的小模型也能取得显著的形式化提升。

④ 课程学习对比实验

对比了三种训练方案：(a) SFT + 课程学习（难度 1→10 递进）、(b) SFT 无课程（随机顺序）、(c) GRPO + 循环一致性奖励。结果显示课程学习在该任务上无可测量的收益，而 RL 方案在所有测试集上均显著优于 SFT。这可能因为形式化任务中"难度"不是单调的——简单的数学概念可能对应复杂的 Lean4 语法。

ITPO: Implicit Turn-Wise Policy Optimization for Proactive User-LLM Interaction

Haoyu Wang, Yuxin Chen, Liang Luo et al. · arXiv:2603.23550 · cs.LG · Paper Link · Code

背景与动机

多轮人机协作（如自适应辅导、对话推荐、专业咨询）是交互式 AI 服务的核心。但用强化学习优化这类多轮交互面临两大难题：(1) 可验证的中间奖励极度稀疏——通常只有交互结束后才有明确反馈（如学生是否学会）；(2) 用户响应高度随机，同一策略在不同用户身上效果可能截然不同。现有方法要么依赖 token 级奖励（不稳定），要么完全使用结果奖励（稀疏）。

核心方法详解

① 隐式过程奖励模型 (Implicit Process Reward Model)

ITPO 的核心是从稀疏的结果信号中推导出逐轮的过程奖励。给定一个 \(T\) 轮对话 \(\tau = (a_1, u_1, a_2, u_2, ..., a_T, u_T)\)，结果奖励为 \(R(\tau)\)。ITPO 学习一个隐式价值函数 \(V_\phi\)，使得第 \(t\) 轮的过程奖励可以由差分推导：

$$r_t = V_\phi(\tau_{\leq t}) - V_\phi(\tau_{\leq t-1})$$

其中 \(\tau_{\leq t}\) 是到第 \(t\) 轮为止的对话前缀。价值函数通过 TD 学习训练：

$$\mathcal{L}_V = \sum_{t=1}^{T} \left(V_\phi(\tau_{\leq t}) - \gamma V_\phi(\tau_{\leq t+1}) - R(\tau) \cdot \mathbb{1}[t=T]\right)^2$$
② 逐轮信号的归一化与鲁棒性

与 token 级奖励不同，逐轮信号在更粗粒度上操作，天然具有更低的方差。为进一步增强稳定性，ITPO 引入批次归一化：

$$\hat{r}_t = \frac{r_t - \mu_B}{\sigma_B + \epsilon}$$

其中 \(\mu_B, \sigma_B\) 是 mini-batch 内所有轮次奖励的统计量。这种归一化使得不同长度的对话可以在统一尺度下比较，避免长对话的过程奖励因为更多的累积步数而天然偏高。

③ 与 PPO/GRPO/RLOO 的集成

ITPO 推导出的逐轮奖励 \(\hat{r}_t\) 可以直接替换任何 RL 算法中的奖励信号。以 GRPO 为例，优势函数变为：

$$\hat{A}_t^{\text{GRPO}} = \frac{\hat{r}_t - \mu_G}{\sigma_G}$$

作者证明这种集成在 PPO（使用 \(\hat{r}_t\) 作为 per-step reward）、GRPO（组内归一化）和 RLOO（leave-one-out baseline）三种算法上都一致优于使用结果奖励的对应版本。关键原因：更密集的奖励信号加速了信用分配。

④ 语义对齐验证

为验证推导出的逐轮偏好是否合理，作者设计了人工标注实验：让 3 名标注员对 200 段多轮对话的每一轮标注质量分（1-5），然后计算 ITPO 推导的 \(\hat{r}_t\) 与人类标注之间的 Spearman 相关系数。结果显示 \(\rho = 0.71\)（p < 0.001），证明隐式推导的奖励与人类判断在语义上高度一致。

场景：AI 辅导员教学生解一元二次方程 \(x^2 - 5x + 6 = 0\)。

PrefPO: Pairwise Preference Prompt Optimization

Rahul Singhal, Pradyumna Tambwekar, Karime Maamari · arXiv:2603.19311 · cs.CL · Paper Link · Code

背景与动机

提示工程（Prompt Engineering）虽然有效但劳动密集，催生了自动化提示优化方法。然而现有方法（如 TextGrad、MIPRO、GEPA）存在三大问题：(1) 需要标注数据集（往往不可得）；(2) 生成的提示冗长且重复——平均比原始提示长 14.7×或含 34% 重复内容；(3) 存在"提示 hacking"现象，即优化器博弈评估标准而非真正提升质量。

核心方法详解

① 基于偏好的提示评估 (Preference-based Evaluation)

PrefPO 不使用绝对分数评估提示质量，而是使用成对偏好比较。给定两个候选提示 \(p_A, p_B\) 及其对应输出 \(o_A, o_B\)，LLM 判别器按照自然语言标准 \(c\) 表达偏好：

$$\text{Pref}(p_A, p_B | c) = \text{LLM}_{\text{judge}}(o_A, o_B, c) \in \{A, B, \text{tie}\}$$

这种设计受 RLHF 中 Bradley-Terry 偏好模型的启发：人类（和 LLM）更擅长做相对比较而非绝对评分。

② 迭代优化循环

PrefPO 在每次迭代中：(1) 生成多个候选提示变体；(2) 执行任务并收集输出；(3) 通过成对比较选出最佳变体；(4) 将偏好反馈给 LLM 优化器生成下一轮候选。形式化为：

$$p^{(t+1)} = \text{LLM}_{\text{opt}}(p^{(t)}, \text{feedback}^{(t)}, c)$$

其中反馈包含：赢的示例为什么更好、输的示例的具体弱点。这种"元反馈"使优化器不仅知道方向，还理解原因。

③ 提示卫生控制 (Prompt Hygiene)

为解决提示膨胀和重复问题，PrefPO 在优化过程中引入长度和多样性约束：

$$\mathcal{C}_{\text{hygiene}} = \lambda_L \cdot \max(0, |p| - L_{\max}) + \lambda_D \cdot \text{Dup}(p)$$

其中 \(L_{\max}\) 是最大允许长度，\(\text{Dup}(p)\) 是重复段检测率。实验表明 PrefPO 将膨胀和重复问题减少了 3-5×。

④ 无标签优化能力

PrefPO 的一大优势是不依赖标注数据。在无标签设置下，LLM 判别器完全基于自然语言标准 \(c\) 进行成对比较。实验显示无标签的 PrefPO 在 9 个 BBH 任务中的 6 个上紧追有标签版本（差距 < 2%），这意味着只需一个初始提示和一段文字描述期望即可开始优化。

Hubert M. Pysklo, Artem Zhuravel, Patrick D. Watson · arXiv:2602.11224 · cs.SE · Paper Link · Submitted to KDD 2026

背景与动机

评估 LLM Agent 在真实企业 API 任务上的表现面临根本性权衡：沙盒化方法控制了环境变量但牺牲了真实性；而使用真实服务的方法具有生态效度但难以标准化。此外，现有基准通常通过 API 调用日志或参数匹配来评估——但这些方法过于脆弱，无法捕捉"不同路径达到相同目标"的等价行为。Agent-Diff 试图同时解决这两个问题。

核心方法详解

① 状态差异合约 (State-Diff Contracts)

Agent-Diff 的核心创新是将任务成功定义为"是否实现了预期的环境状态变化"，而非"是否调用了正确的 API"。形式化定义：

$$\text{Pass}(\tau) = \mathbb{1}[\Delta S(\tau) \supseteq \Delta S^*]$$

其中 \(\tau\) 是 Agent 的执行轨迹，\(\Delta S(\tau) = S_{\text{after}}(\tau) - S_{\text{before}}\) 是实际状态变化，\(\Delta S^*\) 是预期状态变化。关键在于不规定过程——只要最终状态满足预期，Agent 可以使用任意 API 组合。

② 容器化 API 沙盒

为每个评估实例创建企业 API 的容器化副本。沙盒架构包含三层：

  • API 层：真实企业 API 的 Docker 镜像，保持完整的接口语义
  • 状态层：每次测试前重置为预定义的初始状态 \(S_0\)，确保可复现性
  • 监控层：记录所有状态变化 \(\Delta S\)，用于自动化评分

这种设计使得 Agent 与真实 API 交互（保持生态效度），同时环境完全受控（保持可复现性）。

③ 代码执行评估范式

与传统的"选择正确工具"评估不同，Agent-Diff 要求 Agent 通过编写和执行代码来完成 API 任务：

$$a_t = \text{LLM}(\text{task}, \text{docs}, \text{history}_{每一步 Agent 生成 Python 代码，代码在沙盒中执行，执行结果反馈给 Agent。这种评估更接近真实开发场景，要求 Agent 理解 API 文档、处理错误响应、管理状态变更。

④ API 文档消融分析

通过消融实验量化 API 文档对 Agent 性能的贡献。设 \(P_{\text{full}}\) 为完整文档下的通过率，\(P_{\text{none}}\) 为无文档下的通过率，文档贡献度：

$$\text{DocGain}(M) = \frac{P_{\text{full}}(M) - P_{\text{none}}(M)}{P_{\text{full}}(M)}$$

实验发现不同模型的文档依赖程度差异显著：Claude-3.5 的 DocGain = 0.31（对文档依赖较低），而 GPT-4o 的 DocGain = 0.58（高度依赖文档）。

背景与动机

LLM Agent 已展现出在复杂任务中推理和规划的能力，但一个关键问题尚未解答：它们能否在不确定环境下进行有效的长期资源分配？与短期反应式决策不同，CFO 级别的资源分配需要：(1) 在长时间跨度内承诺稀缺资源；(2) 平衡相互竞争的目标；(3) 为未知的未来需求保持灵活性。现有 Agent 基准（如 WebArena、SWE-bench）都是短期任务导向的，无法评估这种长期战略能力。

核心方法详解

① EnterpriseArena 模拟器

构建了一个跨越 132 个月（11 年）的企业运营模拟环境，每个月为一个决策步骤。模拟器整合四类信息源：

  • 财务数据：真实上市公司的匿名化资产负债表、损益表、现金流量表
  • 商业文档：季度报告、市场分析、竞争情报
  • 宏观信号：利率变化、GDP 增长率、行业景气指数
  • 操作规则：经专家验证的财务约束（如最低流动性比率、最大杠杆率）

每月 Agent 需决定预算分配 \(\mathbf{b}_t = (b_t^1, ..., b_t^K)\)，满足 \(\sum_k b_t^k \leq B_t\)，其中 \(B_t\) 是当月可用资金。

② 部分可观测性与信息成本

环境是部分可观测的：Agent 无法直接看到完整状态，必须使用"预算工具"来获取信息。每次工具调用消耗资金：

$$B_t' = B_t - c_{\text{tool}} \cdot n_{\text{calls}}$$

这引入了一个关键博弈：获取更多信息可以做出更好的决策，但信息获取本身消耗资源。最优策略需要平衡信息价值（Value of Information）和资源保留：

$$\text{VOI}(\text{query}) = \mathbb{E}[\text{Utility}_{\text{informed}}] - \mathbb{E}[\text{Utility}_{\text{uninformed}}] - c_{\text{tool}}$$
③ 多维评估体系

评估指标包含三个维度：

  1. 生存率：是否在 132 个月周期内避免了破产（资产 < 0 或流动性危机）
  2. 累积回报：\(\sum_{t=1}^{132} r_t\)，其中 \(r_t\) 包含收入增长、成本控制、投资回报
  3. 风险调整回报：Sharpe 比率 \(= (\mu_r - r_f) / \sigma_r\)，惩罚高波动性决策
④ 规模效应失效

一个反直觉的发现：更大的模型并不一定更好。作者分析原因为：大模型在单步决策质量上确实更优，但在 132 步的长序列中更容易陷入过度分析（information seeking）陷阱：

$$\text{InfoCost}(\text{large}) = 1.7 \times \text{InfoCost}(\text{small})$$

大模型平均每月使用的工具调用次数是小模型的 1.7×，过度消耗资源导致在后期出现现金流危机。

核心方法详解

① POMDP 建模优先级分配

RL-RH-PP 将动态优先级分配建模为部分可观测马尔可夫决策过程（POMDP）。在每个时间步 \(t\)，观测为：

$$o_t = (\text{pos}_t^{1:N}, \text{goal}_t^{1:N}, \text{congestion}_t, \text{map})$$

动作为智能体的优先级排序 \(\sigma_t \in \mathcal{S}_N\)（\(N\) 个智能体的一个排列），优先级高的智能体先规划路径，后续智能体需避让。奖励函数基于系统吞吐量：

$$r_t = \sum_{i=1}^{N} \mathbb{1}[\text{agent}_i \text{ reached goal at step } t]$$
② 注意力自回归优先级解码

使用基于注意力机制的编码器-解码器网络自回归地生成优先级排序。编码器对所有智能体的观测进行编码：

$$h_i = \text{Encoder}(o_t^i), \quad H = [h_1, ..., h_N]$$

解码器在每一步选择一个尚未被选中的智能体作为下一个优先级：

$$P(\sigma_t(k) = i | \sigma_t(1:k-1)) = \text{softmax}\left(\frac{q_k \cdot H_{\text{remaining}}^T}{\sqrt{d}}\right)_i$$

这种自回归设计允许后续选择依赖前面的选择，捕捉智能体间的依赖关系。

③ 滚动时域规划 (Rolling Horizon)

由于终身 MAPF 是无限期的，RL-RH-PP 采用滚动时域：在每个决策点只规划未来 \(W\) 步（时域窗口），然后在执行一定步数后重新规划。

$$\pi^* = \arg\max_\pi \sum_{t=0}^{W} \gamma^t \cdot r_t$$

窗口大小 \(W\) 是关键超参数：\(W\) 过小导致短视行为（路径频繁变更），\(W\) 过大则计算开销过大。实验确定 \(W = 10\) 为最优平衡点。

④ 拥堵感知优先级策略

解释性分析揭示了 RL-RH-PP 学到的策略模式：模型倾向于优先处理位于拥堵区域的智能体（给予高优先级），并将其重定向到低拥堵路径：

$$\text{Priority}(i) \propto \text{Congestion}(\text{pos}_i) + \alpha \cdot \text{Urgency}(i)$$

其中 \(\text{Congestion}(\text{pos}_i)\) 是位置 \(i\) 的局部智能体密度，\(\text{Urgency}(i)\) 是智能体到目标的剩余距离。这种"让拥堵者先走"的策略有效缓解了瓶颈。

S4CMDR: A Metadata Repository for Electronic Health Records

Jiawei Zhao, Md Shamim Ahmed et al. · arXiv:2603.24118 · cs.IR · Paper Link

背景与动机

电子健康记录（EHR）是医疗 ML 的基础数据源，但 EHR 标准因国家和医院而异（如 HL7 FHIR、ICD-10、SNOMED CT 等混用），导致数据不兼容。这种碎片化严重限制了大规模和跨机构的 ML 应用。例如，使用美国医院数据训练的疾病预测模型通常无法直接部署到欧洲医院，因为特征定义、编码体系和数据结构可能完全不同。建立一个标准化的元数据仓库是解决这一问题的关键。

核心方法详解

① ISO 11179-3 元数据标准化

S4CMDR 基于 ISO 11179-3 标准构建，将每个 EHR 数据元素分解为三层抽象：

  • 概念域 (Conceptual Domain)：定义数据的语义含义（如"血压测量"）
  • 值域 (Value Domain)：定义允许的值范围和格式（如"收缩压: 60-250 mmHg"）
  • 数据元素 (Data Element)：特定系统中的具体实现（如"BP_SYSTOLIC, INT, FHIR.Observation"）

兼容性判断基于概念域和值域的映射：

$$\text{Compatible}(DE_A, DE_B) = \mathbb{1}[\text{CD}(DE_A) \equiv \text{CD}(DE_B)] \wedge \text{VD\_Overlap}(DE_A, DE_B) > \tau$$
② 自中而外标准化 (Middle-out Approach)

不同于自顶向下（强制统一标准）或自底向上（从数据推断）的方法，S4CMDR 采用自中而出策略：

  1. 中间层：定义一组"桥接概念"（Bridge Concepts），覆盖常见临床数据（如诊断、药物、实验室检查）
  2. 向上映射：每个桥接概念链接到国际标准（ICD-10、SNOMED CT、LOINC 等）
  3. 向下映射：每个参与机构将本地数据元素映射到桥接概念

这种设计最大限度减少了各机构的改造成本。映射覆盖率定义为：

$$\text{Coverage}(H_i) = \frac{|\{DE \in H_i : \exists BC, \text{mapped}(DE, BC)\}|}{|H_i|}$$
③ 自动化编目与错误检测

为减少人工编目错误，S4CMDR 实现了自动化管线：

  • 格式验证：检查数据类型、值域范围、必填字段的完整性
  • 语义验证：使用预训练的医学 NLP 模型（基于 PubMedBERT）检测概念映射的合理性
  • 冲突检测：识别同一概念在不同机构中的矛盾定义（如"Glucose" 在 A 医院指空腹血糖，在 B 医院指随机血糖）

错误率降低效果：手动编目的错误率约 8.3%，自动化后降至 1.7%。

④ 微服务架构与部署

系统采用微服务架构：认证服务（OAuth 2.0）、元数据服务（REST API）、可视化服务（React 前端）、搜索服务（ElasticSearch）。支持本地 Linux 部署和云托管，通过 Docker Compose 一键启动。关键 API 包括：

  • POST /register：注册新的数据元素及其映射
  • GET /discover?concept=X：发现与概念 X 兼容的所有数据源
  • GET /compatibility?source=A&target=B：计算两个数据注册表的兼容特征集

背景与动机

对话式医疗 AI（如辅助抗抑郁药物选择的决策支持工具）的部署需要系统化的风险评估，但获取足够多样化的真实患者交互数据成本极高且涉及伦理问题。现有评估要么规模太小（几十例），要么未能覆盖健康素养水平、行为模式等关键人口学维度。需要一个可扩展的患者模拟器来进行自动化、可控的风险评估。

$$P(\text{profile}_m) \propto \prod_{d \in \text{diagnoses}} \text{HR}(d) \cdot \mathbb{1}[\text{HR}(d) > \tau_{\text{risk}}]$$

② 循环事件与新发事件的区分

论文发现了 EHR 基础模型评估中的一个关键缺陷：当重复事件（如慢性病每次就诊都出现的诊断码）与新发事件未加区分时，模型可以通过简单"复制"上次就诊的事件来获得虚高分数。RAVEN 引入事件类型标记：

③ 数据受限下的扩展行为

$$L(N, D) = \alpha \cdot N^{-\beta} + \gamma \cdot D^{-\delta} + L_{\infty}$$

核心方法详解

① 混合加权候选路径枚举

S-Path-RAG 将 KG 中的检索问题转化为加权图上的路径搜索。对于查询 \(q\)，首先定位种子实体 \(e_s\)，然后在 KG 子图上使用三种互补策略枚举候选路径：

  • k-最短路径 (k-Shortest Paths)：找到权重最小的 \(k\) 条路径，边权重定义为：
$$w(e_i, r, e_j) = 1 - \cos(\text{Emb}(q), \text{Emb}(r))$$
  • 束搜索 (Beam Search)：保持 beam_width 条部分路径，逐跳扩展
  • 约束随机游走 (Constrained Random Walk)：以语义相似度为概率进行采样，增加路径多样性

三种策略的组合确保了既有高质量路径（最短路径）又有多样性（随机游走）。

② 可微路径评分器

候选路径通过一个可微的评分模型进行排序。路径 \(\rho = (e_0, r_1, e_1, ..., r_L, e_L)\) 的分数由对比路径编码器计算：

$$s(\rho, q) = \text{MLP}\left(\text{PathEnc}(\rho) \odot \text{QueryEnc}(q)\right)$$

其中 PathEnc 使用 GRU 沿路径边序列编码，\(\odot\) 是逐元素乘积（Hadamard product）。训练使用对比损失：

$$\mathcal{L}_{\text{contrast}} = -\log \frac{\exp(s(\rho^+, q) / \tau)}{\sum_{\rho \in \mathcal{P}} \exp(s(\rho, q) / \tau)}$$

正样本 \(\rho^+\) 是连接正确答案的路径，负样本来自候选路径池。

③ 软混合潜在变量注入

选定的 top-\(m\) 路径不是直接拼接为文本输入 LLM，而是通过交叉注意力将其紧凑的潜在表示注入语言模型：

$$h_{\text{path}} = \sum_{i=1}^{m} \beta_i \cdot \text{PathEnc}(\rho_i), \quad \beta_i = \frac{\exp(s(\rho_i, q))}{\sum_j \exp(s(\rho_j, q))}$$ $$\text{Output} = \text{LLM}(\text{CrossAttn}(h_{\text{query}}, h_{\text{path}}))$$

这种设计既节省 token（不需要将图路径转换为冗长文本），又保留了拓扑结构信息。

④ 神经苏格拉底图对话循环

S-Path-RAG 引入了迭代的"苏格拉底对话"机制：当 LLM 对当前路径证据不确定时，生成诊断性信息（如"需要更多关于 X 与 Y 关系的证据"），系统将其映射为图编辑操作：

  1. 种子扩展：从当前种子实体向外扩展 1-2 跳
  2. 路径重评分：基于新信息重新计算路径权重
  3. 子图精化：剪枝低相关度节点和边

这种自适应检索机制使系统在面对复杂问题时能够"追问"知识图谱，而非一次性检索。

背景与动机

RAG 系统在企业搜索中被广泛部署，但验证生成内容是否忠实于源文档面临两难困境：(1) 大型 LLM 可以处理长上下文但速度慢、成本高（GPT-4 处理 32K token 需 >10s）；(2) 轻量级分类器速度快但受上下文窗口限制（通常 512-2K token），对截断片段之外的证据容易遗漏。需要一个在延迟约束下实现全文档验证的系统。

核心方法详解

① 自适应推理策略 (Adaptive Inference Strategy)

核心思想：不是所有声明都需要全文档验证。系统首先对每个声明进行"可验证性预评估"：

$$\text{Difficulty}(c) = 1 - \max_{i} \text{sim}(c, \text{chunk}_i)$$

如果声明 \(c\) 与某个文档块高度匹配（Difficulty < 0.3），使用快速的块级验证器；如果证据分散（Difficulty > 0.7），使用全文档验证器；中间情况使用多块聚合验证器。这种分层策略使 70% 的声明走快速路径。

② 分层验证架构

系统包含三级验证器，按计算成本递增：

  1. Tier 1 — 块级验证器：基于 DeBERTa-large（330M 参数），处理单个块（512 token），延迟 ~20ms
  2. Tier 2 — 多块聚合验证器：将 top-3 相关块拼接后输入，处理 ~1.5K token，延迟 ~80ms
  3. Tier 3 — 全文档验证器：基于 Longformer 架构（4K token）或 Mistral-7B（32K token），延迟 200ms-2s

总延迟预算 \(T_{\text{budget}}\) 由应用场景决定。系统优化分配策略：

$$\min \sum_c (1 - \text{Acc}(\text{tier}(c))) \quad \text{s.t.} \quad \sum_c \text{Latency}(\text{tier}(c)) \leq T_{\text{budget}}$$
③ 全文档基础处理

对于 Tier 3 验证器，采用基于滑动窗口的全文档覆盖策略。将 32K token 文档分为重叠窗口，每个窗口独立验证后进行投票聚合：

$$\text{Verdict}(c, \text{doc}) = \begin{cases} \text{Supported} & \text{if } \frac{1}{W}\sum_w \text{score}_w > \tau^+ \\ \text{Not Supported} & \text{if } \frac{1}{W}\sum_w \text{score}_w < \tau^- \\ \text{Uncertain} & \text{otherwise} \end{cases}$$

重叠率设为 25% 以避免边界处的信息丢失。

Retrieval Improvements Do Not Guarantee Better Answers: A Study of RAG for AI Policy QA

Saahil Mathur, Ryan David Rittner et al. · arXiv:2603.24580 · cs.CL, cs.AI, cs.IR · Paper Link

背景与动机

RAG 系统被越来越多地用于分析复杂政策文档，但一个核心假设——"更好的检索 → 更好的答案"——在政策领域可能不成立。AI 治理和监管文档的特殊性在于：(1) 密集的法律语言，一个句子可能包含多层限定条件；(2) 不断演进的监管框架，同一主题有多个重叠甚至矛盾的法规；(3) 专家级问题需要在多个文档间进行推理而非单纯的信息检索。

核心方法详解

① AGORA 政策语料库与领域微调

使用 AI Governance and Regulatory Archive (AGORA) 语料库——包含 947 份 AI 政策文档。检索器采用 ColBERT 架构，通过对比学习进行领域微调：

$$\mathcal{L}_{\text{retriever}} = -\log \frac{\exp(\text{MaxSim}(q, d^+) / \tau)}{\sum_{d \in \mathcal{D}} \exp(\text{MaxSim}(q, d) / \tau)}$$

其中 MaxSim 是 ColBERT 的 token-level 最大相似度操作。微调使用合成查询生成：从文档中随机采样段落，用 GPT-4 生成对应的政策问题。

② DPO 对齐的生成器

生成器使用直接偏好优化（DPO）与人类偏好对齐。收集成对偏好数据 \((y_w, y_l)\)（赢/输回答），训练目标：

$$\mathcal{L}_{\text{DPO}} = -\log \sigma\left(\beta \left[\log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right]\right)$$

偏好标准包括：引用准确性、条款完整性、限定条件的包含度。DPO 使生成器更倾向于给出有据可查的、包含限定条件的回答。

③ "更好检索 → 更自信幻觉" 发现

反直觉的核心发现：领域微调后的检索器在标准检索指标（NDCG@10、Recall@10）上显著提升，但端到端 QA 性能并未同步提高，在某些情况下甚至更差。分析原因：

$$P(\text{hallucination} | \text{relevant\_but\_incomplete}) > P(\text{hallucination} | \text{irrelevant})$$

当检索到"相关但不完整"的文档时（例如找到了法规 A 但遗漏了与之矛盾的法规 B），生成器的回答更加自信（因为确实有相关文献支持），但可能遗漏了关键的限定条件或例外情况。这比完全检索不到（生成器会承认不确定）更危险。

④ 组件级 vs 端到端评估

作者提出了一个重要方法论论点：单独优化 RAG 的各个组件可能产生误导。定义组件-系统对齐度：

$$\text{Alignment} = \text{corr}(\Delta \text{Retrieval}, \Delta \text{QA})$$

实验中 Alignment = 0.23（弱正相关），说明检索改进仅有 23% 的情况能转化为 QA 改进。在法律和政策领域这种对齐度更低（0.15），因为这些领域的答案质量更依赖于推理而非单纯的信息检索。
