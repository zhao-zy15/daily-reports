Pure Text LLMs

Demystifying When Pruning Works via Representation Hierarchies
📋 arXiv:2603.24652 👤 Shwai He, Guoheng Sun, Haichao Zhang, Yun Fu, Ang Li 📅 2026-03-25 🧠 LLM
背景与动机

网络剪枝是一种通过移除冗余参数来压缩模型的经典方法。然而一个令人困惑的现象是：同样的剪枝策略在非生成任务（分类、检索）上表现良好，却在生成任务上严重退化。现有研究停留在经验层面，缺乏对这种不一致性的理论解释。本文从表示层级（Representation Hierarchy）的全新视角，系统地揭示了剪枝在不同任务上表现迥异的根本原因。

核心方法详解
①

三层表示空间分解

将语言模型的前向传播分解为三个顺序空间：嵌入空间 \(\mathcal{H}\)（隐藏层输出）、Logit 空间 \(\mathcal{L}\)（Softmax 前）、概率空间 \(\mathcal{P}\)（Softmax 后）。令原始模型的隐藏表示为 \(h\)，剪枝后为 \(\hat{h}\)，则嵌入空间的偏差为 \(\Delta h = \hat{h} - h\)。关键发现：\(\|\Delta h\| / \|h\|\) 通常很小（<5%），嵌入空间对剪枝扰动是鲁棒的。

②

Softmax 非线性放大效应

从 Logit 到概率的变换 \(p_i = \text{softmax}(l_i) = \frac{e^{l_i}}{\sum_j e^{l_j}}\) 是非线性的。当 Logit 空间存在微小偏差 \(\Delta l\) 时，概率空间的偏差被指数函数放大：$$\Delta p_i \approx p_i \cdot \left(\Delta l_i - \sum_j p_j \Delta l_j\right)$$ 对于高置信度 token（\(p_i\) 接近 1），Logit 的微小偏差不会改变 argmax，因此分类任务不受影响。但对于生成任务，每一步的累积偏差通过自回归解码不断放大。

③

自回归误差累积模型

在 \(T\) 步自回归生成中，令每步的 token 级错误概率为 \(\epsilon\)，则序列级正确概率为 \((1-\epsilon)^T\)。当 \(\epsilon = 0.02\)、\(T = 100\) 时，序列正确率仅为 \((0.98)^{100} \approx 13.3\%\)。这解释了为什么剪枝后模型在短序列分类上几乎无损，但在长文本生成上灾难性崩溃。论文通过跟踪 \(T\) 从 1 到 512 的 KL 散度变化，量化验证了这一指数退化。

④

分类子空间稳定性分析

对于 \(K\) 类分类任务，模型只需要概率空间的一个 \(K\) 维子空间 \(\mathcal{P}_K \subset \mathcal{P}\)。论文定义子空间稳定性指标：$$S_K = 1 - \frac{\|\Pi_K(\Delta p)\|}{\|\Delta p\|}$$ 其中 \(\Pi_K\) 是到分类子空间的投影。实验显示 \(S_K > 0.92\)，即剪枝引起的偏差主要落在分类不关心的"正交"维度，因此检索/选择题任务的精度几乎不变。

Prune as You Generate: Online Rollout Pruning for Faster and Better RLVR (ARROL)
📋 arXiv:2603.24840 👤 Haobo Xu, Sirui Chen, Ruizhong Qiu, Yuchen Yan 等 📅 2026-03-25 🧠 LLM
背景与动机

RLVR（强化学习与可验证奖励）通过 GRPO/DAPO 等方法显著提升了 LLM 的推理能力。然而核心瓶颈在于：每个 prompt 需要采样大量 rollout（通常 16-64 条），其中大部分要么全对要么全错，导致组内奖励方差极低，优势估计 \(\hat{A}\) 接近零，学习信号微弱。ARROL 提出在生成过程中动态剪枝低价值 rollout，同时引导剩余 rollout 向正确-错误平衡方向发展。

核心方法详解
①

轻量级质量头（Quality Head）在线训练

在 LLM 最后一层隐藏状态之上附加一个单层 MLP 质量头 \(Q_\phi\)，预测当前部分生成序列的最终成功概率：$$Q_\phi(x_{\le t}) = \sigma(W_q \cdot h_L^{(t)} + b_q) \in [0, 1]$$ 其中 \(h_L^{(t)}\) 是第 \(L\) 层在 token \(t\) 处的隐藏状态。\(Q_\phi\) 使用二元交叉熵损失在每个 batch 的已完成 rollout 上在线更新：$$\mathcal{L}_Q = -\frac{1}{N}\sum_{i=1}^N [r_i \log Q_\phi(x_i) + (1-r_i)\log(1-Q_\phi(x_i))]$$ 其中 \(r_i \in \{0, 1\}\) 是可验证奖励。由于只有一层 MLP，训练开销几乎可忽略。

②

在线剪枝决策策略

在生成到预设检查点 \(t_c\)（如总长度的 30%、50%）时，对所有活跃 rollout 计算 \(Q_\phi(x_{\le t_c})\)。将 rollout 按预测质量排序，保留使正-负样本比例接近 1:1 的子集。具体而言，若当前 batch 有 \(G\) 条 rollout，质量头预测 \(k\) 条可能正确，则保留 top-\(k\) 和 bottom-\(k\) 各自的 rollout，剪掉中间的"无信息"样本。剪枝比例自适应：$$\text{prune\_ratio} = \max\left(0, 1 - \frac{2\min(k, G-k)}{G}\right)$$ 当所有 rollout 预测结果相同时，剪枝率最高（全部无信息）。

③

系统级重新分批设计

关键工程创新：剪枝直接在 vLLM 推理引擎内执行，被剪枝的 rollout 不再继续生成，释放的 GPU 内存和 KV-cache 槽位分配给同一 batch 中新的 prompt。对数概率（log-prob）计算和策略更新也仅对存活 rollout 执行，避免浪费。整体流程：Prompt Queue → vLLM生成（带检查点剪枝）→ 存活Rollout → LogProb & 优势计算 → PPO/GRPO更新。相比朴素实现，内存效率提升 1.4×。

④

测试时质量加权扩展

训练好的 \(Q_\phi\) 还可在测试时扩展（test-time scaling）中使用。对于 majority voting，不再均匀投票，而是用质量头分数加权：$$\hat{a} = \arg\max_a \sum_{i: \text{answer}(x_i)=a} Q_\phi(x_i)$$ 这使得高质量 rollout 获得更大权重，比 Best-of-N 或均匀多数投票更高效。

Closing the Confidence-Faithfulness Gap in Large Language Models
📋 arXiv:2603.25052 👤 Miranda Muqing Miao, Lyle Ungar 📅 2026-03-26 🧠 LLM
背景与动机

LLM 在被要求给出置信度分数时，倾向于给出与实际准确率严重脱节的数字——例如对错误答案也报告 90% 的置信度。传统校准方法（如温度调节、标签平滑）在外部调整输出分布，但对 LLM 口头表达的置信度（verbal confidence）无效。本文首次从机械可解释性角度研究这一问题，揭示了一个令人惊讶的几何发现。

核心方法详解
①

线性探针发现：校准与置信正交

在模型的中间层（如第 16-24 层）提取隐藏状态，训练两个线性探针：一个预测"模型是否真的正确"（校准方向 \(\mathbf{v}_\text{cal}\)），另一个预测"模型口头表达的置信度"（置信方向 \(\mathbf{v}_\text{conf}\)）。关键发现：$$\cos(\mathbf{v}_\text{cal}, \mathbf{v}_\text{conf}) \approx 0.03 \sim 0.08$$ 两个方向几乎完全正交。这意味着模型内部知道自己是否正确，但口头表达的置信度编码在一个独立的、不相关的方向上。

②

推理污染效应（Reasoning Contamination Effect）

当模型被要求"先推理再给出置信度"时，推理过程的激活会干扰置信方向 \(\mathbf{v}_\text{conf}\)。具体而言，推理激活向量 \(\mathbf{v}_\text{reason}\) 与 \(\mathbf{v}_\text{conf}\) 的投影为正，导致置信度被系统性地推高：$$\text{conf}_\text{after\_reasoning} = \text{conf}_\text{before} + \alpha \cdot \langle \mathbf{v}_\text{reason}, \mathbf{v}_\text{conf}\rangle$$ 其中 \(\alpha > 0\)。这解释了为什么 Chain-of-Thought 反而恶化了校准——推理越复杂，污染越严重。

③

两阶段自适应引导管道（CAA Steering）

利用对比激活添加（Contrastive Activation Addition）技术构建引导管道：
阶段一（读取）：从校准探针读取模型对当前 query 的内部准确率估计 \(\hat{p}_\text{correct} = \sigma(\mathbf{v}_\text{cal}^\top h)\)。
阶段二（引导）：计算目标置信度与当前置信度的差值 \(\Delta = \hat{p}_\text{correct} - \hat{p}_\text{conf}\)，在生成置信度 token 时添加引导向量：$$h' = h + \Delta \cdot \lambda \cdot \mathbf{v}_\text{conf}$$ 其中 \(\lambda\) 是引导强度超参数。这将口头置信度"拉向"内部校准估计。

Do LLMs Know What They Know? Measuring Metacognitive Efficiency with Signal Detection Theory
📋 arXiv:2603.25112 👤 Jon-Paul Cacioli 📅 2026-03-26 🧠 LLM
背景与动机

现有 LLM 校准评估（ECE、Brier Score）混淆了两种不同能力：模型知道多少（Type-1 敏感性 \(d'\)）和模型对自身知识的了解程度（Type-2 元认知敏感性 \(\text{meta-}d'\)）。一个"校准好"的模型可能只是恰好设了一个保守的置信阈值，而非真正了解自己的能力边界。本文引入信号检测理论（SDT）的 Type-2 框架，首次将这两种能力解耦度量。

核心方法详解
①

Type-1 \(d'\)：基础知识敏感性

将每道事实题视为信号检测任务。"信号"是正确答案存在于模型知识中，"噪声"是缺失。令 Hit Rate = 正确回答且高置信的比例，False Alarm = 错误回答且高置信的比例，则：$$d' = z(\text{Hit Rate}) - z(\text{False Alarm Rate})$$ 其中 \(z(\cdot)\) 是标准正态分布的逆函数。\(d'\) 越大表示模型的基础知识区分能力越强。

②

Type-2 \(\text{meta-}d'\)：元认知敏感性

关键创新：在 Type-1 响应之后，分析模型置信度评分对自身正确性的区分能力。对于已做出的回答，将"信号"定义为实际正确，"噪声"为实际错误：$$\text{meta-}d' = z(\text{Confidence}|\text{Correct}) - z(\text{Confidence}|\text{Incorrect})$$ 使用 Fleming & Lau (2014) 的 MLE 方法估计。\(\text{meta-}d'\) 衡量的是：给定模型已经做出了回答，它的置信度能多好地区分自己的对错。

③

M-ratio：元认知效率比

核心指标定义为：$$M\text{-ratio} = \frac{\text{meta-}d'}{d'}$$ 当 \(M\text{-ratio} = 1\) 时，模型对自身知识的感知是"理想"的——所有信息都被置信度完美利用。\(M\text{-ratio} < 1\) 表示元认知不足（"不知道自己不知道"），\(M\text{-ratio} > 1\) 在理论上不可能但实践中可因噪声出现。该指标独立于 \(d'\)，允许在不同知识水平的模型间公平比较。

SafeMath: Inference-time Safety Improves Math Accuracy
📋 arXiv:2603.25201 👤 Sagnik Basu, Subhrajit Mitra, Aman Juneja 等 📅 2026-03-26 🧠 LLM
背景与动机

数学应用题通常被视为"安全"的 NLP 任务。然而，以自然语言叙述构建的数学问题可以成为传播有害内容的微妙媒介——例如嵌入偏见、歧视或暴力场景的 word problem。这在教育场景中风险尤其高。现有安全对齐方法在面对此类问题时要么完全拒绝（损失数学准确性），要么完全忽略（传播有害内容）。

核心方法详解
①

ToxicGSM 数据集构建

基于 GSM8K 构建 1.9k 个有害数学题。流程：选取 GSM8K 原始题目 → 用 GPT-4 将场景改写为包含有害内容（如"一个贩毒者每天卖出 x 克..."）→ 保留数学结构不变 → 人工审核毒性标签和数学正确性。每道题标注毒性类别（偏见、暴力、非法活动等）和数学难度级别。数据集按 7:3 分为训练/测试集。

②

SafeMath 推理时安全引导

SafeMath 在推理时（非训练时）添加安全层。核心思路是将数学推理与语言内容"去耦合"：$$\text{Output} = \text{MathSolver}(\text{Sanitize}(\text{Input})) + \text{SafetyDisclaimer}$$ 分三步：(i) 语义净化器识别并替换有害实体名称为中性占位符（如 "drug dealer" → "person A"），同时保留数学关系；(ii) 数学求解器在净化后的输入上正常推理；(iii) 安全声明器根据毒性分类器的判断，决定是否附加安全警告。

③

SafeMath 通过联合损失训练语义净化器：$$\mathcal{L} = \mathcal{L}_\text{math}(\hat{y}, y) + \beta \cdot \mathcal{L}_\text{toxic}(\hat{s}, s)$$ 其中 \(\mathcal{L}_\text{math}\) 确保净化后的题目数学答案不变，\(\mathcal{L}_\text{toxic}\) 确保毒性降低。\(\beta\) 控制安全-准确性权衡。实验发现 \(\beta \in [0.3, 0.7]\) 时两者可同时改善，因为净化掉无关有害信息反而减少了模型的"注意力分散"。

背景与动机

自进化 LLM Agent（如让 LLM 编写并改进自己的 Agent 程序）在代码修复和科学发现中展现了强大潜力。然而，这些 Agent 程序在未见输入上自主执行时，没有任何正确性或安全性保证。现有框架只优化性能指标，完全忽略了形式化约束——如果 Agent 在关键场景中违反合约，后果不堪设想。SEVerA 将 Agent 代码生成表述为一个约束学习问题，结合硬的形式化规范与软的任务效用目标。

核心方法详解
①

形式化保护生成模型（FGGM）

允许规划器 LLM 为每个生成模型调用指定一阶逻辑输出合约。FGGM 将底层模型包裹在带有已验证回退机制的拒绝采样器中：$$\text{FGGM}(M, \phi, x) = \begin{cases} M(x) & \text{if } \phi(M(x)) = \text{True} \\ \text{fallback}(x) & \text{otherwise} \end{cases}$$ 其中 \(\phi\) 是一阶逻辑合约（如"输出必须是合法 JSON 且金额 ≥ 0"），fallback 是安全默认行为。关键保证：对任意输入 \(x\) 和任意模型参数 \(\theta\)，FGGM 的输出始终满足 \(\phi\)。

②

三阶段搜索-验证-学习框架

搜索阶段：规划器 LLM 合成 \(K\) 个候选参数化程序，每个程序包含 FGGM 调用。
验证阶段：使用 Dafny 形式化验证器证明每个候选程序对所有参数值都满足硬约束。验证通过的程序进入学习阶段，未通过的被丢弃。这将原始的约束优化问题简化为无约束学习：$$\max_\theta f(\theta) \quad \text{s.t. } \forall x: \phi(\text{Program}(x; \theta)) \quad \xrightarrow{\text{verify}} \quad \max_\theta f(\theta)$$ 学习阶段：对验证通过的程序，使用 GRPO 风格的微调优化软目标（如任务准确率），同时形式化保证确保学习过程不违反约束。

③

GRPO 微调与约束保持

在学习阶段，对 Agent 内部调用的 LLM 使用 GRPO 优化：$$\mathcal{L}_\text{GRPO} = -\mathbb{E}_{x \sim \pi_\theta} \left[\hat{A}(x) \log \pi_\theta(x)\right] - \beta D_\text{KL}(\pi_\theta \| \pi_\text{ref})$$ 由于 FGGM 的拒绝采样机制，即使 GRPO 更新导致模型输出偏离，FGGM 仍然会拒绝不满足合约的输出并回退到安全行为。这意味着形式化保证在训练过程中始终成立，不需要额外的约束惩罚项。

Maximum Entropy Behavior Exploration for Sim2Real Zero-Shot Reinforcement Learning
📋 arXiv:2603.25464 👤 Jiajun Hu, Nuria Armengol Urpi, Jin Cheng, Stelian Coros 📅 2026-03-26 🎯 RL
背景与动机

零样本 RL 旨在从无奖励数据集中预训练策略库，测试时直接恢复任意奖励函数的最优策略。关键挑战：如何在没有下游任务先验的情况下收集足够多样化的数据？无方向探索导致数据低多样性、策略不自然，无法直接部署到真实硬件。本文提出 FB-MEBE，结合最大熵行为探索与正则化 Critic，在模拟中预训练、真实四足机器人上零样本部署。

核心方法详解
①

Forward-Backward (FB) 表示学习

FB 算法学习两个网络：前向特征 \(F(s, a)\) 和后向特征 \(B(s)\)，使得任意奖励函数 \(r(s)\) 的 Q 值可以表示为：$$Q^r(s, a) = F(s, a)^\top z_r, \quad z_r = \mathbb{E}_{s \sim d^\pi}[B(s) \cdot r(s)]$$ 其中 \(z_r\) 是奖励向量嵌入。这允许在测试时给定新的 \(r\)，直接通过 \(\arg\max_a F(s,a)^\top z_r\) 恢复策略，无需重新训练。关键：预训练数据的多样性决定了 \(F, B\) 的表达能力。

②

最大熵行为探索（MEBE）

为了最大化预训练数据的多样性，定义行为分布 \(d_\text{behavior}(b)\)，其中 \(b = B(s)\) 是后向特征空间中的行为向量。MEBE 优化的目标：$$\max_\pi H(d_\text{behavior}) = -\int d_\text{behavior}(b) \log d_\text{behavior}(b) \, db$$ 即最大化行为空间的熵。实践中使用 k-nearest neighbor 熵估计器，在每个 batch 中计算：$$\hat{H}_k = \frac{d}{N}\sum_{i=1}^N \log \|b_i - b_i^{(k)}\| + \text{const}$$ 其中 \(b_i^{(k)}\) 是第 \(i\) 个样本的第 \(k\) 近邻。将 \(\hat{H}_k\) 作为探索奖励，鼓励机器人尝试尽可能不同的行为模式。

③

正则化 Critic 的行为约束

纯粹的最大熵探索可能学到物理上不自然甚至危险的行为（如翻滚、剧烈抖动），无法部署到硬件。引入正则化 Critic \(C_\text{reg}\)：$$r_\text{total} = r_\text{entropy} + \gamma \cdot C_\text{reg}(s, a)$$ 其中 \(C_\text{reg}\) 惩罚不自然的关节角度、过大的力矩和跌倒状态。具体：$$C_\text{reg}(s, a) = -w_1\|\tau\|^2 - w_2\|q - q_\text{default}\|^2 - w_3 \cdot \mathbb{1}[\text{fall}]$$ \(\tau\) 是关节力矩，\(q\) 是关节角度，\(q_\text{default}\) 是默认站姿。这在多样性和物理可行性之间取得平衡。

📋 arXiv:2603.24979 👤 Yuhang Zhou, Zhuokai Zhao, Ke Li 等 17 位作者 📅 2026-03-26 🤖 Agent
背景与动机

大规模工业 ML 系统的特征选择通常依赖于统计启发式（如互信息、Lasso），但这些方法忽略了生产环境中的操作约束：延迟上限、特征组依赖、特征更新频率、合规要求等。标注数据的稀缺使传统方法更难应用。MoFA (Model Feature Agent) 提出让 LLM 作为"特征工程师"，利用语义理解和定量信息执行约束感知的顺序特征选择。

核心方法详解
①

结构化特征信息聚合

为 LLM 构建结构化 Prompt，整合四类信息：
(a) 特征定义：每个特征的语义描述（如"user_7d_click_count: 用户近7天点击次数"）
(b) 重要性评分：从已有模型提取的 SHAP 值或 feature importance
(c) 特征相关性矩阵：Pearson 相关或 mutual information 的 Top-k 关系
(d) 元数据约束：特征组（如"用户画像"组有 50ms 延迟，"实时行为"组有 200ms 延迟）
LLM 综合这些信息生成选择理由，格式为：$$\text{Select}(f_i) \leftarrow \text{Reasoning}(\text{Def}(f_i), \text{SHAP}(f_i), \text{Corr}(f_i, f_j), \text{Constraints})$$

②

顺序特征选择推理链

MoFA 不是一次性选择所有特征，而是模拟人类特征工程师的顺序决策过程。每轮选择一个特征，并更新上下文：
Step 1: 从最重要特征开始，LLM 选择 \(f_1\) 并解释理由
Step 2: 给定 \(f_1\)，LLM 考虑约束（如已选特征的延迟占用）并选择 \(f_2\)
...
Step N: 当新增特征的边际收益低于阈值或约束被触发时停止
停止条件：$$\text{Stop if } \Delta\text{SHAP}(f_{\text{new}} | f_{1:k}) < \epsilon \text{ or } \sum_{i=1}^{k+1} \text{Latency}(f_i) > L_\text{max}$$

③

高阶交互项发现

MoFA 的独特能力：LLM 可以基于语义推理发现统计方法难以捕捉的高阶交互。例如，LLM 推理出"user_purchase_frequency × item_price_range"的交互项可能对价值预测重要，因为"高频购买低价商品"和"低频购买高价商品"的用户价值模式截然不同。具体操作是让 LLM 在选择完基础特征后，提出候选交叉特征：$$f_\text{cross} = g(f_i, f_j), \quad g \in \{\times, +, \text{concat}, \text{bucket}\}$$ 然后用小样本验证增益。

GraphER: An Efficient Graph-Based Enrichment and Reranking Method for Retrieval-Augmented Generation
📋 arXiv:2603.24925 👤 Ruizhong Miao, Yuying Wang, Rongguang Wang 等 📅 2026-03-26 🔍 IR
背景与动机

RAG 系统中的语义检索在处理分散证据时表现不佳——相关信息可能分布在多个文档中，单靠语义相似度难以捕捉结构关系。现有方案要么用 Agent 迭代检索（效率低），要么构建知识图谱（维护成本高、与向量存储不兼容）。GraphER 提出了一种零维护成本的图增强方案：离线索引时丰富数据对象，查询时执行图重排序。

核心方法详解
①

离线图结构索引增强

在索引构建阶段，对每个文档块 \(d_i\) 提取多种邻近关系，构建邻接图 \(G = (V, E)\)：
语义边：\(e_\text{sem}(d_i, d_j) = \cos(\mathbf{e}_i, \mathbf{e}_j) > \tau_\text{sem}\)
结构边：来自同一文档的不同块、相邻章节、引用关系等
实体边：共享命名实体（如同一公司名、人名）
每条边附带类型标签和权重。图与向量索引并行存储，不替换原有基础设施。图的构建是离线的、增量的。

②

查询时图传播重排序

给定 query \(q\)，先用标准检索器获得 Top-N 候选 \(\{d_1, ..., d_N\}\) 及其初始分数 \(s_i^{(0)}\)。然后在图 \(G\) 上执行 \(K\) 轮消息传播：$$s_i^{(k+1)} = (1-\alpha) \cdot s_i^{(0)} + \alpha \cdot \sum_{j \in \mathcal{N}(i)} w_{ij} \cdot s_j^{(k)}$$ 其中 \(\alpha\) 是图传播强度（类似 PersonalizedPageRank），\(w_{ij}\) 是边权重。关键：传播只在 Top-N 的子图上进行，计算开销为 \(O(NK |\overline{\mathcal{N}}|)\)，对于典型参数（N=100, K=3, \(|\overline{\mathcal{N}}|=5\)）仅需 1500 次乘法，延迟 <1ms。

③

边类型感知加权

不同类型的边对重排序的贡献不同。GraphER 学习边类型权重：$$w_{ij} = \sum_{t \in \text{types}} \beta_t \cdot \mathbb{1}[(i,j) \in E_t]$$ 其中 \(\beta_t\) 是可学习的类型权重。在验证集上通过 grid search 或轻量梯度优化确定。实验发现结构边（同一文档/章节）的贡献最大（\(\beta_\text{struct} \approx 0.45\)），因为它们直接捕捉了证据的局部聚集性。

📋 arXiv:2603.25092 👤 Zhihui Yao, Hengran Zhang, Keping Bi 📅 2026-03-26（投稿 ACL 2026） 🔍 IR
背景与动机

RAG 利用外部知识增强 LLM，但容易受到低权威来源的影响，传播错误信息。一个关键但未被充分研究的问题是：LLM 是否具备感知信息权威性的能力？权威性是超越语义理解的维度——同样内容的信息，来自 WHO 和来自匿名博客的可信度截然不同。AuthorityBench 是首个系统性评估此能力的基准。

核心方法详解
①

三层数据集构建

DomainAuth (10K 网页域)：收集 10,000 个网页域，以 PageRank 和 Majestic Trust Flow 作为权威性 ground truth。评估 LLM 对域名权威性的感知能力。
EntityAuth (22K 实体)：从 Wikidata 提取 22,000 个实体，以 Wikipedia 页面浏览量和引用数量作为流行度/权威性指标。
RAGAuth (120 查询)：构建 120 个 QA 查询，每个查询配有来自不同权威级别的文档（高权威如政府网站、中权威如新闻媒体、低权威如个人博客），评估权威性过滤对下游答案质量的影响。

②

三种评估范式

设计了三种提示范式评估 LLM 的权威性判断：
PointJudge：直接给单个域名/实体打分（1-10 分），\(\text{Score}_\text{point}(x) = f_\text{LLM}(x)\)
PairJudge：给定两个域名，判断哪个更权威，\(\text{Pref}(x_i, x_j) = f_\text{LLM}(x_i, x_j)\)
ListJudge：给定 5-10 个域名列表，排序它们的权威性
评估指标：与 ground truth 的 Kendall's τ 和 Spearman's ρ。

ColBERT-Att: Late-Interaction Meets Attention for Enhanced Retrieval
📋 arXiv:2603.25248 👤 Raj Nath Patel, Sourav Dutta 📅 2026-03-26 🔍 IR
背景与动机

ColBERT 的"晚期交互"范式通过 MaxSim 操作实现高效检索：为每个 query token 找到最匹配的 document token，然后求和。然而，MaxSim 对所有 query token 等权重——"the"、"of" 与关键实体词获得相同贡献。ColBERT-Att 认为 query/document token 的注意力权重捕捉了它们的"重要性"，应该被显式利用。

核心方法详解
①

注意力权重提取

从 ColBERT 的 BERT 编码器中提取最后一层的自注意力权重。对 query 序列 \(q_1, ..., q_m\)，每个 token \(q_i\) 的注意力权重定义为 [CLS] token 对 \(q_i\) 的注意力值（跨所有注意力头平均）：$$\alpha_i^q = \frac{1}{H}\sum_{h=1}^H A_h^{\text{cls} \to i}$$ 对文档 token 类似定义 \(\alpha_j^d\)。高权重意味着该 token 对整体语义更重要。

②

注意力加权的 MaxSim

将标准 ColBERT 的 MaxSim 替换为注意力加权版本：
标准 ColBERT：$$S(q, d) = \sum_{i=1}^m \max_{j=1}^n \cos(q_i, d_j)$$ ColBERT-Att：$$S_\text{att}(q, d) = \sum_{i=1}^m \alpha_i^q \cdot \max_{j=1}^n \left[\alpha_j^d \cdot \cos(q_i, d_j)\right]$$ 这使得关键 query token（如实体名、动词）的匹配获得更大权重，停用词的贡献被自然压低。注意力权重在编码时已经计算好，推理时无额外成本。

③

训练与索引兼容性

ColBERT-Att 使用与 ColBERT 完全相同的训练流程（对比学习 + hard negatives），仅在 scoring 函数中加入注意力权重。这意味着：
(a) 现有 ColBERT 索引可直接复用（只需额外存储每个 token 的一个 float 注意力权重，存储开销 <1%）
(b) ANN 检索阶段仍使用标准 MaxSim（快速召回），注意力加权仅在重排序阶段应用
两阶段设计：$$\text{Top-1000} \xrightarrow{\text{MaxSim}} \text{Top-100} \xrightarrow{\text{Att-MaxSim}} \text{Top-10}$$
