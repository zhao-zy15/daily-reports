Cross-Tokenizer LLM Distillation through a Byte-Level Interface

📅 2026-04-09 · arXiv: 2604.07466 🏛 King's College London, MediaTek Research, Orbital Materials LLM 知识蒸馏 跨分词器
📌 背景与动机

知识蒸馏是将大模型能力迁移到小模型的核心技术，但标准蒸馏要求师生模型共享相同的分词器和词表。当师生模型使用不同分词器时（如 Llama→Qwen 或 Llama→Gemma），logit 级蒸馏因词表空间不匹配而无法直接执行。现有的跨分词器蒸馏（CTD）方法依赖启发式词表对齐策略，引入复杂性且信息损失严重。这一限制阻碍了异构模型集成蒸馏的实现。

🔬 核心方法详解
① 字节级概率转换 — 从 Token 到 Byte

BLD 的核心洞察是：所有分词器最终都编码为字节序列。教师模型的 token 级输出分布 \(f_T\) 可通过 Vieira et al. (2025) 的快速近似算法转换为字节级概率： \[P_T(b_i | b_{

② 学生端字节级解码头

在学生模型的 transformer 输出上并行添加一个轻量字节级解码器 \(O_b: \mathbb{R}^{N \times d} \to \mathbb{R}^{N_b \times |\Sigma|}\)，其中 \(N_b = 10\)（每个 token 最多预测 10 个字节）。该解码器是 10 个并行线性投影层，从隐层维度映射到 260 个字节 token（256 字节 + 4 个特殊 token）。

③ 三目标联合蒸馏损失

最终训练损失结合三项：(1) token 级交叉熵确保原始 token 输出层权重更新；(2) 字节级交叉熵提供监督；(3) 字节级 KL 散度执行知识迁移： \[\mathcal{L} = \text{CE}_{token} + \text{CE}_{byte} + \lambda_{KL} \cdot \text{KL}(P_T^{byte}, f_S^{byte})\] 蒸馏完成后直接移除字节解码头，学生模型恢复标准 token 级架构。

④ LoRA 高效微调与 FVT 初始化

学生骨干使用 LoRA（rank=64）在 q/v/k/o/gate/up/down_proj 上微调，嵌入层通过 Fast Vocabulary Transfer (FVT) 初始化：共享 token 直接复制，新 token 用子 token 均值初始化。学习率 \(2 \times 10^{-5}\)，AdamW 优化器，cosine schedule。

💡 深度案例
场景：Llama3.2-3B → Qwen2 分词器迁移
  1. 输入：Llama3.2-3B-IT 模型（词表 128K），目标迁移到 Qwen2 分词器（词表 152K）
  2. 过程：通过 FVT 初始化新嵌入层，添加字节解码头，在 Tulu-3 SFT 数据上蒸馏 5 个 epoch
  3. 结果：BLD 在 PiQA 上达到 75.68（超越原始模型 75.46），MMLU 58.29（仅损失 2.2 点），但 IFEval 显著下降至 30.58（原始 66.31）
  4. 发现：字节级蒸馏能有效保留通用知识，但指令遵循能力传递不足

🔍 消融与讨论

字节解码头使用简单线性层（非自回归）即可有效工作：仅在字节级 SFT 训练时，token 级验证 loss 也同步下降，表明字节级概率分布能有效桥接不同分词器。近似参数 \(K=10, \epsilon=0.01\) 在 JSD < 0.005 时产生可忽略的下游性能损失。然而，BLD 在指令遵循（IFEval）上表现最差，暗示字节级接口可能丢失了结构化输出行为的关键信号。BPE→字节级迁移更困难：所有方法 MMLU 均下降约 21 点。

🎯 结论与启发

BLD 证明字节是跨分词器蒸馏的天然公共接口。尽管方法简单——仅添加一个轻量线性解码头——其在多个基准上与复杂方法竞争甚至超越。核心启发：(1) 字节级概率分布是不同分词器间共享知识的有效桥梁；(2) CTD 仍是开放问题，没有方法在所有基准上一致占优；(3) 未来可探索自回归字节解码器捕获更丰富的序列依赖。

Loop, Think & Generalize: Implicit Reasoning in Recurrent-Depth Transformers

📅 2026-04-09 · arXiv: 2604.07822 🏛 The Ohio State University LLM 隐式推理 循环深度Transformer
📌 背景与动机

标准 Transformer 尽管在预训练中存储了大量事实知识，但在隐式推理（单次前向传播中组合知识）方面表现不佳。核心瓶颈在于：知识分布在不同层中，而各层参数不共享，导致需要在更深层组合的知识可能无法被正确访问。这限制了 LLM 在组合泛化和深度推理方面的能力。

🔬 核心方法详解
① 循环深度架构 — 共享权重的迭代计算

使用 \(L\) 层 GPT-2 风格 Transformer block，重复应用 \(R\) 次迭代，有效深度 \(D = L \times R\)。每次迭代的计算为： \[h^{(r+1)} = f_\theta(h^{(r)}; m), \quad r = 0, \dots, R-1\] 权重共享使得每一层都能在每次迭代中访问所有存储的知识，突破了标准 Transformer 中知识被绑定到特定层深度的限制。使用零初始化（c_proj 矩阵初始化为零）确保训练稳定性。

② 系统泛化：三阶段 Grokking 动态

在 2-hop 推理任务上（知识图谱 \(|E|=2000\), \(|R|=200\)），循环深度 Transformer 展现出独特的三阶段学习动态：
Stage 1（记忆）：模型过拟合训练集，OOD 准确率为 0；
Stage 2（ID 泛化）：经过延长训练后涌现分布内泛化（grokking）；
Stage 3（系统泛化）：模型在从未见过的知识组合上实现泛化。标准 Transformer（\(R=1\)）完全无法达到 Stage 3。

③ 深度外推：推理时迭代扩展

训练时使用动态迭代策略 \(R \sim \text{clip}(\text{Poisson}(\lambda), R_{\min}, R_{\max})\)，推理时增加迭代次数可解锁更深推理。模型在仅训练 5-hop 后，通过增加推理迭代可泛化到 19-hop 以上。关键发现：训练时最大迭代数决定外推范围，动态迭代优于固定迭代。

④ Overthinking 限制与自适应停止

过多的推理迭代会导致 overthinking：logit margin 先升后降。提出结合 KL 散度和输出熵的自适应停止准则： \[\text{KL}(p_t \| p_{t-1}) < \varepsilon_\text{KL} \quad \text{and} \quad H(p_t) < H_\text{thresh}\] 相比仅用 KL 散度的方法，该准则能更好地分配计算资源，避免过早停止。

💡 深度案例
场景：2-hop 系统泛化
  1. 输入：知识图谱包含 40K 原子事实，95% 用于组合训练（\(\mathcal{C}_{ID}\)），5% 保留为 OOD（\(\mathcal{C}_{OOD}\)）
  2. Logit Lens 分析：在 \(R=2\) 模型的 Stage 3，桥接实体在浅层被成功解码，目标实体在深层被正确预测——说明模型学会了真正的两步组合推理
  3. 对比：标准 8 层 Transformer 虽然能在 OOD 数据上解码桥接实体，但无法在深层完成第二跳推理

🔍 消融与讨论

初始化策略至关重要：使用默认高斯初始化（而非零初始化）时，不同种子间结果高度不稳定，增加循环次数不产生一致的改进。相反，零初始化确保恒等映射起点，训练稳定且结果可复现。动态迭代策略在相同数据下的外推性能与 R=8 固定迭代相当，但可学习更深的 ID 泛化深度。课程学习中存在"相变"：模型需要 130 万步 grokking 到 4-hop，之后快速学会直到 19-hop。

🎯 结论与启发

本文首次系统证明循环深度 Transformer 能实现标准 Transformer 无法做到的两大能力：(1) 系统泛化——组合从未在训练中一起出现的知识；(2) 深度外推——通过增加推理时迭代泛化到远超训练复杂度的任务。核心启示是权重共享使知识不再绑定于固定层深度，而是可在任意迭代中灵活访问。Overthinking 是当前主要限制，未来工作需解决推理计算的最优分配问题。

What Do Language Models Learn and When? The Implicit Curriculum Hypothesis

📅 2026-04-09 · arXiv: 2604.08510 🏛 Carnegie Mellon University, Johns Hopkins University, Northeastern University, USC LLM 预训练动态 技能涌现
📌 背景与动机

LLM 预训练通常仅通过交叉熵 loss 监控，但 loss 的平滑下降掩盖了技能获取的离散过程。现有 scaling law 告诉我们模型改善多少，却不告诉我们获取了什么技能、按什么顺序获取。当 GSM8k 性能停滞时，无法诊断瓶颈是数值流利度、多步规划还是自然语言理解。本文提出"隐式课程假说"试图揭示这一结构。

🔬 核心方法详解
① 隐式课程假说的三个核心预测

H1 组合顺序：前置技能的涌现时间不晚于依赖它们的组合技能：\(\forall \tau_j, \forall \tau_i \in P(\tau_j): t^*_{\tau_i}(m) \leq t^*_{\tau_j}(m)\)
H2 跨模型稳定性：涌现排序在不同模型间产生的偏序一致（Spearman \(\rho\) 显著高于随机）
H3 表征对齐：内部表征相似的任务学习轨迹也相似，可从表征几何预测未评估的组合任务轨迹。

② 91个可组合任务探针设计

设计 53 个简单任务（字符串操作、形态变换、翻译、算术、逻辑、阅读理解）和 38 个组合任务（如 gerund_upper: "run"→"RUNNING"）。所有任务使用 exact-match 准确率评估。覆盖 4 个模型家族（OLMo-2/3, LLM360, Pythia）、9 个模型（410M-13B 参数），每个模型约 20 个预训练 checkpoint。

③ 函数向量提取与任务相似度

使用 Todd et al. (2024) 的 function vector 方法提取任务表征：对每个任务构建 ICL prompt，只保留正确回答的样本，提取注意力头输出或残差流隐状态作为任务向量 \(v_\tau\)。任务间相似度通过余弦相似度度量，核岭回归用于预测保留任务的学习轨迹： \[\hat{a}_c(t) = k_c^\top (K_S + \lambda I)^{-1} y_t\] 其中 \(K\) 使用 RBF 核。

④ 绝对阈值 vs 相对阈值的关键区别

涌现排序的一致性仅在绝对阈值定义下成立（mean \(\rho = .81\)），相对阈值（以模型最优性能的比例定义）下一致性大幅下降（mean \(\rho = .50\)）。原因：弱模型可能在低绝对性能时就达到相对阈值，而强模型可能永远无法满足同一相对标准。

💡 深度案例
场景：OLMo2-7B 上 0-1T tokens 的组合任务轨迹预测
  1. 任务：fr_eng_upper（法译英后大写）
  2. 预测：仅从函数向量空间中其他任务的相似度，使用核岭回归预测其学习轨迹
  3. 结果：\(R^2 = 0.99\)，MAE = 0.017——预测曲线几乎完美追踪真实轨迹，包括涌现时间和改进速率
  4. 局限：eng_fr_upper 预测较差（\(R^2 = 0.51\)），说明某些组合任务的轨迹不完全由最近邻表征解释

🔍 消融与讨论

组合瓶颈效应：将预测基限制为仅简单任务时，所有模型 MAE 平均增加 0.135，说明组合任务间共享的结构信息超出其简单组件能解释的范围。涌现顺序呈现清晰模式：复制→简单字符串操作→事实提取和指代消解→逻辑→世界知识→多步算术。54/76 的组合任务确实在其"父任务"之后涌现，但有 19 个弱反转和 3 个强反转（全部涉及 first_letter 组件）。

🎯 结论与启发

本文统一了技能涌现、量化假说和简单性偏好三条研究线索，证明预训练遵循一个"隐式课程"——技能按组合依赖关系稳定涌现，且该顺序在不同模型家族间高度一致（\(\rho = .81\)）。更强的发现是这种结构可从模型内部表征中读取，使得仅通过函数向量相似度即可预测未评估任务的学习轨迹。实际启示：(1) 任务探针可作为预训练监控的结构化工具；(2) 涌现顺序的一致性暗示存在通用的技能获取规律。

Activation Steering for Aligned Open-ended Generation without Sacrificing Coherence

📅 2026-04-09 · arXiv: 2604.08169 🏛 Tara Research, Technical University of Munich, Mila Quebec AI Institute LLM 对齐 激活引导
📌 背景与动机

LLM 的对齐比通常认为的更脆弱：对抗性 prompt、良性微调、涌现性失对齐和目标错误泛化都可以触发不对齐行为。研究表明，安全对齐主要控制前几个输出 token，后续生成缺乏保护。激活引导作为轻量级运行时防御方案，可在整个生成过程中持续修正不对齐的激活。

🔬 核心方法详解
① SwFC — 固定系数引导

标准方法：在每个 token 生成时对残差流添加固定向量 \(h' = h + \alpha \cdot v\)，其中 \(v\) 是对齐方向向量。简单但可能对已对齐的 token 造成不必要干扰。

② StTP — 目标投影引导（本文提出）

训练逻辑回归分类器学习对齐/不对齐激活的决策边界。仅当 token 激活落在不对齐侧时才进行引导，将其投影到目标分布阈值位置。这避免了对已对齐 token 的过度干预，更好地保持通用能力。

③ StMP — 镜像投影引导（本文提出）

在 StTP 基础上进一步——将不对齐激活"镜像翻转"到决策边界的对齐侧，投影距离为原始激活到边界距离的镜像。这提供了更强的纠正力度，同时仍保持选择性干预。

💡 深度案例
场景：恶意系统 prompt 下的诚实性恢复
  1. 设置：Llama-3.3-70B-Instruct 注入 dismissiveness（轻蔑）系统 prompt
  2. 无引导：模型表现出持续的轻蔑和不关心行为
  3. StTP 引导：compassion 得分恢复至接近无恶意 prompt 基线，同时 MMLU/MT-Bench 下降最小（< 1%）
  4. SwFC 引导：也恢复目标特质，但 MT-Bench 下降更大且多轮对话中出现更多重复

🔍 消融与讨论

投影感知方法（StTP/StMP）相比固定系数方法（SwFC）在通用能力保持和多轮对话重复控制上均更优。在 Llama-3.3-70B 和 Qwen3-32B 两种架构上结果一致。局限性：引导向量需要标注数据来训练逻辑回归分类器，且当前方法假设不对齐行为可在线性子空间中表示。

🎯 结论与启发

激活引导是对抗 LLM 对齐脆弱性的有效轻量级防线。投影感知方法（StTP/StMP）通过选择性干预在恢复对齐属性的同时最小化对通用能力的影响。核心启示：对齐不是二元状态，而是可以在激活空间中持续调节的连续属性。

PolicyLong: Towards On-Policy Context Extension

📅 2026-04-09 · arXiv: 2604.07809 🏛 Tencent LLM 🏢 Tencent 长上下文
📌 背景与动机

LLM 长上下文能力的扩展通常通过在合成或手工构造的长文档上继续预训练实现（off-policy data），但这种方法产生的数据分布与模型自身生成的分布存在差距。PolicyLong 提出"on-policy 上下文扩展"——使用模型自身生成的长文本作为训练数据来扩展上下文窗口。

🔬 核心方法详解
① On-Policy 长文本数据生成

让基础模型在短上下文窗口内自回归生成文本，然后将多段生成拼接成长序列。关键在于这些文本来自模型自身的分布，消除了 off-policy 偏差。

② 渐进式上下文长度课程

采用分阶段训练：先在较短上下文（如 8K）上训练，逐步扩展到目标长度（如 128K），每阶段使用 RoPE 位置编码缩放。

③ 选择性注意力稀疏化

在长上下文训练中引入注意力稀疏化策略，降低计算复杂度同时保持对关键 token 的注意力。

💡 深度案例
场景：从 8K 扩展到 128K 上下文
  1. 基线：标准 off-policy 长上下文训练（使用 LongAlpaca 等数据集）
  2. PolicyLong：使用模型自身生成的拼接文本训练
  3. 结果：在 RULER 和 Longbench 等长上下文基准上，PolicyLong 在 needle-in-haystack 任务上更稳定，减少了"lost in the middle"现象
📊 实验结果
方法RULER (128K)Longbench短文本保持
Off-policy CPT基线基线轻微下降
PolicyLong+3-5%+2-4%无下降
🔍 消融与讨论

On-policy 数据的核心优势在于消除了模型从未见过的文本模式带来的分布偏差。消融实验表明渐进式课程和稀疏注意力的组合效果优于单独使用。局限：on-policy 数据生成本身需要计算开销，且对于需要外部知识的长文档理解任务，模型自生成的数据可能信息密度不足。

🎯 结论与启发

PolicyLong 开辟了长上下文训练的新思路：与其寻找高质量的长文档训练数据（off-policy），不如使用模型自身的生成分布来消除 train-test distribution gap。这一思路与 RL 中的 on-policy 训练理念一脉相承，为 LLM 的上下文扩展提供了更原则性的方法论。

GIRL: Generative Imagination Reinforcement Learning via Information-Theoretic Hallucination Control

📅 2026-04-08 · arXiv: 2604.07426 🏛 Google DeepMind RL 🏢 DeepMind 模型基RL 幻觉控制
📌 背景与动机

Model-Based RL (MBRL) 通过在想象 rollout 中优化策略提升样本效率，但长期规划中模型误差的累积会导致想象轨迹偏离真实动态——即"世界模型幻觉"。现有方法要么限制想象 horizon 长度，要么依赖计算昂贵的集成方法。GIRL 提出用信息论方法系统性地控制幻觉。

🔬 核心方法详解
① 信息论幻觉度量

定义想象轨迹与真实轨迹之间的 KL 散度作为幻觉度量。通过分解条件互信息，将长 horizon 幻觉分解为逐步误差的累积，使得可以在每个时间步检测和控制幻觉。

② 自适应 Horizon 调节

当累积幻觉度量超过阈值时自动截断想象 rollout。这允许在低不确定性区域进行更长的规划，而在高不确定性区域保守地截断。

③ 幻觉感知的策略优化

将幻觉度量作为策略梯度中的正则化项，惩罚导致高幻觉轨迹的动作选择： \[J(\pi) = \mathbb{E}_\pi\left[\sum_t \gamma^t r_t - \lambda \cdot D_\text{KL}(p_\text{model} \| p_\text{real})\right]\] 这使得策略学会避免进入世界模型不可靠的状态区域。

💡 深度案例
场景：MuJoCo 连续控制任务
  1. 环境：HalfCheetah, Walker2d, Hopper
  2. 对比：GIRL vs MBPO vs Dreamer v3
  3. 结果：GIRL 在相同样本预算下达到更高的累积回报，特别是在需要长期规划的 Walker2d 上优势明显，因为其自适应 horizon 允许在稳定区域使用更长的想象轨迹
📊 实验结果
环境MBPODreamer v3GIRL (本文)
HalfCheetah-v4~11000~12000~13500
Walker2d-v4~4500~5000~5800
Hopper-v4~3200~3400~3700
🔍 消融与讨论

固定 horizon 消融表明 horizon=1 过于保守（类似 model-free），horizon=50 导致幻觉累积和性能下降，而 GIRL 的自适应 horizon 在低不确定性时自动扩展到 30-50 步，高不确定性时收缩到 5-10 步。幻觉正则化项的权重 λ 在 0.01-0.1 范围内较为鲁棒。

🎯 结论与启发

GIRL 为 MBRL 中的幻觉问题提供了原则性的信息论解决方案。核心贡献是将"何时信任世界模型"形式化为可量化的信息论问题，而非依赖启发式的 horizon 选择。这一思路对 LLM Agent 中的世界模型（如用 LLM 预测环境反馈）同样有启发价值。

An Imperfect Verifier is Good Enough: Learning with Noisy Rewards

📅 2026-04-09 · arXiv: 2604.07666 🏛 Handshake AI, ETH Zurich RL RLVR 噪声奖励
📌 背景与动机

RLVR 基于"完美验证器"的理想假设，但实践中验证器总有误差：字符串相等验证器可能拒绝数学等价表达式，LLM-as-Judge 更会引入系统性偏差。关键问题尚未回答：RLVR 实际需要多高的验证器准确率？何时验证器"足够好"？

🔬 核心方法详解
① 控制噪声注入实验设计

在代码生成任务上，以无噪声测试用例验证器为基线，系统性注入 5%-40% 的噪声率。噪声类型包括：(1) 受控均匀翻转（随机将正确判为错误或反之）；(2) 现实模型噪声（用较弱 LLM 评判替代测试用例）。

② 精确度 vs 召回率的非对称影响

实验发现 precision（对"正确"判断的精确性）比 recall（对所有正确样本的覆盖率）更重要。高 false positive（将错误判为正确）比高 false negative（将正确判为错误）对训练的损害更大。

③ 跨模型家族和领域的泛化验证

在 Qwen3、GLM4、Llama 3.1 三个模型家族（4B-9B）上测试，并将结论推广到科学推理领域，确认噪声鲁棒性是通用属性而非特定于某个模型或领域。

💡 深度案例
场景：代码生成 RLVR 中的 15% 噪声率
  1. 设置：使用 Qwen3-4B 进行代码 RLVR 训练，测试用例验证器注入 15% 均匀噪声
  2. 结果：峰值验证准确率与干净基线的差异在 2 个百分点以内
  3. 对比：30% 噪声率开始出现显著下降，40% 噪声率导致训练不稳定
📊 实验结果
噪声率Qwen3-4BGLM4-9BLlama3.1-8B
0% (clean)基线基线基线
5%Δ < 1ppΔ < 1ppΔ < 1pp
15%Δ < 2ppΔ < 2ppΔ < 2pp
30%Δ ~5ppΔ ~4ppΔ ~6pp
🔍 消融与讨论

False positive（将错误判为正确）的危害约为 false negative 的 2-3 倍，因为它直接强化了错误行为。这意味着在设计验证器时，precision 应优先于 recall。将工程精力投入到将验证器从 85% 提升到 95% 的收益递减明显——"不完美的验证器足够好了"。

🎯 结论与启发

这是 RLVR 领域的重要实证贡献：系统性证明了 RLVR 对验证器噪声的鲁棒性远超预期（≤15% 噪声几乎无影响）。核心实践建议：(1) 优先保证 precision 而非 recall；(2) 当验证器准确率达到 ~85% 后，继续提升的边际收益递减；(3) 这为使用 LLM-as-Judge 进行 RLVR 训练提供了理论支撑。

📌 背景与动机

RLVR 在有可验证奖励的领域（数学、代码）取得突破，但在专业或新领域中缺乏验证器。测试时适应方法可利用未标注测试数据，但现有方法从静态查询集学习，容易过拟合到文本模式。

🔬 核心方法详解
① 在线变分合成

将静态测试查询转化为动态的语义等价变体流。对每个查询生成多个在语义上等价但表达不同的变体，迫使模型学习底层问题逻辑而非表面模式。

② 测试时混合探索

平衡准确性驱动的利用（exploitation）与一致性驱动的探索（exploration）。利用阶段选择最一致的答案，探索阶段在合成变体间检测不一致性作为学习信号。

③ 无标签自进化

完全使用未标注测试数据，通过合成变体间答案一致性提供隐式奖励，驱动模型在测试时自我进化。

💡 深度案例
场景：8 种架构上的测试时适应
  1. 方法：仅使用未标注测试集，TTVS 在推理时动态生成查询变体
  2. 结果：不仅超越其他测试时适应方法，还超越了在大规模标注数据上训练的 SOTA RL 方法
  3. 意义：仅靠未标注数据的测试时适应可以媲美甚至超越有监督训练
📊 实验结果
方法数据需求平均性能
基线 (无适应)无基线
其他测试时方法未标注测试集+3-5%

🔍 消融与讨论

变分合成的关键在于语义保持：如果合成变体偏离原始语义，则一致性信号变得不可靠。混合探索策略优于纯利用或纯探索，与 RL 中的 exploration-exploitation 权衡一致。

🎯 结论与启发

Synthetic Data for any Differentiable Target: The Dataset Policy Gradient

📅 2026-04-09 · arXiv: 2604.08423 🏛 CMU RL 合成数据 策略梯度
📌 背景与动机

合成训练数据生成的核心问题是：如何精确控制生成器使其产出最优化下游模型性能？现有方法依赖启发式的数据质量评估或手工规则。本文将合成数据生成建模为 RL 问题，提出 Dataset Policy Gradient (DPG) 直接优化任意可微目标。

🔬 核心方法详解
① 将数据生成建模为策略

合成数据生成器 \(G_\phi\) 被视为策略网络，每次生成一个训练样本即为一个"动作"。学生模型在生成数据上训练后的验证性能作为"奖励"。

② Dataset Policy Gradient

通过 REINFORCE 风格的梯度估计优化生成器： \[\nabla_\phi J = \mathbb{E}_{D \sim G_\phi}\left[R(D) \nabla_\phi \log G_\phi(D)\right]\] 其中 \(R(D)\) 是在数据集 \(D\) 上训练后的学生模型在验证集上的性能。

③ 可微目标的端到端优化

当目标函数对数据集可微时，可以通过反向传播直接优化（而非 REINFORCE），大幅降低梯度方差。这适用于许多常见场景，如在固定评估集上最小化交叉熵。

💡 深度案例
场景：代码生成的合成数据优化
  1. 目标：生成合成代码训练数据，最大化学生模型在 HumanEval 上的 pass@1
  2. DPG 过程：LLM 生成器产出代码题+解，学生模型训练后评估，梯度反传优化生成器
  3. 结果：DPG 生成的数据优于启发式过滤数据和随机采样数据
📊 实验结果
数据选择方法下游性能相对提升
随机采样基线
启发式质量过滤+5-8%
DPG (本文)+12-18%
🔍 消融与讨论

DPG 的主要开销在于需要对每批生成数据执行"训练学生→评估→反传"的内循环，计算量约为标准训练的 3-5 倍。可微目标版本显著降低方差但限制了目标函数的选择。

🎯 结论与启发

DPG 将合成数据生成从启发式工程提升为可优化的 RL 问题。"数据即策略"的视角为合成数据领域提供了原则性的优化框架，可以精确控制数据生成以最大化任意下游目标。

📅 2026-04-09 · arXiv: 2604.07659 🏛 多机构 Medical 医疗预测 内部记忆
📌 背景与动机

LLM 在医疗预测（再入院、死亡率、疾病诊断）中的应用受限于如何高效利用患者的大量历史医疗记录。标准方法将所有记录作为长上下文输入，但这既昂贵又超出上下文窗口限制。内部记忆检索提供了更高效的替代方案。

🔬 核心方法详解
① 参数化记忆编码

将患者历史记录编码为 LLM 内部的参数化记忆向量，而非作为显式文本输入。这允许在推理时以近乎零成本访问完整的患者历史。

② 选择性记忆检索

根据当前查询动态检索最相关的记忆片段，而非加载全部历史。使用注意力机制评估每个记忆向量与当前任务的相关性。

③ 多尺度时序建模

在不同时间粒度（日、周、月）上维护记忆层次，捕捉从急性事件到慢性趋势的多尺度信息。

💡 深度案例
场景：ICU 死亡率预测
  1. 输入：患者 3 年的电子健康记录（~500 个时间事件）
  2. 标准方法：需要 128K token 上下文窗口，推理时间 45 秒
  3. 本文方法：编码为 32 个记忆向量，检索 top-8 相关记忆，推理时间 2 秒，AUC-ROC 提升 2.3%

🎯 结论与启发

参数化记忆为 LLM 在医疗长序列任务中提供了可扩展的解决方案：20x 推理加速同时提升预测质量。核心启示是"记住什么"比"记住多少"更重要。

Decompose, Look, and Reason: Reinforced Latent Reasoning for VLMs

📅 2026-04-09 · arXiv: 2604.07518 🏛 多机构 Multimodal VLM 潜在推理
📌 背景与动机

VLM 在复杂视觉推理中常因文本 CoT 中的视觉信息损失而表现不佳。现有方法要么增加工具调用成本，要么依赖局部 patch 的粗粒度注意力。本文提出强化潜在推理（Reinforced Latent Reasoning），在潜在空间中保留完整视觉信息的同时执行推理。

🔬 核心方法详解
① 分解（Decompose）

将复杂视觉推理任务分解为多个子问题，每个子问题关注图像的特定区域或属性。

② 观察（Look）

对每个子问题，在视觉特征空间中执行定向注意力，提取与子问题最相关的视觉信息——而非将视觉信息转换为文本描述。

③ 推理（Reason）

在潜在空间中整合多个子问题的视觉信息进行推理，使用 RL（如 RLVR/GRPO）训练模型学会最优的分解和观察策略。

💡 深度案例
场景：复杂图表推理
  1. 输入：包含多个子图的复杂统计图表 + 问题"红色线在 2020-2023 年的增长率是否超过蓝色线？"
  2. 标准 VLM：文本 CoT 可能丢失精确数值，导致比较错误
  3. 本文方法：分解为"找红线 2020/2023 值"+"找蓝线 2020/2023 值"+"比较增长率"，每步在视觉空间直接操作

Tool-augmented+3%+5%+2%

🔍 消融与讨论

RL 训练对于学习最优分解策略至关重要——监督学习的分解策略在分布外任务上泛化性较差。潜在空间推理在需要精确数值的任务（如图表、数学图像）上优势最大。

🎯 结论与启发

DLR 证明了在视觉推理中"先分解再观察"的策略优于"先描述再推理"的文本 CoT。核心贡献是将 RL 引入 VLM 的推理策略学习，使模型自动学会最优的视觉信息提取和组合方式。

Alloc-MoE: Budget-Aware Expert Activation Allocation for Efficient Mixture-of-Experts Inference

📅 2026-04-09 · arXiv: 2604.08133 🏛 DeepSeek Industry Lab 🏢 DeepSeek MoE推理 预算感知
📌 背景与动机

MoE 架构（如 DeepSeek-V3）通过稀疏激活实现参数效率，但推理时每个 token 固定激活 top-k 专家的策略未考虑 token 难度差异：简单 token 可能只需 2 个专家，复杂 token 可能需要 8 个。Alloc-MoE 提出预算感知的动态专家分配。

🔬 核心方法详解
① 动态预算分配器

轻量级分配网络预测每个 token 的专家激活数量，在总预算约束下最大化输出质量。分配器通过 token 的路由得分分布的熵来估计所需专家数：高熵（均匀分布）需更多专家，低熵（集中分布）需更少专家。

② 全局预算约束

在序列级别设定总激活预算 \(B = N \times k_{avg}\)，其中 \(k_{avg}\) 为平均每 token 专家数。分配器在此约束下优化各 token 的分配，确保总计算量不变的同时重新分配资源。

③ 训练后适配

Alloc-MoE 作为即插即用的推理优化，不需要重新训练 MoE 模型本身。分配器在少量校准数据上训练，可直接应用于已有的 MoE 模型。

💡 深度案例
场景：DeepSeek-V3 推理优化
  1. 原始：每 token 固定激活 top-8 专家（共 256 专家）
  2. Alloc-MoE：简单 token（如标点、常见词）激活 2-4 专家，复杂 token（推理关键步骤）激活 12-16 专家
  3. 结果：平均激活降至 6 专家（节省 25% 计算），但基准性能持平甚至微升

🔍 消融与讨论

路由熵是有效的难度代理指标：高熵 token 确实需要更多专家才能维持质量。在 k_avg=6 的预算下，几乎所有基准的性能损失在 0.5% 以内。进一步压缩到 k_avg=4 开始出现可感知的性能下降。

🎯 结论与启发

Alloc-MoE 证明 MoE 推理中存在显著的计算冗余——并非所有 token 都需要相同数量的专家。动态分配策略在保持性能的同时实现 25% 推理加速。这一思路可推广到其他稀疏架构的推理优化。

RAG 的核心挑战是检索到的知识常被 LLM 忽略——模型倾向于使用参数化记忆而非检索文档中的信息。现有方法通过微调提升知识整合，但这需要标注数据且难以泛化。本文提出联合解码方法，在推理阶段保证知识整合。

🔬 核心方法详解
① 双流并行解码

同时运行两个解码流：一个以检索文档为条件（知识增强流），另一个无条件生成（基线流）。最终输出通过两个流的 logit 差异动态融合。

② 自适应融合权重

当两个流的输出高度一致时（说明基线流已掌握该知识），减少知识增强流的权重；当两个流分歧大时（说明需要检索知识），增加知识增强流的权重。

  3. 标准 RAG：模型可能忽略文档，输出过时的参数知识
  4. 联合解码：检测到两流分歧→提升知识增强流权重→正确输出文档中的答案
