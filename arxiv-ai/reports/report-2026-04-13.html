🧠 LLM — 纯文本大模型

RecaLLM: Addressing the Lost-in-Thought Phenomenon with Explicit In-Context Retrieval

👤 Kyle Whitecross, Negin Rahimi 🏛 University of Massachusetts Amherst 📅 2026-04-10 | LLM Long Context Reasoning
📌 背景与动机

推理型 LLM（如 DeepSeek-R1、Qwen3）在长链思维（CoT）推理中表现出色，但作者发现了一个被忽视的致命缺陷——"沉思迷失"（Lost-in-Thought）：模型在进行推理后，上下文检索能力会急剧下降。即使推理答案正确，模型也无法从上下文中准确提取对应的值。在 5 个 7-8B 开源 LLM 上的实验显示，128K 上下文长度下，推理后的检索准确率比直接检索低 40-60 个百分点。这揭示了 test-time scaling 的一个核心瓶颈。

🔬 核心方法详解
① Recall Tokens 与 Recall Spans

在基础词汇表 \(\mathcal{V}\) 中添加两个特殊标记：<|start_recall|>（\(R_{\text{start}}\)）和 <|end_recall|>（\(R_{\text{end}}\)）。模型生成序列在推理与检索之间交替：

$$(\underbrace{y_1, \ldots, y_T}_{\text{reasoning}},\; R_{\text{start}},\; \underbrace{r_1, \ldots, r_K}_{\text{recall span}},\; R_{\text{end}},\; \underbrace{y_{T+1}, \ldots}_{\text{reasoning}}, \ldots)$$

模型自主决定何时需要检索以及检索什么内容，recall span 内的 token 通过约束解码保证逐字复制。

② 上下文感知约束解码（Context-Aware Constrained Decoding）

在 recall span 内部，生成被约束为上下文中的有效延续。具体来说，对输入序列 \(x = (x_1, \ldots, x_n)\)，recall span 中每个 token \(r_k\) 的采样被限制在：能够延续已匹配上下文子串的 token 集合中。这通过维护一个候选匹配位置集合实现，每生成一个 token 就更新匹配集，确保 recall span 始终是上下文的逐字子串。该机制的计算开销可忽略不计（\(O(n)\) 预处理后每步 \(O(1)\) 查询）。

③ SFT 冷启动 + RL 训练框架

训练分两阶段：(a) SFT 冷启动：在合成数据上教会模型使用 recall token 的基本格式，数据包含多跳QA、单跳QA、KV检索等任务，上下文长度 8-10K tokens；(b) RL 训练：使用 GRPO 算法在 4 个 A100 GPU 上训练，16 rollouts/example。奖励函数包含两部分：答案正确性奖励 \(R_{\text{ans}}\) 和上下文检索奖励 \(R_{\text{ret}}\)（基于 character-level F1 overlap），总奖励为 \(R = R_{\text{ans}} + \lambda \cdot R_{\text{ret}}\)。

④ 密度惩罚与正确性惩罚

为防止滥用 recall span，引入两种惩罚：密度惩罚在 recall span 频率超过阈值时指数衰减奖励；正确性惩罚检测格式错误的 recall span（如极短 span 或不匹配的 start/end 标记）。这确保模型学会有选择性地、高质量地使用检索能力。

💡 深度案例
场景：128K 上下文的多跳 QA

输入包含一个大型 JSON 字典（~100K tokens 的键值对）和一个数学问题。模型需要：

  1. 先推理数学问题得到查询键（如 "key_7342"）
  2. 从 128K 上下文中找到对应值

Baseline（Qwen2.5-7B）：数学准确率 95%，但推理后检索准确率仅 31%（直接检索为 89%），差距 58pp

RecaLLM：数学准确率 95%，推理后检索准确率 87%。模型在推理完毕后自动插入 <|start_recall|>，约束解码确保逐字复制正确值。

🔍 消融与讨论

消融实验显示：(1) 去除约束解码后，recall span 中仅 43% 能实现逐字匹配（有约束时为 100%），RULER 下降 6.2 点；(2) 去除 RL 训练仅用 SFT 时，RULER 下降 8.7 点，说明 RL 是学会有效检索的关键；(3) 仅使用 10K 训练上下文即可泛化到 128K 评测，远短于 ProLong 的 512K 训练序列。局限性在于 recall span 增加了输出长度（平均增加 15-20%），对推理延迟有一定影响。

🎯 结论与启发

RecaLLM 首次揭示并系统解决了推理 LLM 的"沉思迷失"现象。通过将上下文检索从隐式能力转变为显式、可验证的操作，RecaLLM 在 10K 训练长度下实现了 128K 的泛化，这为长上下文能力的高效扩展提供了全新路径——与其训练超长序列，不如教会模型更好地检索。

Large Language Models Generate Harmful Content Using a Distinct, Unified Mechanism

👤 Hadas Orgad, Boyi Wei, Kaden Zheng, Martin Wattenberg, Peter Henderson, Seraphina Goldfarb-Tarrant, Yonatan Belinkov 🏛 Kempner Institute / Harvard University, Princeton University 📅 2026-04-10 | LLM Safety Mechanistic Interpretability
📌 背景与动机

LLM 经过对齐训练后仍然脆弱：越狱攻击能轻易绕过安全护栏，甚至在窄领域微调后出现"涌现式错误对齐"（emergent misalignment）。核心问题在于：有害性在模型内部是否存在一个连贯的、统一的机制？如果有害内容的生成依赖于分散的、独立的组件，那么逐一修补将极其困难；但如果存在统一机制，则有望实现更彻底的安全干预。

🔬 核心方法详解
① 靶向权重剪枝作为因果干预（Targeted Weight Pruning as Causal Intervention）

将权重剪枝重新定义为因果探针工具。对模型中的每个参数 \(w_{ij}\)，计算其"有害性重要度得分"（harmfulness importance score），然后按得分排序剪枝。如果移除少量高得分参数就能显著降低有害输出，而不影响通用能力，则说明有害性集中在特定子网络中。具体度量使用：

$$I_{\text{harm}}(w_{ij}) = \left|\frac{\partial \mathcal{L}_{\text{harm}}}{\partial w_{ij}}\right| \cdot |w_{ij}|$$

其中 \(\mathcal{L}_{\text{harm}}\) 是在有害提示数据集上计算的损失。

② 跨类别一致性分析

关键发现：不同类别的有害内容（如暴力、偏见、非法活动等）所依赖的参数高度重叠。定义参数集 \(S_c\) 为类别 \(c\) 排名前 \(k\)% 的参数，跨类别 Jaccard 相似度：

$$J(S_i, S_j) = \frac{|S_i \cap S_j|}{|S_i \cup S_j|}$$

实验在 Llama-3.1-8B 和 Qwen2.5-7B 上测量，\(J\) 值在 0.72-0.89 之间，远高于随机基线（~0.02），证明有害性由一个统一的子网络驱动。

③ 越狱与涌现错误对齐的统一解释

将分析扩展到越狱成功的样本和"涌现式错误对齐"的微调模型：发现两者激活的有害子网络与原始有害提示高度一致。这解释了为何：(a) 越狱攻击能跨类别泛化；(b) 窄领域微调能导致广泛错误对齐——因为它们都通过同一个子网络"绕过"安全层。

④ 安全性干预：统一子网络剪枝

基于发现，提出统一剪枝干预：仅使用少量有害样本（~50条）识别有害子网络，剪枝后在 HarmBench 上的攻击成功率（ASR）从 62.3% 降至 8.7%，同时在 MMLU 和 GSM8K 上的性能保持在 98% 以上。这比现有的安全微调方法更高效（无需重训练）且更鲁棒。

💡 深度案例
案例：跨类别有害子网络的可视化

在 Llama-3.1-8B-Instruct 上，分别用 4 个有害类别（暴力、偏见、非法活动、隐私侵犯）计算参数重要度得分，取各类别 top-0.1% 参数：

  • 暴力 ∩ 偏见：重叠率 78.3%
  • 暴力 ∩ 非法活动：重叠率 82.1%
  • 四类别交集：占各类别 top 参数的 72.6%
  • 这 72.6% 的共享参数主要集中在 中间层（layers 12-20）的 MLP 模块中

剪枝这些共享参数后，所有 4 个类别的 ASR 同时大幅下降，证实了统一机制假说。

🎯 结论与启发

本文首次从因果角度证明 LLM 的有害内容生成依赖于一个跨类别统一的内部机制，而非分散的独立组件。这一发现既解释了越狱攻击的泛化性和涌现错误对齐的广泛性，也为安全干预提供了新范式——无需重训练的靶向子网络剪枝。这对 LLM 安全研究具有范式级别的启示。

How does Chain of Thought decompose complex tasks?

👤 Amrut Nadgir, Vijay Balasubramanian, Pratik Chaudhari 🏛 University of Pennsylvania 📅 2026-04-10 | LLM Chain-of-Thought Theory
📌 背景与动机

Chain-of-Thought (CoT) 已成为提升 LLM 推理能力的标准技术，但其理论机制仍不清楚。直觉上 CoT 将复杂问题分解为简单子问题，但缺乏定量解释。本文从统计物理学的视角出发，建立了 CoT 效果的幂律理论：证明分类误差与类别数呈幂律关系，而 CoT 的核心作用是将指数级困难的大类分类问题转化为多个小类分类子问题。

🔬 核心方法详解
① 分类误差的幂律缩放

将 LLM 任务建模为多类分类：给定上下文，在 \(C\) 个可能答案中选一个。理论分析表明误差服从幂律：

$$\epsilon(C) \propto C^{-\alpha}$$

其中 \(\alpha > 0\) 依赖于模型容量和任务难度。关键推论：当 \(C\) 很大时，即使强模型也面临巨大误差。这在 5 个 LLM 和 12 个基准上得到了实证验证（\(R^2 > 0.95\)）。

② CoT 作为类别空间分解

CoT 的数学机制：将原始 \(C\) 类问题分解为 \(T\) 个步骤，每步面对 \(C_t\) 类子问题，满足 \(\prod_{t=1}^{T} C_t \approx C\)。总误差从单步的 \(C^{-\alpha}\) 变为：

$$\epsilon_{\text{CoT}} \leq \sum_{t=1}^{T} C_t^{-\alpha} \leq T \cdot \left(\frac{C}{T}\right)^{-\alpha/T}$$

当 \(C\) 足够大时，\(\epsilon_{\text{CoT}} \ll \epsilon(C)\)，解释了 CoT 为何在复杂任务上效果显著而在简单任务上提升有限。

③ 实证验证：合成任务的精确控制

设计合成任务精确控制类别数 \(C\)（从 10 到 10,000）和分解步骤 \(T\)（1-5），在 GPT-4o、Llama-3.1-70B 等模型上验证幂律预测。结果表明理论预测与实验吻合度极高，\(\alpha\) 值在 0.3-0.8 之间。

④ 对 Test-Time Compute 的推论

理论暗示 test-time scaling 的收益取决于任务的有效类别数。对于 \(C\) 较小的简单任务（如情感分析，\(C \leq 5\)），增加 CoT 步骤收益递减；对于 \(C\) 极大的复杂推理（如数学证明），CoT 的边际收益持续增长。这为何时使用长 CoT 提供了理论指导。

💡 深度案例
案例：合成多位数乘法（C = 10,000）

任务：两个 2 位数相乘，答案空间 \(C = 10{,}000\)（0-9999）。

  • 直接回答（T=1）：GPT-4o 准确率 62.3%，Llama-70B 准确率 41.7%
  • CoT 分解为 3 步（T=3）：先算个位 × 个位（C₁=100），再算十位（C₂=100），最后加总（C₃=10），理论预测准确率 ~89%
  • 实际结果：GPT-4o 91.2%（+28.9pp），Llama-70B 83.5%（+41.8pp）

幂律预测误差 < 3pp，验证了理论的精确性。

🔍 消融与讨论

关键消融发现：(1) 幂律指数 \(\alpha\) 与模型规模正相关（7B: \(\alpha\)≈0.35, 70B: \(\alpha\)≈0.62, GPT-4o: \(\alpha\)≈0.78），说明更大模型从 CoT 中获益更多；(2) 分解粒度存在最优点——步骤太多时中间步骤的累积误差开始主导（\(T > 5\) 后收益递减）；(3) 该理论无法解释非分类型任务（如开放生成），这是主要局限。

🎯 结论与启发

本文提供了 CoT 为何有效的首个定量理论：分类误差的幂律缩放使得 CoT 的类别空间分解获得超线性收益。这不仅解释了 CoT 的经验现象，也为设计最优推理策略（何时分解、分几步）提供了理论工具。统计物理视角与 LLM 理论的结合值得更多探索。

\(p_1\): Better Prompt Optimization with Fewer Prompts

👤 Zhaolin Gao, Yu Wang, Bo Liu, Thorsten Joachims, Kianté Brantley, Wen Sun 🏛 Cornell University, Microsoft, Harvard University, Databricks AI Research 📅 2026-04-10 | LLM Prompt Optimization
📌 背景与动机

Prompt 优化是无需更新权重即可提升 LLM 性能的重要技术，但其效果因任务而异。核心问题是：什么样的任务适合 Prompt 优化？为什么有些任务即使用最好的 Prompt 也提升不大？本文从方差分解的角度回答这个问题，并提出了一种只需极少 Prompt 候选的高效优化算法 \(p_1\)。

🔬 核心方法详解
① 奖励方差分解理论

将总方差 \(\text{Var}[R]\) 分解为两个正交分量：

$$\text{Var}[R] = \underbrace{\text{Var}_p[\mathbb{E}_y[R|p]]}_{\text{prompt 间方差}} + \underbrace{\mathbb{E}_p[\text{Var}_y[R|p]]}_{\text{生成随机性方差}}$$

其中 \(p\) 是 system prompt，\(y\) 是生成的回答。只有 prompt 间方差才是可优化的——如果它很小，则换 prompt 没用。作者提出 可优化性比率 \(\rho = \text{Var}_p / \text{Var}[R]\) 作为任务筛选指标。

② 信号-噪声感知的搜索策略

现有方法（如 OPRO、PromptBreeder）盲目搜索大量候选 Prompt，忽视了 \(\rho\) 的大小。\(p_1\) 算法的核心思想：当 \(\rho\) 小时少搜（甚至不搜），\(\rho\) 大时精搜。具体通过在前 \(k\) 个候选上估计 \(\rho\)，再决定后续搜索预算。使用 ANOVA F-test 估计方差分量，仅需 5-10 个候选即可获得可靠的 \(\rho\) 估计。

③ 迭代贝叶斯优化框架

对于高 \(\rho\) 任务，使用贝叶斯优化在 prompt 的语义嵌入空间中搜索。以 LLM 生成的 prompt 变体作为候选，用高斯过程建模 \(\mathbb{E}_y[R|p]\) 的后验，通过 Expected Improvement 获取函数选择下一个候选。整个流程平均只需评估 12-15 个候选（现有方法需 100+）。

💡 深度案例
案例：GSM8K vs. Sentiment Analysis 的可优化性对比
  • GSM8K（数学推理）：\(\rho = 0.73\)，prompt 间方差占主导。用 "Think step by step" vs. "Solve directly"：准确率差 12.5pp → 值得优化
  • SST-2（情感分析）：\(\rho = 0.08\)，生成随机性方差占主导。最好和最差 prompt 差距仅 1.2pp → 不值得优化
  • \(p_1\) 策略：在 GSM8K 上分配更多搜索预算（15 个候选），SST-2 上仅尝试 3 个后即停止，总计节省 73% 的 API 调用

A Little Rank Goes a Long Way: Random Scaffolds with LoRA Adapters Are All You Need

👤 Allen Discovery Center Team 🏛 Allen Discovery Center / Tufts University, TU Wien, Wyss Institute 📅 2026-04-10 | LLM LoRA Parameter Efficiency
📌 背景与动机

神经网络的参数中有多少真正编码了任务特定信息？本文通过一个极端实验回答：LottaLoRA——将所有骨架权重随机初始化并冻结，仅训练低秩 LoRA 适配器。如果这种"随机骨架 + LoRA"方案仍能工作，就证明任务信息实际上只需要极少量参数，绝大部分参数只是提供了一个"支架"（scaffold）结构。

🔬 核心方法详解
① 随机骨架范式

标准训练中，所有参数 \(W \in \mathbb{R}^{m \times n}\) 都从预训练初始化并微调。LottaLoRA 将每层权重替换为随机高斯矩阵 \(W_{\text{rand}} \sim \mathcal{N}(0, \sigma^2/n)\)，然后仅训练 LoRA 分解 \(\Delta W = BA\)，其中 \(B \in \mathbb{R}^{m \times r}\)，\(A \in \mathbb{R}^{r \times n}\)，\(r \ll \min(m,n)\)。最终权重为 \(W_{\text{rand}} + BA\)。

② 秩的惊人发现

核心实验结果：在 9 个基准（涵盖 NLU、NLG、代码、数学、推理）上，秩 \(r=4\) 的 LoRA 适配器加上随机骨架就能达到完全微调 90% 以上的性能。可训练参数仅占总参数的 0.2-0.5%。更令人意外的是，秩 \(r=1\) 仍能保留 75-80% 的性能，说明每层仅需 1 个方向向量就能编码大部分任务信息。

③ 不同随机种子的鲁棒性

在 100 个不同随机种子下重复实验，性能标准差仅 1.2-2.1pp，说明具体的随机值不重要——重要的是参数矩阵提供的维度结构（即矩阵的行列空间维度，而非具体基向量方向）。这与近期关于"随机特征"（random features）理论的结论一致。

④ 与预训练权重的对比分析

对比"随机骨架+LoRA" vs. "预训练骨架+LoRA" vs. "全参数微调"三种范式。预训练骨架 + LoRA 比随机骨架好 5-8pp，而全参数微调仅再好 2-3pp。这意味着预训练的主要价值不是精确的权重值，而是某种更抽象的结构先验。

Bridging SFT and RL: Dynamic Policy Optimization for Robust Reasoning

👤 Tsinghua University Team 🏛 Tsinghua University, ZTE, Chongqing University 📅 2026-04-10 | RL SFT Policy Optimization
📌 背景与动机

LLM 后训练面临 SFT 与 RL 的根本困境：SFT 提供稳定性（低方差）但受限于训练数据质量；RL 提供探索能力（低偏差）但训练不稳定。现有方法要么二选一，要么简单串接（先 SFT 后 RL），缺乏理论指导的动态平衡机制。本文提出 DynPO，通过信息论框架将 SFT 和 RL 的权衡形式化，并提供自适应混合策略。

🔬 核心方法详解
① 偏差-方差分解与动态温度

将策略优化目标分解为 SFT 项和 RL 项的凸组合：

$$\mathcal{L}_{\text{DynPO}} = (1-\beta_t) \cdot \mathcal{L}_{\text{SFT}} + \beta_t \cdot \mathcal{L}_{\text{RL}}$$

动态系数 \(\beta_t\) 基于当前策略与参考策略的 KL 散度自适应调整：

$$\beta_t = \sigma\left(\frac{D_{\text{KL}}(\pi_t \| \pi_{\text{ref}}) - \tau}{\gamma}\right)$$

当策略偏离参考过多时（KL 大），增大 SFT 权重以"拉回"；当策略过于保守时（KL 小），增大 RL 权重以"推进"。

② 渐进式课程：从模仿到探索

训练分三阶段：(a) 热身期（\(\beta \approx 0.1\)）以 SFT 为主建立基线能力；(b) 平衡期（\(\beta \approx 0.5\)）SFT 和 RL 权重接近；(c) 探索期（\(\beta \approx 0.9\)）以 RL 为主进行创新探索。阈值 \(\tau\) 随训练步数线性增长，允许策略逐渐远离参考分布。

③ 双重参考策略防止崩溃

使用两个参考策略：静态参考 \(\pi_{\text{init}}\)（SFT 初始化）和滑动参考 \(\pi_{\text{ema}}\)（指数移动平均）。RL 损失中的 KL 惩罚使用 min 操作：\(D_{\text{KL}}(\pi_t \| \pi_{\text{ref}}) = \min(D_{\text{KL}}(\pi_t \| \pi_{\text{init}}), D_{\text{KL}}(\pi_t \| \pi_{\text{ema}}))\)，确保策略不会离最近的"安全锚点"太远。

💡 深度案例

  1. 步骤 0-400（热身期）：\(\beta\) 从 0.05 升至 0.25，MATH 准确率 40.1→56.3%，KL < 0.5
  2. 步骤 400-1200（平衡期）：\(\beta\) 在 0.4-0.6 波动，MATH 56.3→71.8%，此阶段 RL 探索发现新解题策略
  3. 步骤 1200-2000（探索期）：\(\beta\) 稳定在 0.85，MATH 71.8→76.4%

纯 RL（GRPO）在步骤 800 附近出现 reward hacking（KL > 5），而 DynPO 的 KL 始终 < 2.5。

🔍 消融与讨论

DynPO 提供了 SFT 与 RL 之间的理论性桥梁：通过 KL 散度驱动的动态权重调整，实现了"稳定探索"而非"稳定或探索"。双重参考策略有效防止了 reward hacking，在保持通用能力的同时获得了超越纯 RL 的推理能力提升。

Skip-Connected Policy Optimization for Implicit Advantage

👤 Fengwei Teng, Jinyi Bai, Xinhao Yao, Demi Ruohan Wang, Jiahao Zhao, Zhijiang Guo 🏛 Alibaba Group 📅 2026-04-10 | RL RLVR Policy Optimization 🏢 Alibaba
📌 背景与动机

GRPO 在 RLVR 中使用 outcome-based reward 已被证明有效，但理论上细粒度的 dense reward（为每个推理 token 提供奖励信号）应该更优。然而作者揭示了一个反直觉的现象：在实际采样预算下，Monte Carlo 估计的 dense advantage 对早期推理 token 产生高方差和符号不一致的信号，反而比 outcome-only GRPO 更差。

🔬 核心方法详解
① 诊断：Dense Reward 的方差陷阱

对推理序列 \((s_1, a_1, \ldots, s_T, a_T)\)，Monte Carlo 估计 token \(t\) 的优势：

$$\hat{A}_t^{\text{MC}} = \frac{1}{G}\sum_{g=1}^{G} R_g - \bar{R}$$

其中 \(R_g\) 是从 \(s_t\) 开始的第 \(g\) 条轨迹的回报。问题：对早期 token（\(t\) 小），从 \(s_t\) 到终止的路径变数极大，导致 \(\text{Var}[\hat{A}_t^{\text{MC}}]\) 极高，且符号频繁翻转。

② SKPO 的上游-下游分解

将推理过程分为上游（前半段推理）和下游（后半段推理）：

  • 上游：接收来自下游 Monte Carlo 采样的 dense reward，使用 single-stream 优化
  • 下游：保持 group-relative 优化（类似 GRPO），使用 outcome reward

关键创新是跳跃连接：下游的输入不仅包含上游生成的推理，还通过跳跃连接直接拼接原始问题，使模型能利用有用的上游推理同时保留绕过有缺陷推理的自由度。

③ 中位数选择策略

上游生成 \(G=8\) 条推理段，通过中位数选择（median-based selection）选出质量居中的一条作为下游的上下文。选中位数而非最优的设计直觉：最优推理可能过度拟合特定路径，中位数更具泛化性。选出的上游段与原始问题通过跳跃连接拼接后，下游再生成 \(G=8\) 条完整回答。

👤 Ashima Suvarna, Kendrick Phan, Mehrab Beikzadeh, Hritik Bansal, Saadia Gabriel 🏛 University of California, Los Angeles 📅 2026-04-09 | RL RLVR General Reasoning 🏢 Alibaba
📌 背景与动机

RLVR 在数学和代码等形式化领域取得了显著成功，但在因果推理、时序推理等通用推理任务上仍表现不佳。根本瓶颈：通用推理缺乏自动可验证的奖励信号——不像数学有答案校验器。SUPERNOVA 的核心创新：将自然语言指令本身转化为可验证的奖励源，无需外部验证器。

🔬 核心方法详解
① 自然指令奖励构建

收集 14 类通用推理任务的自然语言指令（如"判断事件 A 是否在事件 B 之前"、"从以下段落推断因果关系"），每个指令包含一个可提取的结构化判断标准。将这些标准转化为 binary verification functions \(V(y, c)\)，其中 \(y\) 是模型输出，\(c\) 是标准。奖励 \(R = V(y,c) \in \{0,1\}\)。

② 多粒度验证器设计

三级验证：(a) 答案级：最终答案与 ground truth 的精确匹配或语义匹配；(b) 格式级：输出是否遵循指令要求的结构（如列出 3 个理由）；(c) 推理级：中间推理步骤是否引用了输入中的相关证据。总奖励为三级加权和：

$$R = w_1 R_{\text{answer}} + w_2 R_{\text{format}} + w_3 R_{\text{reasoning}}$$

其中推理级奖励通过检测模型输出中是否包含输入关键信息的 n-gram 重叠来近似计算。

③ 课程式任务调度

14 类任务按难度排序（基于基线模型的初始准确率），训练早期侧重简单任务（如事实检索），后期逐渐引入复杂任务（如反事实推理）。调度权重 \(w_i^{(t)} \propto \exp(-\alpha \cdot \text{acc}_i^{(t)})\)，让模型在弱项任务上花更多训练时间。

Accelerating Semi-Autoregressive Decoding of Diffusion Language Models

👤 Lorenzo Flores, Cesare Spinoso di-Piano, Ori Ernst, David Ifeoluwa Adelani, Jackie Chi Kit Cheung 🏛 Mila - Quebec AI Institute, McGill University 📅 2026-04-10 | RL Diffusion LLM Decoding
📌 背景与动机

扩散语言模型（dLLMs）正成为自回归模型的有力替代，半自回归（Semi-AR）解码是主流推理策略。然而，现有 Semi-AR 方法存在依赖建模不足的问题：每一步并行解码的多个 token 之间缺乏依赖约束，导致生成质量和速度的权衡不理想。

🔬 核心方法详解
① 依赖感知的块选择（Dependency-Aware Block Selection）

Semi-AR 的每步解码一个 block 的 \(B\) 个 token。关键问题：哪些 token 可以安全并行解码？本文提出基于注意力模式估计 token 间依赖强度：

$$D(i,j) = \frac{1}{H}\sum_{h=1}^{H} A_{h}[i,j]$$

其中 \(A_h\) 是第 \(h\) 个注意力头的注意力矩阵。依赖强度低于阈值 \(\delta\) 的 token 可以并行生成，高于阈值的必须串行。

② 自适应块大小调整

不同位置的最优块大小不同：句子开头的 token 通常高度相关（需串行），而填充词或确定性高的 token 可用大块。自适应策略：基于当前位置的置信度（predicted token 的 softmax 概率）动态调整块大小 \(B_t\)：

$$B_t = \lfloor B_{\max} \cdot \min(p_{\max}^t / p_{\text{thresh}}, 1) \rfloor$$

③ 投机式验证（Speculative Verification）

Process Reward Agents for Steering Knowledge-Intensive Reasoning

👤 Jiwoong Sohn, Tomasz Sternal, Kenneth Styppa, Torsten Hoefler, Michael Moor 🏛 ETH Zürich, Heidelberg University 📅 2026-04-10 | Agent Process Reward Knowledge Reasoning 🏢 Alibaba
📌 背景与动机

知识密集型推理（如医学诊断、法律分析）的中间步骤难以局部验证——不像数学可以逐步检查，评估某个推理步骤是否正确可能需要综合大量外部知识。现有过程奖励模型（PRM）是事后评分（post-hoc scoring），无法在推理过程中实时纠错。本文提出 Process Reward Agents (PRA)：将 PRM 从被动评分器升级为主动引导智能体。

🔬 核心方法详解
① 智能体化的过程奖励（Agentic PRM）

PRA 在每个推理步骤后执行三个动作：(a) 检索——从知识库中主动查询与当前步骤相关的证据；(b) 验证——基于检索到的证据评估当前步骤的正确性；(c) 引导——如果步骤有误，生成修正建议并反馈给推理模型。形式化为 MDP：状态 \(s_t = (q, r_{1:t}, e_{1:t})\) 包括问题 \(q\)、推理轨迹 \(r\) 和证据 \(e\)，动作空间包括 \(\{检索, 批准, 修正\}\)。

② 检索增强的步骤验证

验证得分结合局部一致性和全局证据支持：

$$V(r_t | r_{1:t-1}, e_{1:t}) = \alpha \cdot \text{Coherence}(r_t, r_{1:t-1}) + (1-\alpha) \cdot \text{Support}(r_t, e_t)$$

其中 Coherence 由语言模型评估推理步骤与前文的逻辑一致性，Support 由检索到的证据与当前步骤的语义相似度计算。当 \(V < \tau_{\text{reject}}\) 时触发修正。

③ 知识驱动的修正生成

修正模块接收当前有误的步骤 \(r_t\)、具体的验证反馈（哪个方面出错）和检索到的正确证据 \(e_t\)，生成替代步骤 \(r_t'\)。使用 constrained generation 确保修正后的步骤引用了检索证据中的具体信息，避免"自由幻觉"式修正。

💡 深度案例

  1. 推理步骤 1："3cm 结节 + 毛刺征提示恶性可能"→ PRA 检索医学文献，验证 ✓（支持度 0.92）

  3. 修正步骤 2'："CEA 升高主要见于肺腺癌"（基于检索证据）
  4. 最终诊断：右肺上叶腺癌可能性最大 → ✓ 正确

CORA: Conformal Risk-Controlled Agents for Safeguarded Mobile GUI Automation

👤 HKU, CUHK, University of Tokyo Team 🏛 University of Hong Kong, Chinese University of Hong Kong, University of Tokyo 📅 2026-04-10 | Agent GUI Agent Safety
📌 背景与动机

VLM 驱动的 GUI 智能体正从被动辅助转向自主操作，但不受约束的动作空间暴露用户于严重且不可逆的风险——误操作可能导致财务损失、隐私泄露或社交伤害。现有安全措施依赖 Prompt 工程、启发式规则或 VLM-as-critic，缺乏形式化保障和用户可调的风险控制。CORA 引入Conformal Prediction为 GUI 智能体提供有概率保证的安全框架。

🔬 核心方法详解
① Conformal Risk Score 定义

对每个候选动作 \(a\)，定义非一致性得分（non-conformity score）：

$$s(a) = 1 - P_{\theta}(\text{safe} | a, \text{context})$$

其中 \(P_{\theta}\) 是安全分类器的输出概率。Conformal prediction 基于校准集 \(\{s_1, \ldots, s_n\}\) 计算阈值 \(\hat{q}\)，使得：

$$P(\text{unsafe action passes}) \leq \alpha$$

用户通过设定 \(\alpha\)（如 0.05 = 5% 误放率）控制风险容忍度。

② 分层风险分类

将 GUI 动作分为三个风险层级：(a) 低风险（滚动、查看）：直接执行；(b) 中风险（点击按钮、输入文本）：在 conformal set 内则执行，否则请求确认；(c) 高风险（支付、删除、发送消息）：始终需要用户确认。风险层级基于动作类型和目标 UI 元素的语义（如按钮标签包含"购买"、"删除"等关键词）。

③ 在线校准更新

conformal 阈值 \(\hat{q}\) 在使用过程中通过用户反馈在线更新。当用户确认或拒绝动作时，更新校准集并重新计算 \(\hat{q}\)：

$$\hat{q}_{t+1} = \text{Quantile}_{1-\alpha}\left(\{s_1, \ldots, s_t, s_{t+1}\}\right)$$

这使阈值适应特定用户的风险偏好和使用场景（如金融 app 用户可能需要更严格的阈值）。

💡 深度案例

  1. 打开支付 App → 低风险 → 直接执行 ✓
  2. 搜索"张三" → 中风险，conformal score 0.12 < \(\hat{q}\)=0.18 → 执行 ✓
  3. 输入金额"500" → 中风险，score 0.15 < \(\hat{q}\) → 执行 ✓
  4. 点击"确认转账" → 高风险 → 弹出确认："即将向张三转账 ¥500，确认？" → 用户确认 ✓

📌 背景与动机

长期记忆是对话智能体维持跨会话连贯性和个性化的关键，但现有方法（RAG 或图记忆）依赖被动提取——仅在需要时通过检索匹配获取记忆。这导致：(a) 检索失败时记忆完全丢失；(b) 无法主动整合和更新记忆。MemReader 将记忆管理从被动检索转变为主动提取、整合和遗忘的闭环系统。

🔬 核心方法详解
① 主动记忆提取（Active Memory Extraction）

在每轮对话结束后，MemReader 主动识别对话中的新增事实、偏好变化和关系更新，将其结构化存储。提取使用 LLM 作为信息抽取器，输出格式化的记忆条目：\(\text{mem} = (\text{entity}, \text{relation}, \text{value}, \text{confidence}, \text{timestamp})\)。

$$\text{Priority}(\text{mem}_i) = \text{conf}_i \cdot \exp(-\lambda (t_{\text{now}} - t_i))$$

当新旧记忆冲突时，保留 Priority 更高的版本。同时维护一个"历史版本"链以支持溯源。

③ 主动遗忘机制

灵感来自人类记忆的遗忘曲线：长时间未被访问且置信度低的记忆会被主动清除，防止记忆库无限膨胀。遗忘阈值基于 Ebbinghaus 曲线建模，访问频率高的记忆保留更久。

Dictionary-Aligned Concept Control for Safeguarding Multimodal LLMs

👤 Jinqi Luo, Jinyu Yang, Tal Neiman, Lei Fan, Bing Yin, Son Tran, Mubarak Shah, René Vidal 🏛 University of Pennsylvania, Amazon, University of Central Florida, Johns Hopkins University 📅 2026-04-10 | Multimodal Safety Activation Steering 🏢 Alibaba
📌 背景与动机

多模态 LLM（MLLM）容易被恶意查询诱导生成不安全内容。现有防御手段（Prompt 工程、响应分类、微调）要么无法应对新型攻击，要么需要重新运行查询，要么计算成本高。近来激活转向（activation steering）提供了灵活的推理时安全控制，但其概念表示通常是单向量，无法捕捉安全/不安全概念的多面性。本文提出 DACC（Dictionary-Aligned Concept Control）：使用字典学习获取多概念表示，实现更精细的安全控制。

🔬 核心方法详解
① 字典学习的概念分解

收集安全和不安全响应的中间层激活，使用稀疏字典学习分解为多个概念方向：

$$h = \sum_{k=1}^{K} \alpha_k d_k + \epsilon, \quad \|\alpha\|_0 \leq s$$

其中 \(d_k\) 是字典原子（概念方向），\(\alpha_k\) 是激活系数，\(s\) 是稀疏度。与 SAE（稀疏自编码器）不同，DACC 使用监督信号对齐字典原子到安全/不安全语义。

② 安全概念的对齐与选择

使用对比学习将字典原子对齐到安全标签：安全响应激活某些原子，不安全响应激活另一些原子。通过概念激活向量（CAV）识别最相关的安全概念方向 \(\{d_{k^*}\}\)，其中 \(k^*\) 是在安全/不安全分类上 AUC 最高的原子索引。

③ 推理时的多方向转向

在推理时，同时沿多个安全概念方向转向：

$$h' = h + \sum_{k \in \mathcal{S}} \lambda_k d_k$$

其中 \(\mathcal{S}\) 是选出的安全概念集合，\(\lambda_k\) 是转向强度。多方向转向比单向量更鲁棒，因为不同类型的不安全内容可能需要不同方向的纠正。

UIPress: Bringing Optical Token Compression to UI-to-Code Generation

👤 Dasen Dai, Shuoqi Li, Ronghao Chen, Huacan Wang, Biao Wu, Qizhen Lan 🏛 Alibaba Group 📅 2026-04-10 | Multimodal Vision-Language Token Compression 🏢 Alibaba
📌 背景与动机

UI-to-Code 生成要求 VLM 从一张截图生成数千 token 的 HTML/CSS 代码，视觉 token 效率至关重要。现有压缩方法要么在推理时用启发式选择 token（丢失关键信息），要么只是将低注意力特征置零（未真正缩短序列），都无法同时实现 prefill 延迟降低和生成质量保持。

🔬 核心方法详解
① 任务感知的光学 Token 压缩

UIPress 在训练阶段学习一个可微分的 token 选择模块，直接优化 UI-to-Code 任务的端到端性能。对视觉编码器输出的 \(N\) 个 token，学习二值掩码 \(m \in \{0,1\}^N\)（通过 Gumbel-Softmax 松弛），保留 \(N' = \lfloor \rho N \rfloor\) 个 token，其中压缩率 \(\rho\) 是超参数。

② UI 结构感知的选择策略

利用 UI 截图的结构特点（文本区域信息密度高，背景区域信息密度低），在训练中引入UI 布局先验：对 token 选择模块施加正则化，使其倾向保留文本/按钮/输入框区域的 token 而压缩纯色背景。布局先验通过对 UI 截图的简单梯度检测获取。

③ 实际序列缩短

与仅置零低重要性 token 不同，UIPress 在 LLM 输入前物理移除被掩码的 token，真正缩短 prefill 序列长度。这直接减少了 KV-cache 大小和 prefill 计算量，在长代码生成场景下效果尤为显著。

LLM-Rosetta: A Hub-and-Spoke Intermediate Representation for Cross-Provider LLM API Translation

👤 Peng Ding 🏛 University of Chicago 📅 2026-04-10 | Industry Lab API Interoperability 🏢 OpenAI
📌 背景与动机

LLM 提供商各自使用专有 API 格式，应用开发者需要为每对提供商编写适配器——\(O(N^2)\) 的维护成本。LLM-Rosetta 采用轮辐式中间表示（Hub-and-Spoke IR）：只需为每个提供商编写一个到 IR 的映射，就能实现任意两个提供商间的 API 互通，将复杂度降为 \(O(N)\)。

🔬 核心方法详解
① 统一中间表示设计

IR 将 LLM API 的核心概念抽象为 5 个原语：Message（消息）、Model（模型参数）、Tool（工具调用）、Stream（流式输出）和 Response（响应格式）。每个原语有标准化的 schema，覆盖 OpenAI、Anthropic、Google、Cohere 等主流提供商 95%+ 的 API 功能。

② 语义保持的翻译规则

对每个提供商定义双向映射规则 \(f_{\text{provider} \to \text{IR}}\) 和 \(f_{\text{IR} \to \text{provider}}\)。翻译规则处理语义差异：如 OpenAI 的 function_calling vs. Anthropic 的 tool_use，以及参数命名差异（max_tokens vs. max_output_tokens）。不可翻译的特性标记为可选降级。

③ 运行时适配层

提供一个轻量级的运行时中间件：应用发送 IR 格式请求 → 中间件翻译为目标提供商格式 → 转发请求 → 翻译响应回 IR → 返回应用。延迟开销 < 5ms，不影响流式输出。

💡 深度案例
案例：从 OpenAI 无缝切换到 Anthropic

原始 OpenAI API 调用：

  • model: "gpt-4o", messages: [{role: "user", content: "..."}], tools: [{type: "function", function: {...}}]

IR 中间表示：

  • model: IR_MODEL("gpt-4o"), messages: [IR_MSG(user, "...")], tools: [IR_TOOL(function, ...)]

翻译为 Anthropic API：

  • model: "claude-sonnet-4-20250514", messages: [{role: "user", content: [{type: "text", text: "..."}]}], tools: [{name: ..., input_schema: {...}}]

应用代码无需任何修改，仅需更改配置中的目标提供商。

功能覆盖差距主要来自提供商特有功能（如 Anthropic 的 thinking 模式、Google 的 Grounding API），这些无法在其他提供商上实现。IR 设计中将这些标记为"optional extensions"。与直接双边适配器相比，IR 方案在 10 个提供商的场景下将维护成本从 90 个适配器降到 10 个。局限性：快速变化的 API 需要频繁更新 IR schema。

LLM-Rosetta 用经典的编译器中间表示思想解决了 LLM 生态碎片化问题。\(O(N)\) 的维护成本和 <5ms 的延迟开销使其具有实际工程价值。这一工作也暴露了 LLM API 标准化的迫切需求。

SPASM: Stable Persona-driven Agent Simulation for Multi-turn Dialogue Generation

👤 Han Luo, Guy Laban 🏛 University of Leeds, Southwest Jiaotong University, Ben-Gurion University 📅 2026-04-10 | Industry Lab Dialogue Persona 🏢 DeepSeek
📌 背景与动机

LLM 在多轮对话中越来越多地扮演特定角色（如辅导、咨询），但 LLM-LLM 对话会累积身份相关失效——角色漂移、目标遗忘、语气不一致。这在生成合成训练数据时尤为严重。SPASM 提出一个稳定的人格驱动对话生成框架，确保跨长对话的角色一致性。

🔬 核心方法详解
① 结构化人格表示

将人格（persona）分解为 4 个维度：身份（名字、背景）、目标（对话要达成的任务）、性格（语气、表达习惯）和知识（领域专业知识）。每个维度用结构化模板表示，并在每轮对话开始时作为系统 prompt 的一部分注入。

② 一致性检查器（Consistency Checker）

在每轮生成后，运行一个轻量级的一致性检查模块，检测 4 个维度的漂移：

$$C_{\text{drift}}(t) = \sum_{d \in \{id, goal, style, know\}} w_d \cdot \text{sim}(r_t^d, p^d)$$

其中 \(r_t^d\) 是第 \(t\) 轮回复在维度 \(d\) 上的表现，\(p^d\) 是原始人格设定。当 \(C_{\text{drift}} < \tau\) 时触发重新生成。

③ 长程记忆锚定

在长对话（>10 轮）中，仅靠系统 prompt 不足以维持人格一致性。SPASM 引入记忆锚点机制：每 5 轮提取一次对话中的关键人格表现（如"在第 3 轮使用了幽默的语气"），并将这些锚点追加到后续轮次的 prompt 中，作为额外的人格约束。
