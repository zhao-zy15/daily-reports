LLM-1 · SAECache: Not All Tokens Are Worth Caching — Learning Semantic-Aware Eviction for LLM Prefix Caches

📅 2026-05-12 · arXiv:2605.18825
🏛 Peking University, FirestAI, Tsinghua University（共 6 个机构）
LLMKV CacheServing

① 背景与动机

Prefix cache 是现代 LLM serving 的关键优化——把不同请求间共享的 prompt 前缀 KV 状态复用，跳过 prefill 计算。但 GPU 显存稀缺，必须有淘汰策略。生产中普遍使用的 LRU 等策略把所有 KV 块视为同质：只看访问时间。本文作者用真实流量数据（ShareGPT + AgentBank）测得一个被忽视的事实——不同 token 类型的复用率差距高达 42 倍：system prompt 复用率 92.3%，user prompt 30.8%，response 27.8%，tool output 23.0%，而 chain-of-thought 推理痕迹仅 2.2%。把一个 system prompt 块和一个 CoT 块按相同优先级保留，等于浪费宝贵的 HBM。此外，多轮会话的复用几乎 100% 是 session-local，inter-turn 时间间隔服从 session 特定的 log-normal 分布（chat: P50=63s, P99=633s；agentic: P50=6s, P99=79s）——固定参数完全无法兼顾。

② 核心方法详解

① 多队列分区架构（Multi-Queue Routing）

SAECache 用基于规则的 partition 把每个 KV 块路由到三类专用队列：单轮模板队列（system prompt + tool def）、多轮历史队列（user/response/tool output）、CoT 推理队列。每个队列维护独立的优先级度量：

$$\text{Priority}(b) = w_{\text{type}}(t_b) \cdot w_{\text{queue}}(q_b) \cdot f(\Delta t_b, \tau_q)$$

其中 $w_{\text{type}}$ 是 token-type 权重（系统 prompt > user > response > CoT），$w_{\text{queue}}$ 是队列级权重，$f(\Delta t_b, \tau_q)$ 是与 log-normal session timescale $\tau_q$ 匹配的时间衰减函数。

② 语义感知的 token 加权（Online Reuse Weighting）

不依赖人工调参，作者把权重 $w_{\text{type}}(t)$ 设为在线学习参数。每当一个块被命中时，给该类型的权重一个正梯度；每当一个块被淘汰却随后被请求未中，给一个负梯度。形式上：

$$w_{\text{type}}^{(k+1)}(t) = w_{\text{type}}^{(k)}(t) + \eta \cdot [\mathbb{1}_{\text{hit}}(b) - \lambda \cdot \mathbb{1}_{\text{miss-after-evict}}(b)]$$

其中 $\eta$ 是学习率，$\lambda$ 是惩罚因子。整个机制不需要 offline profiling。

③ Tail-First eviction（结构性保护）

实证发现：相同 prefix 模板下，淘汰 prefix 头部块比淘汰尾部块更致命——因为头部一旦被驱逐，整段共享 prefix 就重新需要从头 prefill。SAECache 在每个队列内部强制 tail-first eviction，即同等优先级下优先驱逐尾部块。

④ Inter-Turn 自适应（log-normal 时间窗）

对每个 session 类型，SAECache 拟合 log-normal 分布 $\log \Delta t \sim \mathcal{N}(\mu_q, \sigma_q^2)$，并把生存期 $\tau_q$ 设为 $\exp(\mu_q + k\sigma_q)$（论文用 $k=2$，覆盖到 P95）。这样 agentic 工作流（短间隔）的块快速淘汰为后续高频请求让位，而 chat 工作流的块得到更长保留。

③ 深度案例

场景：一个混合 chat + agentic 的 LLM 服务，GPU KV 池仅能容纳 50% 的活跃前缀。
  • 请求 A（chat，刚结束一轮，下一轮预计 60s 后）：含 1000 token system prompt + 500 token 历史对话 + 200 token 当前 CoT 推理
  • 请求 B（agentic，每 8s 来一轮）：含 800 token system + tool def + 300 token tool output
LRU 行为：按时间排序，把刚生成完的 A 的 CoT 块也放到队尾"高优先级"；当 B 的 tool def 块被淘汰时，下一次 B 必须重 prefill 1100 token。
  1. SAECache 路由：A 的 system → 单轮模板队列（保留权重高）；A 的 CoT → 推理队列（权重 0.02 倍，立即可弃）；B 的 system + tool def → 单轮模板队列。
  2. 权重更新：online 学习 200 个请求后，CoT 权重 → 0.022，system 权重 → 0.923，差距 42×。
  3. 结果：A 的 CoT 块第一个被淘汰，B 的 tool def 永久保留，B 命中率从 LRU 的 41% 提升到 78%；端到端 TTFT 从 850ms 降到 360ms（2.36× 加速）。

⑤ 消融与讨论

消融实验显示三个组件的独立贡献：去掉 token-type weighting 损失 23% 增益；去掉自适应 queue weighting 损失 39%；去掉 tail-first 损失 11%。三者均不可或缺。局限：在线学习需要 ~500 个请求的 warm-up，冷启动期收益较小；CoT 复用率统计基于公开数据集 ShareGPT，对于代码补全等"高 CoT 复用"场景未必准确。

⑥ 结论与启发

SAECache 的核心洞见——"语义结构 + 时间分布 + 在线学习" 三轴联合驱动 KV cache 淘汰——为 LLM serving 系统打开了新的优化维度。它呼应了 LLM-2 Spherical KV 的理念：把 KV 看作 rate–distortion 分配问题；但走的是更朴素的工程路线：不需要改 attention kernel，只需在 serving runtime 加 200 行代码。对于阿里、字节、火山等推理平台，这是近期可落地的最高 ROI 改进点之一，预计将在下一波 vLLM/SGLang/TensorRT-LLM 优化中被吸收。

LLM-2 · Spherical KV: Angle-Domain Attention and Rate-Distortion Retention for Memory-Bounded LLM Inference

📅 2026-05-13 · arXiv:2605.18856
🏛 Synopsys, McGill University, Amazon（含 Google DeepMind, BITS Pilani 等共 7 个机构）
LLMKV CacheLong Context

① 背景与动机

长上下文解码瓶颈早已不是 FLOPs，而是 KV cache 体积、HBM 带宽和峰值显存。已有手段（windowing/eviction、KV quantization、offload）都"移动了瓶颈而非移除"：eviction 均匀丢历史，quantization 需 dense 重构付"reconstruction tax"，offload 把瓶颈搬到 PCIe。作者提出根本问题：能否把 KV memory 当成带几何先验的 rate–distortion 分配问题？注意力本质是 方向匹配（query–key 余弦相似度），magnitude 只是标量调制器。如果能在 critical path 上只读"方向编码"且按未来效用分配 bit，就能同时降低 bytes/token 和 HBM traffic。

② 核心方法详解

① 球面参数化（Spherical Parameterization）

每个 key vector $k \in \mathbb{R}^{d_h}$ 被分解为 $(r, \phi) = (\|k\|_2, \Phi(\hat{k}))$，其中 $\hat{k} = k / \|k\|_2$ 是单位向量，$\Phi: S^{d_h-1} \to \mathbb{R}^{d_h-1}$ 是球面到角度的映射（论文用 hyperspherical coords）。存储时：

$$\text{store}(k) = (\underbrace{Q_r^t(r)}_{b_r^t \text{ bits}}, \underbrace{Q_\theta^t(\Phi(\hat{k}))}_{(d_h-1) b_\theta^t \text{ bits}})$$

radius 与 angle 用不同 bitwidth独立量化——这是关键创新：radius 信息量低（10–20% energy），可极端压缩。

② Angle-Domain Attention（无 dense 重构）

核心难点：常规 quantized KV 在 attention 计算时必须先 dequant 出 dense vector 才能算点积。Spherical KV 直接在 angle code 上计算 logits：

$$\text{logit}(q, k) = \|q\| \cdot r \cdot \cos(\theta_q, \theta_k) \approx \|q\| \cdot r \cdot \text{CosFromAngles}(\hat{q}, c_\theta, t)$$

其中 CosFromAngles 是一个对角度码 $c_\theta$ 的查表/位运算实现的余弦递推，不需要还原出 $\hat{k}$。这彻底跳过了 "unpack/dequant/rebuild dense vector" 步骤，HBM 读取减少为 compact stream 直接 fuse 到 GPU kernel 中。

③ Rate–Distortion Retention（保留 + 精度联合分配）

给定全局比特预算 $B$，每个 (token, head, layer) item $i$ 的 tier 选择 $t_i \in \mathcal{T} = \{t_0, \ldots, t_K\}$（$t_0 = $ drop）由以下贪心解：

$$\max_{\{t_i\}} \sum_i V_i(t_i) \quad \text{s.t.} \quad \sum_i R(t_i) \le B, \quad i \in \mathcal{P} \Rightarrow t_i \ne t_0$$

其中 $V_i(t_i)$ 是该 item 在 tier $t_i$ 下的"未来效用估计"（用 prefill 阶段的 attention statistics 作 proxy），$R(t_i) = (d_h-1)b_\theta^t + b_r^t + b_{\text{meta}}^t$ 是该 tier 的 bit cost，$\mathcal{P}$ 是 protect set（如最近 token、attention sink）。算法用 value/bit ratio 排序后单次贪心。

④ Tier-Homogeneous Paging（kernel-realized）

每个 GPU 页面只包含单一 tier 的 KV，配以 32-byte 固定 header（tier id + scale + flags）。SoA（Struct-of-Arrays）布局保证 coalesced HBM 读取。这是把压缩从"理论 bytes/token 节省"翻译成"实测 tok/s 提升"的关键——避免了 mixed-tier page 导致的 warp divergence。

③ 深度案例

设定：Llama-3.1-8B，context length L=128K，dense KV ≈ 4·L·H·dh bytes/token ≈ 256 bytes/token。
  1. Prefill：对每个 (token, head) 计算 r 和 hyperspherical angles。Controller 用 prefill 中 token 的 attention probability $\bar{p}_i$ 作为 future utility proxy。
  2. Tier 分配：4 个 tier — t0=drop, t1=(bθ=2, br=4), t2=(bθ=4, br=6), t3=(bθ=8, br=8)。Sink tokens + 最近 1K 强制 t3；attention 高的 mid-context 走 t2；attention 接近零的远端走 t0 drop。
  3. 结果：有效 bytes/token 从 dense 256 降到 ~28（≈11%），HBM bytes/generated-token 减少约 6×；同时 angle-domain kernel 无 dequant 开销，throughput 在 128K 上 vs Dense KV 提升 2-3× tok/s。
  4. iso-quality 保护：quality gap δQ < 0.8 ppl points vs Dense baseline，远低于 KVQuant (≈3.2) 和 H2O (≈2.5) 在 128K 场景下的退化。

⑤ 消融与讨论

消融：(i) SphKV (Angle-only) 仅做角度域 attention 不做 RD retention，提升不显著；(ii) SphKV (RD-only) 仅做 rate-distortion 分配但需 dense 重构，省了 bytes 但 HBM 仍重；(iii) Full SphKV 两个机制 非可加，组合后才打出 dominant frontier shift。局限：球面映射的 angle code 需要 prefill 阶段统计 $V_i$，对纯流式无 prefill 场景需另设 estimator；论文未给真实开源权重模型的实测 wall-clock 数字（如 Llama-3.1-8B 在 H100 上的 tok/s），仅给 Pareto envelope，复现需要自行 implement angle-domain kernel。

⑥ 结论与启发

Spherical KV 与 LLM-1 SAECache 形成"双管齐下"：SAECache 解决"哪些 prefix block 该保留"（系统层），Spherical KV 解决"保留的 KV 该怎么编码"（算子层）。两者完全正交可叠加。更深层的启示是"attention 是 directional——所以 KV 编码应该 align with attention's geometry"——这呼应了 2026 年 5 月以来的多个 KV 优化工作（如 OSCAR 的协方差旋转、RTPurbo 的 RoPE 子空间投影）。Spherical KV 把方向编码与精度分配统一在一个 rate-distortion 框架内，可能成为下一代 long-context inference engine 的标配。

LLM-3 · Multi-Token Residual Prediction: 让扩散 LLM 一次前向解多 token，无损 1.42× 加速

📅 2026-05-12 · arXiv:2605.18817
🏛 New York University, NYU Shanghai, Nous Research（+ Modal）
LLMDiffusion LMSpeculative Decoding

① 背景与动机

Diffusion Language Models（DLMs，如 SDAR、LLaDA）通过迭代去噪生成文本，提供 parallelism 与 quality 之间的折中。每步通过 confidence threshold 控制揭示多少 token，但 quality 随每步 unmask token 数 $r$ 单调下降——baseline $r=2$ 在 MATH500 上比 $r=1$ 损失 7.0 pt 准确率（4B 模型）。能不能"更激进地多揭示，但用一个轻量模块预测下一步 logit 的修正"？作者的关键经验观察：相邻两步去噪的 logit 差异 $\delta_t = \ell_{t-1} - \ell_t$ 的 RMS 仅为完整 logits 的 ~20%（$k=1$），远比从头预测简单。

② 核心方法详解

① Logit Residual 的存在性（实证 + 理论）

定义 per-step logit residual：

$$\boldsymbol{\delta}_t = \ell_{t-1} - \ell_t \in \mathbb{R}^{L \times V}$$

实证（GSM8K + SDAR 1.7B/4B/8B）：RMS$(\boldsymbol{\delta}_t)$ 在 $k=1$ 时为 RMS$(\ell_t)$ 的 ~20%，$k=16$ 时仍 < 100%。理论：partially masked states $\boldsymbol{x}_T, \ldots, \boldsymbol{x}_0$ 是 Markov chain，相邻状态仅由 unmask 集合 $\mathcal{R}_t$ 改变，所以 $f(\boldsymbol{x}_{t-1})$ 与 $f(\boldsymbol{x}_t)$ 的差异由该 chain transition 上界。

② MRP 模块结构

MRP 是 2-3 层 transformer 模块 $g_\phi$，输入是 backbone 当前的 hidden states $\boldsymbol{h}$ 和当前 partially masked sequence $\boldsymbol{x}_t$，输出 hidden residual $\boldsymbol{\Delta}$ 与 logit residual $\boldsymbol{\delta}$：

$$(\boldsymbol{\Delta}, \boldsymbol{\delta}) = g_\phi(\boldsymbol{x}_t, \boldsymbol{h})$$

训练时 backbone 冻结，仅训练 $g_\phi$，使用 KL divergence 对齐 teacher logits：

$$\mathcal{L} = \text{KL}\big( \text{softmax}(\ell_{t-1})_{\text{masked}} \big\| \text{softmax}(\ell_t + \boldsymbol{\delta})_{\text{masked}} \big)$$

③ Direct Decoding 模式（牺牲一点 quality 换更高速）

每次 backbone forward 后，运行 $K$ 步 MRP 迭代：每步 $\boldsymbol{h} \leftarrow \boldsymbol{h} + \boldsymbol{\Delta}, \ell \leftarrow \ell + \boldsymbol{\delta}$，按阈值 $\tau$ 揭示新位置。一次 backbone forward 等价于 $K+1$ 步去噪，引入小近似误差。

④ Speculative Decoding 模式（无损加速）

把 MRP 解出的 $K$ 步 token 当作 draft，让 backbone 一次 forward 做验证（采用 Self Speculative Decoding 验证框架）。Verified pass 部分提交，错误部分回退。K=3 时实现 1.42× 无损加速。

③ 深度案例

SDAR-1.7B 解一道 GSM8K 数学题：
  1. Base (r=1)：每步揭示 1 token，accuracy 78.7%，TPS 267.4。
  2. Base (r=2)：贪心多揭示 2 token，TPS 提升到 1.86×（497），但 accuracy 暴跌到 70.8%（-7.9 pt），不可接受。
  3. Direct K=1：用 MRP 预测 1 步残差，accuracy 77.5%（-1.2 pt），TPS 1.50×（401）。Quality 损失温和。
  4. Spec K=3：MRP 解 3 步，backbone 验证。accuracy 77.8%（与 base 平齐 -0.9 pt），TPS 1.20×。在 8B 上 Spec K=3 反而比 base accuracy 略高（90.7% vs 91.0%）—— verification 严格 lossless。
  5. 实操关键：对代码（HumanEval）任务 direct 模式 accuracy 跌得多（48.2% vs 53.7%），因为 code token 分布转移更陡——这是 MRP 的 known limitation。speculative 模式则把这个问题完全消除。

⑤ 消融与讨论

残差 vs 直接蒸馏：同一架构、同一数据下，用残差目标训 vs 直接蒸馏目标训，在 $K=4$ 时残差版本平均提升 +65 pt accuracy——证明利用相邻 logit 相似性的归纳偏置是有效的，而非简单地"训了一个 draft model"。MRP 深度 sweep：2-3 层是 sweet spot，超过 4 层 accuracy 平台、throughput 下降——验证残差信号确实低复杂度。局限：code generation 在 direct 模式损失较多（code token 分布更陡）；MRP 训练数据需要来自 backbone 的 hidden states，跨 backbone 迁移性弱。

⑥ 结论与启发

MRP 对扩散 LLM 提出了"speculative decoding for diffusion"的范式，类似于 autoregressive Medusa/EAGLE：把昂贵的 backbone 一次前向当作 oracle，用轻量 head 解多步。更深层的启示：大模型推理中，"邻近 distribution 高度相似"是普遍现象（既适用于 autoregressive 邻近 token，也适用于 diffusion 邻近 step），只要能 cheap 地刻画残差，就能换 throughput。对于希望部署扩散 LLM（如 SDAR、Mercury Coder、Inception）的厂商，MRP 是近期可立刻接入 SGLang/vLLM 的 1.4× 免费加速。

RL-1 · LambdaPO: Lambda 风格的 Policy Optimization，用成对偏好结构替代 GRPO 的标量优势

📅 2026-05-19 · arXiv:2605.19416
🏛 Pinterest, Facebook (Meta), University of Michigan（+ Mississippi State, CMU, Emory）
RLGRPOMath Reasoning

① 背景与动机

GRPO 已成为 reasoning LLM 的 RLHF 主流——通过 group 内 reward 归一化避开 value critic。但其优势函数定义为对单一 group mean 的偏离，把 group 内 G 条轨迹之间的丰富关系压缩成一个标量，这是"关系信息瓶颈"。例如 4 条 reward 分别为 [1, 1, 0, 0] 的轨迹与 [0.7, 0.5, 0.3, 0.1] 在 GRPO 看来给出几乎相同的优势分布，但前者其实是"两强两弱"，后者是"严格排序"——后者对优化更有价值。在 rank-sensitive 的 reward landscape（如数学推理 partial credit）上，GRPO 抹去了 partial preference 信号，导致 training 后期出现 catastrophic policy collapse（论文 Figure 3：1200 步后 reward 趋零）。

② 核心方法详解

① Pairwise Decomposed Advantage (PDA)

对 group $\{o_1, \ldots, o_G\}$ 中的轨迹 $o_i$，定义：

$$A_\lambda(o_i) = \frac{1}{G-1} \sum_{j \ne i} \Delta R_{ij} \cdot \sigma\!\left(\frac{\Delta s_{ij}}{\tau}\right)$$

其中 $\Delta R_{ij} = R(o_i) - R(o_j)$ 是 reward 差，$\Delta s_{ij} = \log \pi_\theta(o_i|q) - \log \pi_\theta(o_j|q)$ 是策略 log-prob 差，$\sigma(\cdot)$ sigmoid，$\tau$ 温度。静态 baseline 被换成动态 peer-conditioned reference——每对比较都有自己的 confidence 权重。

② 三种 Gradient Kinematics

由 PDA 代回 policy gradient theorem：

$$\nabla_\theta J \propto \sum_{j \ne i} \underbrace{(R_i - R_j)}_{\text{direction}} \cdot \underbrace{\sigma\!\left(\frac{s_i - s_j}{\tau}\right)}_{\text{confidence gate}} \cdot \nabla_\theta \log \pi_\theta(o_i)$$

三种工作机制：(I) Model Confusion：$s_i \approx s_j$ 时 $\sigma \approx 0.5$，让 reward 差驱动学习；(II) Misalignment：模型对 worse trajectory 反而自信（$s_j \gg s_i$ 但 $R_i > R_j$），sigmoid → 0 但 log-odds 梯度产生强校正（LambdaRank 机制）；(III) Convergence：策略已对齐，梯度自然衰减，防止 over-optimization。

③ Semantic Density Reward（密集化稀疏奖励）

仅有 binary correctness 太稀疏。作者额外加 ROUGE-L F1 作为推理密度奖励：

$$R(o) = R_{\text{acc}} + \lambda R_{\text{fmt}} + \beta R_{\text{sem}}, \quad R_{\text{sem}}(o_i) = \text{ROUGE-L}_{\text{F1}}(o_i, y^*)$$

这使得即使最终答案错，但推理痕迹接近 ground truth solution 的轨迹也能拿到部分 reward。

④ Self-Annealing 温度调节

温度 $\tau$ 作为 spectral regularizer：$\tau$ 太小（$\tau=1$）梯度 deterministic，不稳；$\tau$ 太大（$\tau \ge 5$）sigmoid 饱和趋平，退化回 mean aggregation。论文 ablation 显示 $\tau=1.5$ 是 sweet spot（59.60% avg，比 baseline 57.72% 高 +1.88 pt）。

③ 深度案例

Qwen3-1.7B 在 AIME24 上的 group rollout：
  1. Group：G=8 条 trajectory，reward 为 [1.0, 0.7, 0.5, 0.3, 0.2, 0.1, 0.0, 0.0]。
  2. GRPO 优势：group mean = 0.35，所以 advantages = [+0.65, +0.35, +0.15, -0.05, -0.15, -0.25, -0.35, -0.35]。前 3 条都得正向更新，后 5 条都被打压——但 trajectory 2 与 3 之间的 0.2 reward 差距被无差别处理。
  3. LambdaPO PDA：对 $o_1$ vs $o_2$，$\Delta R = 0.3$，$\Delta s$ 视 policy 而定。如果 policy 对 $o_1$ 略更自信（$\Delta s = 0.2$），则 $\sigma(0.2/1.5) = 0.53$，贡献 $0.3 \times 0.53 = 0.159$。对 $o_1$ vs $o_8$（reward 差 1.0），$\sigma$ 接近 1.0，贡献完整 1.0。这样每对都按 confidence 加权聚合。
  4. 结果：1800 步训练后 GRPO 因策略坍塌 reward 跌至 0，LambdaPO 稳定在 ~0.55。AIME24 accuracy GRPO 71.67% → LambdaPO 73.33%。

⑤ 消融与讨论

⑥ 结论与启发

LambdaPO 把信息检索领域的 LambdaRank 思想移植到 LLM RL，本质上把"标量 baseline"扩展为"$\binom{G}{2}$ 对偏好"。这与 DMPO（RL-4）的"前向 KL → 分布匹配"思想相互呼应——两者都意识到 standard GRPO 对 group 内信息的处理过于粗粒度。对 LLM post-training 工程的启示：reward design 应当保留 partial credit 信号，而 advantage estimator 应当保留 ranking 信号，二者不能在 group mean 这一步同时被压扁。LambdaPO 是 plug-and-play 替换 GRPO advantage module，几乎零额外训练成本，值得开源社区跟进。

RL-2 · SAGE: Shaping Anchors for Guided Exploration in RLVR of LLMs

📅 2026-05-15 · arXiv:2605.18864
🏛 KAIST (Korea Advanced Institute of Science and Technology)
RLRLVRExploration

① 背景与动机

RLVR（Reinforcement Learning with Verifiable Rewards）能可靠提升 pass@1，但 pass@k 往往不增——这意味着 RLVR 不是真正学到新推理能力，而是把基模本有的某条 reasoning mode 锐化了。已有理论（Wu et al., 2025）证明：标准 reverse-KL RLVR 的empirical support 永远是 reference model support 的子集（Theorem 4.2）——即不可能发现 base model 完全没见过的解。常见对策（弱化 KL、用 forward KL、annealed KL）虽能扩 coverage 但会让训练不稳定。本文目标：保留 reverse-KL 的稳定性同时让 empirical support 真正扩展。

② 核心方法详解

① Pseudo-KL：让 anchor 可以非归一化

标准 RLVR 目标 $J(\theta) = \mathbb{E}_{x,y\sim\pi_\theta}[r(x,y) - \beta D_{KL}(\pi_\theta \| \pi_{\text{ref}})]$ 强制 anchor 是归一化分布。作者引入pseudo-KL：

$$\tilde{D}_{KL}(p \| f) = \mathbb{E}_{y \sim p}\left[\log \frac{p(y)}{f(y)}\right]$$

$f$ 可以是任意 positive unnormalized function。这保留 stationary proportional solution 但允许 anchor 用非归一化的"引导函数" $q(x,y)$ 进行 shaping：

$$J'(\theta) = \mathbb{E}\left[r(x,y) - \beta \tilde{D}_{KL}(\pi_\theta \| q \cdot \pi_{\text{ref}})\right]$$

② 三类 SAGE 引导函数

token-level 因子化：$q(x, y) = \prod_t q(y_t | x, y_{<t})$。Pseudo-KL 展开：

$$\tilde{D}_{KL}(\pi_\theta \| q \cdot \pi_{\text{ref}}) = D_{KL}(\pi_\theta \| \pi_{\text{ref}}) - \mathbb{E}_{x,y\sim\pi_\theta}\!\left[\sum_t \log q(y_t | x, y_{<t})\right]$$

三种 $q$ 设计：(1) Random：均匀噪声；(2) Token-level：基于 token 出现频率；(3) Branch-level（推荐）：基于 token entropy

$$q_{\text{Branch}}(y_t | x, y_{<t}) = 1 + \gamma (H_t - \tau)_+ \quad \text{where} \quad H_t = -\sum_v \pi_\theta(v|x, y_{<t}) \log \pi_\theta(v|x, y_{<t})$$

仅在高熵分支点（推理 fork）激活探索，低熵确定 token 不扰动。

③ Empirical Support Expansion 定理

Theorem 4.3：若 $q(x, y^\star) > 0$ 对某个 $y^\star \in \mathcal{C} \setminus \text{supp}_\epsilon(\pi_{\text{ref}})$（即 reference model 不会生成的正确 completion），SAGE 可以让 $\pi_{\text{SAGE}}$ 把 $y^\star$ 纳入 empirical support。这意味着 SAGE 能真正发现 base model 没见过的解，不是单纯锐化。

④ 在线计算无开销

所有 intrinsic signals（token entropy）可以从 rollout forward pass 中直接拿到，不需要额外计算。SAGE 与 GRPO 算力开销几乎一致，是 plug-and-play 升级。

③ 深度案例

Qwen2.5-Math-7B 在 AIME 上：
  1. Base：pass@1 = 0.033, pass@256 = 0.393。说明 base model 有 39.3% 题目能在 256 次采样中至少找到一次正解。
  2. GRPO 标准训练 1000 步：pass@1 升到 0.053（+0.020 pt），但 pass@256 几乎不变（0.407，+0.014 pt）。这是典型 mode sharpening：把已有 mode 收敛得更紧，但没扩 coverage。
  3. SAGE-Branch：把 q(y_t) 设为 entropy-gated multiplier。当模型在某个 reasoning step 出现高熵（如选择哪个公式应用）时，给予更多探索奖励。训练 1000 步后：pass@1 = 0.063（vs GRPO 0.053），pass@256 = 0.427（vs GRPO 0.407） — 双向都提升。
  4. Branch trigger：典型触发点是"设 x=?"或"应用引理?" 这种 token，模型的 next-token entropy 显著高于普通 token（如 H=2.5 nats vs 普通 H=0.3）。

⑥ 结论与启发

SAGE 提供了 RLVR 探索-利用 tradeoff 的新理论框架：把 KL anchor 从"归一化分布"扩展到"未归一化引导函数"，理论上证明 empirical support 可以严格扩张。这与 LambdaPO（RL-1, pairwise advantage）、DMPO（RL-4, forward KL distribution matching）一同构成了 2026 年 5 月 RLVR 三大方向：分别从 advantage estimator、optimization objective、exploration shaping 三个角度治理 GRPO 的 mode collapse 问题。SAGE 的"在高熵点扰动"实践非常工程友好——它不需要额外网络、不需要 critic、不改变 GRPO loop，仅在 token-level 加一个 entropy-gated multiplier 即可。

RL-3 · GoLongRL: 能力导向的长上下文 RLVR 配方，开源 23K 数据集 + TMN-Reweight 算法

📅 2026-05-19 · arXiv:2605.19577
🏛 Kuaishou Technology (快手), University of Chinese Academy of Sciences
RLLong ContextOpen Dataset

① 背景与动机

当前长上下文 RL 配方（如 QwenLong-L1.5、AEPO）有两大问题：(i) 数据侧过度集中于 UUID chain、chunk QA 等"复杂检索路径"，summarization / ranking / aggregation / structured reasoning 等关键能力没有直接 supervision；(ii) 算法侧标准 GRPO 在异构多任务下出问题——per-prompt 归一化扭曲 advantage，EM/F1/NDCG/ROUGE-L 的 variance 差异导致 high-variance task 贡献过大梯度。本文提出 GoLongRL 配方：23K 任务多样性 RLVR 数据集 + TMN-Reweight 算法。完全开源，code + data + 模型权重都在 GitHub。

② 核心方法详解

① 9 种任务的能力分类（Capability Taxonomy）

基于 LongBench Pro 拆出 9 个能力维度：单文档理解、跨文档推理、长 retrieval、规则归纳、对话记忆、文档求和、ranking、aggregation、结构化推理。每个任务配最适合的 reward function（EM / F1 / NDCG@k / ROUGE-L / 数值精度等）。这是关键创新——不再把所有任务的 reward 折叠成单一 indicator。

② 四阶段数据构建 Pipeline

(P1) Source Corpus Collection（Project Gutenberg books + arXiv CC0 + PMC Open Access）；(P2) Task-Oriented Filtering & Assignment；(P3) 双轨道样本构建——open-source 14K（domain-rich：法律案例、财报、多轮对话）+ synthetic 9K（DeepSeek-V3.2 / Gemini-2.5-Pro 生成 QA pairs，Qwen3-4B 做难度分箱）；(P4) Iterative Refinement——通过 benchmark diagnostics 找 capability gap，针对性补充样本。

③ TMN-Reweight：Task-level Mean Normalization with difficulty-adaptive Reweighting

(a) Task-level Normalization：把 per-prompt $\sigma_u$ 换成 task-level RMS 标准差 $\sigma_T$：

$$\hat{A}_{u,i} = \frac{R_{u,i} - \mu_u}{\sigma_T}, \quad \sigma_T = \sqrt{\frac{1}{|T|}\sum_{u \in T} \sigma_u^2}$$

这消除跨 task 的 reward 尺度差异但保留 per-prompt 难度差异。

(b) Difficulty-adaptive Weighting：用 smoothed pass rate $\bar{p}_u$ 加权：

$$w_u = \exp(-\alpha \bar{p}_u)$$

$\bar{p}_u$ 越低（任务越难）权重越大——把梯度引向 informative hard positive samples。

④ 反污染（Decontamination）

用 13-gram overlap 检测 + 跨 6 个 benchmark 的反污染过滤。保证 OOD 测评公正。

RL-4 · DMPO: Beyond Mode Collapse — Distribution Matching for Diverse Reasoning

📅 2026-05-19 · arXiv:2605.19461
🏛 Tongji University, Shanghai AI Laboratory, Fudan University（+ ZJU, CUHK）
RLMode CollapseForward KL

① 背景与动机

GRPO 等 on-policy RL 算法存在系统性 mode collapse：一旦发现一条高 reward trajectory，概率质量迅速集中到该单一解，停止探索替代策略。本文证明这不是实现 bug，而是 reverse-KL 优化目标的必然结果——reverse KL 是 mode-seeking 的，会"奖励第一个找到的成功解"。理想方案是 forward KL（mode-covering），但它需要从 reward-weighted target distribution $p^*(\tau) \propto \exp(r(\tau)/\alpha)$ 采样，需计算指数大的 partition function $Z$（如 TSP-20 有 $\sim 10^{18}$ 条 tour），全局求 forward KL 不可行。

② 核心方法详解

① 理论诊断：GRPO 即 reverse KL

论文 Theorem 1 证明：standard on-policy RL 最大化 expected reward 等价于最小化 reverse KL $D_{KL}(\pi_\theta \| p^*)$，其中 $p^*(\tau) \propto \exp(r(\tau)/\alpha)$。当 $\alpha \to 0$，$p^*$ 退化为 Dirac delta on highest-reward trajectory，policy gradient 不可避免地收敛到 mode-seeking。GRPO 中的 $D_{KL}(\pi_\theta \| \pi_{\text{ref}})$ 只防 reference drift，不能解决朝高 reward mode 的塌缩。

② Group-Level Forward KL 近似

核心 trick：放弃全局 forward KL，在 group level（$G$ 条 sampled trajectories 之间）做 forward KL。定义 group-conditional Boltzmann target：

$$p(o_i | \mathcal{O}) = \frac{\exp(r(o_i)/\alpha)}{\sum_{j=1}^G \exp(r(o_j)/\alpha)}$$

这只需 group-level partition $Z_{\text{group}}$，trivially computable。

③ Length-Normalized Policy Distribution

policy 的 group 分布需要规避 length bias（$\pi_\theta(o) = \prod \pi_\theta(o_t)$ 指数衰减）。用 length-normalized log-likelihood：

$$\varphi(o_i) = \frac{1}{|o_i|} \sum_{t=1}^{|o_i|} \log \pi_\theta(o_{i,t} | o_{i,<t}, x)$$

$$q_\theta(o_i | \mathcal{O}) = \frac{\exp(\varphi(o_i))}{\sum_{j=1}^G \exp(\varphi(o_j))}$$

④ DMPO 目标函数

最小化 group-level forward KL：

$$\mathcal{L}_{\text{DMPO}} = \mathbb{E}_{x, \mathcal{O} \sim \pi_\theta} D_{KL}\big(p(\cdot|\mathcal{O}) \| q_\theta(\cdot|\mathcal{O})\big) = -\mathbb{E}\sum_i p(o_i | \mathcal{O}) \log q_\theta(o_i | \mathcal{O}) + \text{const}$$

梯度对每条 trajectory 推动 policy 的 length-normalized log-prob 朝 reward-weighted target 对齐。不需要 group mean baseline，不需要 critic，与 GRPO loop 等价计算成本。

③ 深度案例

TSP-20 NP-hard 实例：
  1. Setup：20 城市 TSP，存在 ~1018 条 tour，多个 local optima 长度仅相差 1-2%。
  2. GRPO 训练 1000 步：policy 100% 输出同一条 tour（mode collapse），SR=87%（学到约束），但 QR=67.4%（没找到更短解）。
  3. DMPO 训练同样步数：group 内 8 条 sample 在 5+ 不同 tour 区域分布。SR=98.5%（约束学得更好），QR=73.1%（找到更短解）。多样性带来质量提升而非牺牲。
  4. Mode 数：DMPO 在 trained policy 下 sample 100 条 tour，能 cover 12+ distinct local optima；GRPO 仅 1-2 个。

  • LambdaPO：从 advantage estimator 角度治
  • SAGE：从 anchor/KL regularizer 角度治
  • DMPO：从 KL direction（reverse → forward）角度治

Agent-1 · ReBel: Rewarding Beliefs, Not Actions — 长程 Agent 的信念一致性信用分配

📅 2026-05-19 · arXiv:2605.20061
🏛 National University of Defense Technology, IGDL Lab, Xiamen University
AgentRLVRLong Horizon

① 背景与动机

LLM Agent 在长程交互任务（ALFWorld 房间任务、WebShop 网购）中面对部分可观测环境。Agent 必须从不完整观察持续推断 latent state，小推断误差会累积成 belief drift：Agent 自信地认为"钥匙在 box 里"，但打开后发现 box 是空的；如果策略不修正这个 belief，agent 会反复检查错误位置，trajectory 走得越来越偏。终端 reward 太稀疏，无法定位是 action 选错还是 state estimation 错——两者在 POMDP 下紧耦合，standard GRPO 只在 episode 级别归一化 reward，无法分离这两类错误，sample efficiency 极低。

② 核心方法详解

① 显式 Belief State 建模

在标准 LLM agent 输出（thought + action）之外，要求模型同时输出结构化 belief state $b_t$：

$$\pi_\theta(a_t, b_t | h_t) = \pi_\theta(\text{thought, action, belief} | \text{history})$$

belief $b_t$ 是显式可解析的 JSON（如 ALFWorld 中的 "key_location"、"target_object"、"progress"）。这把 implicit hidden state 提升为可监督的显式向量。

② Belief-Consistency 自监督 Reward

核心创新：把 belief 与后续 observation 的不一致性转化为 dense reward 信号。形式上：

$$r_t^{\text{belief}} = -\mathbb{1}[b_t \text{ 与 } o_{t+1:t+k} \text{ 不一致}] \cdot \lambda_b$$

比如 belief 说"钥匙在 box1"，下一步 observation 显示 box1 empty，则 belief 与 observation inconsistent，扣 $\lambda_b$ reward。这是不需要外部 verifier 的自监督信号，dense 程度比 terminal reward 高 10-50×。

③ Belief-Aware Grouping for Advantage

标准 GRPO 按 prompt 分组归一化 advantage，但在 POMDP 中相同 prompt 下 agent 的 belief 可能完全不同（trajectory 走到不同 subspace）。ReBel 改为按 belief state $b_t$ 聚类分组，在 belief 相似的 trajectory 之间做 advantage 归一化：

$$\hat{A}_{i,t} = \frac{r_{i,t} - \mu_{b(i,t)}}{\sigma_{b(i,t)}}, \quad b(i,t) = \text{cluster}(b_{i,t})$$

这显著降低 variance——agent 不会因为某个 belief subset 普遍 reward 低而被错误打压。

④ SFT Warm-Start on Belief-Annotated Data

RL 前先用 SFT 在 belief-annotated 数据上预训练，让模型先学会输出格式良好的 belief JSON。这是工程必要步骤——cold start RL 上模型不会输出合理 belief，自监督 reward 无法启动。论文 Appendix E.2 量化 isolated contribution。

③ 深度案例

ALFWorld：在卧室中找到并清洁苹果，放到厨房柜台。
  1. Step 1：Agent 输出 thought "查看桌子"，action look at desk，belief {apple_location: "desk", phase: "search"}。Observation：桌上没苹果。
  2. Belief check：predicted belief 说苹果在桌上，但 obs 不一致 → 扣 $\lambda_b = 0.1$ reward。这个 dense 信号让 agent 立即学到 "随便猜 belief 会被惩罚"。
  3. Step 2：Agent 修正 belief：{apple_location: "unknown, need to search drawer", phase: "search"}。打开抽屉发现苹果。Belief 与 obs 一致，无扣分。
  4. Belief-aware grouping：在 group 内 8 条 rollout，按 belief cluster 分组——cluster A "认为苹果在 desk" (4 条) 全部失败，cluster B "认为 apple_location unknown 主动 search" (4 条) 全部成功。在 cluster B 内部归一化 advantage 反而能鼓励早期就 belief honest 的策略。

Agent-2 · TriMem: Rethinking How to Remember — Beyond Atomic Facts in Lifelong LLM Agent Memory

📅 2026-05-19 · arXiv:2605.19952
🏛 Hong Kong Baptist University, UT Austin, Shanghai Jiao Tong University（+ Sydney AI Center）
AgentMemoryLifelong

① 背景与动机

LLM Agent 在长程多轮交互（如 LoCoMo benchmark：数十轮对话跨越数月）下，必须把过去 history 压缩进 memory 系统。现有 memory 方案（Mem0、A-Mem、MemoryOS、LightMem 等）都用 "atomic facts" 表示：用静态人工设计 prompt 把对话抽取成 fact tuple，retrieval 基于相似度匹配。本文指出 atomic facts 表示有三个根本问题：(i) 修饰语丢失——"working with trans people, helping mental health" 被抽成 "Caroline wants to pursue counseling"，限定语丢；(ii) 反例信息无法表示——"Melanie 不太可能再去 roadtrip 因为这次出事了"，否定信息原子 fact 无法捕获；(iii) 多 fact 整合推理弱——分散事实需要 holistic profile 才能正确推理。

② 核心方法详解

① 三粒度共存表示（Tri-Granularity Memory）

TriMem 维护三类共存表示：

  • Raw dialogue segments（按 source identifier 锚定）：保留原始上下文 fidelity
  • Extracted atomic facts：传统 fact tuple，用于高效相似度检索
  • Synthesized profiles：把分散 facts 聚合成 holistic semantic understanding，用于深度推理

retrieval 时根据 query 类型决定调用哪个粒度——简单 lookup 用 fact，复杂推理用 profile，需 verbatim 引用用 raw。

② TextGrad-based 自演化 Prompt 优化

关键创新：不依赖人工 prompt engineering。用 TextGrad 把 extraction prompt 和 profiling prompt 视为可优化变量，通过 response quality feedback 反向 textual gradient 更新：

$$p_e^{(t+1)} = p_e^{(t)} \oplus g_e^{(t)}, \quad p_\psi^{(t+1)} = p_\psi^{(t)} \oplus g_\psi^{(t)}$$

其中 $\oplus$ 是 prompt edit 算子，$g$ 是基于响应质量的文本梯度。无任何参数更新即可 lifelong 进化，prompt 随经验越来越好。

③ Profile 合成机制（Holistic Semantic）

对每个 entity（如 "Caroline"），TriMem 周期性合成 profile：整合该 entity 的所有 atomic facts + 相关 raw dialogue 片段 → 一段连贯的 1-2 段 profile 文本，包含：背景、当前状态、modifier qualifications、可能负面信息。Profile 解决了 atomic fact 表示丢修饰语和反例信息的问题。

④ Multi-Granularity Retrieval Routing

给定 query，TriMem 先用 LLM 判断 query 类型：verbatim lookup（"Caroline 那次说了什么"）→ raw；factual lookup（"Caroline 的职业是什么"）→ fact；understanding-based（"Melanie 还会再去 roadtrip 吗"）→ profile。然后只检索对应粒度，token 开销可控。

③ 深度案例

LoCoMo 反例（论文 Figure 1 中提到的）：
  1. Query："Melanie 是否会很快再去一次 roadtrip？"
  2. Raw dialogue："Hey Caroline, that roadtrip this past weekend was insane! We were all freaked when my son got into an accident."
  3. Atomic fact 提取："Melanie's son got into an accident in the roadtrip last weekend."
  4. Atomic-only memory 答："Not mentioned at all." ❌（fact 没显式说"不会再去"）
  5. TriMem profile 答："Likely no; since this one went badly." ✅（profile 整合了 "insane"、"freaked"、"accident" 等情绪 + 上下文信号）
  6. OpenDomain F1：GPT-4.1-mini + TriMem = 43.88 vs SimpleMem 24.50 (+19.38 pt)，差距全部来自这类 reasoning 友好的 holistic 表示。

  • 不要只存 extracted facts；要保留 raw 片段 + holistic profile 三层
  • 不要靠人工写 extraction prompt；用 TextGrad 类的自演化机制
  • retrieval 路由要按 query type 动态选择粒度

Agent-3 · Formal Skill: 把 Markdown skill 升级为可编程运行时，46.8K token 完成 SWE 任务

📅 2026-05-19 · arXiv:2605.19604

① 背景与动机

当前 LLM agent 的 "skill" 大多是 Markdown 文件（Anthropic Skill、OpenAI Operator skill、Cursor rules）：自然语言写一段操作流程，依赖模型自己解读 + 自我约束。但实际部署遇到三大问题：(i) 约束不可执行——"修改前请备份"这种规则模型可能跳过；(ii) 状态不显式——agent 不知道自己在哪个 phase，无法 recover；(iii) 工具暴露过宽——所有工具一直可见，导致 token 浪费且诱发误用。本文提出 Formal Skill：把 Markdown skill 升级为带 schema、executable hooks、phase state machine 的可编程运行时插件。

② 核心方法详解

① Action Plane（动作平面）

④ Token Efficiency Mechanism

  • 用 hooks 把"安全约束"做成 unbypassable 的 execution gate，不依赖模型自律
  • 用 phase state 把工具暴露动态收缩，显著节省 token
  • 用 skill-local runtime state 让 recovery 不依赖模型 reasoning

结合 ReBel（Agent-1，让 agent 显式输出 belief 状态）+ TriMem（Agent-2，多粒度 memory）+ Formal Skill（执行约束运行时），我们看到 2026 年 5 月 LLM agent 工程范式的明确演进方向：从"全靠模型自律"走向"约束硬编码 + 状态显式 + 工具受限"。这是工业可靠性的必经之路。

Medical-1 · MediLongChat: 跨会话长程医学对话评测基准，G-Eval 4.84 综合最佳

📅 2026-05-19 · arXiv:2605.19766
🏛 South-Central Minzu University (中南民族大学), Singapore Management University
MedicalLong DialogueBenchmark

① 背景与动机

有效的 healthcare agent 必须能回溯并推理患者的纵向病史（longitudinal medical history）。但现有医学对话数据集（NoteChat、MSC、Conversation Chronicles、LoCoMo）都聚焦单次孤立交互，无法捕捉跨会话推理（cross-session reasoning）。真实临床数据又受隐私和伦理约束。本文提出 MediLongChat：用 LLM 知识引导的三阶段合成 pipeline，构造高质量、长时间跨度、真实情感色调的医学对话数据集，并定义三个 benchmark task（in-dialogue / cross-dialogue / synthesis reasoning）评估 agent 的 memory capability。

② 核心方法详解

① Stage 1: Synthetic Patient Profile 构造

用 LLM 基于医学知识库（ICD-10 + UMLS + 临床指南）合成 patient profile：基础人口学（年龄、性别、职业）、主病轨迹（如 type-2 diabetes 5 年）、并发症轨迹（如 retinopathy、neuropathy）、accompanying conditions（hypertension, dyslipidemia）。每个 profile 是一个 longitudinal timeline，覆盖数月至数年。

② Stage 2: Multi-Turn Dialogue per Encounter

对 timeline 上每个 encounter（如 "3 个月 follow-up"），用 LLM 生成医生-患者多轮对话。引入 structured priors（症状描述方式、医生 questioning style、患者 emotional tone）避免对话过于"教科书式"。Stage 2 比 Stage 1 给更大表达自由度——Faithfulness 略降（0.601 vs 0.635）换 Diversity 提升（0.5447 vs 0.4381）。

③ Stage 3: Longitudinal History Integration

把多个 encounter 对话整合为连贯纵向历史。关键：在新对话开始时，自然引用之前 encounter 的关键事件（如 "上次复诊您提到血糖控制改善了，这次..."），并随时间推移引入新症状、新检查结果、新药物调整。这是 MediLongChat 区别于其他多轮数据集的核心——cross-session 连贯性而非孤立 session。

④ Benchmark Tasks（评估 memory capability）

三个任务：(1) In-dialogue Reasoning：当前 session 内事实推理；(2) Cross-dialogue Reasoning：跨 session 信息整合（"对比上次和这次的 HbA1c"）；(3) Synthesis Reasoning：基于完整纵向历史的高阶诊断推理（"基于 6 次复诊，病程进展如何，下一步建议"）。Multi-dimensional evaluation：用 Faithfulness（基于 source）、Coherence、Diversity 自动指标 + 4 个 LLM judge ensemble。

③ 深度案例

典型 MediLongChat patient timeline：
  1. Profile：58岁男性，T2DM 8年，最近确诊 diabetic retinopathy 早期。
  2. Encounter 1（半年前）："医生，最近视物有点模糊。" → 检查 → 确诊视网膜病变，开始 anti-VEGF + 加大二甲双胍剂量。
  3. Encounter 2（3个月前）："上次您说要监测血压，最近测了 145/90，担心。" → 加 ACEI。
  4. Encounter 3（当前）："视物比上次好一些，但晚上起夜增多。" → cross-session 触发：医生提示需排查 ACEI 加重 nocturia 还是糖尿病 nephropathy 进展。
  5. Cross-dialogue Reasoning 任务："基于这位患者三次就诊，nocturia 增加最可能的原因是什么？" 标答需要整合 Encounter 2 的 ACEI 用药史 + Encounter 3 的 nocturia 主诉 + Encounter 1 的 diabetic complications 风险。
  6. Synthesis Reasoning："过去 6 个月这位患者的病情整体走向？" 需要整合三次会诊的 trend 信息。

Medical-2 · Prompting Language Effect: 英语 vs 法语提示对 5 个 LLM 临床诊断准确率的影响

📅 2026-05-19 · arXiv:2605.19173
🏛 Data Clinic, Nantes University Hospital, France
MedicalEquityMultilingual

① 背景与动机

LLM 临床诊断辅助大多在英语评测，其他语言下的可靠性几乎未知。但部署到法国、西班牙、德国等非英语医疗系统时，提示语言可能显著影响诊断质量。本文用 180 个临床短文（vignettes，覆盖 16 个医学专科），由 2 位执业医生用 18 分制评估 5 个 LLM（o3、DeepSeek-R1、GPT-4-Turbo、Llama-3.1-405B-Instruct、BioMistral-7B），系统性测量英语 vs 法语 prompting 的 diagnostic reasoning 与 accuracy 差异。这是 LLM 临床部署equity & access 议题的核心证据。

② 核心方法详解

① 实验设计（21,600 评分 items）

$$n_{\text{items}} = 180 \text{ vignettes} \times 2 \text{ raters} \times 5 \text{ LLMs} \times 2 \text{ languages} \times 6 \text{ score components} = 21{,}600$$

180 个临床短文 + 双盲双评。每个 LLM 输出由两位医生独立打分，6 个 score component：differential diagnosis（鉴别诊断）、reasoning quality（推理质量）、logical structure（逻辑结构）、internal validity（内在效度）、justification（依据充分性）、final accuracy（最终诊断准确性）。Overall score 0-18。

② 统计分析方法（高规格）

用linear mixed models (LMM) 建模：

$$\text{Score}_{ijklm} = \beta_0 + \beta_1 \cdot \text{Language}_l + u_{i(\text{vignette})} + u_{k(\text{rater})} + \epsilon$$

vignette 和 rater 作为 random intercept，language 作为 fixed effect。p-value 用 one-sided（EN > FR）+ Bonferroni adjusted（k=5 多重比较）。Inter-rater reliability 用 ICC（two-way random, single measures, absolute agreement）+ quadratic-weighted Cohen's kappa。Wald 95% CI + bootstrap 2000 resamples。

③ Sensitivity Analysis

对 residual normality 用 Shapiro-Wilk test（all $p < 0.001$，皆非正态），所以补充 paired Wilcoxon signed-rank test 作 sensitivity。Spearman 秩相关 + Fisher's z-transformation 测试 EN vs FR 的 reasoning–accuracy 相关性差异。所有分析在 R 4.5.1 + lme4 + lmerTest + irr 包完成。

④ 6 个 sub-score 的分解

Overall 18 分由 6 component 各 0-3 分构成。这样可以定位 language gap 是来自哪个 reasoning 环节：是 differential diagnosis 列得少？还是 logical structure 乱？还是 justification 弱？精细分解 + LMM 是这篇论文方法学最大亮点。

③ 深度案例

Llama-405B 在英法两种 prompt 下的表现差异：
  1. Vignette（中文翻译）："45岁男性，3 天来右上腹痛伴黄疸、低热。BMI 32，5 年前接受过 cholecystectomy。Lab: WBC 14×109, ALT 320, AST 280, ALP 580, T-Bil 6.8 mg/dL, D-Bil 5.2 mg/dL。考虑诊断？"
  2. EN prompt：Llama-405B 输出 ① choledocholithiasis (residual stone in CBD) ② cholangitis ③ Klatskin tumor。差异性鉴别完整，logical structure 清晰，引用了"post-cholecystectomy bile duct dilation" 机制。
  3. FR prompt：同模型输出 ① cholécystite (但患者已无胆囊!) ② cholangite ③ tumeur des voies biliaires。出现逻辑硬伤——把 post-cholecystectomy 病人诊断为"胆囊炎"，忽略了 history。
  4. 评分差距：EN 17/18, FR 12/18，差 5 分。这反映 LMM 中 Llama-405B 的 mean diff 0.91 [95% CI 0.66, 1.17]，是所有模型中差距最大。

⑥ 结论与启发

这篇论文是 2026 年 LLM 临床部署最重要的政策相关论文之一。它用高规格统计方法（LMM + Bonferroni + bootstrap CI + sensitivity Wilcoxon）证明：

  • 大多数 LLM 在非英语 prompt 下有可测量、有统计学显著性的诊断质量退化。
  • 即使是医学专用模型（BioMistral-7B）也无法免疫。
  • 只有 OpenAI o3 实现了真正的 cross-lingual parity。

Multimodal-1 · VLM-CapCurriculum: Decoupling Perception and Reasoning for VLM Post-Training

📅 2026-05-19 · arXiv:2605.20177
🏛 Amazon, UC Santa Cruz, University of Waterloo
MultimodalVLMRL Post-Train

① 背景与动机

近期 VLM（Vision-Language Model）研究强调 long chain-of-thought reasoning，但作者发现：VLM 在视觉任务上的瓶颈主要是 visual perception 不足，而非 reasoning 不行。即使 reasoning 链条无限长，如果模型"看错了"图像内容，结论必然错。当前主流 VLM post-training 把 perception + reasoning 数据混合训练（merged training），结果两者互相干扰——perception 数据被 reasoning loss 稀释、reasoning 数据被 perception 模式压制。作者提出：把 visual perception → visual reasoning → textual reasoning 解耦成三个独立阶段进行 curriculum training。

② 核心方法详解

① 三阶段 Curriculum（Stage 1 → 2 → 3）
  • Stage 1: Visual Perception——专注 object detection / scene description / OCR 等基础视觉感知，数据为 caption + bounding box + structured visual QA
  • Stage 2: Visual Reasoning——基于 perception 的多步推理（geometry, chart, table reasoning）
  • Stage 3: Textual Reasoning——纯文本 chain-of-thought reasoning，强化已有 LLM reasoning capability

顺序不能颠倒——perception 是 reasoning 的 scaffold。

② Perception 更适合 RL 而非 caption-based SFT

关键发现：Stage 1 用 RL with verifiable visual rewards（如目标检测 IoU、OCR exact match）显著优于用 caption SFT。理由：caption SFT 让模型学到"如何描述图像"，但 RL 让模型学到"如何精确感知图像 attribute"。论文用 GRPO 风格的 RL 训练 Stage 1。

③ Vision Encoder Freezing 策略

论文提出 Mixed Strategy：Stage 1 unfreeze vision encoder（让 encoder 适应 perception），Stage 2 freeze（防 reasoning 训练破坏 encoder），Stage 3 unfreeze（让 encoder 微调 to align with textual reasoning）。这平衡了 plasticity 与 stability。

实验显示：staged training 本身（无论 encoder freezing 策略）比 merged training 普遍高 1-2 pt avg accuracy，证明"curriculum 是主驱动而非 encoder 配置"。

④ Capability-Based Curriculum vs Difficulty-Based Curriculum

论文指出 capability-based staging 是一个新的 curriculum 维度，与传统 difficulty-based curriculum（easy → hard problem）正交，二者可组合：在 Stage 3 内再按 problem difficulty curriculum，achieve additive gain。

③ 深度案例

Geometry math problem："在三角形 ABC 中，∠A=60°，AB=8，AC=6，求 BC。" 配图显示三角形。
  1. Merged-trained Qwen3-VL-8B："By law of cosines, BC² = 64 + 36 - 2·8·6·cos(60°)..." → 但模型把图中 AB 看成了 9（perception 错误），导致最终答 √(81+36-54) = √63 ≠ √52。
  2. Staged-trained：Stage 1 让模型先精确感知 "AB=8, AC=6, ∠A=60°"。Stage 2 在精确 perception 之上做 reasoning：BC² = 64+36-48 = 52 → BC = 2√13。正确。
  3. 累计效果：论文 Table 9 显示 Qwen3-VL-8B 在 MVista 上从 base 72.40% → 75.90%（+3.5 pt）、MVerse 31.09% → 43.78%（+12.69 pt）、WeMath 50.86% → 56.10%（+5.24 pt）。
  4. 关键观察：论文 abstract 指出 "1.5% higher reasoning accuracy with 20.8% shorter reasoning traces"——精确感知后，模型不需要长推理来"猜测" image attribute，CoT 自然变短。
