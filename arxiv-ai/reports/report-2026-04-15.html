🧠 LLM — 大语言模型

Why Supervised Fine-Tuning Fails to Learn: A Systematic Study of Incomplete Learning in Large Language Models
📅 2026-04-11 🏛 UNSW, Peking University, Tencent
LLMACL 2026 OralSFT Analysis
📌 背景与动机

监督微调（SFT）是将大语言模型适配到下游任务的标准方法，然而作者发现了一个被忽视的系统性问题：即使训练收敛后，模型仍然无法正确复现其自身训练数据的一个子集。这一现象被命名为不完全学习现象（Incomplete Learning Phenomenon, ILP）。此前的研究主要关注数据质量、学习率调度等表层因素，而对于"为什么模型看过数据却学不会"缺乏根因诊断。本文首次对 ILP 进行了系统性形式化研究，覆盖多个模型家族（Qwen、LLaMA、OLMo）、多个领域和数据集。

🔬 核心方法详解
① 未学习样本检测（Unlearned Sample Detection）
为了可靠地识别哪些训练样本未被有效学习，作者提出将 SFT 数据的自由文本响应转换为多选题（MC）格式。对于每个 SFT 实例，原始响应保留为正确选项，同时构建语义合理但不正确的干扰项。训练收敛后，用微调模型在 MC 接口上重新评估整个训练集。定义训练集准确率为： $$\text{Acc} = \frac{1}{N}\sum_{i=1}^{N}\mathbb{1}(\hat{y}_i = y_i)$$ 为增强鲁棒性，对每个样本进行 \(K\) 次重复采样（默认 \(K=5\)），通过一致性得分判定：当 \(K\) 次均错时标记为"未学习"。实验发现 ILP 普遍存在：Qwen2.5-7B 在 SFT 后仍有约 15-25% 的训练样本始终无法正确回答。
② 五大根因诊断框架
通过受控分析，作者识别了五种导致不完全学习的反复出现的根因：
  1. 基座模型知识缺失：预训练模型本身缺乏回答所需的前置知识，SFT 无法从无中生有
  2. SFT 与预训练知识冲突：SFT 监督信号与基座模型内化的知识相矛盾（如时效性差异）
  3. SFT 数据内部不一致：训练数据内部存在互相矛盾的标注
  4. 左侧遗忘（Left-side Forgetting）：在多阶段微调中，后续训练覆盖了先前学到的知识
  5. 不充分优化：稀有或复杂模式未获得足够的梯度更新
每种根因通过可观测的训练和推理信号进行诊断，形成了一个诊断优先（diagnostic-first）的处理框架。
③ 知识状态探测（Knowledge-State Probing）
作者设计了知识状态探测器来判断未学习样本属于哪种根因。核心思路是对比基座模型和 SFT 模型在同一问题上的表现：
  • 基座模型也错 → 知识缺失（根因1）
  • 基座模型对、SFT 模型错 → SFT-预训练冲突（根因2）
  • SFT 数据中存在矛盾标注 → 内部冲突（根因3）
  • 早期检查点对、后期错 → 左侧遗忘（根因4）
  • 增加训练轮数后改善 → 不充分优化（根因5）
④ 针对性干预策略
为每种根因设计因果干预方案：
  • 知识增强持续预训练（CPT）：对根因1的样本，先用相关语料进行 CPT 补充缺失知识，再进行 SFT
  • 冲突校准：对根因2/3，检测并解决 SFT 数据与预训练/内部的知识冲突
  • 动态重采样：对根因4，采用动态重采样策略缓解序列训练中的遗忘
  • 聚焦训练：对根因5，对检测到的未学习样本增加训练强度
📋 深度案例
SFT-预训练知识冲突案例（根因2）

🔍 消融与讨论

消融研究表明：五种根因的贡献并非均匀，知识缺失（根因1）占 ILP 的 35-40%，是最大瓶颈；SFT-预训练冲突（根因2）占约 20-25%。有趣的发现是，简单增加训练轮数（从3轮到10轮）只能减少约 5% 的 ILP，说明大部分不完全学习并非"训练不够"而是结构性障碍。左侧遗忘在多阶段微调场景中尤为突出，动态重采样可将其缓解约 60%。论文的局限在于 MC 转换可能引入评估偏差，且当前框架主要验证在 7B 级别模型上。

💡 结论与启发

本文首次系统性地揭示并量化了 SFT 的"不完全学习"现象——即使训练收敛，模型仍有约 20% 的训练数据学不会。通过五大根因诊断和针对性干预，ILP 率可降低至约 10%。这一发现对 SFT 实践有重要指导意义：在微调前应先评估基座模型的知识覆盖度，清理 SFT 数据中的冲突，并为不同根因采取不同的干预策略，而非盲目增加数据量或训练轮数。被 ACL 2026 录用为 Oral 论文。

Self-Calibrating Language Models via Test-Time Discriminative Distillation
📅 2026-04-11 🏛 University of Hamburg
LLMCalibrationTest-Time
📌 背景与动机

大语言模型普遍存在系统性过度自信的问题：即使在经常回答错误的问题上，模型也表达高度确定性。现有校准方法要么需要标注验证集、要么在分布偏移下退化、要么推理成本过高。近期研究发现，LLM 内部其实已包含了一个比其外显输出更好校准的信号——当被问"这个答案正确吗？"时，"True"的 token 概率比直接生成的置信度更加可靠。但这个判别信号如何高效地反哺给生成分布，一直未被解决。

🔬 核心方法详解
① 判别置信度提取（Discriminative Confidence Extraction）
给定问题 \(q\) 和模型生成的候选答案 \(a\)，构造验证提示：「Is the following answer to the question correct? Q: \(q\), A: \(a\). Answer True or False.」
判别置信度定义为： $$p_{\text{disc}}(a|q) = P(\text{True} \mid q, a)$$ 作者验证了在多个模型（LLaMA-3-70B、Qwen2.5-72B、Mistral-Large）上，\(p_{\text{disc}}\) 的 ECE（期望校准误差）比生成概率 \(p_{\text{gen}}\) 低 40-60%，证实了判别信号的校准优势。
② 测试时判别蒸馏（Test-Time Discriminative Distillation, TTDD）
核心创新在于在推理时（无需额外训练）将判别置信度蒸馏回生成分布。对于每个测试样本：
  1. 先用模型以温度 \(T\) 采样生成 \(K\) 个候选答案 \(\{a_1, ..., a_K\}\)
  2. 对每个候选计算判别置信度 \(p_{\text{disc}}(a_k|q)\)
  3. 用归一化后的判别置信度作为软标签，对生成分布进行 KL 散度最小化：
$$\mathcal{L}_{\text{TTDD}} = D_{\text{KL}}\left(\tilde{p}_{\text{disc}}(\cdot|q) \| p_\theta(\cdot|q)\right)$$ 其中 \(\tilde{p}_{\text{disc}}\) 是在候选集上归一化的判别置信度。这等价于在测试时将判别模型的校准知识转移给生成模型。
③ 高效近似：无训练版本
为避免测试时反向传播，作者提出了一种无训练近似：直接用判别置信度对候选答案进行加权投票，而非更新模型参数： $$\hat{a} = \arg\max_{a \in \mathcal{A}} \sum_{k=1}^{K} \tilde{p}_{\text{disc}}(a_k|q) \cdot \mathbb{1}(a_k = a)$$ 这种方法只需 \(K\) 次额外前向传播（验证步），计算开销比完整 TTDD 低 90% 以上，且在大多数场景下性能接近。
④ 自适应温度选择
采样温度 \(T\) 直接影响候选多样性。过低的 \(T\) 导致候选重复，过高导致质量下降。作者提出自适应策略：根据模型在问题上的初始不确定性（用生成 entropy 度量）动态选择 \(T\)： $$T^* = T_{\text{base}} \cdot \left(1 + \alpha \cdot H(p_\theta(\cdot|q))\right)$$ 当模型不确定性高时提高温度以增加候选多样性，反之降低温度保证质量。
📋 深度案例

判别信号成功捕捉到了模型的过时知识并校正了输出

🔍 消融与讨论

消融显示候选数量 \(K\) 的效果呈对数增长：\(K=5\) 即可获得约 80% 的收益，\(K=20\) 后边际收益很小。自适应温度相比固定温度提升约 1.5% 准确率。在分布外测试集（ARC-Challenge、CommonsenseQA）上，TTDD 的校准优势更加明显（ECE 改善 2-3x），表明判别信号具有良好的分布泛化性。主要局限是推理延迟增加（\(K\) 倍验证成本），以及对极长生成任务（如文章写作）不适用。

💡 结论与启发

TTDD 揭示了一个重要洞察：LLM 内部已经拥有比其生成输出更好的校准信号，关键在于如何在推理时将这个判别信号高效地反哺给生成过程。这种"自我蒸馏"范式无需额外训练数据或标注，为 LLM 的可信度提升提供了一条优雅的路径。

Think in Sentences: Explicit Sentence Boundaries Enhance Language Model's Capabilities
📅 2026-04-11 🏛 SUSTech (南方科技大学)
LLMArchitectureSentence-Level
📌 背景与动机

标准 Transformer 语言模型以 token 为粒度进行自回归生成，每一步预测都在 token 级别进行。然而，人类思维更倾向于以句子为单位组织信息——我们先构思一个完整的想法，再将其表达为语句。现有模型缺乏对句子边界的显式建模，导致在长程推理、段落连贯性和结构化输出等任务上表现不佳。本文探索了一个简单但有效的假设：通过显式引入句子边界表示，能否增强 LM 的能力？

🔬 核心方法详解
① 句子边界 Token 注入
在训练语料的每个句子末尾注入一个特殊的 [SB]（Sentence Boundary）token。与标准的句号不同，[SB] 被赋予专门的嵌入向量，且参与注意力计算。句子化后的序列为： $$\mathbf{x} = [t_1, ..., t_{n_1}, \text{[SB]}_1, t_{n_1+1}, ..., t_{n_2}, \text{[SB]}_2, ...]$$ 关键设计：[SB] token 的嵌入在训练过程中与其他 token 一起端到端学习，不需要任何额外标注。
② 句子级注意力聚合
在 Transformer 的自注意力层中，每个 [SB] token 作为其所在句子的聚合节点。作者引入了一种分层注意力机制：
  • 句内注意力：句子内的 token 之间正常进行全注意力
  • 句间注意力：跨句子的 token 优先通过 [SB] token 进行信息传递
实现上，在注意力权重中引入句子边界偏置项： $$\text{Attn}'(Q, K) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}} + \lambda \cdot \mathbf{B}_{\text{sb}}\right)$$ 其中 \(\mathbf{B}_{\text{sb}}\) 是一个学习的偏置矩阵，鼓励跨句注意力流过 [SB] 节点。\(\lambda\) 控制句子边界效应的强度。
③ 句子级表示用于下游任务
[SB] token 的隐状态天然形成句子级表示，可直接用于：
  • 文档摘要：用 [SB] 表示的余弦相似度进行句子重要性排序
  • 推理链分析：追踪 [SB] 表示的信息流动路径
  • 段落连贯性评估：相邻 [SB] 表示的平滑度作为连贯性指标
④ 训练策略：渐进式句子边界引入
为避免突然引入 [SB] token 对预训练模型造成冲击，采用渐进式策略：
  1. Phase 1：在继续预训练初期，[SB] 的注意力权重用残差连接衰减（\(\alpha=0.1\)）
  2. Phase 2：逐步增大 \(\alpha\) 至 1.0，让模型适应句子边界信号
  3. Phase 3：全强度训练，[SB] token 完全参与所有注意力计算
📋 深度案例
GSM8K 数学推理案例

问题：「Sally earns $12 per hour. She worked 8 hours on Monday and 6 hours on Tuesday. How much did she earn?」

标准模型（token级）：
「Monday: 12 × 8 = 96. Tuesday: 12 × 6 = 72. Total = 96 + 72 = 158」→ ❌ (计算错误)

句子边界迫使模型在每个推理步骤后"暂停"并整合信息，减少中间步骤的错误传播

🔍 消融与讨论

消融实验显示，句子级注意力偏置 \(\lambda\) 的最优值约为 0.3-0.5，过大会限制灵活的跨句直接注意力。渐进式训练相比直接引入 [SB] 提升约 2 分。有趣的发现：可视化 [SB] 表示的注意力模式显示，模型确实学会了在句子边界处进行"思考暂停"——[SB] token 的注意力权重分布比普通 token 更加分散，表明它在聚合全局信息。局限性在于句子检测需要额外预处理，且在代码、表格等非自然语言场景中句子边界定义不明确。

💡 结论与启发

这项工作表明，一个简单的结构先验——显式的句子边界——就能显著增强 LM 的推理和连贯性能力。句子作为人类思维的基本单位，在 LM 中引入对应的计算结构具有天然的合理性。这为"如何让 LM 以更接近人类的粒度思考"提供了一个轻量但有效的方向。

SpecMoE: A Fast and Efficient Mixture-of-Experts Inference via Self-Assisted Speculative Decoding
📅 2026-04-11 🏛 DeepSeek
LLMMoEInference
📌 背景与动机

Mixture-of-Experts (MoE) 模型以其高效的参数利用成为大模型的主流架构（如 DeepSeek-V3、Mixtral）。然而，MoE 的推理瓶颈在于：每一步需要动态路由并激活不同专家，导致专家加载成为延迟主要来源，尤其在 offloading（专家卸载到 CPU/SSD）场景下。推测解码（Speculative Decoding）是加速自回归推理的有效技术，但传统方法需要单独的小模型作为 drafter，增加了系统复杂度。本文提出 SpecMoE，一种自辅助推测解码方法，利用 MoE 模型自身的子集专家作为高效 drafter。

🔬 核心方法详解
① 专家子集作为 Draft 模型
MoE 模型的路由机制天然将输入分配到 \(K\)（通常 2-4）个活跃专家。SpecMoE 的核心洞察是：仅使用最活跃的 1 个专家即可作为轻量 drafter，生成候选 token 序列： $$\hat{y}_{t+1:t+\gamma} = \text{AutoReg}(\mathbf{x}_{:t}; \theta_{\text{top-1}})$$ 其中 \(\gamma\) 是推测长度，\(\theta_{\text{top-1}}\) 表示仅激活 top-1 专家的模型子集。Top-1 专家推理的 FLOPs 约为完整 top-\(K\) 推理的 \(1/K\)。
② 自适应推测长度
固定推测长度 \(\gamma\) 在不同 token 位置的接受率差异很大。SpecMoE 引入自适应机制： $$\gamma^* = \arg\max_{\gamma} \left\{ \gamma \mid \prod_{i=1}^{\gamma} p_{\text{accept}}(\hat{y}_{t+i}) > \tau \right\}$$ 其中接受概率由 top-1 专家和 top-\(K\) 专家输出分布的 KL 散度估算： $$p_{\text{accept}}(\hat{y}) \approx \exp(-D_{\text{KL}}(p_{\text{top-1}}(\cdot) \| p_{\text{top-K}}(\cdot)))$$ 当 top-1 和 top-K 输出高度一致时（KL 散度小），推测更长的序列；当分歧大时提前截断。
③ 专家预取流水线
SpecMoE 利用推测阶段的计算时间来预取验证阶段所需的专家权重：
  1. Draft 阶段使用 top-1 专家生成候选
  2. 同时异步预取其余 \(K-1\) 个专家权重到 GPU
  3. 验证阶段直接使用已预取的全部 \(K\) 个专家
这种流水线设计将专家加载延迟与 draft 计算重叠，实现了"零等待"的专家就绪。在 offloading 场景下，这是性能增益的最大来源。
④ 无损验证与修正
验证步骤确保输出与原始 MoE 模型完全等价。对于 draft 生成的每个 token \(\hat{y}_i\)，使用标准的推测解码拒绝采样： $$\alpha_i = \min\left(1, \frac{p_{\text{full}}(\hat{y}_i | \mathbf{x}_{:t+i-1})}{p_{\text{draft}}(\hat{y}_i | \mathbf{x}_{:t+i-1})}\right)$$ 以概率 \(\alpha_i\) 接受 draft token，否则从修正分布 \(p_{\text{full}} - p_{\text{draft}}\) 中重采样。这保证了输出分布与原始模型数学上等价。

① 静态全局量化位置策略

Reason Only When Needed: Efficient Generative Reward Modeling via Model-Internal Uncertainty
📅 2026-04-11🏛 UNSW, Tencent
RLReward ModelEfficiency
📌 背景与动机

生成式奖励模型(GRM)通过 Chain-of-Thought 提示增强 LLM 的推理能力来评估回答质量。但 GRM 有两个关键缺陷：(1) CoT 对所有输入无差别应用，简单任务也承担不必要的推理成本；(2) 生成式推理的不稳定性导致同一输入的评分波动大。本文提出根据模型内部不确定性动态决策是否需要推理。

🔬 核心方法详解
① 不确定性感知路由
对输入先执行一次快速直接评分（无 CoT），从模型内部隐状态提取不确定性信号。定义不确定性为最后一层隐状态的 entropy：$$U(\mathbf{x}) = -\sum_v p(v|\mathbf{h}_L) \log p(v|\mathbf{h}_L)$$ 当 \(U(\mathbf{x}) > \tau\) 时触发 CoT 推理，否则直接使用快速评分。阈值 \(\tau\) 通过验证集上的 F1 最优点自动确定。
② 多粒度评分融合
对需要 CoT 的样本，使用三级评分融合：快速评分 \(s_{\text{fast}}\)、CoT 评分 \(s_{\text{cot}}\)、以及 CoT 推理过程中的中间步骤一致性得分 \(s_{\text{consist}}\)：$$s_{\text{final}} = \alpha s_{\text{fast}} + \beta s_{\text{cot}} + (1-\alpha-\beta) s_{\text{consist}}$$
③ CoT 稳定性训练
为减少 CoT 的随机波动，引入一致性正则化：对同一输入的 \(K\) 次 CoT 采样，最小化评分方差：$$\mathcal{L}_{\text{consist}} = \frac{1}{K}\sum_{k=1}^K (s_k - \bar{s})^2$$ 与标准奖励建模损失联合训练。
④ 自适应推理预算分配
在 Best-of-N 采样或 RLHF 训练中，将节省的推理预算重新分配给高不确定性样本：允许它们使用更长的 CoT 或多次采样取均值，实现预算最优分配。

Policy Split: Incentivizing Dual-Mode Exploration in LLM Reinforcement Learning with Dual-Mode Entropy Regularization
📅 2026-04-12🏛 Harbin Institute of Technology, Zhejiang University, ByteDance
RLExplorationEntropy
📌 背景与动机

LLM 的 RL 训练（如 GRPO、PPO）中，探索-利用权衡是核心挑战。现有方法使用单一熵正则化：高熵鼓励探索但降低生成质量；低熵保证质量但容易陷入局部最优。本文提出 Policy Split：将策略显式拆分为探索模式和利用模式，分别使用不同的熵正则化强度。

🔬 核心方法详解
① 策略双模式分裂
将策略 \(\pi_\theta\) 拆分为两个模式，由二项伯努利变量 \(z \sim \text{Bernoulli}(\epsilon)\) 控制：$$\pi_{\theta}(y|x) = \epsilon \cdot \pi_{\theta}^{\text{explore}}(y|x) + (1-\epsilon) \cdot \pi_{\theta}^{\text{exploit}}(y|x)$$探索模式使用高温度 \(T_{\text{exp}}\)，利用模式使用低温度 \(T_{\text{base}}\)。两个模式共享参数但温度不同。
② 双模式熵正则化
对两种模式分别施加不同的熵约束：$$\mathcal{L}_{\text{entropy}} = -\lambda_{\text{exp}} H(\pi^{\text{explore}}) - \lambda_{\text{base}} H(\pi^{\text{exploit}})$$ 其中 \(\lambda_{\text{exp}} \gg \lambda_{\text{base}}\)。探索模式被强制保持高熵以发现新策略，利用模式被鼓励低熵以优化已知好策略。
③ 模式切换的自适应调度
训练初期探索比例 \(\epsilon\) 较高（0.3-0.5），随训练进行按余弦退火降至 0.05-0.1：$$\epsilon_t = \epsilon_{\min} + \frac{1}{2}(\epsilon_{\max} - \epsilon_{\min})(1 + \cos(\pi t / T))$$
④ 探索经验的选择性蒸馏
探索模式生成的高回报样本被筛选后注入利用模式的训练集：只有回报高于利用模式当前均值 1σ 的探索样本才被保留，确保"好的探索结果被利用"。

Low-rank Optimization Trajectories Modeling for LLM RLVR Acceleration
📅 2026-04-12🏛 Renmin University of China, China University of Mining and Technology

📌 背景与动机

RLVR（RL with Verifiable Rewards）已成为提升 LLM 推理能力的关键技术（如 DeepSeek-R1），但训练成本极高。本文发现 RLVR 训练过程中，模型参数的变化轨迹存在显著的低秩结构：参数更新的有效维度远低于参数总维度。基于此观察，提出通过建模低秩优化轨迹来加速 RLVR 训练。

🔬 核心方法详解
① 优化轨迹的低秩分解
RLVR 训练 \(T\) 步后的参数增量 \(\Delta\mathbf{W}_T = \mathbf{W}_T - \mathbf{W}_0\) 可分解为：$$\Delta\mathbf{W}_T \approx \mathbf{U}_r \mathbf{\Sigma}_r \mathbf{V}_r^T$$ 其中秩 \(r \ll \min(d_1, d_2)\)。实验发现，RLVR 训练 1000 步后，前 8 个奇异值捕获了 95%+ 的参数变化量。
② 轨迹预测加速
基于低秩假设，用前 \(T_0\) 步的真实轨迹拟合低秩子空间 \((\mathbf{U}_r, \mathbf{V}_r)\)，然后在该子空间内外推后续步骤：$$\Delta\mathbf{W}_{T_0 + t} \approx \mathbf{U}_r \cdot f(t) \cdot \mathbf{V}_r^T$$ 其中 \(f(t)\) 是在低秩子空间中训练的轻量预测网络（2层 MLP）。
③ 子空间内的高效 RL 更新
将 RL 梯度投影到低秩子空间内进行更新，每步只需更新 \(r \times (d_1 + d_2)\) 个参数而非 \(d_1 \times d_2\)：$$\nabla_{\mathbf{U}} \mathcal{L} = \nabla_{\mathbf{W}} \mathcal{L} \cdot \mathbf{V}_r, \quad \nabla_{\mathbf{V}} \mathcal{L} = \mathbf{U}_r^T \cdot \nabla_{\mathbf{W}} \mathcal{L}$$ 当 \(r=8\)，7B 模型每步训练的梯度内存降低约 98%。
④ 混合训练策略
前 \(T_0\) 步（约总步数的 20%）使用全参数 RL 训练以建立子空间，之后切换到子空间投影训练。每隔 \(\Delta T\) 步重新拟合子空间基以适应优化方向的渐变。

🏢 Sanity Checks for Agentic Data Science
📅 2026-04-12🏛 Microsoft Research, OpenAI
AgentIndustry LabData Science
📌 背景与动机

LLM-based data science agents（如 OpenAI Codex、Data Interpreter）在自动化数据分析中展现出巨大潜力。然而，这些 Agent 的输出缺乏可靠的质量保证——它们可能生成看似合理但实际错误的分析结论。本文来自 Microsoft Research 和 OpenAI 的联合研究，提出了一套针对 Agentic Data Science 的完整性检验（Sanity Checks）框架，系统地识别 agent 输出中常见的错误模式。

🔬 核心方法详解
① 错误分类法（Error Taxonomy）
通过分析 1000+ 个 agent 数据分析案例，总结 5 类系统性错误：(1) 数据泄漏：特征工程中无意使用目标变量信息；(2) 分布假设违反：在非正态数据上使用参数检验；(3) 因果混淆：将相关性描述为因果关系；(4) 粒度不匹配：聚合层级与分析问题不一致；(5) 结果幻觉：agent 描述了代码未实际产生的结论。
② 自动化检验管道
设计 12 种自动化检验器，每种针对特定错误类型。例如：
  • LeakageChecker：检查特征矩阵与目标变量的相关性是否在合理范围
  • DistributionChecker：在统计检验前自动进行正态性/方差齐性检验
  • CausalLanguageChecker：NLI 模型检测结论中是否使用了因果语言
③ 代码-结论一致性验证
最关键的检验：agent 生成的代码实际输出是否与其自然语言结论一致。方法是重新执行代码，提取数值结果，与结论中的陈述进行事实核查：$$\text{Consistency}(c, r) = \text{NLI}(\text{code\_output}(c), \text{claims}(r))$$ 发现约 18% 的 agent 输出存在代码-结论不一致。
④ 人机协作修正循环

📅 2026-04-11🏛 South China University of Technology, South China Agricultural University

📌 背景与动机

多模态推理模型（如 GPT-4o、Gemini）在进行视觉推理时频繁产生幻觉——生成与图像不符的描述或推理。现有方法大多从输出端修正（如 RLHF 后处理），但幻觉在推理链的哪个环节产生？能否在源头阻止？本文提出"认知枢纽点（Cognitive Pivot Points）"理论：幻觉通常在推理链的特定转折点产生，通过在这些枢纽点注入"视觉锚定"可以有效修正。

🔬 核心方法详解
① 认知枢纽点检测
分析 VLM 推理过程中每一步的视觉注意力权重分布。定义枢纽点为注意力从视觉 token 急剧转向语言 token 的步骤：$$\text{PivotScore}(t) = \frac{A_{\text{visual}}(t-1) - A_{\text{visual}}(t)}{A_{\text{visual}}(t-1)}$$ 当 PivotScore > θ 时，模型在该步"脱离"了视觉信息，进入纯语言推理模式——这正是幻觉最容易产生的时刻。
② 视觉锚定注入
在检测到的枢纽点处，强制将注意力权重的一部分重新分配回视觉 token：$$A'_{\text{visual}}(t) = (1-\beta) A_{\text{visual}}(t) + \beta \cdot A_{\text{visual}}(t-1)$$ 其中 β 是锚定强度（默认 0.3）。这相当于在模型即将"脱离视觉"时拉回它的注意力。
③ 推理链一致性校验
在每个枢纽点后，对当前推理步骤进行视觉一致性验证：提取该步中提到的视觉属性（颜色、位置、数量等），与图像中实际属性进行对比。不一致的步骤被标记并触发局部重推理（仅重新生成该步，不回滚整条链）。
④ 轻量级在线部署
枢纽点检测和视觉锚定都在注意力层内实现，无需额外模型或后处理步骤。推理延迟增加仅 5-8%。可作为插件应用到任何 Transformer-based VLM 上。

视觉生成（文生图/视频）的奖励模型大多基于简单的美学评分或 CLIP 对齐分数，无法捕捉生成内容的逻辑一致性（如"一只猫坐在红色椅子上"中猫和椅子的空间关系是否正确）。本文提出用推理型奖励来扩展视觉生成——不仅评分"好看"，还评分"合理"。

🔬 核心方法详解
① 推理型视觉奖励模型
将 VLM（如 InternVL）改造为推理奖励模型。给定生成图像和文本提示，模型输出结构化评分：$$R = [R_{\text{aesthetic}}, R_{\text{align}}, R_{\text{spatial}}, R_{\text{count}}, R_{\text{attr}}]$$ 每个维度都伴随 CoT 推理解释（如"图中有2只猫，但提示要求3只 → 数量不匹配 → R_count=0.3"）。
② 训练时 RL 优化
将推理奖励作为 diffusion model 的 RL 信号：$$\mathcal{L}_{\text{RL}} = -\mathbb{E}_{z \sim p_\theta}\left[\sum_d w_d \cdot R_d(x_{\text{gen}}, \text{prompt})\right]$$ 其中 \(w_d\) 是各维度权重。相比单一 CLIP 分数，多维推理奖励可以针对性地优化薄弱维度（如空间关系）。
③ 测试时 Best-of-N 推理缩放
生成 N 张图像，用推理奖励模型选出最优：$$x^* = \arg\max_{x_i} \sum_d w_d \cdot R_d(x_i, \text{prompt})$$ 关键发现：随着 N 的增加，推理奖励的选择质量对数增长——\(N=16\) 时效果接近 \(N=64\)。
④ 奖励维度的自适应权重
不同提示需要不同的评分权重。例如，"3只猫在桌上"侧重数量和空间；"一幅梵高风格画"侧重美学。根据提示的语义分析自动调整 \(w_d\)。

🔬 核心方法详解
① 三角色设计
  • Tier 1 — 主 Agent（π_agent）：负责执行动作，使用标准 agent 提示（观察→思考→行动循环）
  • Tier 2 — 摘要模型（π_sum）：将冗长的环境观察压缩为简洁摘要，降低 agent 的上下文负担
  • Tier 3 — 评审模型（π_critic）：在行动执行前评估合理性，检测潜在错误并建议修正
三个角色共享同一套模型权重，通过不同的系统提示切换角色。
② 编排调度器

③ 角色特化微调
虽然使用同一模型，但通过多任务微调增强各角色能力：训练数据混合 agent 轨迹(60%)、摘要对(20%)、critique 对(20%)。使用特殊 token 区分角色：[AGENT]、[SUMMARIZER]、[CRITIC]。
④ 自适应上下文管理
π_sum 的摘要能力使得有效上下文长度扩展 3-5x。当对话历史超过模型窗口的 70% 时，自动触发摘要压缩，保留关键信息。

🔬 核心方法详解
① 校准崩溃的量化
测量三种校准指标在 RLHF 训练过程中的变化：ECE（期望校准误差）、Brier Score、以及自适应校准误差(ACE)。发现 RLHF 的前 500 步几乎不影响校准，但 500-2000 步期间 ECE 从 ~8% 急剧恶化至 ~25%，此后稳定在高位。
② 逢迎-校准因果链
通过受控实验建立因果链：reward model 偏好自信表达 → RLHF 训练鼓励高确定性输出 → 模型对所有问题都表达高置信 → 校准崩溃。形式化表示为：$$\text{ECE}_{\text{post-RLHF}} = \text{ECE}_{\text{base}} + \lambda \cdot |\text{Sycophancy Score}| + \epsilon$$实验确认 λ > 0 且统计显著（p < 0.001）。
③ 后校准（Post-hoc Recalibration）尝试
测试了温度缩放、Platt Scaling 和保序回归三种后校准方法。发现温度缩放在轻度逢迎下有效（ECE 降至 ~12%），但在重度逢迎下失效（ECE 仍 >20%），因为逢迎改变了置信度分布的形状而非单纯的缩放。
④ 校准感知的 RL 训练
提出修改 reward 函数，加入校准惩罚项：$$R_{\text{cal}} = R_{\text{base}} - \mu \cdot |\text{confidence}(y) - \text{accuracy}(y)|$$ 在保持 RLHF 对齐效果的同时约束校准退化。实验显示可将 ECE 恶化控制在 +5% 以内。

🏢 MathAgent: Adversarial Evolution of Constraint Graphs for Mathematical Reasoning Data Synthesis
📅 2026-04-12🏛 Huawei, Tsinghua University, Harbin Institute of Technology
AgentIndustry LabData Synthesis
📌 背景与动机

数学推理数据的质量和多样性是 LLM 数学能力的瓶颈。人工标注成本高、覆盖有限；LLM 自合成容易陷入模式坍缩（mode collapse），生成大量相似的简单题目。本文来自华为+清华+哈工大团队，提出 MathAgent：通过对抗演化约束图来合成结构多样、难度可控的高质量数学推理数据。

🔬 核心方法详解
① 约束图表示
将数学题目抽象为约束图 \(G = (V, E, C)\)：节点 \(V\) 为数学变量，边 \(E\) 为变量间关系，约束 \(C\) 为等式/不等式条件。例如"鸡兔同笼"：节点={鸡数, 兔数, 总头数, 总脚数}，边={头数=鸡+兔, 脚数=2鸡+4兔}，约束={头数=35, 脚数=94}。
② 对抗演化机制
由两个 Agent 协作：Proposer（𝒜_P） 生成约束图变异（添加节点/边/约束），Critic（𝒜_C） 评估变异后题目的质量和求解唯一性。演化目标函数：$$\max_{\mathcal{A}_P} \min_{\mathcal{A}_C} \mathbb{E}_{G' \sim \mathcal{A}_P(G)}[\text{Quality}(G') + \lambda \cdot \text{Diversity}(G', \mathcal{D})]$$ Proposer 追求最大化质量+多样性，Critic 负责淘汰不合格的变异。
③ 难度控制
约束图的拓扑特征直接映射到题目难度：图深度 → 推理步数，图宽度 → 并行子问题数，环结构 → 需要方程组求解。通过控制图的拓扑参数精确控制合成题目的难度分布。
④ 解耦设计：蓝图与语言化分离
先生成约束图蓝图（纯结构），再由 LLM 将蓝图包装为自然语言题目。这种解耦确保了结构多样性不受语言表达风格的限制。
📋 深度案例
约束图演化实例

种子题：「甲乙两人各走一段路，甲速度5km/h，乙速度3km/h，同时出发，甲比乙早到30分钟。求路程。」
约束图：V={d, v₁=5, v₂=3, Δt=0.5h}, E={d=v₁t₁, d=v₂t₂, t₂-t₁=Δt}
Proposer 变异：添加"中途休息"节点 + "速度变化"边 → 新约束图
