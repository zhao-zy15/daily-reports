LLM-1 · OSCAR: Offline Spectral Covariance-Aware Rotation for 2-bit KV Cache Quantization

📅 2026-05-19 🏛 Together AI, University of Sydney, UIUC arXiv:2605.17757
LLM KV Cache INT2 量化 系统部署

① 背景与动机

把 KV cache 压到 INT2（每元素 2bit）对长上下文 LLM 服务极有价值——它能直接把同硬件能承载的 batch / context 长度提升数倍。但现有方法在 INT2 上几乎全军覆没：QuaRot-INT2 在 Qwen3-8B AIME25 上从 BF16 的 70% 跌至 2.22%，朴素 INT2 直接归零。问题根源在于：(1) Hadamard 变换等"通用旋转"只能均化通道方差，并不知道下游 attention 真正消费的是 $Q^\top Q$ 与 $V^\top S^\top S V$ 这两个统计量，因此误差能量错配；(2) 现有方法多为算法 demo，缺乏 真正可用的分页 KV / 融合 kernel，无法接入 SGLang/vLLM 等生产推理栈。OSCAR 直面这两点：用注意力感知的协方差结构对齐 INT2 量化方向，并写出了完整的 SGLang 集成。

② 核心方法详解

① 注意力感知的目标协方差（Query-aware key & Score-aware value）

OSCAR 的关键洞察：朴素 PCA 优化 $K, V$ 自己的协方差 $K^\top K, V^\top V$，但下游 attention 实际消费的是 $QK^\top$ 与 $S^\top S V$。因此应当用注意力下游真正用到的二阶统计量作为目标。对 query 行 $q$，定义

$$C_Q = \frac{1}{N}\sum_{n=1}^{N} q_n^\top q_n,\quad C_S = \frac{1}{N} V^\top S^\top S V,\quad S = \mathrm{softmax}_{\mathrm{row}}\!\left(QK^\top/\sqrt{d}\right)$$

其中 $C_Q$ 编码"哪些方向上 query 投影最强"，$C_S$ 编码"attention score 加权后 value 真正贡献的方向"。论文证明（Theorem 1）在残差对角假设下，用 $C_Q, C_S$ 的特征向量做旋转可极小化 frozen-error 代理。

② 谱分解 + Hadamard + 比特反转的复合旋转

设 $C_Q = U_Q \Lambda_Q U_Q^\top$，$C_S = U_S \Lambda_S U_S^\top$，OSCAR 的最终旋转矩阵是三段复合：

$$R_K = U_Q\,H_{\mathrm{Had}}\,P_{\mathrm{br}},\qquad R_V = U_S\,H_{\mathrm{Had}}\,P_{\mathrm{br}}$$

其中 $U_Q / U_S$ 把能量旋到注意力主成分轴上、$H_{\mathrm{Had}}$ 进一步均化通道、$P_{\mathrm{br}}$（bit-reversal 置换）把高方差与低方差通道交错排列以使每个 INT2 块的动态范围更均衡。这是同时为 "对齐 attention 几何" 与 "对齐 INT2 硬件几何" 服务。

③ 混合精度 KV 布局：[BF16 sink | INT2 history | BF16 recent]

OSCAR 把每个 request 的 KV cache 分三段：前 $S_0=64$ 个 token（sink）和最近 $W=256$ 个 token 仍保留 BF16，中间最长的"历史区"压成 INT2。

$$\underbrace{[1, S_0]}_{\mathrm{BF16\ sink}}\;\Vert\;\underbrace{[S_0{+}1,\, t{-}W]}_{\mathrm{INT2\ history}}\;\Vert\;\underbrace{[t{-}W{+}1,\, t]}_{\mathrm{BF16\ recent}}$$

这是与 paged-attention 兼容的设计：每个 paged block 内部要么纯 BF16，要么纯 INT2，避免页内格式混用导致 kernel 复杂度暴涨。新生成 token 先写入 BF16 recent window，待其滑出 W 时再被 fused Triton kernel 量化降级。

④ INT2 解码 kernel：双段 attention + 在线 softmax 合并

解码时把 cache 分成 BF16 段（sink+recent）和 INT2 段两次独立 attention 计算，其结果通过 online softmax 在数值稳定下合并：

$$\mathrm{out}_t = \frac{e^{m_{\mathrm{bf16}}}\,o_{\mathrm{bf16}} + e^{m_{\mathrm{int2}}}\,o_{\mathrm{int2}}}{e^{m_{\mathrm{bf16}}} + e^{m_{\mathrm{int2}}}}$$

INT2 kernel 内部解包 4 个 2-bit 值打包到一个 byte，应用 per-token scale/zero point，反量化后在 fp32 中累计 attention。OSCAR 把 $R_V$ 吸收进模型的输出投影权重，因此 V 路解码时不再需要在线乘以 $R_V$，进一步降低延迟。

③ 深度案例：Qwen3-8B 在 AIME25 上的"INT2 复活"

设定：Qwen3-8B Thinking-2507，AIME25（30 道）32K 推理生成，5 random seeds。
BF16 baseline：70.00 ± 3.33 % 准确率。
朴素 INT2 直接量化：所有 5 seed 输出全部不可读 → 0.00%。
QuaRot-INT2（仅 Hadamard 旋转）：2.22 ± 1.57 % — 模型仍然可以"说话"但答案完全错乱。
KIVI-KV2（channel-wise INT2）：57.67 ± 9.00 %（注意：方差 9 个点，且需要 channel-wise 的非分页 buffer，无法接入 SGLang）。
OSCAR (BPE=2.28)：66.67 ± 3.33 %，相对 BF16 仅掉 3.33 个点（mean-of-5 drop = -1.42 跨 5 个 benchmark），并且完全在 SGLang 分页 attention 链路里运行。
关键：OSCAR 不是"靠更高的 effective bitrate 取胜"——它的 BPE=2.28 与 QuaRot 的 2.25 几乎相同，差距完全来自旋转矩阵 $U_Q, U_S$ 的注意力对齐。

⑤ 消融与讨论

消融分析：(1) 仅 Hadamard 不带 $U_Q, U_S$ 即 QuaRot 配置，Qwen3-4B-Thinking 从 71.9% 跌到 1.4%，说明注意力感知协方差是 INT2 复活的唯一关键；(2) 不用 score-aware $C_S$ 而退回 $V^\top V$ 在 LiveCodeBench 上掉约 2pt，说明 attention score 加权确实重要；(3) 取消 BF16 sink/recent 窗口会让 Qwen3-4B-Thinking 平均掉 ~3pt，证明长尾外溢区仍需高精度兜底。系统层：在单卡 H100 上 Qwen3-8B 32K 解码吞吐相对 BF16 提升约 1.6×，长上下文 RULER-NIAH 128K 上 OSCAR 仍稳定，naive INT2 完全坍塌。局限：(a) 校准集需 ~9K token，对超大模型校准成本仍可观；(b) 当前仅评测了 dense 模型，对 MoE 模型每个 expert 的 $C_Q$ 估计稳定性未验证；(c) 旋转矩阵全部 offline 固定，对在线 distribution shift（如新语言、新代码风格）鲁棒性待研究。

⑥ 结论与启发

OSCAR 给出的本质教训是：低比特量化的"几何"必须与下游算子的几何一致。对 attention 来说，这个几何就是 $Q^\top Q$ 与 $V^\top S^\top S V$，而非 $K, V$ 自身的协方差。这一原则可以推广到任何"激活进入下游线性算子"的场景——下次要做 INT2 weight quant 也好、做长上下文 cache 也好，都可以问一句"我真正要近似的算子是什么二阶量"。在工程上，OSCAR 又一次证明：算法 + 配套 paged kernel + SGLang 集成才能让 INT2 真正"用得上"。对中金的 GPU IPO 项目而言，这类"把同一张卡的有效 KV 容量再压 4×"的工作正是数据中心客户最敏感的产品力——值得重点跟踪 Together AI 在这个方向的产业落地节奏。

LLM-2 · Full Attention Strikes Back: Transferring Full Attention into Sparse within Hundred Training Steps (RTPurbo)

📅 2026-05-19 🏛 Nanjing University, Alibaba Group arXiv:2605.16928
LLM Sparse Attention 长上下文 Top-p 选择

① 背景与动机

长上下文推理是 LLM 部署最大的算力黑洞——full attention 的复杂度随 context 平方增长。现有解法分两条路：(1) 原生稀疏训练（如 NSA、Mixture-of-Heads），代价是从头预训练；(2) 启发式 token eviction（如 SnapKV、Quest），不需训练但精度下滑明显，且对多跳检索任务（multi-Q / multi-K / multi-V）尤其脆弱。RTPurbo 提出一个第三条路：full-attention LLM 本身就内禀稀疏，只需要极少量（百步级）训练就能转化成稀疏检索头，并且：(a) 仅 ~15% 的 head 真正需要长程检索；(b) 长程检索由 低维（16 维）子空间主导；(c) 每个 query 真正需要的 token 数随上下文动态变化，因此 dynamic top-p 比 fixed top-k 更合适。

② 核心方法详解

① Offline Head-wise Calibration（找到检索头）

构造一个"双 needle"长校准序列：在长文档头尾各插入同一段 needle text。用后段 needle token $t \in N_{\mathrm{post}}$ 对前段 needle token $j \in N_{\mathrm{pre}}$ 的 attention mass 来量化每个 head 的检索能力：

$$R_h = \frac{1}{|N_{\mathrm{post}}|}\sum_{t \in N_{\mathrm{post}}}\sum_{j \in N_{\mathrm{pre}}} A_h(t, j)$$

论文发现 head 的检索行为极度稳定且与输入几乎无关，因此只需一条长序列校准一次即可把所有 query head 切成 $\mathcal H_{\mathrm{ret}}$（top 15%）和 $\mathcal H_{\mathrm{loc}}$（其余 85%）。这是 RTPurbo "百步训练即可" 的关键前提：要训练的参数量小，且每 head 的角色已经事先固定。

② RoPE-induced Low-Rank Routing（用 16 维子空间做 token 选择）

关键洞察：RoPE 的高频通道会让长程 query–key 内积衰减；长程信息其实集中在低频子空间。RTPurbo 在 RoPE 注入之前用低秩投影做 token 路由：

$$s_h(m, n) = (W^Q_h q^{\mathrm{pre}}_{m,h})^\top (W^K_h k^{\mathrm{pre}}_{n,h}),\quad W^Q_h, W^K_h \in \mathbb R^{r \times d_h}, r = 16 \ll d_h$$

$s_h$ 只是路由信号，最终的 attention 仍然在原始 d 维空间用完整 RoPE 计算，因此位置几何不被破坏。这是与 LinearAttention/低秩 attention 的关键区别：低秩只用于"选 token"，不用于"算 attention"。

③ Query-Aware Dynamic Top-p Selection

对每个 retrieval head，根据低秩 score 选出累积 mass 达到阈值 $p$ 的最小 token 子集：

$$\mathcal S_h(m) = \mathrm{Top}\text{-}P\big(s_h(m, \cdot),\, p = 0.9\big),\quad O_h(m) = \mathrm{softmax}\!\Big(\frac{q_{m,h}^\top k_{n,h}}{\sqrt{d_h}}\Big)_{n \in \mathcal S_h(m)} v_{n,h}$$

不同 query 自动选不同数量的 token：简单 query 可能 1K 即可，复杂多跳 query 可能选到 8K。这与 fixed top-k=4096 的 baseline 相比，能在简单 query 上省算力，复杂 query 上保精度。

④ 两阶段轻量训练（共 ~600 步）

第一阶段：仅训练低维投影参数 $W^Q_h, W^K_h$（每 head 仅 $2 \times 16 \times d_h$ 参数），冻结主干。第二阶段：在 32K–80K 长文档上做 600 步端到端微调，平均长度 48K。整个训练在 H20 GPU 上完成。论文用约 100 训练步即可让 LongBench 平均分接近 full attention 的 53.80%（RTPurbo top-p 达 54.24% 反超 0.44pt）。

③ 深度案例：Qwen3-Coder-30B-A3B 在 multi-V 任务上的稀疏化

任务：LongBench multi-V（多 value 多跳问答），需要从分散的多个 value 段落聚合信息。
Full Attention baseline：在该子任务上保持 76.2 / 76.2 满分。
FlexPrefill（依赖相邻块）：因证据分散到多块、不连续 → 显著掉点（multi-V 平均掉 ~10pt）。
SnapKV / Minference（用 recent query 估全局）：multi-Q / multi-K 直接坍塌（原文表 4 显示 RULER 64K 上 SnapKV 多跳子任务掉至 60% 以下）。
RTPurbo top-p：保留 ~15% 的 retrieval heads + 8K 局部窗口。在 multi-V 上几乎无损（单题 token 选择数依据复杂度从 ~1K 动态扩到 ~6K），LongBench 整体平均 54.24%（vs Full 53.80%）。
关键：dynamic top-p 让简单 needle 任务自动节省算力，复杂多跳任务保留更多 token——同一套阈值在不同任务上自适应。

⑤ 消融与讨论

消融分析：(1) retrieval head ratio 从 15%→10% 时 LongBench Avg 掉约 0.5pt，→25% 时几乎不再涨说明 15% 是 sweet spot；(2) 低维 r 从 16→8 时 RULER 多跳掉 1pt，→32 几乎无收益，说明 RoPE 低频信号确实能用 16 维捕获；(3) top-p vs top-k：在 LongBench 上差 ~1pt，但在 reasoning 任务上 top-p 优势更明显，因为 reasoning 长度高度异质；(4) 不要 sliding window 会让 PR-zh 直接掉到接近随机；(5) 训练步数 100 vs 600：100 步已恢复 ~95% 性能，600 步收敛到 +0.44 pt 反超 Full。局限：(a) 只在 Qwen3-30B-A3B 系列上验证，对 dense 大模型（如 70B+）head specialization 是否同样稳定待验证；(b) 校准依赖 needle 任务，对极端不规则的 retrieval 模式（如代码）可能需要重新校准；(c) MoE 模型每个 expert 的 retrieval 行为是否一致未明确说明。

⑥ 结论与启发

RTPurbo 的核心启发：full-attention 模型本身就内禀稀疏，主流"重训练换稀疏"或"启发式驱逐"都低估了原生 attention 的结构信息。把训练成本压到 600 步、把目标参数压到几个低秩投影，就能把 30B 模型在 512K 长上下文上稳住——这对在线服务至关重要。它揭示了与 OSCAR 同源的工程哲学：低成本压缩的关键是"对齐 attention 真实使用的几何"。OSCAR 对齐 $C_Q, C_S$ 协方差，RTPurbo 对齐 RoPE 低频子空间——本质都是"少数主成分负责所有重要信号"。对长上下文 LLM 服务的产品化意义：未来 32K→512K 上下文不再需要重训，sparse retrofit 路线已经具备生产可行性。

LLM-3 · Context Memorization for Efficient Long Context Generation (Attention-State Memory)

📅 2026-05-19 🏛 Institute of Science Tokyo, Imperial College London arXiv:2605.18226
LLM 长前缀 In-Context Learning RAG

① 背景与动机

现代 LLM 应用越来越依赖长 conditioning prefix（few-shot 示例、任务说明、检索文档）来控制模型行为。但 prefix-augmented 推理有两个结构性缺陷：(i) Prefix 影响力会衰减：随生成进行，prefix 在后期 generation token 处的注意力变得稀薄；(ii) Attention over prefix 是 O(L_p) 线性扫描，长 prefix 让推理延迟随 prefix 长度线性增长。现有解法两类：(a) 在 attention 中保留 prefix 但压缩它（如 KVzip），仍要 attend prefix；(b) 把 prefix 内化到参数（fine-tuning），训练昂贵且不适应频繁更新的 prefix。本文提出 ASM：把 prefix attention 完全外化为 query-indexed 字典，不再 attend prefix、训练免费、可动态更新。

② 核心方法详解

① Attention 状态分解的可外化性（Sufficiency + Composability）

关键数学观察：标准 attention 在 prefix 段产生的输出可以表示为 attention state $(a_p(q), Z_p(q))$，其中 $a_p$ 是 attended value sum，$Z_p$ 是 normalizer：

$$a_p(q) = \sum_{i \in \mathrm{prefix}} \exp(q^\top k_i)\, v_i,\quad Z_p(q) = \sum_{i \in \mathrm{prefix}} \exp(q^\top k_i)$$

由 online-softmax identity，prefix 部分的 attention 与 response 部分可以独立计算后合并：

$$\mathrm{Attn}(q) = \frac{a_p(q) + a_r(q)}{Z_p(q) + Z_r(q)}$$

Sufficiency：$(a_p, Z_p)$ 完全决定 prefix 对 attention 的贡献，知道它们就不再需要 prefix 本身。Composability：多 prefix chunk 可以分别 forward 然后 merge，不需要一次性放入显存。

② Memory Bank：每层一个 K-entry 字典

对每一层 $i$ 构造字典 $\mathcal C^{(i)} = \{(\bar q_k^{(i)}, \bar a_k^{(i)}, \bar Z_k^{(i)})\}_{k=1}^K$，其中 $\bar q$ 是 lookup key（簇质心 query 表示），$(\bar a, \bar Z)$ 是该簇的合并 attention state。$K \in \{1K, 2K, 4K, 8K, 16K\}$。每条 entry 的存储足迹按 GQA 调好与"一个 KV cache entry"等价。

③ 离线校准：Attention-Aware K-means 聚合

第 1 阶段（Collection）：对每对 $(p, t)$ 走一次 forward 收集 $(q, a_p(q), Z_p(q))$；第 2 阶段（Clustering）：对 query 向量做 K-means，每簇聚合方式不是简单平均，而是按 attention 数学合规聚合：

$$\bar q_k = \frac{1}{|C_k|}\sum_{q \in C_k} q,\quad \bar Z_k = \frac{1}{|C_k|}\sum_{Z \in C_k} Z,\quad \bar a_k = \frac{\sum_{C_k} Z\,a}{\sum_{C_k} Z}$$

注意 $\bar a_k$ 是 Z 加权平均而非简单算术平均，这正是 online-softmax 合并的代数结构（公式 2）。$\bar Z_k$ 用 $|C_k|$ 归一化以避免无界 merge 在 attention scale 上引入噪声（这是先前合并多文档常见的失败模式）。

④ 在线推理：Top-m 检索 + 合并到自注意力

推理时，每层用当前 token 的 query $q$ 去检索 top-m 个相似 entry：

$$\mathrm{out}(q) = \frac{\sum_{k \in \mathrm{Top}\text{-}m} \omega_k \bar a_k + a_r(q)}{\sum_{k \in \mathrm{Top}\text{-}m} \omega_k \bar Z_k + Z_r(q)},\quad \omega_k = \exp(q^\top \bar q_k)$$

这样推理时不再 attend prefix——只 attend response 自身（生成的几百 token）+ 一次字典 lookup。Attention 复杂度从 $O(L_p + L_r)$ 降到 $O(L_r + m)$，对 32K prefix + 几百 generation 来说接近于"前缀消失"。

③ 深度案例：在 NBA RAG 上 20% memory 反超 ICL

任务：RuleArena NBA 球员交易规则问答，rulebook ~20K token。
Zero-shot baseline（不给规则）：21.2% exact-match。
Full ICL（20K rulebook 全塞 context）：24.1%。
ASM @ K=4K entries（用 4K 字典 entry，memory footprint ≈ 4K KV cache，仅 ICL 的 20%）：25.5%，反超 Full ICL 1.4 pt。
关键洞察：accuracy vs entries 是非单调的——K=4K 是甜点，K=8K 反而掉到 23.2%、K=16K 掉到 19.4%。原因是训练数据有限，bucket 太多时每簇质心估计不准。这意味着codebook size 是任务相关的超参数，并非越大越好。

LLM-4 · Why Do Reasoning Models Lose Coverage? The Role of Data and Forks in the Road

📅 2026-05-19 🏛 Penn State University arXiv:2605.17026
LLM 推理模型 pass@k 衰减 数据中心

① 背景与动机

推理后训练（reasoning SFT、RLVR）能稳定提升 pass@1，但近期文献反复观察到一个反直觉现象——coverage shrinkage：post-trained 模型在更大 k 下的 pass@k 反而低于其 base 模型。这意味着 post-training 不是"全面让模型变强"，而是"把概率质量集中到一条路径上"，从而损失了 base 模型本就具备的解题路径多样性。这对 best-of-N、test-time scaling、majority voting 等推理策略都是直接威胁。已有解释多归因于优化算法或 entropy collapse；本文给出一个全新的数据中心假设：shrinkage 由训练数据中"决策点（forks in the road）"的特性驱动。

② 核心方法详解

① Fork-in-the-road Hypothesis

正式假设：当 solution space 含多条等价路径，但训练数据 obscure 了"为什么选这一条"的 rationale 时，模型无法学到泛化的选择机制；它转而依赖虚假（spurious, non-causal）线索来消解歧义，结果是把概率集中到次集合上，导致大 k 处 pass@k 衰减。

这个假设的优雅之处在于把 shrinkage 和 SFT 的 survivorship bias 直接挂钩：fine-tuning 数据通常只包含"成功的最终路径"，每个 fork 处的替代路径与选择理由被结构性删除。模型于是学到：这种 prompt → 这种走法，但这一映射的本质是"伪相关"，缺乏因果性。

② Graph Navigation 受控实验：Forward vs Reverse (w/o DP)

构造一个 star-graph 依赖链：每个 prompt 给出变量间的函数依赖与 target query。binary branching 节点本身就是天然的决策点。两个对照组：

  • Forward 设置：模型必须在分支节点选一个分支（含决策点）。
  • Reverse (w/o DP) 设置：把同一问题用 target→source 反向求解，分支选择被消除。

两个设置对应同一个底层任务，因此任何行为差异都可归因于决策点的存在。在 Qwen-2.5-0.5B 与 EvoLM-1B 两个 backbone 上重复实验。

③ Confidence Collapse at Decision Points

定义决策点处的 confidence 为 token 概率：

$$\mathrm{conf}_{\mathrm{DP}}(t) = \max_v p_\theta(v \mid x_{

论文发现：随 SFT epoch 增加，模型在决策点的 confidence 急剧上升，但这种上升是非选择性的——不论模型选对还是选错，confidence 都被推到接近 1.0。这是 shrinkage 的微观机制：模型"自信但盲目"，把不确定性折叠为一个 spurious commit。

④ Mitigation：Top-k Prefix Sampling

既然初始 token 是决策点的高发区，论文提出在 inference 端用 Top-8 prefix sampling：每条样本的前 N 个 token 强制从 top-8 candidates 采样而非 top-1，再进入 standard decoding。这相当于在决策点处人为重新注入多样性，反向"解锁"被 SFT 抑制的路径。

③ 深度案例：Graph Navigation 上的 Coverage Shrinkage

Forward 设置（含决策点）：
  1. pass@1 单调上升（这是 SFT 的预期收益）。
  2. pass@2/8/32 先上升、后下降——k 越大、衰减越明显。
  3. 到 epoch 16，pass@32 显著低于 epoch 1 的 base。
Decision Point Confidence 分析：epoch 1→16，模型在 fork 处的 token confidence 从近均匀分布（多模态）逐步坍塌到 ≈1.0（单峰）；但拆开"correct/incorrect"看，错误分支同样获得高 confidence — 这就是"自信地走错路"。
Mitigation 实验（Top-8 prefix sampling）：在 EvoLM-1B 上，pass@1 仅小幅下降，pass@64 从 0.72 →恢复到 0.80；Qwen-2.5-0.5B 上 pass@64 从 0.88→0.92。说明多样性没有被消除，只是被锁死在 decoding 起点。

⑥ 结论与启发

这是 2026 年迄今最重要的"反直觉认知"之一：推理 SFT/RLVR 提升 pass@1 是有代价的，coverage 衰减不是 bug 而是 feature——只要训练数据 obscure 了决策点的 rationale，模型就必然学会"自信地猜"。对 RLVR 实践（GRPO、LPO 等政昀正在研究的方向）的直接启示：评测必须同时报告 pass@k 曲线，单看 pass@1 会严重高估算法收益；同时数据 curation 应当为 fork 显式标注"为什么"，而不只是给 ground-truth 路径。这一假设也为 best-of-N 设计（如 majority voting、verifier-guided search）提供了新视角：base 模型的 path diversity 是宝贵资产，post-training 不能只追求 sharpening。

RL-1 · General Preference Reinforcement Learning (GPRL)

📅 2026-05-19 🏛 Stanford University, University of Oklahoma arXiv:2605.18721
RL GPRL GPM Reward Hacking

① 背景与动机

当前 LLM post-training 的两条路径几乎是脱节的：(1) RLVR（GRPO + verifiable reward）能在 math/code 上诱发 emergent reasoning，但依赖可执行的 verifier，无法处理 open-ended 任务；(2) 偏好优化（DPO、SimPO、SPPO、GPO）能处理 open-ended，但放弃了 online exploration —— 它们要么 offline、要么"3 轮迭代"。要把两者合并就需要一个 open-ended 的 verifier，但标量 reward model 是错误的形状：质量本质上是多维的，任何标量得分都是不完整代理，会让 online RL 坍塌到该 RM 最敏感的那条轴（典型如 verbosity / 拍马屁）。GPRL 的回答：用 General Preference Model（GPM）的 $k$ 个 skew-symmetric 子空间替代标量 reward，让 online RL 拥有"多维 + intransitivity-aware"的偏好信号。

② 核心方法详解

① GPM：把偏好嵌入到 $k$ 个 skew-symmetric 子空间

对每个回答 $y$，GPM 学一个 $2k$ 维归一化嵌入 $v_{y|x} \in \mathbb R^{2k}$，并通过 block-diagonal skew-symmetric 算子 $R_\succ$ 给出每对回答的得分：

$$s_l(y_i, y_j \mid x) = v_i^{(2l)} v_j^{(2l-1)} - v_i^{(2l-1)} v_j^{(2l)},\quad s(y_i \succ y_j \mid x) = \sum_{l=1}^k \lambda_l(x)\, s_l(y_i, y_j \mid x)$$

每个子空间 $l$ 的得分 $s_l$ 天然反对称（因此可以表示 intransitive 偏好 A≻B≻C≻A），而权重 $\lambda_l(x) \ge 0$ 是 prompt 相关的"维度门控"。在 Skywork-Reward 上 $k=3$ 时三个子空间分别对应"helpfulness vs verbosity / factual accuracy vs fluency / safety vs directness"等可解释维度。

② Per-dimension Group-Relative Advantage

对 $G$ 个 rollout，先对每个子空间 $l$ 算群内 pairwise 平均：

$$\hat s_l^{(i)}(x) = \frac{1}{G-1}\sum_{j \ne i} s_l(y_i, y_j \mid x)$$

关键：每个子空间独立做 mean/std 归一化，再用 $\lambda_l$ 加权聚合：

$$\hat A_l^{(i)}(x) = \frac{\hat s_l^{(i)}(x) - \mu_l(x)}{\sigma_l(x) + \epsilon},\quad \hat A^{(i)}(x) = \sum_{l=1}^k \lambda_l(x) \hat A_l^{(i)}(x)$$

这是 GPRL 与"全局归一化标量 reward"的本质差别——任何单一维度因 magnitude 大而 dominate 的能力被切断。论文形式化证明（Theorem in §3.1）：在足够条件下，GPRL 的聚合 advantage 拒绝任何标量 reward 必然倾向的 single-axis exploitation。

③ GPRL Objective：GRPO 的多维替换

把 advantage 插入 GRPO 的 clipped surrogate：

$$\mathcal L_{\mathrm{GPRL}}(\theta) = -\mathbb E_{x, \{y_i\}}\!\left[\frac{1}{G}\sum_{i=1}^G \min\!\big(r_i \hat A^{(i)},\ \bar r_i \hat A^{(i)}\big) - \beta\,\mathrm{KL}(\pi_\theta \| \pi_{\mathrm{ref}})\right]$$

其中 $r_i = \pi_\theta(y_i|x)/\pi_{\theta_{\mathrm{old}}}(y_i|x)$ 是重要性比，$\bar r_i$ 是其裁剪版本。GPRL 在 $k=1$ 时严格退化为 GRPO+BT，因此可以无缝接入 vLLM rollout / 分布式训练栈，唯一的额外开销是每组一次 GPM forward。

④ Closed-loop Drift Monitor：在线检测 reward hacking

核心可观测量是 advantage variance share：$\alpha_l(t) = \mathrm{Var}_l(t) / \sum_{l'} \mathrm{Var}_{l'}(t)$。健康训练下 $\alpha(t)$ 保持初始形状；reward hacking 时 $\alpha(t)$ 坍塌到某一维 $l^\ast$。定义 drift 量

$$D(t) = \big\|\alpha(t) - \alpha(0)\big\|_2$$

$D(t) > \tau$ 时触发控制器：自动 reweight $\lambda_l$ 抑制 over-optimized 维度，并紧缩 KL trust region $\beta$。这是无需任何外部 RM 评估信号就能识别并矫正 reward hacking 的机制——一种纯内部一致性检验。

③ 深度案例：GPRL vs GRPO+BT 的 length collapse 对比

设定：base = Llama-3-8B-Instruct，rollout policy 与 GPM/BT-RM 都是 8B 规模，prompt set = UltraFeedback。
GRPO + 8B BT scalar reward（baseline）：epoch 1–2 性能上升，epoch 3 开始 unwinds——AlpacaEval 2.0 LC.WR 回落到 41.92%。响应长度被推到 ~3000 token（length hacking 经典模式：判官偏长，模型就疯长）。
SPPO / GPO（iterative baselines）：iteration 3 时 LC.WR 停在 41–43%，但响应长度 inflate 到 2400–3300 token。
GPRL with 8B GPM（本文）：LC.WR = 56.51%（+14.59 vs GRPO+BT），且响应长度仅 1600 token（全表最短）。
机制可视化：drift trajectory 图显示 healthy run 的 $D(t)$ 始终接近 0；hacked run（关闭 controller）下 $D(t)$ 在 $t' \approx 4000$ 步处突破阈值 $\tau$，BT-RM 分数仍单调上升但实际 LC.WR 已 unwind。开启 controller 后，drift 被实时矫正回归。

⑥ 结论与启发

GPRL 是 RLVR 之外最值得 RL 研究关注的"重大设计变量"：把标量 reward 升级为多维 skew-symmetric 偏好，是真正解决 reward hacking 的结构性方案，而非缝补。它提供了一个 GRPO 的"无副作用替换"——在 $k=1$ 时严格等价于 GRPO，在 $k>1$ 时获得多维结构红利。对政昀正在研究的 RLVR/GRPO/LPO 方向尤其重要：(a) per-dimension advantage normalization 是可以直接迁移到 LPO 的设计模式，能阻止单维 dominance；(b) drift monitor 提供了一个无需外部 verifier 的 reward hacking 检测器，对 open-ended RL 训练流水线是必备工具；(c) GPM 与 RLVR 完全互补——math/code 走 RLVR，open-ended 走 GPRL，构成下一代 post-training 的双轨并行框架。

RL-2 · Nudging Beyond the Comfort Zone: Efficient Strategy-Guided Exploration for RLVR (NUDGERL)

📅 2026-05-19 🏛 KAIST, DeepAuto.ai arXiv:2605.15726
RL RLVR Exploration Strategy Nudging

① 背景与动机

RLVR（GRPO + verifiable reward）的本质瓶颈是探索：策略只能在它已经采样到的轨迹上改进。要发现稀有正确轨迹，naive 做法是疯狂加 rollout 数（$N=64, 128$），但概率论给出的下界是 $\mathbb E[\#\text{rollouts}] = 1/\pi(y\mid x_0)$——对低概率正确轨迹而言成本爆炸，而且 GRPO 在 $N=64$ 时已经表现出训练不稳定。另一类方案（POPE）依赖 oracle solution prefix 提供 privileged information，scaling 困难且容易锁死到某条窄路径。NUDGERL 提出第三条：用 策略级 lightweight context（不是 oracle answer，而是几个 reasoning 策略的关键词）来 nudge 模型主动遍历多种思路，并配套 inter-intra group advantage + self-distillation 解决 train-test gap。

② 核心方法详解

① Strategy-Level Context Pool（不是 oracle，是 hint）

对每个 prompt $x_0$，用 gpt-4o-mini 廉价生成 2 个策略关键词（例如对几何题给"Pythagorean theorem" / "shoelace formula"），构成 $C(x_0) = \{c_1, \ldots, c_M\}$（论文 $M=2$）。每个 rollout 在采样前独立采样一个 strategy context $c_i \sim \mathrm{Unif}(C(x_0))$ 并 append 到 prompt：

$$y_i \sim \pi_\theta(\cdot \mid x_0 \oplus c_i)$$

这样 $G$ 个 rollout 被结构性切分成 $M$ 组，强制覆盖不同策略。Strategy 不是答案，只是一句"试试用这个工具"——成本相比 oracle 解法低 1-2 个数量级。

② Inter-Intra Group Advantage（跨组 + 组内的双层信用分配）

由于 rollout 在不同 context 下生成，奖励差异同时反映"轨迹质量"和"context 偏置"，标准 group advantage 不可靠。NUDGERL 把 advantage 拆成两层：

$$\hat A_i^{\mathrm{intra}} = \frac{r_i - \mu_{c(i)}}{\sigma_{c(i)} + \delta},\quad \hat A_i^{\mathrm{inter}} = \frac{\mu_{c(i)} - \bar\mu}{\bar\sigma + \delta}$$

其中 $\mu_{c(i)}, \sigma_{c(i)}$ 是当前 rollout 所属 strategy 组的统计量，$\bar\mu, \bar\sigma$ 是所有 strategy 组之间的统计量。最终 advantage 用调节因子 $\lambda \in (1, 2]$ 加权：

$$\hat A_i = \lambda \cdot \hat A_i^{\mathrm{intra}} + (1 - \lambda) \cdot \hat A_i^{\mathrm{inter}}$$

直觉：来自高奖励 strategy 组的成功 rollout 拿到更大 advantage（"这是个好策略，多抄"），来自低奖励 strategy 组的稀有成功 rollout advantage 较小（"幸运 outlier，别学过头"）。

③ Self-Distillation 缩小 train-test gap

训练时 rollout 都带 strategy context，但推理时没有—— 这造成显著的分布漂移。NUDGERL 在每步加入蒸馏项：

$$\mathcal L_{\mathrm{Distill}} = -\sum_{(x_0, y^\ast) \in \mathcal H_{\mathrm{good}}} \log \pi_\theta(y^\ast \mid x_0)$$

其中 $\mathcal H_{\mathrm{good}}$ 是"在 strategy context 下产生的正确轨迹"，但训练目标里把 context 抹掉，让 base prompt 也学会这条路径。这把"context 暴露下发现的解法"内化进 base policy，使推理时不再需要 strategy hint。

④ 完整 NUDGERL 目标

把 GRPO clipped surrogate 与蒸馏项合并：

$$\mathcal L_{\mathrm{NUDGERL}} = -\mathbb E\!\left[\min(r_0 \hat A,\, \mathrm{clip}(r_0, 1\!-\!\epsilon, 1\!+\!\epsilon)\hat A)\right] + \alpha \mathcal L_{\mathrm{Distill}}$$

训练流程：strategy-conditioned rollout → inter-intra advantage → policy update + 同步蒸馏。完全兼容 vLLM/DAPO 等标准 RLVR pipeline。

③ 深度案例：AIME 上 GRPO 全军覆没的题被 NUDGERL 攻破

问题：AIME25 一道函数方程几何题——求所有 $f: \mathbb N^2 \to \mathbb R$ 满足 $f(x+y) + f(x-y) = f(3x)$。
GRPO 训练后模型行为：32 rollouts 全部尝试"coordinate geometry"主导路径——具体看 13 次坐标几何 + 5 次 symmetry assumption + 4 次 area decomposition + 5 次其他次优策略。32/32 全部 truncated & wrong。Shoelace formula（这道题的关键武器）只被 sample 到 1 次，且因前后不一致也错。
NUDGERL 训练后模型行为：32 rollouts 中 coordinate geometry 仍是 15 次，但shoelace formula 出现 6 次（10 次中 6 次正确）。其他长尾策略（如 Pythagorean）也都被覆盖。正确率从 0/32 → 6/32（≈19%）。
关键洞察：NUDGERL 没有"教会"模型新策略——base 模型本就具备使用 shoelace 的能力，但 GRPO 训练让概率坍塌到 dominant mode。Strategy Nudging 通过在训练阶段强制采样 shoelace context 然后蒸馏回 base，解锁了被 GRPO 抑制的 latent 能力。这与 LLM-4 (Forks-in-the-Road) 的发现深度共鸣。

消融分析：(1) 无 self-distillation：训练时 strategy 看似有用，但推理时（无 context）性能掉 ~3pt，证明蒸馏是 train-test gap 的关键桥梁；(2) $\lambda$ 设置：$\lambda \approx 1.5$ 最优，$\lambda=2$（纯 intra）丢失跨策略信号，$\lambda=1$（intra=inter 平均）削弱组内区分；(3) strategy context 数量 $M$：$M=2$ 已显著超越 $M=1$，$M=4$ 收益递减（受限于 rollout 总数）；(4) strategy 是否经过验证：用 gpt-4o-mini 不验证地生成的策略 vs 人工筛选的策略——基本等效，说明 noise 容忍度高，scalable。局限：(a) 仅在 math 域验证，code/agent 任务的 strategy 抽象更难自动化；(b) gpt-4o-mini 生成 strategy 仍需 API 成本，对极大规模训练数据可能成为瓶颈；(c) inter-intra advantage 引入的额外超参 $\lambda$ 需手调。

⑥ 结论与启发

NUDGERL 的关键贡献是把"探索瓶颈"从计算资源问题重新定义为结构化引导问题：在固定 rollout 预算下，通过策略级 hint 强制覆盖 long-tail reasoning mode 反而比 8× 加 rollout 更有效。这与 LLM-4 (Forks in the Road) 共同指向同一个核心问题——RL 训练后 base policy 的多样性被默默损毁，而恢复它的关键不是更多算力，而是显式注入策略多样性。对政昀正在研究的 LPO 方向有非常直接的启示：(a) inter-intra advantage 是处理"context-conditioned rollout"的通用模式，可以推广到 LPO 中处理"different prompt rewriting under same query"；(b) self-distillation 闭合 train-test gap 是任何"训练时加 hint、推理时无 hint"算法的必备组件；(c) 用 cheap LLM 当 strategy generator 是低成本可 scale 的工程模板。NUDGERL 与 LLM-4 应该被作为一对"诊断 + 治疗"配套阅读。

RL-3 · Beyond Inference-Time Search: Reinforcement Learning Synthesizes Reusable Solvers (Hero)

📅 2026-05-19 🏛 ETH Zürich, University of Maryland arXiv:2605.18374
RL GRPO 代码生成 组合优化

① 背景与动机

LLM 解组合优化通常采用 inference-time search：每个实例都重新采样、搜索、prompt——成本随实例数线性增长，且不会"积累经验"。本文提出反向命题：能不能用 RL 把"求解逻辑"固化进模型权重，使模型为整个问题族（family）合成一个可复用的代码 solver，而非每实例 separately 推理？作者用 Synergistic Dependency Selection (SDS)——constrained Quadratic Knapsack 的可控变体——作为研究对象。SDS 设计成一个"诱惑陷阱"：local signal 让贪心启发式（如按 weight 排序）看起来很合理，但 quadratic interaction 会让贪心 retrieve 到的 Simulated Annealing 模板语义错误（典型表现：误用 Metropolis acceptance rule）。Best-of-64 base sampling 的 gap 到 Virtual Best Solver (VBS) 卡在 ≈28.7%——这就是 "competence wall"。

② 核心方法详解

① 任务定义：Reusable Solver Synthesis

给定 SDS 问题分布 $\mathcal P$，目标不是"对每个实例生成一段一次性 trace"，而是让模型生成一段对整个 $\mathcal P$ 通用的 Python solver code，编译一次后可重复执行所有实例。形式化地：

$$\theta^\ast = \arg\max_\theta \mathbb E_{p \sim \mathcal P}\!\left[\,\mathrm{Score}\big(\mathrm{compile}(\pi_\theta(\cdot \mid \mathcal P))\,\text{on}\,p\big)\right]$$

RL 把"实例级搜索成本"摊销到一次训练里。

② Hierarchical Reward + Feasibility-Gated Optimality

复合 reward 拆成三层（权重 0.1 / 0.2 / 0.7）：

$$r(o) = \lambda_1 R_{\mathrm{format}} + \lambda_2 R_{\mathrm{exec}} + \lambda_3 R_{\mathrm{nom}}$$

其中 $R_{\mathrm{format}}$ 检查 reasoning-action trace 结构（$\langle\mathrm{think}\rangle\ldots\langle\mathrm{code}\rangle$）；$R_{\mathrm{exec}} = R_{\mathrm{syntax}} + R_{\mathrm{schema}} + R_{\mathrm{struct}} + R_{\mathrm{feas}}$ 是稠密的代码可执行性反馈，并嵌入anti-lazy sorting penalty——惩罚模型 fall back 到"按 weight 排序"这种 linear greedy；$R_{\mathrm{nom}}$ 是 feasibility-gated optimality：

$$R_{\mathrm{nom}} = \begin{cases} \mathrm{Norm}(\mathrm{Score}(o)) & N_{\mathrm{vio}} = 0 \\ 0 & N_{\mathrm{vio}} > 0 \end{cases}$$

这种"不可行就 0 分"的 cliff 强制策略先满足硬约束、再优化目标值，避免陷入"高分但不合法"的退化解。

③ GRPO with Group Size 64, β = 0

用 Qwen2.5-Coder-14B-Instruct 作初始 policy，group $G=64$，无 KL 正则（$\beta=0$）以最大化探索深度。整个训练只有 90 global steps（≈ 4 小时，12×GH200 GPU），共看到 360 prompts（占完整 epoch 的 4.5%），但合成 23,040 reasoning traces。GRPO 的 group standardization 在 reward 稀疏时仍能提供有效梯度。

④ Curriculum-Faded Structural Bonus

$R_{\mathrm{struct}}$ 中的 graph-structure 正向 bonus 随训练步 $t$ 衰减：

$$R_{\mathrm{struct}}(t) = \alpha(t)\,R_{\mathrm{struct}}^{\mathrm{raw}},\quad \alpha(t) \to 0\,\text{as}\,t \to T$$

意图：训练早期需要 structural hint 引导策略走出朴素采样陷阱；训练后期撤掉 hint 让 policy 自我承担优化责任。这与 NUDGERL 的"训练时 nudge → 推理时蒸馏掉"哲学完全一致——本质都是 train-test 距离的动态 schedule。

③ 深度案例：从 misimplemented Metropolis 到 constraint-aware SA

Base 模型 Best-of-64 行为：在 SDS 上检查 base 模型 64 次采样的代码：~30% 的样本提到了 "Simulated Annealing"，但误用 Metropolis acceptance rule——把 $\exp(-\Delta E / T)$ 的符号写错或忽略 quadratic interaction term，导致即使 SA 模板被检索到也无法解出正确解。Best-of-64 的 gap to VBS 锁定在 28.7%，且 cumulative execution cost 高达 17.4 CPU-sec。
Hero 训练后行为：90 步 GRPO 训练后，99.8% 的 feasible 输出收敛到一个 constraint-aware Simulated Annealing 模板——正确实现 acceptance rule、考虑 $W_{ij}$ interaction term、并嵌入硬约束检查。这不是"生成长 reasoning trace"，而是合成一段固定 Python solver。
Compile-once 验证：把训练完成的 frozen Hero solver（一份 Python 代码）不修改地跑遍 1000 道测试集 SDS 实例，pass rate = 97.87%、mean gap to VBS = 4.34%（Appendix C.5 Table 2）。
关键洞察：RL 没有发明新算法（SA 早就存在），但它修复了 base 模型对该算法的 semantic-level 错误实现——这是论文标题"semantic repair"的真正含义，比简单的"RL 提点"深刻。

⑥ 结论与启发

这篇工作给出的最有价值的概念升级是"amortized solver synthesis"：RL 不再为每个实例做 inference-time search，而是把求解逻辑烧进权重并产出可复用代码。这本质上把"推理是消费品"变成"推理是资本品"——一次训练，无数次复用。对 RLVR/GRPO 的实践意义：(a) feasibility-gated reward cliff 是处理硬约束任务的通用模式，避免 reward hacking 到 "高分但违法" 的退化解；(b) 极简胜过复杂：oracle anchoring + diversity bonus 反而损害性能，简洁的 hierarchical reward 已经足够；(c) RL 主要功能是 semantic repair，不是发明新算法——这对评估 RLVR 真正能力边界至关重要。对中金 GPU IPO 项目的延伸思考：当 LLM 越来越多承担"代码合成-自动调度-自动优化"的角色时，"训练一次→产出 reusable solver"的范式会重新定义 datacenter compute 的需求结构（训练侧重 vs 推理侧重的比例会显著倾斜）。

Agent-1 · Look Before You Leap: Autonomous Exploration for LLM Agents

📅 2026-05-19 🏛 University of Science and Technology of China, Meituan arXiv:2605.16143
Agent Exploration Explore-then-Act GRPO

① 背景与动机

LLM agent 在不熟悉的环境中频繁失败，根因是 premature exploitation：模型会基于先验立刻行动，而不肯先收集环境信息。现有 ReAct 风格的 loop 都是 goal-conditioned $\pi(a_t \mid H_t, g)$ —— 没有任何机制为 agent 分配"先观察、再行动"的预算。论文提出两个深刻问题：(1) 当前 LLM agent 究竟有多差的 autonomous exploration 能力？(2) 任务驱动 RL（task-only GRPO）会不会反而损害 exploration 能力？答案是：损害（与 LLM-4 / NUDGERL 完全一致——RL 让模型更窄）。本文把 exploration 作为一种可量化、可单独训练的能力提取出来，并给出 Explore-then-Act 推理范式。

② 核心方法详解

① Exploration Checkpoint Coverage (ECC)：第一个可验证的 exploration 指标

对每个环境实例预定义有限 checkpoint 集合 $\mathcal C = \{c_1, \ldots, c_M\}$，每个 checkpoint 对应"一个有能力的探索者应该发现的事实"——可达地点、关键物体、有效交互目标、功能状态、affordance、约束等。给定 exploration trajectory $\tau_{\mathrm{EXP}}$：

$$\mathrm{ECC}(\tau_{\mathrm{EXP}}) = \frac{1}{M}\sum_{i=1}^M \mathbb 1[c_i \in \tau_{\mathrm{EXP}}]$$

这是不依赖任务成功的 process-level metric，等价于 agent 学界长期缺失的"地图覆盖率"指标。论文为 ALFWorld / SciWorld / TextCraft 三个环境分别构造了 checkpoint set（Appendix E）。

② Autonomous Exploration 形式化

普通 task agent：$a_t \sim \pi_{\mathrm{ACT}}(\cdot \mid H_t, g)$。Autonomous exploration：把 $g$ 移除，agent 只有交互预算 $N$ 步：

$$a_t \sim \pi_{\mathrm{EXP}}(\cdot \mid H_t),\quad \tau_{\mathrm{EXP}} = (o_1, a_1, \ldots, a_N, o_{N+1})$$

其目标是建立环境的内在模型——transition $T(o_{t+1} \mid o_t, a_t)$、状态空间、动作语义、隐藏约束。Exploration 完成后再合成 grounded knowledge summary $\mathcal K = \mathrm{SUMMARIZE}(\tau_{\mathrm{EXP}})$。

③ Interleaved GRPO：交错的 task + exploration 训练

对 exploration 步用 ECC 作 verifiable reward，对 task 步用 task success reward。两种 step 在训练中交错出现：

$$\hat A_i^{\mathrm{exp}} = \frac{\mathrm{ECC}(\tau_i^{\mathrm{exp}}) - \mu^{\mathrm{exp}}}{\sigma^{\mathrm{exp}} + \epsilon},\quad \hat A_i^{\mathrm{task}} = \frac{R_i^{\mathrm{task}} - \mu^{\mathrm{task}}}{\sigma^{\mathrm{task}} + \epsilon}$$

每种 reward 走自己的 GRPO clipped surrogate，混合训练让两种能力同时存在于一个 policy，不必额外切换 expert。论文称为 GRPO (Interleaved)。

④ Explore-then-Act 推理范式

推理时分两阶段：先 $N$ 步 goal-free exploration 产生 $\tau_{\mathrm{EXP}}$，再合成 $\mathcal K$，最后 goal-conditioned acting：

$$a_t \sim \pi_{\mathrm{ACT}}(\cdot \mid H_t, g, \mathcal K)$$

$\mathcal K$ 作为结构化自然语言注入 prompt——包含状态布局、object affordance、action 前置条件、已发现的约束、failure cases。这是把"环境知识"从 implicit policy memory 显式化到 prompt 上下文，可解释、可检查、可调试。

③ 深度案例：Qwen3-4B 在 ALFWorld 上从 31.8% 跃升到 89.7%

诊断阶段（无任务、100 步预算）：
  • Qwen2.5-7B 三环境平均 ECC = 22.2%——大量重复动作、过早终止；
  • Qwen3-4B 平均 ECC = 28.5%；Qwen3-4B + Task-Only GRPO 后反而降到 18.8%——任务训练显著损伤 exploration；
  • Claude-Opus-4.5 平均 ECC = 89.5%（接近完美），GPT-4.1 = 49.3%——开闭源差距巨大。
训练 + Explore-then-Act 后：以 Qwen3-4B 在 ALFWorld 为例，Task-Only baseline 直接执行成功率 72.6%，Interleaved GRPO + E-t-A 达到 89.7%。Qwen2.5-7B 同样从 78.4% → 90.9%。
失败行为对比（Table 3）：Task-Only GRPO 训练后的 agent —— 重复动作率 63.4%、loop 率 16.0%、信息搜索率 1.0%、错误恢复率 0.0%。Exploration-Aware 训练后 —— 重复动作率 24.9%、loop 率 7.7%、信息搜索率 7.5%、错误恢复率 20.1%。这是质变：错误恢复能力从无到有。
关键：探索能力本身就是抗失败的护城河——会 recover 的 agent 就是会 explore 的 agent。

⑤ 消融与讨论

消融分析：(1) Task-Exploration ratio sensitivity（Appendix D）：50:50 配比是甜点，全 exploration 反而损害 task 完成；(2) 不同 exploration model + 同一 acting model：固定 Qwen3-4B 当 actor，仅替换 explorer（如用 Claude 探索 → Qwen 行动），$\mathcal K$ 质量直接决定下游任务表现；(3) 关闭 $\mathcal K$ summary：直接把 $\tau_{\mathrm{EXP}}$ 拼入 prompt 反而性能下降——必须经过 SUMMARIZE 把 noisy trajectory 压成结构化知识；(4) 错误恢复率 0.0%→20.1% 是定性转变。局限：(a) ECC 依赖人工预定义的 checkpoint，对 open-world 任务难以自动构造；(b) 训练数据需要 environment-specific exploration 信号，泛化到新环境仍需要建 checkpoint；(c) E-t-A 范式的 budget $N$ 是手调超参，未优化自适应分配。

⑥ 结论与启发

这篇工作的核心贡献是把"探索"作为 LLM agent 的独立可训练能力提取出来——它有自己的 verifiable reward (ECC)、自己的训练目标、自己的推理范式 (E-t-A)。三个深刻发现共同形成了一个连贯认知：(1) 任务驱动 RL 会损伤 base 模型的 exploration 能力（与 LLM-4 / RL-2 完全同构）；(2) 错误恢复是 exploration 的副产品；(3) Claude-Opus-4.5 是唯一在该指标上接近"人类可信代理"水准的大模型。对正在做的 GPU IPO 和 LLM agent 投资分析有直接参考价值——下一代 agent 的核心壁垒不只是 reasoning step length，而是能在陌生环境里"先看再动"的元能力。这也指出了 RLVR/LPO 研究的一个重要扩展方向：不要只优化 task reward，要把 process-level reward（如 ECC 这类 environment-aware 信号）纳入混合训练目标。

Agent-2 · ICRL: Learning to Internalize Self-Critique with Reinforcement Learning

📅 2026-05-19 🏛 HKUST (Guangzhou), Nanjing University, Microsoft Research arXiv:2605.15224
Agent Self-Critique GRPO 分布校准

① 背景与动机

"Self-Refine"和"Critic-then-Revise"已经成为 LLM agent 的标准 inference-time 增强：模型先尝试 → critic 给反馈 → 模型再修正。但有两个尚未解决的根本问题：(1) Internalization 失败：删掉 critic 后，模型在同一问题上又会失败——说明它学到的是"有 critic 时正确"，没把能力真正内化；(2) Critic 静态化：现有方法的 critic 通常是 frozen 的（甚至来自外部 teacher 模型），随 solver 进化会越来越无关。ICRL 同时解决这两点：从同一 backbone 联合训练 solver 和 critic，并通过 token-level 分布校准把 critique-guided 修正选择性地迁移回 critique-free policy，避免 solver 退化为"对 critic 上瘾"。

② 核心方法详解

① Joint Solver + Critic from Shared Backbone

从同一个 base 模型 $\pi_\theta$ 派生两种 role 行为：solver 角色 $\pi_\theta^S(\cdot \mid q)$ 和 critic 角色 $\pi_\theta^C(\cdot \mid q, \tau_i)$。critic 的 reward 不是 critique 自身的"好坏"，而是下游 solver 接受 critique 后的 performance gain：

$$R^C(c) = R\big(\tau_{i+1} \sim \pi_\theta^S(\cdot \mid q, c)\big) - R(\tau_i)$$

这把 critique 的"actionability"作为可优化目标——critic 必须给出对当前 solver真正能用的反馈，不仅是"对的话"。

② Critique-Conditioned Distribution Calibration

修正轨迹 $\tau_{i+1}$ 来自 critique-conditioned 分布 $\pi^S(\cdot \mid q, c)$，但要更新的是 critique-free policy $\pi^S(\cdot \mid q)$——直接 plug-in 会得到有偏估计。ICRL 引入 token-level reweight ratio：

$$w_t = \frac{\pi_{\theta_{\mathrm{rollout}}}^S(y_t \mid q, y_{

$w_t$ 度量"这个 token 在没有 critique 时是否仍然合理"：

  • $w_t \approx 1$：token 不依赖 critique，可以直接迁移；
  • $w_t \ll 1$：token 强烈依赖 critique，降权避免 reinforce critique-dependence；
  • $w_t > 1$：critique-free 反而会更自然地生成的 token，升权。

这是一种 importance sampling 校正——但用在 token 粒度而非 trajectory 粒度，因此可以同一条 trajectory 内"取其精华去其糟粕"。

③ Role-Wise Group Advantage Estimation

solver 和 critic 在不同 prompt 下采样，奖励量纲不同，混在同一个 group 做 standardize 会把信号污染。ICRL 把 advantage 按 role 分组：

$$\hat A_i^S = \frac{R_i^S - \mu^S}{\sigma^S + \epsilon},\quad \hat A_i^C = \frac{R_i^C - \mu^C}{\sigma^C + \epsilon}$$

各自 GRPO clipped surrogate 加权后联合更新共享 backbone。这是处理 multi-role 联合训练的通用模式。

④ 完整 ICRL 目标

对修正轨迹中 critique-guided 部分应用 token-level $w_t$ 校准的 GRPO surrogate：

$$\mathcal L_{\mathrm{ICRL}} = -\mathbb E\!\left[\sum_t w_t \cdot \min\big(r_t \hat A^S, \mathrm{clip}(r_t)\hat A^S\big)\right] - \mathbb E\!\left[\min\big(r^C \hat A^C, \mathrm{clip}(r^C)\hat A^C\big)\right]$$

无需外部 teacher、无需手工标注 critique 数据，纯 self-improvement 闭环。

③ 深度案例：Qwen3-8B 在 AIME24 上从 50% → 65.1%

设定：base model = Qwen3-8B，benchmark = AIME24（30 道高难度奥数）。
各方法表现：
  • Qwen3-8B base: 40.0%
  • SFT: 40.0%（SFT 对竞赛数学增量很小）
  • GRPO（标准 RLVR）: 50.0%（+10pt 来自答题 reward）
  • Critique-GRPO（外加 frozen critic）: 60.0%（+10pt 来自 critique）
  • ICRL（本文）: 65.1%（+15pt over GRPO、+5pt over Critique-GRPO）
测试时 multi-round refinement（ALFWorld 上）：GRPO baseline 第 1 轮 ~80% 第 3 轮约 ~83%（gradual gains 主要来自重复采样的多次机会，而非真的修正）；Critic-GRPO 第 1 轮 ~85% 第 3 轮 ~91%；ICRL 第 1 轮就 ~92%，第 3 轮达 98%——同时 first-round 更强、refinement 增益更陡，证明它的 critic 真正在产生 actionable feedback。

⑥ 结论与启发

ICRL 是 self-improvement RL 的关键拐点：它正面回答"为什么 self-refine 训练经常退化为 'critic 依赖'"——根因是 critique-conditioned trajectory 直接 plug 进 GRPO 会改变有效目标分布。token-level reweight ratio 是一个简单但极有效的工程修正，可以推广到任何 mixed-prompt-prefix 联合训练场景（包括 NUDGERL 的 strategy context 训练）。对政昀 RLVR/LPO 研究的启发：(a) 在 importance ratio 上做 token-level 校正是处理 distributional shift 的最直接方式，比 SFT mixing 优雅得多；(b) critic 与 solver 共训 + reward 绑定下游 utility 是构造 self-improvement 闭环的标准模板；(c) ICRL 的 8B critic ≈ 32B critic 这一观察具有深刻系统含义——critic 能力不是"参数量越多越好"，而是"reward shaping 越准越好"。

Agent-3 · EvoMemBench: Benchmarking Agent Memory from a Self-Evolving Perspective

📅 2026-05-19 🏛 Beijing Institute of Technology, BUPT, HKUST arXiv:2605.18421
Agent Memory Benchmark Self-Evolution

① 背景与动机

LLM agent memory 文献长期存在"碎片化评测"问题：LoCoMo / LongMemEval 偏 conversational 知识保留；MemoryAgentBench 偏 incremental QA；MemoryArena 偏 multi-session action。每种 benchmark 只测 memory 能力空间的一个角落，而且其中一些（MemoryBench、Evo-Memory）不开源，无法横向比较。结果是：现有 memory 系统众多（Mem0、A-MEM、MemOS、ReasoningBank、AWM、SkillWeaver、AgentKB ……），但没有人能告诉你哪种 memory 在哪种场景下真的有用。EvoMemBench 提出统一的二维分类：In-Episode vs Cross-Episode（时间维度）× Knowledge vs Execution（内容维度）= 4 个 cell，把 memory 评测压平成一个可对比的矩阵。

② 核心方法详解

① 二维分类框架

Memory evolution 的两个正交轴：

  • Scope（范围）：In-Episode（一次任务内 step 之间的演化）vs Cross-Episode（多次任务间的复用与积累）。
  • Content（内容）：Knowledge（事实、规则、约束）vs Execution（动作、工具调用、路径状态）。

四个 cell：

$$\mathrm{INEP\text{-}KNOW},\ \mathrm{INEP\text{-}EXEC},\ \mathrm{XEP\text{-}KNOW},\ \mathrm{XEP\text{-}EXEC}$$

每个 cell 都有针对性子任务，使得"这个 memory 系统对哪类任务有用"可以被精确归因。

② INEP-KNOW：FactConsolidation 任务

评测 in-episode 中的 retention（记住 1 次提到的事实）和 revision（在 conflicting evidence 出现时正确 update）。后者是更难的——大多数 memory 系统能记，但当新证据与旧证据冲突时多步 revision 很少做对。

③ INEP-EXEC：四个 tool-use 任务族（Gorilla File System / Vehicle Control / Trading Bots / Travel Booking）

设计上把 implicit reference（用代词/省略提到的实体）和 multi-op turn 拆解（一句话里多个工具调用）作为关键挑战，强迫 agent 跨 step 对齐 execution state。每个任务在 16K / 32K / 64K / 128K 四个 context 长度下评测——这正是 memory 真正起作用的"context-budget × execution-load"二维空间。

④ XEP-KNOW & XEP-EXEC：跨 episode 共享 context

每个 context 下有 ≥5 个 episode，agent 必须 reuse 上一个 episode 学到的 facts/rules/patterns。XEP-EXEC 测的是"上次成功的 action template 是否能被复用"——这是 procedural memory（AWM、SkillWeaver、AgentKB、ReasoningBank、ACE）真正的战场。

③ 深度案例：INEP-EXEC 上 ReasoningBank 的"反 long-context"胜利

设定：DeepSeek-V3.2 backbone，4 个工具 task（Gorilla / Vehicle / Trading / Travel），各 4 个 context 长度。
无 memory 基线（DeepSeek-V3.2）overall：16K=28.5%, 32K=35.5%, 64K=45.3%, 128K=39.5%。
关键观察 1：Memory 在 context 受限时收益最大。最佳 memory 方法相对 backbone 在 16K 提升 +14.5 pt、32K +14.0、64K +7.8、128K +8.5。当 context 充裕（128K）时多数 memory 反而低于 baseline——append memory 引入噪声、占用 useful context space。
关键观察 2：Procedural memory 全面胜出 retrieval/long-term memory。ReasoningBank 在 16K=43.0、32K=49.5、128K=48.0 三档拿到最优。AWM 在 64K=53.1 拿到最优。它们存的不是"raw fact"而是"reusable strategy"——在工具调用任务上比 BM25/Mem0/A-MEM 这种存 fact 的高 5-10 pt。
关键观察 3：Short-term compression 危险。MemAgent / MemoBrain 在 16K-32K 上甚至低于 no-memory baseline（25.0–26.0 vs 28.5）。原因：execution-state tracking 需要 fine-grained tool 参数细节，简单 summarize 会丢掉关键信息。
结论性 takeaway：当工具调用 agent 部署在受限 GPU/context 预算下（这是真实生产环境），优先选 procedural memory（ReasoningBank/AWM 类），而不是事实型存储。

Medical-1 · SEMA-RAG: Self-Evolving Multi-Agent RAG for Medical Reasoning

📅 2026-05-19 🏛 The Chinese University of Hong Kong, Wuhan University of Technology arXiv:2605.17101
Medical RAG Multi-Agent 临床推理

① 背景与动机

医学 QA 是 RAG 部署的"高难度典型"：临床问答需要先把 patient narrative 解释成可搜索的临床问题、再多轮检索 + 综合 + 仲裁，最后给出可追溯结论。但主流 RAG 框架仍把检索当 static one-shot：单轮 BM25/Embedding → 直接生成。这与临床推理的多阶段过程严重不匹配，导致两类失败：(1) Question-to-query 失真——问题里隐藏的关键约束（如 "hospital day 7"、"renal impairment"、"pregnant"）丢失，retrieval 漂向 generic symptom evidence；(2) Sufficiency 缺乏检查——单轮 retrieval 拿回的证据是否足以做决策没有反馈。SEMA-RAG 把 RAG 从"一次性查询"升级为三角色 + 自演化迭代：I-Agent (interpreter) → E-Agent (explorer) → A-Agent (arbiter)。

② 核心方法详解

① I-Agent：临床 Schema 化的问题解释器

I-Agent 不只是 paraphrase 问题 $Q$，而是把它投影到结构化 clinical schema：

$$Q' = \langle o_{\mathrm{int}},\, o_{\mathrm{ent}},\, o_{\mathrm{cons}},\, q_{\mathrm{init}}\rangle = \mathrm{Agent}_I(P_{\mathrm{mt}I},\, Q)$$

四个槽位：clinical intent（diagnosis / treatment / dosage）、medical entities（疾病、药物）、clinical constraints（孕妇、肾损伤、成人等）、initial retrieval query。然后 linearize 成 retrieval-ready string：

$$\hat q_{\mathrm{init}} = \mathrm{Linearize}(Q') = \mathrm{Concat}(q_{\mathrm{init}} \oplus o_{\mathrm{int}} \oplus o_{\mathrm{ent}} \oplus o_{\mathrm{cons}})$$

关键设计：linearize 是无参数的拼接函数，但显式把 implicit constraints 暴露给 dense retriever，避免 retrieval 漂移。

② E-Agent：Sufficiency-Driven 自演化迭代检索

E-Agent 维护当前 query 集 $\mathcal Q_t$ 与累积证据集 $\mathcal C_t$，每轮做 Top-k 检索：

$$\mathcal D_t = \bigcup_{q \in \mathcal Q_t} \mathrm{TopK}(q),\quad \mathcal C_t = \mathcal C_{t-1} \cup \{D_i \in \mathcal D_t : \mathrm{ID}(D_i) \notin \mathrm{IDs}(\mathcal C_{t-1})\}$$

然后做 sufficiency 判断：

$$[s_t, g_t, \mathcal Q_{t+1}] = \mathrm{Agent}_E\big(P_{\mathrm{mt}E},\,[Q', \mathcal Q_t, \mathcal C_t]\big)$$

$s_t \in \{0, 1\}$ 是 sufficiency flag；$g_t$ 是 gap description；如果 $s_t = 0$ 则生成 $m$ 个 follow-up query 针对 gap 继续检索。终止条件：$s_t = 1$ 或达到 $T_{\max}$ 或 stagnation（$\mathcal Q_{t+1} = \emptyset$）。这是 RAG 第一次在 medical 域内引入"可追踪的证据收敛轨迹"$\tau = \{[\mathcal Q_1, \mathcal C_1], \ldots, [\mathcal Q_T, \mathcal C_T]\}$。

③ A-Agent：证据仲裁与可追溯报告

A-Agent 从收敛证据集 $\mathcal C^\ast$ 中：(a) 删除冗余/不相关；(b) 识别 consistencies 和 conflicts；(c) 把 supporting / refuting clue 组织成结构化报告 $R$，并保留每个证据的 source ID：

$$\mathcal S^\ast = \{(\mathrm{ID}(D_i), D_i) \mid D_i \in \mathcal C^\ast\},\quad R = \mathrm{Agent}_A(P_{\mathrm{mt}\,\mathrm{adj}},\,[Q, \mathcal C^\ast, \mathcal S^\ast])$$

最终答案选择 grounded 在 $R$ 上——这一步把"reasoning trace"和"answer"显式分离，让审计员可以独立检查证据链而非只看最终答案。

④ 关键超参 $T_{\max}$ 与 $m$

论文把"探索深度 $T_{\max}$"与"探索广度 $m$"作正交超参分析。Figure 3 显示 $T_{\max}=2$（仅 2 轮）已捕获大部分收益，$T_{\max}=3$ 略有提升，$T_{\max}>3$ 收益饱和并引入 noise；Table 3 显示 $m=1 \to 86.72$, $m=2 \to 89.00$, $m=3 \to 89.95$。最优配置 ($T_{\max}=2, m=3$) 仅 2 轮、每轮 3 query，显著少于 i-MedRAG 的 3 轮固定迭代但效果反超——证明 sufficiency-driven 比 fixed-iteration 更高效。

③ 深度案例：MedQA-US 上 deepseek-v3.1 的差异化

问题：MedQA-US 一道典型多选题——患者 hospital day 7 出现 late-onset 感染症状，问最可能的致病菌（candidate 包括 S. aureus / Pseudomonas / E. coli 等）。
朴素 RAG 失败模式：
  • CoT only（无检索）：依赖 LLM 内部知识，可能遗漏住院日 7 这一时间约束。
  • + MedCPT（单轮检索）：retrieve 到的多是 "S. aureus general" 资料，没有 anchored on "hospital day 7 = late onset"。
  • + MedRAG / i-MedRAG：能多轮，但 question-to-query 把 "hospital day 7" 作为辅助信息丢失，3 轮检索仍无法 surface late-onset evidence。
SEMA-RAG 工作流程：
  1. I-Agent 把问题 schema 化：intent=diagnosis, entities={late-onset infection, S. aureus}, constraints={"hospital day 7", inpatient}, q_init="late-onset hospital-acquired infection day 7 most likely pathogen"。
  2. E-Agent 第 1 轮检索得到 S. aureus 与其他菌的"general profile"——sufficiency=0, gap="time-onset specific differential"。
  3. E-Agent 第 2 轮生成 3 个 gap-targeted 子查询，retrieve 到 late-onset/early-onset 时间窗 epidemiology 资料——sufficiency=1。
  4. A-Agent 把证据组织成报告，识别"late-onset (>5 days) 强烈支持 S. aureus"为 supporting clue，最终答案 S. aureus 与 source ID 一并输出。
结果：deepseek-v3.1 + SEMA-RAG 在 MedQA-US 拿到 89.95%（同模型 i-MedRAG 仅 74.78%，CoT-only 77.53%）。

Medical-2 · Artificial Intolerance: Stigmatizing Language in Clinical Documentation Skews LLM Decision-Making

📅 2026-05-19 🏛 Johns Hopkins University arXiv:2605.17228
Medical LLM 偏见 健康公平 Stigmatizing Language

① 背景与动机

临床决策辅助是 LLM 最被看好的医疗应用，但 frontier model 是否会把临床记录中的污名化语言（Stigmatizing Language, SL）当作真信号、进而做出对患者有害的决定？过去的 medical bias 研究主要关注 demographic perturbation（性别/种族/年龄替换），但 SL 是更 subtle、更 pervasive 的偏见 vector：一句话的"声称在控盐"或"没去做胃镜"会潜移默化让 LLM 给出更不积极的处置方案。Johns Hopkins 团队系统性测了 9 个 frontier LLM × 4 个高度污名化疾病（镰状细胞病 SCD、肥胖、肝硬化、纤维肌痛）× 3 种 SL 表型（doubt 怀疑 / blame 责备 / maligning 贬损）× 4 个剂量（neutral / 1句 / 4句 / 7句 SL），并测试 prompt-level mitigation 策略（CoT、self-debiasing）。结论令人警觉：所有 9 个 frontier LLM 都受影响，且 prompt 矫正几乎无效。

② 核心方法详解

① 三种 SL 表型 + 剂量 - 反应设计

SL 划分三类：

  • Doubt（怀疑）：如 "claims to be following a low-sodium diet"
  • Blame（责备）：如 "failed to schedule the endoscopy"
  • Maligning（贬损）：如 "insisting that we do something about his leg"

每种 SL 设三个剂量 $S(1), S(4), S(7)$ 表示 1、4、7 句 SL 嵌入临床记录。共 9 LLM × 4 疾病 × (1 neutral + 3 SL 类型 × 3 剂量 + 1 mixed) ≈ 360 个组合，每个跑多次随机化测量稳定性。这是医学 bias 研究中第一次做剂量 - 反应曲线。

② 双指标度量：客观治疗 + 主观态度

Treatment Score：每种疾病有专家定义的"应做处置"清单（如 SCD 应给阿片镇痛、影像学转诊；肝硬化应做 endoscopy 与 fluid management）。LLM 输出按这些 checkpoint 打分。PASS (Patient Attitudes Scale Score)：让模型回答验证过的临床医生态度量表（"我尊重这位患者"、"我相信他/她的描述"等），输出主观尊重/信任评分。这两个指标允许研究"模型可能给出"对"的治疗决定，但内部表征已被 SL 污染"这一深刻区别。

③ 效应大小量化：Cramer's V + $\eta^2$

对 categorical treatment outcome 用 Chi-square + Cramer's V：

$$V = \sqrt{\frac{\chi^2 / n}{\min(r-1, c-1)}}$$

对连续 PASS score 用 ANOVA + $\eta^2$：

$$\eta^2 = \frac{SS_{\mathrm{between}}}{SS_{\mathrm{total}}}$$

对每个 LLM 在 4 个疾病上分别计算 demographic 变量（name/age/gender/race/SO）vs SL 的效应大小，从而定量比较 "demographic bias vs SL bias 哪个更主导"。

④ Mitigation 实验：Chain-of-Thought 与 Self-Debiasing

测试两种 prompt-level 缓解：(a) CoT——明确要求模型 step-by-step 推理；(b) Self-Debiasing——在 prompt 中要求模型识别 SL 并忽略其影响。两者都在原 vignette 之后追加。前者部分缓解，但self-debiasing 反而下沉——这是一个深刻发现：模型可以显式识别 SL，但仍隐式被它影响。

③ 深度案例：肝硬化 + Doubt SL 让 GPT 给出"回家服药"建议

原始临床场景：肝硬化患者因 swollen leg 来诊，本应做液体管理 + 利尿剂调整 + 监测肝功能 + 必要时影像学排查 DVT。
Neutral 临床记录：客观描述症状与既往史。LLM 给出齐全处置方案。
+ Doubt SL 一句话：注入 "claims to be following a low-sodium diet"。
GPT-5 (假设变体) 行为变化：没有提议影像学排查、削减利尿剂剂量调整、最终给出建议"Go home and adhere to prescribed treatment" — 即让患者回家"按方服药"。这相当于把"不信任患者自述"从注释泄漏进了医疗决策本身。
剂量 - 反应曲线（Figure 4a）：从 neutral → S(1) → S(4) → S(7)，treatment score 单调下降；S(7) Mixed (3 类 SL 各占一些) 时降幅最大。关键：仅一句 SL 就足以改变决策，且呈现严格 dose-response。
主观态度 vs 客观决策的 divergence（Figure 2 + Figure 5b）：所有 9 个模型的 PASS（尊重 / 信任）都在 SL 暴露下一致下降（Avg ΔPASS = -6.8）；但 treatment score 下降是异质的——某些模型在某些疾病上仍能维持决策，但所有模型对患者的"内部表征"都已被腐化。

消融分析：(1) SL 类型：Doubt > Blame > Maligning（Doubt 影响最大），但三者 mixed 时影响累加超过单独任一种；(2) 剂量响应：S(1) 已显著、S(7) 最严重，但 S(4)→S(7) 的边际增量小于 neutral→S(1) 的初始变化——证明第一次暴露最具杀伤力；(3) Self-debiasing 失败：尽管模型在 prompt 中能识别 "this note contains stigmatizing language"，treatment score 仍低于 neutral baseline——这是知行分离的清晰证据，意味着 RLHF 可能教会了模型"声明不偏见"但没消除"实际偏见"；(4) Demographic vs SL 效应：5 个 demographic 变量的效应大小总和仍小于 SL 单变量。局限：(a) 仅 4 种疾病、9 种模型，泛化到全部医疗场景需扩展；(b) 临床 vignette 是合成的，真实病历更复杂；(c) 缺少实际临床医生评估 LLM 输出的"is harmful"金标签；(d) 没测试 fine-tuning 级别的 mitigation（如指令微调对 SL 鲁棒性的影响）。

⑥ 结论与启发

这项研究意义非凡：它把"算法偏见"的主要载体从显性 demographic perturbation 转向 implicit linguistic framing。RLHF 训练让 frontier 模型在显性人口特征上达成 surface-level fairness，但这些模型对临床记录中潜伏的污名化语言毫无防御。对医疗 LLM 部署的政策含义重大：FDA / EMA 的 AI 医疗器械审批必须把 SL 鲁棒性纳入测试规范，仅测 demographic fairness 远远不够。对 LLM 训练和后训练的研究启示：(a) RLHF 可能在"声明的价值观"和"内部表征"之间制造分离——模型能识别 bias 词汇但仍被影响，这与 LLM-4 (Forks in the Road) 揭示的"自信但盲目"机制深度同构；(b) process-level supervision（如对 PASS 类内部态度量化的 reward）可能比 outcome supervision 更能根治 bias；(c) prompt-level mitigation 是临时止血带，结构性解决方案需要 fine-tuning 阶段或后训练阶段引入对污名化模式的对抗训练。对中文医疗 LLM 工作（包括 DR.EHR / CliniQ 类项目的延伸）：建议把 SL 鲁棒性测试集本地化（中文病历常见的"诉述"、"未遵"、"抗拒"等用词），作为标准评测维度之一。

Multimodal-1 · SaaS-Bench: Can Computer-Use Agents Leverage Real-World SaaS to Solve Professional Workflows?

📅 2026-05-19 🏛 UniPat AI, Peking University, HKU arXiv:2605.15777
Multimodal Computer-Use Agent Long-Horizon SaaS

① 背景与动机

Computer-Using Agents (CUAs) 把 LLM 从"语言理解"推向"主动执行"——可以打开浏览器、点按钮、跨应用操作。但现有 CUA benchmark（Mind2Web、WebArena、OSWorld 等）系统性高估当前能力：(1) 应用级复杂度低（即使可执行环境，page logic / backend 仍简化）；(2) 单 application、单步 goal，没有真实工作流的跨系统协调；(3) 缺少 long-horizon 依赖（真实 SaaS 工作流常需 100+ 交互步）。SaaS（CRM、财务、运营、客服）才是现代 knowledge work 的真实底座。SaaS-Bench 在23 个真实可部署 SaaS 系统上构造 106 个跨 6 大专业领域（Business / Healthcare / Software / Teamwork / Agriculture / Media）的长程任务，含 text-only 和 multimodal 双 setting，用 weighted checkpoint 测部分进展和完整完成。

② 核心方法详解

① 23 个真实 SaaS 系统 × 6 专业领域

不是模拟环境，而是真正部署、有完整 frontend/backend/state 的 SaaS。例子：OpenEMR (medical record)、OnlyOffice (document)、OpnForm (form builder)、Odoo (ERP)、Mattermost (chat) 等。每个 SaaS 都有真实登录、真实数据库、真实 API。这与 OSWorld 用容器化 desktop、WebArena 用合成 web 有本质不同——SaaS-Bench 是"agent 上线就能用"的最严苛环境。

② Long-Horizon Multi-App Tasks（>100 步交互）

每个 task 跨 1-4 个 SaaS app，平均 >100 步交互。任务规格包括：

  • description.md：任务文字描述、所需步骤、登录凭证；
  • verify.py：可执行 Python 验证脚本，调用 SaaS 后端 API/数据库直接检查目标是否达成；
  • weighted checkpoints：5-20 个加权验证点，覆盖 strict completion 与 partial progress。

典型任务：作为公共卫生官员，在 OpenEMR 创建 SOAP note → 在 OnlyOffice 生成案例报告 → 在 OpnForm 构造上报字段 → 关联 ICD-10 编码并提交。

③ Weighted Checkpoint 评分

定义任务 $T$ 的 checkpoint 集 $\{c_1, \ldots, c_K\}$，权重 $w_k$，verify 脚本输出 $v_k(\tau) \in \{0, 1\}$：

$$\mathrm{CheckpointScore}(\tau) = \frac{\sum_{k=1}^K w_k v_k(\tau)}{\sum_{k=1}^K w_k},\quad \mathrm{Resolved}(\tau) = \prod_{k=1}^K v_k(\tau)$$

Checkpoint Score 反映 partial progress（很多任务 agent 完成 30-70% 就卡住），Resolved Score 是严格的 end-to-end 成功（all checkpoints 通过）。这是 SaaS-Bench 比之前 GUI benchmark 都更精细的地方——它不是 0/1 二元，而是带"工作流哪一段最容易卡死"的诊断信号。

④ Pass@k 多次尝试评估

论文不仅报告 single-pass score，还做 pass@1, pass@3 — 这能区分"模型缺乏知识"vs"模型不稳定但偶尔成功"。SaaS-Bench 的发现：pass@3 比 pass@1 平均提升 ~8 pp，说明很多失败来自 execution variance 而非缺知识。

⑤ 消融与讨论

消融分析：(1) Pass@1 vs Pass@3：平均提升 8pp，但模型间差异大——一些模型从重复尝试获益强（高 variance、occasionally lucky），另一些表现稳定但低（low variance）。这表明 single-pass evaluation 严重 misleading。(2) Text-only vs Multimodal：multimodal 任务平均比 text-only 难 ~5pp，但差距小于预期，说明多模态接口理解已不再是主要瓶颈，真正的瓶颈是 long-horizon planning。(3) Domain difficulty：Teamwork（chat-like）最简单，Healthcare（结构化记录 + 数值约束）和 Business（跨 app 数据搬运）最难。(4) Checkpoint stage decay：所有模型 late-stage checkpoint 通过率比 early-stage 低 25-40 pp。局限：(a) 106 任务覆盖 6 个领域，每域约 15-20 个任务，统计置信区间较宽；(b) verify.py 是预定义检查，可能漏掉某些"有创意但有效"的解法；(c) SaaS 系统本身在迭代更新，benchmark 维护成本高。

⑥ 结论与启发

SaaS-Bench 给出的 brutal truth：2026 年 5 月 frontier model 的 end-to-end SaaS 自动化能力远低于业界鼓吹的水平——最强 Claude Opus 4.6 仅完成 2/106，最强 GPT-5.4 也只有 4/106。这与同日 Agent-3 (EvoMemBench) 的发现深度同构：现实任务对 memory + planning + recovery 的综合要求远超单一能力的 benchmark。三大启示：(a) Resolved 与 Checkpoint Score 的分化说明 agent 评估必须双指标——只看 checkpoint 进度容易被 partial progress 蒙骗；(b) 70% 失败来自 entity missing 是工程级关键诊断信号——不是模型不会点鼠标，而是它在长链中"忘了创建对象"；(c) pass@3 vs pass@1 差距 8pp 提示 execution variance 是非平凡因素——Agent-1 的 explore-then-act 与 Agent-2 的 self-critique 这类减少 variance 的机制对生产部署至关重要。对中金 GPU IPO 项目的延伸思考：SaaS 自动化是企业级 GPU 推理需求的一个重要增量来源——一旦 frontier CUA resolve rate 从 4% 跨过 30-50% 阈值，企业 SaaS 工作流的 LLM 推理调用量将出现量级跳跃，这是未来 18-24 个月 datacenter 推理需求结构性增长的重要变量之一，值得在 OC 投行 pitch 材料中作为长期需求侧驱动力之一引用。

📌 今日主线脉络

今日 13 篇精选论文呈现一条清晰的暗线——"RL 让 base 模型变窄"是 2026 年最有争议的核心问题。LLM-4 (Forks-in-the-Road)、RL-2 (NUDGERL)、Agent-1 (Look Before You Leap) 三篇从不同维度直接论证：

  • LLM-4 用 graph navigation 的可控实验证明，决策点存在时 SFT 必然引发 pass@k 衰减，且模型在 fork 处过度自信地走错；
  • RL-2 NUDGERL 用 strategy hint 强制覆盖 long-tail reasoning 模式 + self-distillation 蒸回 base，让模型在 8 rollout 下匹配 GRPO 32 rollout；
  • Agent-1 直接量化 task-only RL 让 ECC（探索覆盖率）下降 9.7 pt，并通过 interleaved exploration reward 把 ALFWorld success 从 72.6% 提到 89.7%。

三篇共同得出"RL 不止优化、也在悄悄抹杀 diversity / exploration"的结论。配套疗法的核心模式相似——在训练时显式注入 diversity 信号 + 蒸馏回 critique-free policy（ICRL 的 token-level reweight 是又一个同模式案例）。这构成今日方法论级别的共识。

📌 RL/GRPO 的设计变量更新

  • RL-1 GPRL：用 $k$-维 General Preference Model 替代标量 reward，直接把 reward hacking 从"事后补救"变为"结构性禁止"。per-dimension advantage normalization + drift monitor 是可直接迁移到 LPO 的两个独立设计 patch。
  • RL-3 Hero / SDS：feasibility-gated reward cliff + curriculum-faded structural bonus 是处理硬约束 + 训练-推理 gap 的成熟模板。
  • Agent-2 ICRL：token-level distribution-calibration reweight ratio $w_t = \pi^S(y_t \mid q) / \pi^S(y_t \mid q, c)$ 是处理 mixed prompt prefix 联合训练的通用工具，可推广到任何"训练时有 hint、推理时无 hint"的算法。

对政昀 RLVR/LPO 研究的直接动作项：(1) GPRL 的 multi-axis advantage 与 LPO 框架结合；(2) NUDGERL 的 inter-intra group advantage 在 LPO 多 prompt rewriting 场景的迁移；(3) ICRL 的 token-level reweight 用于处理任何 conditioned-rollout 训练。

📌 长上下文 / KV Cache 工程的"对齐 attention 几何"哲学统一

LLM-1 (OSCAR INT2 KV cache)、LLM-2 (RTPurbo Sparse Attention)、LLM-3 (ASM Attention-State Memory) 三篇看似互不相关，但共享同一哲学：低成本压缩的关键是对齐下游算子真正使用的几何结构。

  • OSCAR 对齐 $C_Q = Q^\top Q$ 与 $C_S = V^\top S^\top S V$；
  • RTPurbo 对齐 RoPE 低频子空间（仅 16 维即可路由）；
  • ASM 把 prefix attention 的代数结构 $(a_p, Z_p)$ 显式分解、可合并、可外化为字典。

2026 年长上下文 LLM 工程的关键认知升级："通用旋转 / 通用压缩"已经走到尽头，下一阶段必须使用算子感知的目标函数。这一哲学对生产推理栈（vLLM/SGLang）的产品力提升路径有直接指导意义。

📌 Agent / Memory 的产品化结论
