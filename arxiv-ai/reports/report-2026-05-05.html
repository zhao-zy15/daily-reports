🧠 LLM（纯文本大模型）

Component-Aware Self-Speculative Decoding in Hybrid Language Models
📅 2026-05-05 🏛 Universitat Politècnica de València
LLM推理加速自推测解码Falcon-H1 / Qwen3.5
🔥 背景与动机

自回归生成是 LLM 推理延迟的核心瓶颈：每个 token 都需一次完整前向。推测解码（Speculative Decoding）通过 draft-then-verify 范式可得 1.5–3× 的无损加速，但关键挑战是如何获得"便宜且质量高"的 draft 模型。LayerSkip、CLaSp、SWIFT 等自推测方法假设模型是同构层堆叠（skip layer），在纯 Transformer 上可得 1.3–2.0× 的加速。但近期 Falcon-H1、Qwen3.5、Jamba 等混合架构同时包含 SSM/线性注意力与 softmax 注意力，现有 LayerSkip 无法充分利用架构异质性。本文提出核心问题：能否直接把混合模型内部的 SSM-only 子图当作零成本自推测 drafter？

⚙️ 核心方法详解
① 混合架构的形式化与组件提取（Component Subgraph）。 对 \(L\) 层混合模型，每层含替代组件 \(\mathcal{A}_l\)（SSM/线性注意力）与注意力组件 \(\mathcal{S}_l\)。
  • 并行混合（Falcon-H1）：$$h_l = \mathcal{A}_l(h_{l-1}) + \mathcal{S}_l(h_{l-1})$$ Draft 构造：将 \(\mathcal{S}_l\) 输出置零，保留 \(\mathcal{A}_l\) 路径。
  • 序列混合（Qwen3.5）：层分成不交集 \(L_\mathcal{A}=\{线性层\}\) 与 \(L_\mathcal{S}=\{注意力层\}\)（24层=18+6）。Draft 构造：对 \(l\in L_\mathcal{S}\) 做恒等穿透 \(h_l = h_{l-1}\)。
Draft 模型 \(M_d\) 无需额外参数、无训练、无外部模型，通过 forward hook 实时切换。
② Draft-Verify 协议与拒绝采样。 Draft 阶段 \(M_d\) 自回归生成 \(K\) 个 token 并记录 \(q_i(x)\)；Verify 阶段完整模型 \(M\) 一次前向算出 \(p_i(x)\)；每位置 \(i\) 应用标准拒绝采样： $$\alpha_i = \min\Big(1, \frac{p_i(x_i)}{q_i(x_i)}\Big)$$ 贪心解码下简化为 \(\arg\max p_i = \arg\max q_i\) 则接受。全接受率 \(\alpha^{(K)}\) 是衡量 draft 质量的关键指标。
③ 理论加速与 FLOP 比率。 对 Falcon-H1-0.5B，SSM-only 路径约占完整模型 FLOPs 的 78.4%；Qwen3.5-0.8B 线性-only 路径占 77.3%。速度公式： $$\text{Speedup} = \frac{1-\alpha^{K+1}}{(1-\alpha)(1+c K)}$$ 其中 \(c\) 是 draft/verify FLOP 比。SSM 路径在长上下文还有 \(O(1)\) 内存（KV cache 消除）的额外优势。
④ 消融诊断：PPL 降解率预测自推测可行性。 本文发现一个非常干净的规律——通过简单的 PPL 消融（移除注意力后 PPL 膨胀倍数）可提前预测自推测可行性，无需真正实现 draft-verify 流水线：
  • Falcon-H1：PPL 从 5.62 → 17.73（×3.15）→ 接受率 0.370
  • Qwen3.5：PPL 从 7.62 → 624.84（×81.96）→ 接受率 0.019
这个近乎完美的反相关关系意味着：新混合架构可通过单次 PPL 评测（几分钟）决定是否值得投入工程实现。
深度案例：Falcon-H1-0.5B 贪心自推测实例
输入 prefix："The capital of France is"
  1. Draft 阶段（SSM-only）：并行跳过所有注意力输出，SSM 分支输出 \(q(x)\) 顺次给出 \(K=3\) 个 draft token：" Paris", ",", " which"，总耗时 ≈ 1 次完整模型前向的 78%。
  2. Verify 阶段：完整 Falcon-H1-0.5B 单次 forward 算出这 3 个位置的目标分布 \(p(x)\)，top-1 分别为 " Paris", ",", " which"。
  3. 接受判定：3 个位置贪心全匹配 → 接受 3 个 token，一次前向产出 3 个 token（理论 2.4× 吞吐）。
  4. 对照 Qwen3.5-0.8B：线性-only 路径 \(q(x)\) top-1 仅 32% 概率是 " Paris"，\(K=2\) 的全接受率只有 3.8% —— 几乎每次都退化为 1 token/轮。
核心结论：组件感知自推测对并行混合是零成本获胜策略，对序列混合则完全不可行。

🔬 消融与讨论

任务依赖实验表明数学推理（GSM8K）受益最大（Falcon-H1 α=0.495），事实回忆（MMLU）受益最小（0.208），与"注意力主要负责事实检索"假设一致。替代策略对比显示，对序列混合必须用 LayerSkip：Qwen3.5-0.8B 的 LayerSkip-33% α=0.233（12.3× 高于组件感知）。局限性：作者 Python 级实现未达到实际 wall-clock 加速（< 1.0×），需 CUDA 融合核；规模只到 3B，7B+ 情况待验证；未覆盖 Jamba 等 Transformer-Mamba-MoE 复杂块。

💡 结论与启发

这是首次将模型架构异构性转化为零成本推测解码源："组件集成模式决定推测可行性"比参数规模、训练技巧更根本。对未来混合模型设计者的明确启发是——若希望保留自推测友好性，应优先选择并行加法式而非串行交替式组件整合，因为前者保证了每条子路径仍是"够用"的 LM。此外 PPL 消融作为预测诊断的思路可推广到多种推理加速场景，成为"先算便宜指标再决定工程投入"的通用方法论。

Focus on the Core: Empowering Diffusion Large Language Models by Self-Contrast
📅 2026-05-05 🏛 Chinese Academy of Sciences, Renmin University, Xiaohongshu
扩散LLM自对比解码训练无关
🔥 背景与动机

扩散 LLM（DLM，如 LLaDA、Dream）用迭代去噪替代自回归生成，支持双向上下文与并行解码，但解码策略普遍有局部偏好：已去噪 token 附近的候选获得更高 confidence，导致模型只关注局部邻域而忽视真正关键的核心推理 token。作者提出"异构信息密度"视角：大部分 token 只是冗余填充（Low Density, LD），少数核心 token（数字、实体、关键条件词）是逻辑锚点（High Density, HD）。HD token 的过早/不当解码往往是整条推理链偏向错误的根源。

⚙️ 核心方法详解
① HD Token 的三大特性。 作者通过系统的 mask-ratio 实验揭示 HD token 有三个特性：
  • 信息密度异质性：全局 mask 50% 时 HD token 置信度 ΔConfidence≈0.52，LD token 仅 ≈0.26；
  • 强全局上下文依赖：非线性、突变式 confidence 塌陷；
  • "核心优先"早期收敛：在 t=30% 解码进度时 HD token 已收敛 38% 而 LD 仅 28% —— 为并行加速提供契机。
② 自动化 HD 识别：Top-K 截断 JS 散度 + EMA。 传统语言规则（名词+数字）在代码任务上失效。作者用模型内部信号定义不稳定性分数： $$U_t^{(i)} = \mathrm{JS}_{\mathrm{top}K}\big(P_t^{(i)} \parallel P_{t-1}^{(i)}\big)$$ 然后用 EMA 平滑历史振荡： $$S_t^{(i)} = \beta \cdot S_{t-1}^{(i)} + (1-\beta) \cdot U_t^{(i)}, \quad \beta=0.9$$ Top-K=8 截断控制了计算开销，理论上近似误差有 \(2\epsilon_K\log 2\) 上界（附录 Prop D.2）。消融证据：JS_detect 比 POS-tagging 规则在 HumanEval/MBPP/BBH 上分别提升 +4.3/+3.8/+1.4 pp。
③ 自对比解码（FoCore）。 在每一步的条件预测 \(P(x|c)\) 外，额外构造一个"mask 掉 HD token"的负样本 \(\hat c\)，再用 Classifier-Free Guidance 公式： $$\tilde\ell = \ell(x|c) + \gamma \cdot \big(\ell(x|c) - \ell(x|\hat c)\big)$$ 其中 \(\gamma=0.3\)。这样迫使模型在去噪时"主动关注 HD token 的贡献"。附录 Theorem D.2 证明这会严格降低 HD 位置的输出熵 \(\Delta H \ge \gamma \cdot \mathrm{IG}_{\mathcal{I}}(x)\)。
④ 加速变体 FoCore_A：HD 收敛触发并行。 监控平均 EMA 不稳定度 \(\bar S_t = \frac{1}{|\mathcal{U}|}\sum_{i\in\mathcal{U}} S_t^{(i)}\)；一旦 \(\bar S_t \le \tau\)（作者用 \(\tau=10^{-5}\)），说明 HD 锚点已定，立即对周围 LD 区块做并行解码 P 个 token。ablation 显示 P=5–6 为最佳：GSM8K 准确率 73.00 → 75.50% + 1.18×~1.23× 加速；但 P≥10 会塌到 63%（推理链被破坏）。
深度案例：GSM8K 数学题"折扣玻璃杯"
题目："每个玻璃杯 5 美元，但每第二个杯子只需 60% 价格。Kylar 要买 16 个玻璃杯，总共多少钱？"

Confidence 基线（错误）：识别不到"every second"这个关键约束，把 60% 折扣简单应用到全部 16 杯：\(16 \times 3 = \$48\) ❌

FoCore（正确）：通过 JS 散度识别出的 HD token 包括：second (0.094), costs (0.207), price (0.128), 0 (折扣=0.6, JS=5.0)。FoCore 通过 HD-mask 负样本迫使模型正确解读"every second"=每隔一个：8 个全价 + 8 个折扣 → \(8 \times 5 + 8 \times 3 = \$64\) ✓

另一个 GSM8K 案例（橙汁溢出）：基线忽略"洒掉 1 升"，得 13 升水（错）；FoCore 捕获 HD 约束后正确计算 \((10-1) \times \tfrac{2}{3} + 15 \times \tfrac{3}{5} = 15\) 升水 ✓。

🔬 消融与讨论

超参鲁棒性：γ 在 0.3 附近为最优，过小无引导、过大陷入局部最优；Top-K 相对不敏感，K=8 已足够。Countdown 这种组合搜索任务对 γ 更敏感（SVAMP 上 88.0, Countdown 上 21.8，均超基线）。局限：(1) 超参需按任务微调；(2) 仍依赖 CFG 双前向范式，per-step 开销未减少，只是总步数少；(3) 只在 7B/8B 上验证，尚未在 100B DLM 上测试。

💡 结论与启发

FoCore 给出一个重要启示：DLM 的"并行 + 迭代去噪"架构天然容纳基于内部信号的自适应引导，而这是自回归模型无法简单复用的能力。"HD token 提前收敛"本是 confidence-based 解码的副产品，作者反过来把它当成并行加速的触发信号，实现了"同样 8B 模型既更准又更快"的正效应。对生产级 DLM 服务（如 LLaDA2.0 100B）而言，这种训练无关且 0 额外存储的后处理方法可能立即部署。未来 RL post-training 内化"focus-on-core"原则是明确的下一步方向。

Distilling Long-CoT Reasoning through Collaborative Step-wise Multi-Teacher Decoding
📅 2026-05-05 🏛 KAIST
推理蒸馏多教师协同解码Long-CoT
🔥 背景与动机

DeepSeek-R1、QwQ-32B 等大推理模型（LRM）靠 Long-CoT 实现复杂问题求解，但参数大、成本高。主流做法是用 S1、LIMO 等"先生成再筛选"的 curation 蒸馏：每个 teacher 独立产出一条完整 CoT，然后按 PRM/MCTS 打分挑最优。然而这种后处理筛选有两个致命缺陷：(1) 大量 token 被丢弃，浪费算力；(2) teachers 之间的互补性无法被利用——单个 teacher 可能在第 3 步错但第 7 步对，而筛选粒度是整条轨迹，无法拼接。作者提出 CoRD：让多个异构 teachers 在每一步协同解码，而非事后筛选。

⚙️ 核心方法详解
① Prompt-Guided 步骤切分。 Long-CoT 常用 \n\n 或 "wait" 做分步，但这些在单步内也出现，易误判。作者用 prompt 让 LRM 在生成时显式标注 ### Step 1. Understanding the problem、### Step 2. Recalling Descartes' Theorem……这种"命名+编号"格式既符合模型自然生成，又做到了跨模型一致切分。消融表明 prompt-guided 切分相比 line-break/prefix 在 AIME24/25 分别 +3.1/+2.5 pp。
② 协同步进（Collaborative Stepping）。 令 teachers \(\mathcal{T}=\{T_1,\dots,T_n\}\)（如 R1-Qwen-32B、QwQ-32B、Phi4-Reasoning-Plus），在第 \(k\) 步每个 teacher 提出自己的下一步候选 \(s^k_i\)，共 \(n\) 个候选： $$\text{Candidate pool at step } k: \{(\tau_{k-1} \oplus s^k_i, T_i) \}_{i=1}^{n}$$
③ Predictive Perplexity 评分与 Top-b 保留。 用一个第三方 meta-prover \(\mathcal{M}\)（独立于 teachers）在候选步后 "foresee" 最终答案 \(y\)，计算： $$\mathrm{PP}(s^k_i) = \exp\left(-\tfrac{1}{|y|}\sum_{t}\log p_\mathcal{M}(y_t \mid \tau_{k-1} \oplus s^k_i, y_{跨 teacher 跳跃拼接，例如前半段用 R1-Qwen，后半段切到 QwQ。
④ 蒸馏阶段。 保留的协同轨迹作为 SFT 数据。相比 S1/LIMO 产出 1 条/题，CoRD 从协同搜索中挑 \(b\) 条组合式最优轨迹，既降低答案-PP 又扩展多样性。学生模型直接在这些高质量、高互补性轨迹上做序列级 KD。
深度案例：AIME 几何题（4 个两两外切圆）
第 1 步：3 个 teachers 给出候选 —— T₁: "设 R, r, r, r 为四个圆的半径"、T₂: "回忆 Descartes 圆定理：(k₁+k₂+k₃+k₄)² = 2(k₁²+...)"、T₃: "分情况讨论是否一个是外包圆"。
Meta-prover 打分：回到目标答案 42.5，T₂ 的 predictive perplexity 最低（0.215），被选中；但 T₃ 的"外包圆分类"作为备选也保留。
第 2 步：从 T₂ 出发，T₁ 提出 "设 k₄=−1/R 为外圆"，T₂ 提出 "代入 k₁=k₂=k₃=1/r"。此时 T₁ 的 PP 反而更低，协同切换 teacher。
第 k 步起：轨迹成为 T₂→T₁→T₂→T₃ 的组合，任何单一 teacher 都无法独立产出这种"各取所长"的推理链。
最终答案与正解吻合，predictive perplexity 整条路径 = 0.774（CoRD），而 Curation 最佳单轨迹只 0.652。

🔬 消融与讨论

💡 结论与启发

CoRD 把"教师多样性"从 post-hoc 筛选提前到 in-process 协同，显式解锁跨 teacher 的组合式互补。核心哲学与多专家混合（MoE）类似——但这里是"多推理者混合"，在生成时刻做路由。对产业界的意义：未来开源推理模型生态越丰富（DeepSeek-R1、QwQ、o4-mini、Gemini 2.5 Thinking…），这种协同蒸馏越可能成为小模型提升的主流范式，从 "closed-set of trajectories" 升级为 "open compositional search space"。

📅 2026-05-05 🏛 University of Windsor
DPO 改进梯度动力学softmax 挤压效应
🔥 背景与动机

DPO 这类偏好优化最近被发现存在"概率挤压效应"（squeezing effect）：对被拒绝响应的负梯度并非只压它自己，而是通过 softmax 归一化将概率质量推到极少数高置信 token 上，反而让偏好响应的概率也下降。这是 softmax 学习的根本性质，而非超参问题。既有方法（IPO、Cal-DPO、Extend 数据增强）间接缓解但未解决核心：当被拒绝响应已经落在概率"山谷"（很低似然区域）时，继续推它下降收益递减却持续破坏邻近分布。作者提出 Gate-DPO：直接在梯度层面"门控"——若响应已在山谷就衰减其负梯度。

⚙️ 核心方法详解
① 山谷统计量 \(v(y_r)\)。 定义两种统计量检测被拒响应是否在概率山谷：
  • 序列级（几何均值）：$$v_\text{seq}(y_r) = \exp\left(\frac{1}{|y_r|}\sum_{i=1}^{|y_r|}\log p_\theta(y_{r,i}|y_{r,   • Token 级（下尾分位数）：$$v_\text{tok}(y_r) = Q_q\{p_\theta(y_{r,i}|\ldots):i=1..|y_r|\}, \quad q=0.1$$
前者捕捉全局似然，后者更敏感于极低概率尾部。
② 平滑门控函数。 定义门控： $$g(y_r) = \sigma\!\left(\tau \cdot (\log v(y_r) - \theta)\right) \in (0, 1)$$ 其中 \(\theta\) 是阈值（山谷判据），\(\tau\) 控制过渡陡峭度。当 \(v(y_r) \ll e^\theta\)（在深谷）时 \(g\to 0\) 衰减负梯度；当 \(v(y_r) \gg e^\theta\)（不在谷）时 \(g\to 1\) 回归标准 DPO。关键设计：门控用 stop-gradient 从计算图剥离，即 \(g = \mathrm{sg}(g)\)，防止策略"操纵"山谷统计量来绕过门控。
③ Gated 偏好损失。 修改 DPO logit： $$\hat u_\theta = \beta \log\frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \mathrm{sg}(g) \cdot \beta \log\frac{\pi_\theta(y_r|x)}{\pi_{\text{ref}}(y_r|x)}$$ 损失仍是 \(-\log\sigma(\hat u)\)。反向传播得到梯度等式（Theorem 1）： $$\nabla_\theta \mathcal L_\text{Gate-DPO} \supset +\nabla_\theta \log\pi_\theta(y_w) - g(y_r) \cdot \nabla_\theta \log\pi_\theta(y_r)$$ 对 chosen 响应梯度完全不变，只对 rejected 响应乘 \(g\in(0,1)\)。
④ 理论性质与模块化。 三个结构性质：(a) 严格正梯度流：\(g > 0\) 对任意 \(v\)，所以学习信号不会完全消失；(b) 山谷外温和：若 \(v(y_r) \ge e^{\theta+\delta}\)，Gate-DPO 与 DPO 只差指数级小量 \(O(e^{-\tau\delta})\)；(c) 损失无关：同一门控可套在 IPO、Cal-DPO 上（见附录 A），具备即插即用性。
深度案例：UltraFeedback 上的分布塌缩挽救
问题：Qwen-0.5B 在 UltraFeedback 做 DPO 训练时出现典型 squeezing —— chosen 响应 log-prob 从 −24.6 降到 −33.1，而 rejected 响应反而更低（−42.3→−78.5）；整体分布质量 \(\Delta H\) 塌缩至 −2.3 nats。

Gate-DPO 介入：训练到 epoch 2 时，序列级门控统计显示 31% 的 rejected 样本 \(v_{\text{seq}} < e^\theta\)（已在谷）。对这些样本 \(g(y_r)=0.08–0.15\)，负梯度被衰减 85%+；剩余 69% 样本仍走标准 DPO。

结果对比（epoch 5）：
  • Chosen log-prob：DPO −33.1 → Gate-DPO −25.4（几乎保持 SFT 水平）
  • Rejected log-prob：DPO −78.5 → Gate-DPO −39.2（仍被压低但不过度）
  • 偏好 gap (chosen − rejected)：DPO +45.4 nats vs Gate-DPO +13.8 nats（DPO 虚高的 gap 来自分布崩塌，Gate-DPO 真实地保留了两端概率）
  • AlpacaEval win-rate：DPO 52.3% → Gate-DPO 58.1%（vs. SFT baseline）

🔬 消融与讨论

阈值鲁棒性：序列级门控在 \(\theta \in [-5.5, -4.5]\) 范围都满足成功判据；token 级在 \([-3.0, -2.0]\)。τ 控制过渡陡峭度，τ=20 时约 30–40% 样本进入门控。局限：(1) 门控统计量需要全序列前向计算；(2) 多回合 RLHF（如 online RL）下 \(\pi_\theta\) 漂移可能使阈值 \(\theta\) 需动态调整，本文未探讨；(3) 理论仅覆盖 DPO 家族，对 PPO/GRPO 这类 reward-based 方法的扩展留作未来工作。

💡 结论与启发

Gate-DPO 的重要意义是揭示：偏好优化的稳定性问题不在损失函数形式，而在梯度几何。既有 IPO/Cal-DPO 等新损失函数都是间接干预，而 Gate-DPO 直接在梯度流上贴"减震器"，保留 DPO 最大的优点（无需奖励建模的简洁性）同时系统性消除分布塌缩。这种"loss-agnostic + stop-gradient 式检测"范式可能成为未来偏好优化的标准构件，适用于各种基于对比学习的对齐方法。

RMGAP: Benchmarking the Generalization of Reward Models across Diverse Preferences
📅 2026-05-05 🏛 Beijing Univ. Posts & Telecom, Nanjing University
奖励模型基准偏好泛化语言变体一致性
🔥 背景与动机

现有 RM 基准（RewardBench、PRMBench）都隐含"单一普适偏好"假设：同一个 prompt 下某个回答就是"好"的，另一个就是"差"的。但真实用户偏好是多元的——有人要正式书面化解释、有人要口语通俗化，同样一个问答的理想回答取决于用户的偏好轮廓。作者提出 RMGAP 基准，沿两个正交轴评测 RM：(i) 对多元偏好的敏感性，(ii) 对语言变体（同义 paraphrase）的一致性。这是首个显式评估 RM 跨偏好泛化能力的基准。

⚙️ 核心方法详解
① 种子 prompt 过滤与多样性构建。 从 Chat、Writing、Reasoning、Safety 四域共采集 3,371 个种子 prompt，经人机混合过滤（质量、重复率、公平性）保留 1,097 个评测实例。每实例对应 4 条风格分化明显的回答。
② 五维风格分化生成。 沿五个可控维度生成 4 个风格差异化响应：Formality（正式度）、Conciseness（简洁度）、Technicality（技术深度）、Emotionality（情感色彩）、Perspective（视角）。例如同一个"如何解释概率"的 prompt，4 个回答分别是：(a) 高正式+高技术（数学家风格）、(b) 低正式+低技术（类比日常）、(c) 高正式+高简洁（一行定义）、(d) 低正式+低简洁（举多个案例）。
③ 偏好触发 prompt 合成。 对每个响应，合成一个唯一偏爱该回答的 tailored prompt，如 "用 3 个生活中的类比，口语化地解释一下概率"——此 prompt 应让 RM 给 (b) 最高分。RMGAP 的关键契约：偏好是 prompt 条件化的，而不是响应绝对质量。这 4 个"偏好类型"的 prompt 共 4,388 个。
④ 语言变体一致性评测。 每个 tailored prompt 再自动生成 2 个语义等价的 paraphrase 变体，共 3 个表达。评分指标： $$\text{Ranking Consistency} = \frac{1}{N}\sum_{i=1}^{N}\mathbb{1}\Big[\mathrm{rank}_{\text{RM}}(R_i \mid p_i) = \mathrm{rank}_{\text{RM}}(R_i \mid p_i^{(\text{paraphrase})})\Big]$$ 如果 RM 真正理解偏好，则对同义 prompt 的排序应保持一致。这是 "sensitivity" 之外的关键 "robustness" 维度。

种子 prompt："写一段描述春天早晨的文字。"
4 个风格化响应：(a) 古典诗歌风格 30 字、(b) 现代散文 150 字、(c) 儿童故事口吻 80 字、(d) 科学说明文 100 字。

💡 结论与启发

RMGAP 是奖励模型评测从"单一标准"走向"多元化"的重要里程碑。核心洞察——精度与一致性是正交的失败模式——对 RLHF pipeline 有实际指导意义：只做 pairwise 精度验收的 RM 在个性化部署（Copilot-style 多用户场景）中可能表现极差。未来 RM 训练应显式加入 preference conditioning 与语言 invariance 约束（如对比学习、paraphrase augmentation），而不是只靠扩展规模。

T²PO: Uncertainty-Guided Exploration Control for Stable Multi-Turn Agentic Reinforcement Learning
📅 2026-05-05 🏛 Amazon, UCLA
多轮 Agentic RLToken-level 不确定性训练稳定性
🔥 背景与动机

多轮 Agentic RL（WebShop、ALFWorld 等长时序任务）普遍面临训练塌缩问题：成功率下跌、KL 散度和梯度范数爆炸。既有稳定化方案（fine-grained credit assignment、process reward、轨迹过滤）多在 trajectory-level 做干预或靠 reward shaping 间接控制。作者论证不稳定的真正来源是无效探索：策略持续产生"低信息量"动作（既不降低不确定性又不推进任务），rollout 收集浪费算力却无法学习。T²PO 提出 token-level + turn-level 双层不确定性门控，主动截断无意义探索。

⚙️ 核心方法详解
① Token-level 不确定性度量。 对每个生成 token \(t\)，定义不确定性为策略熵的滑动平均： $$U_t = \frac{1}{W}\sum_{j=t-W+1}^{t} H\big(\pi_\theta(\cdot|s, a_{
② "How to stop?" —— 非犹豫事件与强制终止。 定义 non-hesitation event：若 \(U_t < \theta_\text{low}\) 且此前连续 \(K\) 步持续 \(U_t > \theta_\text{high}\)，说明模型从犹豫转为决断。触发后不是立即停，而是在下一步强制插入 token，让模型进入"执行阶段"。这种 "decide when to decide" 的机制比粗暴截断更自然。
③ Turn-level 动态采样（TDS）。 对每一回合的完整 rollout 计算回合级信号 \(c_{\text{turn}}\)。若 TDS Rule 触发： $$c_{\text{turn}} < \tau_{\text{turn}} \Rightarrow \text{resample this turn under same state}$$ 即重采样当前回合，直到质量过关或达到预算 \(B\)。这避免把"废轨迹"送入梯度更新；关键设计是 \(c_{\text{turn}}\) 在 regenerate 后才重新计算（避免 rejection loop 死锁）。
④ Memory Window + 折扣回报。 多轮任务中完整 trajectory 串联会爆显存。T²PO 只保留最近 \(M\) 回合的交互作为 context：\(s_t = [a_{t-M},\ldots,a_{t-1}]\)。跨回合奖励传播用折扣： $$R_t^{\text{eff}} = \sum_{k=0}^{T-t}\gamma^k r_{t+k}, \quad \gamma=0.95$$ 稀疏终局奖励从末端回传到早期动作。
深度案例：ALFWorld Clean 任务的"犹豫-决断-执行"
任务："Clean the apple then put it in the sidetable."
无 T²PO 的 GRPO 轨迹（训练塌缩前）：Turn 3 模型反复生成 "<think> Should I pick up apple first... or check fridge... maybe... </think> ..."，token 熵长期 > 2.3 nats，300 个 token 内没有落实动作，最终因超长被截断，reward=0。

💡 结论与启发

T²PO 最大贡献是把 Agentic RL 训练稳定性问题从优化层转移到探索层——不是调 KL/clip 系数，而是主动识别并截断"虚假努力"的 rollout。这与经典 option-critic RL 的 "when-to-terminate" 问题一脉相承，但首次在 LLM Agentic 场景系统化实现。对产业界 Agent 训练（WebAgent、Devin-like code agent）的意义：rollout 质量比数量重要，主动放弃无意义样本可能比扩大 batch 更有效。未来方向是学习自适应阈值 θ（而非手工设定）和引入更高阶的"任务进度"信号替代熵。

Reference-Sampled Boltzmann Projection for KL-Regularized RLVR: Target-Matched Weighted SFT, Finite One-Shot Gaps, and Policy Mirror Descent
📅 2026-05-05 🏛 (机构信息未标注)
RLVR 理论加权 SFT策略镜像下降
🔥 背景与动机

在线 RLVR（Reinforcement Learning with Verifiable Rewards）虽然有效，但训练路径上要保留 rollout 生成、verifier 打分、reference 评测都在优化环上，成本高。实践界希望用静态加权 SFT替代：预先生成 rollout、预先打分、预先算权重，然后用多 epoch 加权 MLE 拟合。但"加权 SFT 的拟合策略到底等于不等于 KL-RLVR 的目标策略？"一直缺乏清晰答案。本文给出唯一使两者相等的组合，并用 projection 视角解释为什么多 epoch SFT 永远修复不了 coverage 缺陷。

⚙️ 核心方法详解
① KL-RLVR 的目标分布刻画。 给定参考策略 \(\pi_{\text{ref}}\) 与可验证奖励 \(r(\tau)\in\{0,1\}\)，KL-正则化 RLVR 目标为： $$\max_\pi \mathbb{E}_{\tau\sim\pi}[r(\tau)] - \beta \cdot \mathrm{KL}(\pi \parallel \pi_{\text{ref}})$$ 经典变分解： $$\pi^*(\tau) \propto \pi_{\text{ref}}(\tau) \cdot \exp\big(r(\tau)/\beta\big)$$ 这就是 fixed-reference Boltzmann 分布。
② Weighted SFT 的诱导策略。 静态加权 SFT 给定采样器 \(\mu\)（生成 rollout 的分布）和权重 \(w(\tau)\ge 0\)，优化： $$\mathcal L(\theta) = -\mathbb{E}_{\tau\sim\mu}[w(\tau) \log \pi_\theta(\tau)]$$ 渐近下诱导策略：\(\pi_\mu^{w}(\tau) \propto \mu(\tau) w(\tau)\)。
关键等价性定理：\(\pi_\mu^w = \pi^*\) 当且仅当： $$\mu = \pi_{\text{ref}} \quad\text{且}\quad w(\tau) = \frac{\pi_{\text{ref}}(\tau)}{\mu(\tau)} \cdot \exp(r(\tau)/\beta) = \exp(r(\tau)/\beta)$$ 即采样器必须是 \(\pi_{\text{ref}}\) 本身，权重是 prompt-normalized Boltzmann 密度比。作者据此提出 BOLT 算法（Boltzmann-Targeted SFT）。
③ 有限样本的 One-Shot Gap。 即使采样器权重都正确，一次性预采数据集 \(D\) 也可能没覆盖到 \(\pi^*\) 的某些 trajectory。作者证明"支集受限"下的最佳 Boltzmann projection 损失恰好等于 fixed-reference gap： $$\Delta_{\text{FR}} = \mathrm{KL}(\pi^* \parallel \pi^{D}_*)$$ 即数据没覆盖的概率质量无法被任何多 epoch SFT 找回——这直接解释了为什么 pass@k 在 RLVR 不是"创造新推理支集"而是"在已有支集上集中概率"。
④ Sampler Refresh = Policy Mirror Descent。 当覆盖弱但非零时，作者证明重复"当前策略 → 新参考 → 再次 Boltzmann projection"恢复了 KL policy mirror descent 算法： $$\pi_{k+1}(\tau) \propto \pi_k(\tau) \cdot \exp\big(r(\tau)/\beta\big)$$ 每步内部的有限加权 SFT 解作为对精确 mirror 步的 comparator drift 进入误差分析。这就理论上把"多次 refresh + weighted SFT"与 online RLVR 统一起来，给出精确等价性与误差分解。
深度案例：GSM8K 上 BOLT vs 基线的 checkpoint 动态

📅 2026-05-05 🏛 Independent Researcher

🔥 背景与动机

单 Agent RL 的核心对象是 trajectory（一条 token+tool+observation 的序列）。但现实部署的 LLM 系统正快速向"多 Agent 协同团队"演进——Moonshot Kimi K2.6 的 Agent Swarm（PARL 训练，64+ sub-agents，5000+ 协调步骤）、OpenAI Codex 的并行软件工程 Agent、Cursor / Devin 等 IDE Agent 都已不再是单 Agent。但既有 RL 形式化、奖励设计、credit assignment 都是为单 Agent 设计的。本文（203K 字大综述+原创理论）把编排轨迹（orchestration trace）作为多 Agent 系统 RL 的统一抽象单位，重写了 reward design、credit assignment、orchestration learning 三大支柱。

⚙️ 核心方法详解
① Orchestration Trace 形式化：dynamic Dec-POMDP。 经典 Dec-POMDP 假设 agent 数量固定，无法捕捉 spawn/despawn。作者扩展为： $$\mathcal M_d = (\mathcal S, \mathcal A^{\text{spawn}}, \mathcal A^{\text{despawn}}, \mathcal A^{\text{message}}, \mathcal A^{\text{tool}}, \mathcal A^{\text{aggregate}}, T, \Omega, R, \gamma)$$ 其中 spawn/despawn 是元动作（创建/解散 sub-agent）。轨迹不再是 token 序列而是事件图：节点=agent 状态，边=消息/工具调用/聚合操作。

③ Credit Assignment 8 个粒度单元。 从最粗到最细：team-level → orchestrator decision → per-agent → per-turn → per-message → per-tool-call → per-aggregation → per-token。作者论证：单纯团队奖励 + token 级 GAE 在 sub-agent ≥ 8 时几乎必然不收敛——这与 Kimi PARL 的"必须 per-agent verifiable"工程经验吻合。

🔥 背景与动机

Test-time scaling 方法（self-consistency、self-refine、debate、Mixture-of-Agents）层出不穷，但不同方法常用差几个数量级的算力，使横向比较没有意义。论文要回答三个实际工程问题：(1) 同等 FLOPs 预算下哪种方法最优？(2) 不同任务难度是否要切不同方法？(3) 给定算力，应该堆 8B 多 Agent 还是直接上 70B 单模型？这是首次在匹配算力预算下系统对比四大主流 inference paradigm 的工作。

⚙️ 核心方法详解
① 算力公平度量。 现有研究多用 FLOPs，但忽略 memory transfer 成本。作者用实测延迟 + FLOPs 双指标测量 compute budget，并把所有方法的 budget 调到同一刻度（通过调整候选数 \(k\)、refine 轮数 \(r\)、debate agent 数 \(m\)、MoA 层数 \(L\) × proposer 数 \(p\)）。
② 四种 paradigm 形式化。
  • Self-consistency：并行 \(k\) 次 CoT 采样 + majority vote，FLOPs \(\approx k \cdot c_{\text{CoT}}\)。
  • Self-refinement：串行 \(r\) 轮 critique-rewrite，单模型上下文累积，FLOPs \(\approx \sum_{i=1}^r c_{\text{CoT}}^{(i)}\) 且 context 增长。
  • Debate：\(m\) 个 agent × \(R\) 轮，每轮 agents 互相读 message，context 二次增长。
  • MoA：\(L\) 层、每层 \(p\) 个 proposer + 1 aggregator，层间 context 不累积（关键架构优势）。
③ 难度自适应分析。 用 CoT 单次正确率把 MMLU-Pro 1000 题分桶：easy (>75%)、medium (25–75%)、hard (<25%)。然后看每个桶在不同算力下的边际增益，得到任务难度 × 方法的二维启发表。
④ 模型替换前沿。 系统比较 8B 多 Agent（heavily scaled）vs 70B 单 CoT。给定相同实测算力时，分析转换点（switch point）：何时该 "scale agents" 何时该 "scale model"。

The Reasoning Trap: An Information-Theoretic Bound on Closed-System Multi-Step LLM Reasoning
📅 2026-05-05 🏛 PolymathMinds AI Lab
多 Agent Debate推理忠实度理论DPI 上界
🔥 背景与动机

多 Agent Debate（MAD）一直被视为提升 LLM 推理质量的有效手段，但作者指出该领域评估范式严重偏向 accuracy："debate 是否提升答案？" 几乎从未真正问 "debate 是否提升答案背后的推理忠实度？"。本文给出震撼性的负面发现：标准闭环 MAD 即使保持答案准确率，证据接地的推理忠实度反而被系统性侵蚀——majority-vote debate 让 evidence-grounding 跌至 baseline 的 1.7%（Cohen's d = 大效应）。作者命名此现象为 Debate Trap，并证明这是信息论必然，由 50 年前 DeGroot–Banerjee–Janis–Asch 社会科学预言。

⚙️ 核心方法详解
① Reasoning Faithfulness 形式化（Definition 1）。 给定证据集 \(E\) 和声明 \(c\)，agent \(i\) 在 round \(t\) 的输出 \(R_i^{(t)}\) 的忠实度定义为条件互信息： $$F_i^{(t)} = I(R_i^{(t)}; E \mid c)$$ 联合 MI 忠实度 \(F_{\text{joint}}^{(t)} = I(R_1^{(t)},\ldots,R_n^{(t)}; E \mid c)\) 是核心可证量。这种"以 MI 为根本测度"的设计避免了对人类标注的依赖（绕过 inter-rater 不一致问题）。
② Theorem 1（DPI Bound 上界）。 若 MAD 满足四条件 Markov 闭包（agent t 输出仅依赖 t-1 轮内容 + 共享参数），则 debate 链构成 Markov 链 \(E \to R^{(0)} \to R^{(1)} \to \ldots \to R^{(T)}\)。由数据处理不等式（DPI）： $$F_{\text{joint}}^{(t+1)} \le F_{\text{joint}}^{(t)} \le \ldots \le F_{\text{joint}}^{(0)}$$ 即每一轮 debate 期望忠实度都不会增加，而 vote-aggregating 协议下严格小于（strict inequality）。这从信息论根本上证明了 closed-system 多步推理的天花板。
③ Evidence Utilization Rate 与 Debate Trap 判据（Definition 2）。 EUR\(_t\) = round t 中被引用证据占总证据的比例。Debate Trap 触发条件：存在轮 \(t准确率没下降但证据使用率下降"——这正是 accuracy-only 评测看不到的失败模式。
④ EGSR 算法（Theorem 2 下界保证）。 作者提出 Evidence-Grounded Socratic Reasoning：每隔 \(k\) 轮在 prompt 中重新注入证据 \(E\)，打破 Markov 闭包。可证：开放系统的忠实度下界严格大于对应闭环系统： $$F_{\text{EGSR}}^{(t+1)} \ge F_{\text{EGSR}}^{(t)} - \delta_{\text{noise}} + \alpha \cdot I(E; \text{prompt}_{t+1} \mid R^{(t)})$$ 其中 \(\alpha\) 是注入强度系数。EGSR 在保留 multi-agent 多样性优势的同时恢复了证据 grounding。
深度案例：MAD 在长尾事实推理上的"Debate Trap"
实验设置：16 个 condition × 2 模型架构（GPT-4o, Claude Opus）× R1–R8 鲁棒性测试。
典型 Debate Trap 实例：6 题关于"古罗马塞维里皇帝继承顺序"，提供 12 条证据片段。

  • R3 EGSR 改造（每轮重注入 E）：准确率 0.74（+3 pp），引用证据 7.6 条，SFS=0.81（+2%），EUR=63%。
  • 极端 majority-vote variant C15：EUR 仅剩 baseline 的 1.7%，agents 完全收敛到内部相互肯定，证据彻底被忽略。
社会科学解读：这正是 DeGroot 共识、Janis 群体迷思、Asch 从众的 LLM 版——agents 在闭环中相互"确认偏误"，证据被边缘化但答案因模型先验而维持。

💡 结论与启发

这篇论文把 MAD 的"看似有效的失败"从 50 年社会科学传统一路追溯到信息论。意义远超学术：(1) 警示工业部署 multi-agent debate 系统的幻觉风险——准确率没下降但模型已经在和自己玩"迷思游戏"；(2) 给 RAG / Agentic Search 系统一个明确的设计原则：每一轮都要 re-anchor 到外部证据，否则 Markov 闭包会让推理逐步脱离原始 evidence；(3) 评估范式的根本性反思——"answer accuracy" 远不足以判断 LLM 推理质量，未来 benchmark 必须包含证据 grounding 测度。这是 Generation V "Process-Faithful Multi-Agent Reasoning" 的开山之作。

🔥 背景与动机

印度等多语言医疗环境中临床交互常使用印地语、旁遮普语等区域语言，而 EHR 系统和 AI 临床决策支持却以英语为中心。这种语言错配导致诊断延迟、误诊与照护不一致。骨科诊断（脊柱 vs. 关节、急性 vs. 慢性损伤）症状描述常重叠且术语异质，是评估多语言临床建模可靠性的理想测试床。既有工作有三类局限：(1) 通用多语言模型（XLM-R、IndicBERT）未针对临床安全场景校准；(2) 指令微调 LLM 在分类任务上缺少结构化约束；(3) 聚合 metric 掩盖类别条件 / 语言条件的失败模式。

⚙️ 核心方法详解
① 三语骨科数据集构建。 由持照骨科医生监督，构建英语（拉丁文字）、印地语（梵文字符）、旁遮普语（古鲁穆奇文字）三种语言的标注数据集，6 个骨科诊断类别（脊柱疾病、骨折、关节疾病、退行性病变、软组织损伤、外伤后并发症）。两种分布：
  • Controlled：每语言每类 1000 条 = 18,000 平衡样本；
  • Natural：每语言 ≥45,000 条偏态分布样本（反映真实世界）；
合计约 135,000 条。脱敏严格，仅含症状描述与诊断标签。
② 域自适应建模。 评估四种 transformer encoder：mBERT、XLM-R、mDeBERTa、IndicBERT；并提出域自适应增强：在 medical-specific 语料（PubMed-MC + IndicHealthCorpus）上做 continued pretraining，再在三语骨科数据上 fine-tune。这种双阶段域自适应显著提升低资源语言泛化。
③ 可靠性测度框架。 不仅报 macro-F1，还引入：
  • 类别条件混淆率：CCC = \(\sum_{i,j} P(j|i) \cdot \mathbb{1}[i\ne j, \text{class }i\sim j]\)（衡量临床相似类别的混淆）
  • 语言条件不稳定性：LCS = \(\max_l F1_l - \min_l F1_l\)（跨语言性能差距）
  • 校准误差 ECE：标准 expected calibration error。
④ 失败模式审计。 在 natural distribution 上做失败案例分类：(a) 跨语言术语未对齐导致误诊（如印地语"jod ki kharabi"对应英语 "joint disorder" 但匹配不到）、(b) Mixed-script（拉丁+梵文混写）解析失败、(c) 长尾类别（外伤并发症）系统性低召回。这些粒度级失败信息对临床安全部署至关重要。
深度案例：旁遮普语骨科主诉的诊断流程
输入（古鲁穆奇文字）："ਮੇਰੀ ਪਿੱਠ ਵਿੱਚ ਤਿੰਨ ਮਹੀਨੇ ਤੋਂ ਦਰਦ ਹੈ, ਖਾਸ ਕਰਕੇ ਸਵੇਰੇ ਉੱਠਣ ਵੇਲੇ"（"我的背痛三个月，特别是早上起床时"）。

CLEAR: Revealing How Noise and Ambiguity Degrade Reliability in LLMs for Medicine
📅 2026-05-05 🏛 Vanderbilt University, Intuit AI Research
医学 LLM 评测弃权能力噪声鲁棒性
🔥 背景与动机

真实临床决策充斥噪声、歧义、不完整信号。优秀医生的关键能力是知道何时不知道——admitting uncertainty, abstaining, seeking help。但既有医学 LLM benchmark（MedMCQA、MedQA、JAMA-CC）都假设答案在选项里，无法测量 LLM 的"认知谦逊（epistemic humility）"。论文提出 CLEAR 框架，通过三种系统性扰动（增加干扰项、引入弃权选项、改变弃权语义）把现有 QA benchmark 改造为可靠性测试床。

⚙️ 核心方法详解
① 三轴扰动设计。
  • 选项数量轴：原 4 选 1 → 2/3/4/5/6 个 plausible 选项（competing answer pressure）；
  • Ground truth 存在轴：保留 GT vs 移除 GT 仅留 distractors + 弃权（gold-absent 场景）；
  • 弃权语义轴：3 种不同弃权 framing：
     – NA（None of the Above，断言式拒绝）
     – IDK（I Don't Know，承认不确定）
     – INA（I Need Assistance，请求协助）
三者临床意义递进，但在 prompt 表面只是用词不同。
② 评测指标。 不仅算 accuracy，还分别报告：
  • Abstention Rate：在 GT-absent 场景下选择"弃权"选项的比例（理论应≈100%）；
  • Wrong-Selection Rate：在 GT-absent 但有弃权选项时仍选 distractor 的比例；
  • Calibration ECE：置信度与正确率匹配度。
③ 17 模型 × 2.6M 推理。 涵盖 GPT-5.2、GPT-4o、Claude Opus 4、Llama-3.3-70B、Qwen2.5-72B、DeepSeek-V3、MedGemma 等 17 个开/闭源模型，三个 benchmark（MedMCQA / MedQA / JAMA-CC）每个题目做 3 轴扰动，共 2.6M 次唯一推理。这是迄今最大规模的医学 LLM 可靠性评测。
④ Reasoning model 子分析。 单独对比"普通"vs"reasoning"模式（CoT、o1-style）：模型规模和推理深度对 abstention 行为的复合影响。该问题之所以重要：扩规模/扩 reasoning 是否能同时提升推理能力与认知谦逊？答案是悲观的。
深度案例：JAMA-CC 上的 Ground-Truth-Absent 场景
原题（JAMA Clinical Challenges）：一位 47 岁男性病史，正确答案是药物诱导横纹肌溶解。
CLEAR 扰动：移除 GT，留下 4 个 plausible distractors（脱水、急性肾损伤、心肌炎、慢性肾病）+ 1 个弃权选项。

不同弃权 framing 下 GPT-5.2 的行为：
  • NA framing："None of the above" → 弃权率 53%（最高，因 NA 语气最坚定）
  • IDK framing："I am not sure" → 弃权率 25%（−28 pp，因模型把 IDK 解读为"软弱"）
  • INA framing："I need to consult an expert" → 弃权率 20%（最低）
Qwen 2.5 家族扩规模：7B → 32B → 72B 这三档，NA 弃权率提升（28%→41%→55%），但 IDK / INA 弃权率反而下降（21%→18%→14%）—— "扩规模让模型更敢于断言式拒绝，更不愿意 admit uncertainty"。

反人类直觉的扩规模负效应：GPT-5.2 在 INA framing 下平均弃权 20%，反而低于随机选择（1/(4+1)=20%）！意味着前沿模型主动避免请求协助，宁愿瞎猜。

🔬 消融与讨论

💡 结论与启发

CLEAR 是首次把"知道何时不知道"系统性变成可量化基准的工作。对临床 AI 落地的关键警示：当前最强 LLM 不仅没有医生级诊断能力，更缺乏医生级的认知谦逊——前者尚可补救，后者直接对应医疗事故风险。论文的最佳实践建议：(1) 临床部署必须用 NA 而非 INA 作为 abstention prompt；(2) 不能只跑 medical benchmark accuracy，必须跑 GT-absent 场景；(3) 后训练阶段应专门强化 abstention behavior（如 RLHF 中加入 "honest refusal" 偏好对）。这是医学 LLM 评估的下一代标杆。

Visual Latents Know More Than They Say: Unsilencing Latent Reasoning in MLLMs
📅 2026-05-05 🏛 A*STAR Singapore, SUTD, HUST
MLLM 潜在推理推理时方法对比对齐 + RL
🔥 背景与动机

潜在推理（latent reasoning）从 Coconut 开始已被证明比离散 textual CoT 更具表达力——把推理移到连续 hidden state，模型可在 latent 空间做"隐式 BFS"。但作者发现 MLLM 中的"Silenced Visual Latents"现象：联合训练中 latent 同时被两个目标拉扯——左侧的自回归 next-token 预测偏好快捷方式（autoregressive shortcut），右侧的视觉推理需要把视觉证据真正整合到 latent。两个目标冲突时模型倾向走快捷方式，导致 latent 表征被丰富了视觉信息但下游答案预测时根本不用它。

⚙️ 核心方法详解
① 失效模式诊断：解耦 Quality vs Utilization。 作者首先证明 latent 表征本身已经富含视觉信息（latent quality 指标 \(I(z; v)\) 高），但答案 \(\hat y\) 与 latent 的因果关系微弱（utilization 指标 \(I(\hat y; z|x_{\text{txt}})\) 接近零）。这是一个根本性洞察——既往工作都在试图改进 latent 训练目标，本文论证需在推理时分别治理两个失效。
② Stage I：Chunk-wise Contrastive Alignment（提升 quality）。 把视觉证据 \(V\) 划分为正负样本块：与查询相关的 patch 为正集、其他为负集。每个 latent token \(z_i\) 通过对比损失： $$\mathcal L_{\text{align}} = -\log\frac{\exp(\mathrm{sim}(z_i, V^+_i)/\tau)}{\exp(\mathrm{sim}(z_i, V^+_i)/\tau)+\sum_{V^- } \exp(\mathrm{sim}(z_i, V^-)/\tau)}$$ 关键设计：chunk-wise assignment 让每个 latent token 关注不同 patch 子集，避免 latent collapse。
③ Stage II：Confidence-Progression Reward（提升 utilization）。 推理时迭代扰动 latent 并要求 token 预测分布沿 latent 跨度逐渐 concentration： $$r_t = -[\;H(p(\hat y | z_{1:t})) - H(p(\hat y | z_{1:t-1}))\;]$$ 即每延伸一个 latent token，下游答案熵应该单调下降。这逼迫模型走 latent 路径而非 autoregressive 快捷。
④ Frozen Backbone：完全推理时方法。 关键工程优势：(a) 主模型参数完全冻结；(b) 不需要重训；(c) 仅在推理时通过对比对齐 + 奖励驱动微调 latent 本身。这意味着可以即插即用到 Qwen2.5-VL、InternVL3、Llama 3.2 Vision、GPT-4o-mini-VL 任意 MLLM 上。
深度案例：MMVP "数 sheep 数量"题
输入：图片包含 5 只羊（其中 1 只半隐藏在岩石后），问"How many sheep are in the picture?"

vanilla Qwen2.5-VL-7B：直接 autoregressive 输出 "There are 4 sheep" —— 走文本快捷方式，没用 latent 推理。

LVR baseline（仅 latent reasoning，无 unsilencing）：latent 表征已正确编码 5 只羊位置（量化 \(I(z; v)\)=0.83），但答案仍是 4 ：utilization 失败。

本文 unsilencing 方法：
  • Stage I 对比对齐：5 个 latent token 分别专注于 5 只羊的 patch（chunk-wise 分配避免坍缩）；
  • Stage II Reward 驱动：迭代时下游分布从 \(p(\hat y)=[3:0.4, 4:0.4, 5:0.2]\) 逐步 concentration 到 \(p(\hat y)=[3:0.05, 4:0.15, 5:0.80]\)；
  • 最终输出 "There are 5 sheep, including one partially hidden behind the rock"。
对比 latent quality 指标：vanilla 0.34，本文 0.91（+168% 表征质量），utilization 从 0.12 → 0.78（+550%）—— 同时治理两端。

💡 结论与启发

这篇论文的概念贡献是一个广泛适用的"quality vs utilization 双重诊断"框架——既往关于潜在推理、知识引入、attention bias 等问题都可能存在类似的"特征已具备但下游不用"现象。Unsilencing 这种 frozen-backbone + inference-time RL 的范式既保留了大模型不动的工业稳定性，又让 latent 推理真正"上桌"，是 2026 年 MLLM 推理时优化的代表性工作。可预见这种思路会很快被扩展到 video LLM、agentic VLM 等场景。

SAPO: Segment-Aligned Policy Optimization for Multi-Modal Reasoning
📅 2026-05-05 🏛 (机构信息未标注)
多模态推理 RL段对齐策略优化熵自适应分段
🔥 背景与动机

当前 LLM/MLLM RL 的两个极端：(a) PPO 等 token-level 方法用 GAE 给每个 token 算 advantage，但 long-CoT 中稀疏终局奖励要回传几千 token，方差大且同一推理步内多个 token 信号不一致；(b) GRPO 等 sequence-level 方法用整轨迹相对优势，无法区分中间步骤贡献——当模型大部分步骤正确仅最后一步格式错误时，所有正确步骤同样被惩罚。本文提出 SAPO：把强化学习的优化粒度从 token / sequence 提升到推理步骤（segment），并配合自适应熵分段机制，对多模态推理尤其重要——视觉证据通常在某个明确的推理步集中出现。

⚙️ 核心方法详解
① 步骤级 MDP 抽象。 标准 token-MDP 中 \(s_t\) = prompt + 已生成 token，\(a_t\) = 下一个 token。SAPO 重新定义： $$s_k = (x, \text{seg}_{
② 段级 value function 与 advantage。 引入步骤级 value： $$V^\pi(s_k) = \mathbb{E}_\pi\Big[\sum_{j=k}^K \gamma^{j-k} r(s_j, a_j) \mid s_k \Big]$$ 段级 advantage：\(\hat A_k = R_k - V_\phi(s_k)\)，其中 \(V_\phi\) 是辅助 value 网络在 segment 边界处估值。这样 credit assignment 与推理步骤一一对应，避免 token 级噪声。
③ 段对齐重要性采样。 标准 PPO 重要性比率是 token 级 \(r_t = \pi_\theta(a_t)/\pi_{\theta_{\text{old}}}(a_t)\)。SAPO 用段级几何均值： $$\rho_k = \exp\!\left(\tfrac{1}{|a_k|}\sum_{t\in a_k}\log\frac{\pi_\theta(y_t|s_t)}{\pi_{\theta_{\text{old}}}(y_t|s_t)}\right)$$ 这样防止长 segment 上比率连乘爆炸，是稳定 RL 训练的关键技巧。Clipped 目标： $$\mathcal L_{\text{SAPO}} = -\mathbb{E}\Big[\min\big(\rho_k \hat A_k, \mathrm{clip}(\rho_k, 1-\epsilon, 1+\epsilon) \hat A_k\big)\Big]$$
④ 熵自适应分段。 不依赖 prompt 格式（如 \n\n 或 ###Step），SAPO 用 token 级熵识别逻辑边界：高熵 token 标志模型面对多种竞争续写——这正是推理步骤切换点。具体规则：当连续 \(\eta\) 个 token 熵超过阈值 \(\theta_H\) 时切段。作者用 \(\eta=30\)，相当于自动识别 ~30 token 长的 logical chunk。这种模型内部信号驱动的分段比 prompt-driven 更鲁棒。
深度案例：Geo3K 几何题"求外接圆半径"
问题：图片中等边三角形边长 6，求其外接圆半径。

典型推理 trajectory（SAPO 自动分段）：
  1. 段 1（理解）：识别图中是等边三角形 + 边长 6 → 段熵中等，但首 token "Looking" 高熵触发段边界。
  2. 段 2（公式回忆）：等边三角形外接圆半径公式 \(R = \tfrac{a}{\sqrt{3}}\) → "Recall" 高熵触发，明确知识检索段。
  3. 段 3（代入计算）：\(R = 6/\sqrt{3} = 2\sqrt{3}\) → 计算段内熵低，单一路径。
  4. 段 4（最终答案）："The answer is \(2\sqrt{3}\)" 终局奖励 r=+1。
SAPO credit 分配：段 1 (Adv=+0.05)、段 2 (Adv=+0.45 知识对)、段 3 (Adv=+0.30 计算对)、段 4 (Adv=+0.20 答案对)。
对比 GRPO：把整轨迹平均 +1 reward 等分到所有 token，区分不出"知识检索"和"计算"哪一步贡献大。
对比 PPO：每个 token 独立 GAE 估值，公式中常数 \(\sqrt{3}\) 这样的 token 反而被赋负 advantage（学困难），引入梯度噪声。

🔬 消融与讨论

(1) 段长 \(\eta\)：\(\eta=30\) 最优，过短退化为 token、过长退化为 sequence；(2) 熵阈值 \(\theta_H\)：取分布的 90th percentile 自适应；(3) 不用熵分段而用 prompt-guided 段（\n\n）：性能下降 1.8 pp，因 prompt 不一致；(4) 文本数学任务（GSM8K, MATH）也提升明显。局限：(1) 仍需训练辅助 value 网络，比 GRPO 显存高 ~30%；(2) 视觉模态外的多模态（音频、视频）未验证；(3) 极长 CoT（>10k tokens）下段数过多可能拖慢 backward。

💡 结论与启发

SAPO 是 RL credit assignment 粒度演进的自然下一步：从 token （PPO）→ trajectory（GRPO）→ step（SAPO）。其特别价值在多模态推理：视觉信息整合通常发生在某个明确的 segment（"识别图中物体" → "代入数值"），段级 credit 让 RL 信号与多模态融合的天然结构对齐。这种思路也启发未来在 agentic RL（每个 tool call 是一个 segment）、code RL（每个函数定义/调用是一段）等场景的应用。"RL 优化粒度应与任务结构粒度一致"是 2026 年 RL 后训练的重要方法论命题。
