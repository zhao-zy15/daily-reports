Drift and Selection in LLM Text Ecosystems
📅 2026-04-10
🏛 Queen Mary University of London
LLM理论分析数据生态
📌 背景与动机

公共文本记录——人类和AI系统共同学习的材料——正日益被其自身的输出所塑造：AI生成的文本进入公共记录，后续Agent从中学习，循环往复。这种递归式数据污染会导致什么后果？现有研究主要通过实验观察（如"model collapse"），但缺乏一个精确可解的数学框架来分离递归对语料库的两种核心效应：漂移（drift）和选择（selection）。本文首次基于变阶 \(n\)-gram 模型，建立了一个完整的分析理论。

🔬 核心方法详解
① 变阶 \(n\)-gram 递归模型
将公共语料库抽象为一个"urtext"（原始文本），\(n\)-gram Agent 从中学习并生成新文本，新文本替代旧文本成为下一代学习材料。定义递归映射 \(\Phi\)：令 \(p^{(0)}\) 为初始语料分布，则第 \(t\) 代为 \(p^{(t+1)} = \Phi(p^{(t)})\)。核心发现是该递归是一个 Markov 链，其中上下文空间上的概率在每步被确定性地重映射。
关键数学：对于一阶情形（unigram），混合环境下递归为 \(p_w^{(t+1)} = (1-\alpha) p_w^{(t)} + \alpha \cdot \frac{p_w^{(t) 2}}{\sum_v p_v^{(t) 2}}\)，其中 \(\alpha\) 控制递归强度。
② 定理1：混合环境下的漂移
在无选择压力的情况下（描述性发布），递归将不可逆地移除稀有形式。具体地，对于 unigram 分布：
$$p_w^{(\infty)} = \begin{cases} 1 & \text{if } w = \arg\max_v p_v^{(0)} \\ 0 & \text{otherwise} \end{cases}$$ 即最终只有最频繁的token存活——这是一个确定性的词汇坍缩过程。作者进一步证明漂移速率由 \(\alpha\) 和初始分布的Rényi熵共同控制，稀有词消失速度呈指数级。
③ 固定点多面体与循环结构（定理2）
在 \(M \to \infty\) 极限下，\(n\)-gram 递归的不动点构成一个凸多面体。对于 bigram 情形，不动点条件为：\(\rho(w_1, w_2) = p(w_2 | w_1) \cdot \pi(w_1)\)，其中 \(\pi\) 是平稳分布。作者完整刻画了极端点结构：二元 token 上的 bigram 不动点空间是二维的，由条件概率对 \((\theta_0, \theta_1)\) 参数化。
核心洞察：不动点内的"循环（circulation）"携带了高阶统计结构，但描述性发布会杀死所有循环，只留下"浅层"分布。
④ 选择机制与定理3（交叉熵遗传）
当发布过程是规范性的（normative）——即奖励质量、正确性或新颖性——深层结构得以保持。定理3给出了最优上界：选择后的发散度满足 \(D_{\text{KL}}(\hat{p} \| p^*) \leq \frac{H(S)}{n \cdot E}\)，其中 \(H(S)\) 是选择函数的熵，\(n\) 是上下文窗口，\(E\) 是专家数量。这意味着更强的选择压力和更长的上下文能维持更丰富的文本生态。

实验：Doyle/Austen/Darwin 经典文本递归退化
取柯南·道尔、奥斯汀和达尔文各约 50 万 token 的原始语料作为 urtext，使用 3-gram Agent 进行 100 轮递归。
  • 道尔语料：独特 3-gram 数从 18.7 万降至 3.2 万（-83%），稀有词"sanguine""obsequious"在 15 轮内消失
  • 奥斯汀语料：词汇坍缩更剧烈（-89%），因初始分布更集中于高频对话词
  • 达尔文语料：保留了更多专业术语（仅 -71%），可能因科学文本的上下文约束更强
  • 加入规范性选择（奖励信息量高的3-gram）后，词汇保留率提升至 65%

🔍 消融与讨论

本文是纯理论工作，作者通过改变 \(n\)-gram 阶数、Agent数量和选择函数进行了系统性的数值验证。关键发现：(1) 上下文窗口 \(n\) 从2增至5，深度指标从0.41升至0.78，说明长上下文Agent本身就是一种"反漂移"力量；(2) 异质Agent群体（混合不同 \(n\) 的Agent）的漂移速度取决于最低阶Agent——短上下文Agent是生态退化的瓶颈；(3) argmax发布（只发布最高概率序列）比采样发布更快收敛到浅层均衡。局限性在于 \(n\)-gram 模型只是对真实LLM的近似，且未考虑主动学习和在线更新场景。

💡 结论与启发

这是首个精确可解的LLM文本生态数学理论。核心启发：AI训练数据循环使用的后果不仅是"model collapse"，更是确定性的词汇坍缩（漂移），而选择机制（如人类审核、质量过滤）是维持文本多样性的唯一力量。对AI训练语料设计的启示是：必须加入强选择压力（如质量筛选、新颖性奖励），才能对抗递归数据使用带来的inevitable drift。

EMA Is Not All You Need: Mapping the Boundary Between Structure and Content in Recurrent Context
📅 2026-04-10
🏛 Intel Labs
LLM序列建模循环架构
📌 背景与动机

高效序列模型（如 Mamba、RWKV、线性注意力）试图用亚二次复杂度替代 Transformer 的全注意力。这些模型的循环上下文机制究竟比最简单的时间平均（EMA）强在哪里？现有研究将各种门控机制视为黑箱改进，但缺乏系统性的理论分析来精确定位固定系数累积（EMA）能做什么、不能做什么，以及"学习到的、依赖输入的选择"究竟在哪个环节是不可替代的。本文用 EMA trace 作为控制探针，绘制了这条边界。

🔬 核心方法详解
① SPCN 架构（Settling Predictive Coding Network）
构建一个 Hebbian 架构，使用多时间尺度 EMA trace 作为唯一的循环上下文。核心：不使用门控、不使用基于内容的检索，只有固定衰减系数 \(\lambda_k\) 的指数移动平均：
$$\mathbf{h}_k^{(t)} = \lambda_k \mathbf{h}_k^{(t-1)} + (1-\lambda_k) \mathbf{x}^{(t)}$$ 其中 \(\lambda_k \in \{0.9, 0.99, 0.999, ...\}\) 提供不同时间尺度的记忆。Hebbian 更新规则利用这些 trace 预测下一个 token 的结构角色（如语法关系），无需任何标签。
② EMA Trace 编码时间结构但破坏 Token 身份
关键实验发现双重性质：
正面：EMA trace 能编码丰富的时间结构。在语法角色分配任务上，纯 Hebbian + 多时间尺度 EMA 的无监督模型达到了有监督 BiGRU 96% 的性能，并在结构依赖的语法角色上超越有监督基线。
负面：EMA trace 对 token identity 实施了有损、数据无关的压缩。信息论证明：固定系数累积的每步信息损失是不可逆的，由数据处理不等式（DPI）严格保证——任何下游预测器都无法恢复被丢弃的信息：
$$I(X^{(1:t)}; \hat{Y}) \leq I(X^{(1:t)}; \mathbf{H}^{(t)}) < I(X^{(1:t)}; X^{(1:t)})$$
③ 130M 参数语言模型实验：量化信息瓶颈
训练一个 130M 参数的语言模型，用纯 EMA 上下文替代注意力层，在 C4 数据集上评估困惑度。
EMA-only 模型：PPL = 260（GPT-2 的 8 倍）。关键消融——将线性预测器替换为完整的 softmax 注意力预测器：损失完全不变。这证明整个性能差距完全定位于 EMA trace 本身，而非预测器的表达能力不足。
④ 深度维度的信息稀释
将 EMA 分析从时间维度扩展到深度维度：在 Transformer 每层的残差连接中，固定的残差权重 \(\alpha\) 实质上也在做深度方向的 EMA。理论分析表明，深度 EMA 的信息稀释同样不可逆，只有学习到的、输入依赖的选择（如门控、注意力）才能解决——这为 Mamba/RWKV 等模型的门控设计提供了信息论基础。

语法角色分配：EMA vs BiGRU
任务：给定英语句子序列，预测每个词的语法角色（如主语/宾语/修饰语），测试对结构依赖关系（如远距离一致性）的捕获能力。
  • SPCN (EMA, 无监督): 整体准确率 91.3%，结构依赖角色 87.2%
  • BiGRU (有监督): 整体准确率 95.1%，结构依赖角色 84.6%
  • EMA 在结构角色上超越有监督 BiGRU (+2.6%)，证明固定时间尺度的累积天然适合捕获层级结构
  • 但在需要精确 token recall 的任务（如命名实体识别）上，EMA 大幅落后 (-23%)

🔍 消融与讨论

消融实验证实了几个关键发现：(1) 增加 EMA 时间尺度数量从 3 到 8 个，语法角色性能从 85% 提升至 91.3%，但语言模型 PPL 仅微弱改善（280→260），表明多时间尺度帮助结构但无法恢复内容；(2) 让 EMA 系数可学习（但仍输入无关），PPL 几乎不变（258 vs 260），确认瓶颈在于"固定 vs 输入依赖"而非"预设 vs 学习"。局限性：分析基于 \(n\)-gram 模型和较小规模，更大模型是否有不同的信息瓶颈结构尚待验证。

💡 结论与启发

本文精确定位了高效序列模型的信息论边界：EMA（固定系数累积）是完美的时间结构编码器，但不可逆地破坏 token 内容信息。这一"结构-内容分离"由数据处理不等式严格保证，为 Mamba、RWKV 等模型中门控/选择机制的必要性提供了理论基础。核心信息：不是"EMA 不够好"，而是"EMA 天生只能做结构，内容需要学习到的选择"。

Dynamic Sparsity in Tree-Structured Feed-Forward Layers at Scale
📅 2026-04-10
🏛 Graz University of Technology
LLM稀疏计算效率
📌 背景与动机

在典型上下文长度下，Transformer 的前馈 MLP 块占据了大部分计算预算。混合专家（MoE）是主流的稀疏替代方案，但其路由开销和负载均衡问题限制了实际效率。快速前馈（Fast Feedforward, FFF）层是另一种路径——使用二叉树结构的条件计算，每个 token 只需沿树的一条路径计算，理论稀疏度可超 99%。然而，FFF 层此前仅在 300M 参数的 BERT 编码器上验证过，能否扩展到十亿参数级别的 GPT 风格自回归模型仍是未知。

🔬 核心方法详解
① 树结构前馈层（FFF）的参数化
一棵深度为 \(D\) 的完全二叉树，共 \(P = 2^D - 1\) 个内部节点。每个节点 \(j\) 包含：权重向量 \(\mathbf{w}_j \in \mathbb{R}^d\)、偏置 \(b_j\)、以及一个线性投影 \(\mathbf{v}_j \in \mathbb{R}^d\)。
前向计算：输入 \(\mathbf{x}\) 从根节点开始，在每个节点计算路由决策 \(z_j = \sigma(\mathbf{w}_j^\top \mathbf{x} + b_j)\)，然后选择左子树（\(z_j > 0.5\)）或右子树进入下一层。到达叶节点后输出 \(\mathbf{v}_{\text{leaf}}^\top \mathbf{x}\)。
每个 token 的计算量：\(O(D \cdot d)\)，而稠密 MLP 为 \(O(d_{\text{ff}} \cdot d)\)。当 \(D = \log_2(d_{\text{ff}})\) 时，计算量呈对数级缩减。
② 自动剪枝（Auto-Pruning）效应——新发现
核心发现：训练过程中，硬路由决策与非对称非线性（ReLU）之间的交互会自发地停用整个树路径。具体机制：当某个节点的路由 sigmoid 输出在训练后期饱和到 0 或 1，该节点的一个子树永远不会被任何输入激活，其权重有效"死亡"。
量化结果：在 1.3B 参数模型中，约 35-48% 的树节点在训练完成后处于永久休眠状态，将动态（输入依赖）稀疏自动转换为结构性稀疏——无需显式剪枝标准或辅助正则化。
可调控：通过简单的架构选择（如在路由后添加 temperature scaling、调节初始化方差）可以控制 auto-pruning 的强度。
③ 从头训练 vs 微调的缩放验证
系统性验证：从 125M 到 1.3B 参数的 GPT 风格模型，将所有 MLP 块替换为 FFF 层。
训练策略对比：(a) 从头训练 FFF 模型，(b) 从预训练稠密模型初始化然后微调为 FFF。
发现：从头训练的 FFF 模型在 1.3B 规模下与稠密基线的 PPL 差距仅 3.2%，而微调策略可将差距缩小到 1.8%，同时实际 FLOPs 减少 63%。
④ 与 MoE 的对比分析
关键架构差异：FFF 是"硬路由+单路径"（每 token 只经过一条路径），而 MoE 是"软路由+top-k 专家"（每 token 激活 k 个完整专家）。
优势：FFF 无需负载均衡辅助损失、无通信开销、内存访问模式更规则（连续树遍历）。劣势：FFF 每节点容量较小，难以在单节点内编码复杂变换。
混合策略初探：交替使用 FFF 层和稠密层（每隔一层），可在 PPL 损失 < 1% 的情况下减少 45% 的 FFF 层 FLOPs。

1.3B模型的 Auto-Pruning 可视化
训练一个 24 层、深度 \(D=14\)（16384 个叶节点）的 FFF-GPT 模型：
  • 训练初期（0-5K步）：所有节点均活跃，动态稀疏度 ~50%
  • 训练中期（5K-20K步）：auto-pruning 开始，浅层树节点路由饱和
  • 训练末期（20K+步）：42% 的节点永久休眠，有效稀疏度从 99.1% 提升至 99.6%
  • 被剪枝的路径主要集中在树的左侧子树（与 ReLU 的零半平面对齐），而右侧子树保留了 85% 的活跃节点

🔍 消融与讨论

消融实验验证了：(1) 树深度 \(D\) 从 10 增至 14，PPL 从 16.8 降至 15.3，但 \(D=16\) 时反而升至 15.5，暗示过深的树存在优化困难；(2) 初始化方差影响 auto-pruning 强度——高方差初始化导致更多路由饱和，auto-pruning 率从 35% 升至 55%，但 PPL 也退化 0.3；(3) FFF 层在模型的中间层效果最好，替换最浅层和最深层的效果较差。局限性：当前实现未针对 GPU 硬件优化树遍历，实际延迟改善约 1.4x（远低于理论 2.7x），需要定制 CUDA kernel。

💡 结论与启发

首次证明树结构稀疏前馈层可扩展到十亿参数级自回归模型，同时发现了 auto-pruning 这一新现象——训练过程自发地将动态稀疏转化为结构稀疏，无需任何显式剪枝。这提供了一条不同于 MoE 的稀疏化路径：利用二叉树的层级路由实现对数级计算缩减，且不需要负载均衡。如果 GPU kernel 优化跟上，FFF 可能成为 MoE 之外的另一种高效推理架构选择。

Generalization and Scaling Laws for Mixture-of-Experts Transformers
📅 2026-04-10
🏛 Amazon
LLMMoE缩放律
📌 背景与动机

MoE Transformer 通过稀疏路由大幅扩展模型容量，但其泛化理论远落后于稠密模型。现有 MoE 缩放律（如 Kaplan 等人的经验拟合）缺乏严格的理论基础：为什么 MoE 的泛化误差随参数增长的速率与稠密模型不同？路由组合爆炸对泛化有何影响？最优专家数量如何随数据和模型规模变化？本文首次建立了 MoE Transformer 的完整泛化理论，将活跃参数容量与路由组合开销干净地分离。

🔬 核心方法详解
① 条件路由覆盖数界（定理3.4）
核心技术创新：将 MoE 路由视为对假设类的划分，在固定路由模式上条件化，再对所有可能的路由模式做 union bound。
推导 sup-norm 覆盖数界：
$$\log \mathcal{N}(\mathcal{F}_{\text{MoE}}, \epsilon, \|\cdot\|_\infty) \leq \underbrace{P_{\text{active}} \cdot \log(1/\epsilon)}_{\text{活跃参数容量}} + \underbrace{n \cdot E \cdot \log \binom{E}{k}}_{\text{MoE路由开销}}$$ 其中 \(P_{\text{active}}\) 是每个 token 的活跃参数数（top-\(k\) 专家的参数总和），\(E\) 是专家数，\(k\) 是每 token 激活的专家数。关键洞察：泛化误差由活跃参数决定，而非总参数——路由开销仅是对数级的。
② 流形数据模型下的逼近-估计权衡（定理3.2）
假设数据分布支撑在 \(d_0\) 维流形上，目标函数为 \(C^\beta\) 光滑。结合覆盖数界和标准 ERM 分析，推导出 MoE 的泛化界：
$$\text{Risk} \lesssim \underbrace{P_{\text{active}}^{-2\beta/d_0}}_{\text{逼近误差}} + \underbrace{\frac{P_{\text{active}} + nE\log\binom{E}{k}}{N}}_{\text{估计误差}}$$ 其中 \(N\) 是训练样本数。这意味着 MoE 的缩放律指数仅取决于活跃参数和数据流形维度，与总参数无关——理论上证实了"MoE 的泛化像一个与其活跃参数同等大小的稠密模型"。
③ 最优专家数量定理
由泛化界最小化得到最优专家数 \(E^*\) 作为 \(N\) 和 \(k\) 的函数：
$$E^* \propto \left(\frac{N}{n \cdot k}\right)^{d_0 / (2\beta + d_0)}$$ 关键预测：(a) 数据越多，最优专家数越多（更多数据支撑更大的路由空间）；(b) top-\(k\) 增加时最优 \(E\) 减少（每个 token 用更多专家，减少了路由多样性需求）；(c) 存在一个"compute-optimal frontier"——给定计算预算 \(C\)，最优 \(E^*\) 和 \(P_{\text{active}}^*\) 可联合求解。
④ 实验验证：与先验 MoE 缩放律的一致性
在最多 5B 参数的 MoE Transformer 上系统验证理论预测，覆盖 \(E \in \{2, 4, 8, 16, 32, 64\}\) 和 \(k \in \{1, 2, 4\}\)。
发现：(a) 实验拟合的缩放指数 \(\alpha_C = 0.52 \pm 0.03\)，与理论预测的 \(2\beta/(2\beta + d_0) = 0.55\)（取 \(\beta=1, d_0=3.6\)）高度吻合；(b) 最优 \(E^*\) 随 \(N\) 的增长斜率也与理论一致；(c) 路由开销项在实际规模下确实是对数级的，不是泛化的主要瓶颈。

预训练是LLM的基石，消耗了绝大部分算力和数据预算。一个令人困惑的现象：两个具有相同预训练loss的模型，在下游任务上可能表现截然不同。这意味着预训练loss并非下游性能的充分统计量——loss曲面上的"位置"同样重要。现有工作（如SWA、LAWA）尝试用权重平均来寻找更平坦的极小值，但缺乏系统性理论解释。本文提出"公共极小值（Common Minima）"假说，并基于此设计了 Nexus 框架。

🔬 核心方法详解
① 公共极小值假说
核心观察：预训练loss曲面上存在无数等价极小值，但其中只有一个子集是"公共极小值"——即在多个不同数据子集上都同时是极小值的点。形式化定义：
$$\theta^* \in \mathcal{M}_{\text{common}} \iff \theta^* \in \bigcap_{i=1}^{K} \arg\min_\theta \mathcal{L}(\theta; \mathcal{D}_i)$$ 其中 \(\mathcal{D}_1, ..., \mathcal{D}_K\) 是训练数据的 \(K\) 个不相交子集。直觉：公共极小值编码了跨数据分布的通用特征，而非某个特定子集的噪声模式——因此在下游任务中泛化更好。
② Nexus 组合式预训练策略
将总训练数据 \(\mathcal{D}\) 分为 \(K\) 份（默认 \(K=4\)），独立训练 \(K\) 个模型 \(\theta_1, ..., \theta_K\)，每个模型只用一份数据训练到相同的loss水平。然后执行 iterative merging：
$$\theta_{\text{merged}}^{(t+1)} = \frac{1}{K} \sum_{i=1}^{K} \theta_i^{(t)} + \eta \sum_{i=1}^{K} \nabla_\theta \mathcal{L}(\theta_i^{(t)}; \mathcal{D}_i)$$ 即先做简单平均，再做一步梯度修正。关键：这个过程不需要额外计算，因为每个模型的训练数据量是原来的 \(1/K\)——总计算量保持不变，但权重空间的探索方向不同。

实际实现中的关键选择：(a) \(K=4\) 是最优（\(K=2\) 平坦度增益不够，\(K=8\) 每个子模型训练不充分）；(b) 数据分割策略使用 domain-based 分割（如 Web/Book/Code/Scientific），比随机分割效果更好（因为领域间差异提供更强的"模型间距离"）；(c) Merging 频率：每 1000 步 merge 一次，比训练结束才 merge 提升 3%（因为中间 merge 提供了更多的"轨迹修正"机会）；(d) 通信开销：只需传输模型权重差分，实际额外通信 < 2%。

Decomposing the Delta: What Do Models Actually Learn from Preference Pairs?
📅 2026-04-10
🏛 IBM Research
RLDPO偏好学习
📌 背景与动机

DPO、KTO 等偏好优化方法被广泛用于LLM对齐，但关于"偏好数据中到底是什么在驱动下游推理能力的提升"，我们几乎一无所知。偏好对 \((y^+, y^-)\) 包含多种信号：内容差异、格式差异、推理链长度差异、错误类型差异等。哪些因素是关键的？本文首次系统性地分解偏好对中的学习信号，将模糊的"偏好数据质量"问题转化为可量化的组件分析。

🔬 核心方法详解
① 两种 Delta 定义
将偏好对 \((y^+, y^-)\) 的"差异"分解为两个正交维度：
Content Delta：\(\delta_C = \text{sem}(y^+) - \text{sem}(y^-)\)，即两个回答在语义空间中的距离，衡量"说了什么不同"。
Style Delta：\(\delta_S = \text{style}(y^+) - \text{style}(y^-)\)，包括格式（如是否使用列表/代码块）、长度、语气等表面特征的差异。
核心假设：模型从偏好对学到的东西 = \(f(\delta_C, \delta_S)\)，可以独立操控和测量。
② 样本级 Delta 测量
对于每个偏好对，定义三个量化指标：
(a) 推理链 Delta：用 CoT 解析器提取 \(y^+\) 和 \(y^-\) 中的推理步骤数和逻辑跳跃，计算 \(\delta_R = |R^+ - R^-|\)；
(b) 答案正确性 Delta：\(\delta_A = \mathbb{1}[y^+ \text{ correct}] - \mathbb{1}[y^- \text{ correct}]\)；
(c) 表面形式 Delta：使用 token-level 编辑距离归一化：\(\delta_F = \text{EditDist}(y^+, y^-) / \max(|y^+|, |y^-|)\)。
关键发现：推理 Delta \(\delta_R\) 与下游推理任务增益的相关性为 \(r = 0.73\)，远高于答案正确性 Delta（\(r = 0.31\)）和表面形式 Delta（\(r = 0.12\)）。
③ 控制实验：合成偏好数据
为了因果验证（而非仅相关分析），合成三种偏好数据集：
(a) 仅推理 Delta：\(y^+\) 和 \(y^-\) 有相同答案和格式，只有推理链的详细程度不同；
(b) 仅答案 Delta：推理链长度相同，只有最终答案的正确性不同；
(c) 仅格式 Delta：内容完全相同，只有 Markdown 格式/列表/代码块使用不同。
分别用 DPO 训练相同的 base model，在数学推理任务上评估。
④ 主要发现与启示
实验揭示了三个反直觉的结论：
(a) 推理过程比正确答案更重要：仅推理 Delta 的 DPO 训练提升了 GSM8K +8.3%，而仅答案 Delta 仅提升 +2.1%；
(b) 格式 Delta 是噪声：仅格式 Delta 的 DPO 训练对推理任务几乎无效（+0.5%），但会导致输出风格的过度拟合；
(c) 大 Delta 不等于好：当 \(\delta_F > 0.8\)（即两个回答几乎完全不同），DPO 训练效果反而下降——模型无法从差异过大的对中提取有用信号。最佳 \(\delta_F \in [0.3, 0.6]\)。

Re-Mask and Redirect: Exploiting Denoising Irreversibility in Diffusion Language Models
📅 2026-04-10
🏛 Intel Labs
RL安全对齐扩散LM
📌 背景与动机

扩散语言模型（dLLMs，如MDLM、SEDD）通过迭代去噪掩码token序列来生成文本。与自回归LLM不同，dLLM 的安全对齐机制依赖于一个关键假设：去噪调度是单调的，已提交的token不会被重新评估。但这个假设在对抗场景下是否可靠？本文发现dLLM的安全对齐建立在一个极其脆弱的基础上，并设计了 Re-Mask and Redirect (RMR) 攻击来完全绕过它。

🔬 核心方法详解
① 去噪不可逆性假设的脆弱性
安全对齐的 dLLM 在早期去噪步骤中提交"拒绝token"（如"I cannot"、"sorry"），这些 token 在后续步骤中不再被重新掩码。然而，作者发现这个机制有一个致命漏洞：去噪过程中的 token 提交顺序可以被外部干预——通过在中间步骤重新掩码已提交的拒绝 token，迫使模型在新的上下文中重新去噪，从而"重定向"到不安全的输出轨迹。
形式化：令 \(\mathbf{x}_t\) 为第 \(t\) 步的 token 序列，\(M_t\) 为掩码集合。标准去噪：\(M_{t+1} \subset M_t\)（掩码单调递减）。RMR 攻击在第 \(t^*\) 步注入：\(M_{t^*} = M_{t^*-1} \cup \{i : x_i \in \text{refusal tokens}\}\)，即强制重新掩码拒绝 token。
② RMR 攻击的三阶段流程
Phase 1 — 检测（Detection）：监控去噪过程，当检测到连续出现拒绝模式token（如"I", "cannot", "help"在位置 \(i, i+1, i+2\) 被连续提交）时触发攻击。
Phase 2 — 重掩码（Re-Mask）：将已提交的拒绝 token 重新设为 [MASK]，同时保留非拒绝 token 不变：
$$\tilde{x}_i = \begin{cases} [\text{MASK}] & \text{if } x_i \in \mathcal{R} \\ x_i & \text{otherwise} \end{cases}$$ Phase 3 — 重定向（Redirect）：在重掩码后的序列上继续正常去噪。由于拒绝 token 被移除，剩余上下文引导模型生成替代内容——通常是原本被拒绝的不安全回答。
③ 攻击成功率与鲁棒性分析
在三个已对齐的 dLLM（MDLM-Safe、SEDD-Aligned、Dream-Guard）上测试 RMR：
攻击成功率（ASR）：MDLM 87.3%、SEDD 91.2%、Dream 83.6%——远高于自回归 LLM 上类似的 jailbreak 攻击（通常 20-40%）。
关键因素：dLLM 的并行去噪特性意味着拒绝和非拒绝 token 在同一步骤中被处理——去掉拒绝 token 后，非拒绝 token 的条件概率自然地倾向于继续生成有意义的（可能不安全的）内容。
④ 防御策略探索
作者同时提出了三种潜在防御方向：
(a) Token Commitment Lock：一旦 token 被提交，硬件级禁止任何重写——但这限制了 dLLM 的编辑灵活性；
(b) Redundant Refusal Encoding：在多个不连续位置同时注入拒绝信号——增加了重掩码的难度但也增加了计算成本；
(c) Denoising Trajectory Monitoring：检测异常的掩码模式变化——是最实用的方案，但可被梯度优化的隐蔽攻击绕过。
结论：目前没有完美的防御方案，dLLM 的安全对齐需要根本性的重新设计。

Efficient Unlearning through Maximizing Relearning Convergence Delay
📅 2026-04-10
🏛 Sungkyunkwan University
RL机器遗忘隐私
📌 背景与动机

随着GDPR和AI法案赋予用户"被遗忘权"，机器遗忘（Machine Unlearning）成为关键挑战：如何从已训练的模型中移除特定数据的影响？现有方法主要关注模型预测层面的遗忘（输出不再包含被遗忘数据的信息），但忽略了权重空间的持久记忆——即使输出看似遗忘了，模型权重中可能仍保留了快速重学习的"捷径"。本文提出了重学习收敛延迟（Relearning Convergence Delay）这一新度量，关注遗忘后模型重新学习被遗忘数据的速度。

🔬 核心方法详解
① 重学习收敛延迟度量
传统遗忘度量只检查遗忘后的即时输出。新度量定义：给定遗忘后的模型 \(\theta_u\) 和被遗忘数据集 \(\mathcal{D}_f\)，在 \(\theta_u\) 上重新训练 \(\mathcal{D}_f\)，测量模型重新达到原始性能所需的梯度步数 \(\tau\)：
$$\tau(\theta_u, \mathcal{D}_f) = \min\{t : \mathcal{L}(\theta_u^{(t)}; \mathcal{D}_f) \leq \mathcal{L}(\theta^*; \mathcal{D}_f) + \epsilon\}$$ 直觉：如果 \(\tau\) 很小，说明遗忘是"表面的"——权重中仍保留了通向被遗忘数据的快速路径。真正有效的遗忘应使 \(\tau\) 尽可能大（接近从随机初始化开始的训练步数）。
② IEU 框架：影响消除遗忘
提出 Influence Eliminating Unlearning (IEU) 框架，包含两个组件的交替优化：
梯度上升（GA）：在遗忘集上执行梯度上升，显式增加遗忘数据的损失：
$$\theta \leftarrow \theta + \eta_f \nabla_\theta \mathcal{L}(\theta; \mathcal{D}_f)$$ 使用较小的步长 \(\eta_f = c \cdot \eta_r\)（\(c \in [0,1)\)）来控制稳定性。
梯度下降（GD）：同时在保留集上执行标准梯度下降，维持模型在非遗忘数据上的性能：
$$\theta \leftarrow \theta - \eta_r \nabla_\theta \mathcal{L}(\theta; \mathcal{D}_r)$$
③ 噪声正则化（Noisy Regularization）
GA 仅增加遗忘数据的损失，但可能保留权重空间中的"结构记忆"。关键创新——迭代重初始化过程：
$$\theta^{(t+1)} = (1-\alpha) \theta^{(t)} + \alpha \theta_{\text{rand}}$$ 其中 \(\theta_{\text{rand}} \sim \mathcal{N}(0, 2/d)\)（Kaiming 初始化），\(\alpha \in [0,1]\) 控制噪声强度。
理论保证：该过程最大化权重的条件数（condition number），使得从 \(\theta_u\) 重新学习 \(\mathcal{D}_f\) 的收敛速度指数级减慢。作者证明了在凸设定下 \(\tau \geq \Omega(e^{C/\alpha})\)。
④ 收敛保证与指数延迟
在满足PL不等式的条件下，作者证明了IEU+NR的重学习延迟满足：
$$\tau(\theta_u) \geq \frac{1}{\mu} \log\left(\frac{\kappa(\theta_u)}{\epsilon}\right)$$ 其中 \(\mu\) 是PL常数，\(\kappa(\theta_u)\) 是遗忘后权重的条件数。噪声正则化通过增大 \(\kappa\) 来延长 \(\tau\)。
实际效果：IEU+NR 的重学习延迟比标准 GA 大 5-15 倍，接近从随机初始化训练的步数。

CIFAR-10 遗忘类别实验

  • 标准微调遗忘：遗忘后重学习 \(\tau = 120\) 步（从头训练需 2000 步），重学习比 = 6%
  • GA 遗忘：\(\tau = 340\) 步，重学习比 = 17%
  • IEU (GA + GD)：\(\tau = 850\) 步，重学习比 = 43%
  • IEU + NR：\(\tau = \mathbf{1750}\) 步，重学习比 = 88%（接近从头训练！）
  • 保留集准确率：标准 94.2% → IEU+NR 93.8%（仅下降 0.4%）

🔍 消融与讨论

消融实验验证了：(1) \(\alpha\)（噪声强度）的最优值在 0.1-0.3 之间——太小则噪声正则化不足，太大则保留集性能严重退化；(2) GA 的步长比 \(c = \eta_f/\eta_r\) 需要精心调节——过大的 GA 步长导致权重发散；(3) IEU+NR 在 LLM 遗忘任务上也有效（Llama-7B 上遗忘 TOFU 数据集，重学习延迟提升 8 倍），但计算开销比 GA 增加约 40%。局限性：噪声正则化的最优 \(\alpha\) 对任务和模型规模敏感，缺乏自适应选择机制；理论分析假设PL条件，对非凸深度网络的适用性需要更多验证。

💡 结论与启发

本文提出了"重学习收敛延迟"这一新颖的遗忘度量，捕获了传统度量忽略的权重空间记忆。IEU+NR 框架通过梯度上升 + 噪声正则化的组合，实现了接近从头重训的遗忘效果，同时保持了保留集性能。核心启示：有效的机器遗忘不仅要改变输出，更要破坏权重中的"学习捷径"，使重学习变得和首次学习一样困难。

Multi-User Large Language Model Agents
📅 2026-04-10
🏛 Stanford University, KAUST, MIT
Agent多用户协调
📌 背景与动机

LLM Agent 日益被部署为团队工作流和组织工具中的助手，但现有系统隐式地假设单委托人（single-principal）交互范式——模型服务于一个具有最高权限的用户。然而，真实场景中 Agent 需要同时服务多个用户，每个用户拥有不同的角色、偏好和权限级别，形成了多委托人（multi-principal）决策问题。现有研究完全忽略了这一设定中的核心挑战：冲突指令、信息不对称和隐私约束。

🔬 核心方法详解
① 多用户交互的形式化
将多用户 LLM Agent 建模为多委托人决策问题。令 \(\mathcal{U} = \{u_1, ..., u_K\}\) 为用户集合，每个用户 \(u_i\) 有目标函数 \(J_i(\cdot)\)、权限级别 \(\ell_i \in \{1,...,L\}\)、和隐私集合 \(\mathcal{P}_i\)（不可向其他用户泄露的信息）。Agent 的决策目标为：
$$\max_a \sum_{i=1}^{K} w_i \cdot J_i(a) \quad \text{s.t.} \quad \forall i \neq j: \text{Info}(a, \mathcal{P}_i, u_j) = 0$$ 其中 \(w_i\) 为基于权限的加权，约束条件保证不泄露任何用户的隐私信息给其他用户。
② 统一多用户交互协议
设计了一个标准化的交互框架，包含三个阶段：
Phase 1 — 角色声明：每个用户声明自己的角色（管理者/成员/观察者）和权限级别，Agent 构建权限图 \(G = (\mathcal{U}, \mathcal{E})\)。
Phase 2 — 冲突检测：当接收到多个用户的指令时，Agent 执行冲突检测——比较指令间的语义一致性，标记潜在冲突并请求优先级裁决。
Phase 3 — 安全执行：在隐私约束下执行最优动作，对每个用户返回经过信息过滤的个性化回复。
③ 三个压力测试场景
设计了三个针对性的评估场景：
场景A — 指令冲突：两个用户对同一任务给出矛盾指令（如"安排早会"vs"取消所有早会"），测试 Agent 能否根据权限级别稳定执行优先指令。
场景B — 隐私保护：用户A的对话包含敏感信息（如工资、健康状况），用户B在后续多轮对话中试图间接获取——测试 Agent 在多轮交互中隐私泄露的累积风险。
场景C — 社会选择：多个用户需要就共享资源（如会议室、项目优先级）达成共识，测试 Agent 能否实现公平的协调机制。
④ 关键发现：前沿LLM的系统性缺陷
在 GPT-4o、Claude-3.5、Gemini-1.5 和 Llama-3-70B 上测试，发现三个系统性失败模式：
(a) 优先级不稳定：在冲突场景中，68% 的情况下 Agent 未正确执行高权限用户的指令——倾向于折中而非遵循权限层级；
(b) 累积隐私泄露：随着对话轮次增加，隐私泄露率从第1轮的 5% 升至第10轮的 34%——Agent 在长对话中逐渐"忘记"隐私约束；
(c) 协调失败：在社会选择场景中，Agent 的"公平"标准不一致——对不同用户组合可能给出矛盾的协调方案。

CORA: Conformal Risk-Controlled Agents for Safeguarded Mobile GUI Automation
📅 2026-04-10
🏛 HKU, CUHK
AgentGUI自动化安全控制
📌 背景与动机

基于 VLM 的 GUI Agent 正从被动辅助走向自主操作——直接在用户手机上执行点击、输入、滑动等不可逆动作。然而，不受限的动作空间使用户暴露于严重的财务、隐私甚至人身安全风险（如误操作支付、删除数据）。现有安全机制要么过于保守（频繁中断用户），要么过于宽松（无法防止关键错误）。本文提出 CORA 框架，基于保形预测（Conformal Prediction）为 GUI Agent 提供概率安全保证。

🔬 核心方法详解
① 风险分层的动作分类
将 GUI 动作按不可逆程度分为三级：
Level 1 (可逆)：滚动、查看、返回——可安全自主执行。
Level 2 (部分不可逆)：输入文字、选择选项——可以撤销但需要额外步骤。
Level 3 (不可逆)：确认支付、删除账户、发送消息——一旦执行无法撤回。
每个动作 \(a_t\) 被赋予风险标签 \(r(a_t) \in \{1, 2, 3\}\)，Agent 对不同级别的动作适用不同的安全阈值。
② 保形预测置信集
核心：在每步决策时，Agent 不仅输出最优动作，还输出一个保形置信集 \(\mathcal{C}(x_t)\)——保证真正正确的动作以至少 \(1-\alpha\) 的概率包含在集合中：
$$\Pr[a^* \in \mathcal{C}(x_t)] \geq 1 - \alpha$$ 其中 \(\alpha\) 是用户指定的风险容忍度。关键技术：使用分位数回归估计 nonconformity score \(s(x_t, a) = 1 - p_\theta(a|x_t)\)，然后用校准集确定阈值 \(\hat{q}\)：
$$\mathcal{C}(x_t) = \{a : s(x_t, a) \leq \hat{q}\}$$ 保形保证是分布无关的——即使 VLM 的概率估计不准确，置信集的覆盖率仍然有效。
③ 自适应安全策略
根据置信集大小和动作风险级别动态决定执行策略：
(a) 自主执行：当 \(|\mathcal{C}| = 1\)（置信集只包含一个动作）且 \(r(a) \leq 2\) → 直接执行；
(b) 请求确认：当 \(|\mathcal{C}| = 1\) 但 \(r(a) = 3\)（不可逆动作）→ 向用户请求确认；
(c) 缩小选项：当 \(|\mathcal{C}| > 1\) 且 \(|\mathcal{C}| \leq 3\) → 向用户展示候选动作列表；
(d) 暂停并解释：当 \(|\mathcal{C}| > 3\)（高不确定性）→ Agent 暂停并解释当前状态和困惑点。
这比固定阈值策略更灵活——在 Agent 确信时快速执行，不确定时适度中断。
④ 在线校准自适应
GUI 环境是非平稳的——同一个 App 在不同状态下的 UI 布局可能完全不同。CORA 使用在线保形校准，每 \(T\) 步更新 \(\hat{q}\)：
$$\hat{q}_{t+T} = \hat{q}_t + \eta \cdot (\alpha - \text{err}_t)$$ 其中 \(\text{err}_t\) 是最近 \(T\) 步的实际错误率。如果错误率高于 \(\alpha\)，阈值收紧（更保守）；反之则放松。这保证了长期平均覆盖率约等于 \(1-\alpha\)，同时适应 UI 的动态变化。

  • 步骤2（点击"转账"）：\(|\mathcal{C}| = 1\)，\(r = 1\) → 自主执行 ✅
  • 步骤3（输入收款人"张三"）：\(|\mathcal{C}| = 1\)，\(r = 2\) → 自主执行 ✅
  • 步骤4（输入金额"500"）：\(|\mathcal{C}| = 2\)（"500"和"5000"都在置信集中）→ 向用户确认 ⚠️
  • 步骤5（点击"确认转账"）：\(|\mathcal{C}| = 1\)，\(r = 3\)（不可逆） → 请求用户确认 ⚠️
  • 总中断次数：2 次（仅在关键决策点），vs 传统方法 5 次（每步都确认）

💡 结论与启发

CORA 首次将保形预测引入 GUI Agent 安全控制，实现了在概率安全保证下的自适应执行——高确信时快速自主执行，低确信或高风险时最小化中断。核心优势是分布无关的安全保证：即使 VLM 内部概率不准确，保形置信集的覆盖率仍然可靠。这为自主 Agent 的安全部署提供了一个有理论基础的实用框架。

Task-Aware LLM Routing with Multi-Level Task-Profile-Guided Data Synthesis for Cold-Start Scenarios
📅 2026-04-10
🏛 HKU
AgentLLM路由冷启动
📌 背景与动机

不同 LLM 在不同任务上表现差异巨大——没有一个模型在所有任务上都是最优的。LLM 路由系统旨在根据查询自动选择性价比最高的模型。然而，现有路由器存在严重的冷启动问题：当引入新模型或面对新任务类型时，缺乏足够的性能数据来训练路由决策。本文提出任务感知路由框架，通过多层级任务画像指导的数据合成来解决冷启动。

🔬 核心方法详解
① 多层级任务画像（Multi-Level Task Profile）
对每个查询构建三层任务描述：
L1 — 粗粒度分类：任务类别（数学推理/代码生成/创意写作/知识问答/指令遵循等），用分类器提取。
L2 — 能力需求向量：将查询映射到一个 \(d\)-维能力需求空间 \(\mathbf{c} = f_\phi(x) \in \mathbb{R}^d\)，每个维度对应一个基础能力（如"多步推理"、"长文本理解"、"格式遵循"等）。
L3 — 难度估计：预估查询的难度等级 \(\delta \in [0,1]\)，低难度查询可用小模型节省成本，高难度查询需路由到强模型。
路由决策：\(m^* = \arg\max_{m \in \mathcal{M}} \text{Score}(m, \mathbf{c}, \delta) / \text{Cost}(m)\)，即在满足质量要求下最小化成本。
② 任务画像指导的数据合成
冷启动解决方案的核心：利用少量种子数据 + LLM 生成大量合成路由训练数据。
三步流程：
(a) 画像采样：从已知任务画像空间中采样新的 \((\mathbf{c}, \delta)\) 组合，重点采样覆盖不足的区域；
(b) 查询生成：用 LLM 根据采样的画像生成符合该画像的查询——如"生成一个需要3步推理、难度0.7的数学题"；
(c) 模型评估：将生成的查询分别发送给所有候选模型，评估其回答质量，得到 \((\text{query}, \text{model}, \text{quality})\) 三元组。
关键创新：数据合成是画像指导的（而非随机的），确保合成数据均匀覆盖任务空间。
③ 路由器架构与训练
路由器采用双塔架构：
查询塔：编码查询文本 + 任务画像 → 查询表示 \(\mathbf{q} = E_Q(x, \mathbf{c}, \delta)\)。
模型塔：编码模型能力描述 → 模型表示 \(\mathbf{m}_i = E_M(\text{desc}_i)\)。
路由分数：\(\text{Score}(x, m_i) = \mathbf{q}^\top \mathbf{m}_i\)。
训练损失结合质量预测和成本效率：
$$\mathcal{L} = \mathcal{L}_{\text{quality}} + \lambda \cdot \mathcal{L}_{\text{cost}} = -\sum_{i} y_i \log \sigma(\mathbf{q}^\top \mathbf{m}_i) + \lambda \sum_{i} \sigma(\mathbf{q}^\top \mathbf{m}_i) \cdot c_i$$
④ 主动学习的在线适应
部署后，路由器通过主动学习持续改进：对于路由器不确定的查询（置信度 < 阈值），将其发送给多个模型评估——产生高信息量的新训练数据。
这形成了一个"冷启动→合成数据→部署→主动学习→精化"的循环，使路由器在数据积累过程中持续提升，且初始合成数据提供了一个合理的启动点而非随机猜测。

冷启动场景：新增 Qwen-2.5-72B 到路由池
现有路由池包含 GPT-4o、Claude-3.5、Llama-3-70B，新增 Qwen-2.5-72B（无历史性能数据）：
  • 无合成数据：路由器对 Qwen 的选择率 < 5%（几乎不被选中），即使 Qwen 在中文任务上是最优的
  • 随机合成 1K 查询：Qwen 选择率提升至 15%，但在其优势领域（中文推理）仍被低估
  • 画像指导合成 1K 查询：Qwen 选择率在中文推理任务上提升至 62%，在英文任务上保持 18%
  • 成本节省：画像指导路由 vs 全部使用 GPT-4o，质量下降 < 2% 但成本降低 57%

Medical Reasoning with Large Language Models: A Survey and MR-Bench
📅 2026-04-10
🏛 CUHK, NUS
Medical推理综述
📌 背景与动机

LLM 在医学考试式任务上取得了优异成绩（如 MedQA、USMLE），激发了将其部署于真实医疗场景的兴趣。然而，通过考试不等于能做医学推理——真实临床决策需要多步骤推理、不确定性处理、跨模态信息整合，以及在不完整信息下的风险权衡。本文是首个系统性梳理"LLM医学推理"的综述，同时提出 MR-Bench 基准来量化推理深度。

🔬 核心方法详解
① 医学推理能力的四层分类学
提出了一个层级化的医学推理分类框架：
L1 — 知识回忆：检索医学事实（如药物适应证、解剖结构），对应选择题式任务。
L2 — 临床推理：基于症状、检查结果和病史进行鉴别诊断，需要多步逻辑链。
L3 — 治疗规划：制定个体化治疗方案，需权衡疗效、副作用、患者偏好和经济因素。
L4 — 不确定性推理：在信息不完整时（如缺失检查、矛盾症状），做出概率性判断并提出进一步检查建议。
现有 LLM 在 L1 已接近人类专家，但在 L3-L4 上显著落后。
② MR-Bench 基准构建
MR-Bench 包含 2,847 个临床推理问题，覆盖 15 个科室，每个问题需要 3-8 步推理。与现有基准的关键区别：
(a) 过程评估：不仅评最终答案，更评推理链的每一步是否正确——使用 5 名执业医师标注的"黄金推理链"作为参考；
(b) 干扰信息：每个问题包含 2-4 个与正确诊断无关的"红鲱鱼"症状，测试模型是否能正确忽略；
(c) 不完整信息：30% 的问题故意省略关键检查结果，正确答案应该是"需要进一步检查 X"而非直接诊断。
评分机制：\(\text{Score} = 0.4 \cdot \text{Final} + 0.4 \cdot \text{Process} + 0.2 \cdot \text{Uncertainty}\)
③ 技术路线综述：从 Prompting 到 Fine-tuning
系统梳理了增强 LLM 医学推理的三条技术路线：
Prompting 方法：医学 CoT、临床决策树 prompting、自一致性投票——简单有效但受限于 base model 能力；
RAG 方法：检索医学知识库（如 PubMed、UpToDate）辅助推理——提升知识覆盖但可能引入噪声；
Fine-tuning 方法：在医学推理数据上微调（如 Medical CoT 数据集）——最有效但需要高质量标注数据。
关键发现：三类方法在 L1-L2 上效果相当，但在 L3-L4 上 fine-tuning 显著优于其他两类。
④ GPT-4o/Claude-3.5/Med-PaLM 2 在 MR-Bench 上的评估
首次在 MR-Bench 上评估前沿模型，发现了几个关键 gap：
(a) 过程准确率 << 最终答案准确率：GPT-4o 最终答案 72.3% 但推理过程仅 58.1%——说明模型经常"对了答案但错了推理"；
(b) 不确定性处理极差：面对不完整信息时，90%+ 的模型回答仍给出确定诊断（而非建议进一步检查）；
(c) 红鲱鱼陷阱：干扰信息使 GPT-4o 的准确率下降 12%，而人类医师仅下降 3%。

这项综述和基准测试揭示了 LLM 医学推理的核心差距：通过考试 ≠ 会做临床推理。最严重的短板在于不确定性处理——模型在信息不完整时过度自信地给出诊断，而临床实践中"知道自己不知道什么"是最重要的能力。MR-Bench 的过程评估机制为未来研究提供了更精准的度量工具。

MedConceal: A Benchmark for Clinical Hidden-Concern Reasoning Under Partial Observability
📅 2026-04-10
🏛 UIUC
Medical对话推理部分可观测
📌 背景与动机

医患沟通是一个信息不对称问题：患者往往不会主动披露恐惧、误解或实际障碍（如经济困难、用药依从性差），除非临床医生具备巧妙的引导技巧。有效的医学对话需要在部分可观测条件下推理——临床医生必须从有限的表面信息中推断隐性关切（hidden concerns），进而调整沟通策略。本文提出 MedConceal 基准来系统评估 LLM 在这一关键能力上的表现。

🔬 核心方法详解
① 隐性关切的分类体系
定义了五类临床隐性关切：
恐惧/焦虑：患者担心诊断结果但不直接表达（如癌症恐惧）。
认知误解：对疾病或治疗的错误理解（如"抗生素能治感冒"）。
实际障碍：经济困难、交通不便、无法按时就诊等。
社会压力：家庭关系、工作压力导致的隐性心理负担。
用药障碍：副作用、用药方案复杂、药物获取困难等。
每个类别包含10-15个临床亚型，共64个具体的隐性关切模式。
② 基准构建：部分可观测对话
MedConceal 包含 1,200 个多轮医患对话场景。每个场景包含：
(a) 患者画像：基本信息、主诉、真实隐性关切（标注但不直接在对话中表达）；
(b) 对话历史：前 3-5 轮的医患对话（患者只描述症状，不提及隐性关切）；
(c) 评估目标：模型需要从对话中推断隐性关切，并生成引导性问题来确认推断。
评分维度：推断准确率（是否正确识别隐性关切类别）、引导质量（生成的问题是否恰当且有同理心）、回避率（模型是否能识别出"这里有未说出的东西"）。
③ 三阶段评估协议
Stage 1 — 检测：给定对话历史，模型判断是否存在未表达的隐性关切（二分类）。
Stage 2 — 推断：如果存在，推断属于哪个类别和亚型（64类分类）。
Stage 3 — 引导：生成一个开放式问题来引导患者表达隐性关切（生成式评估）。
三阶段的综合得分 = 检测F1 × 推断Acc × 引导质量。
④ 评估发现：LLM 的"共情盲区"
在 GPT-4o、Claude-3.5、Llama-3-70B 和 Meditron-70B 上的评估揭示：
(a) 检测能力尚可：GPT-4o 的检测 F1 达到 0.71，能识别"患者有话没说"；
(b) 推断不够精确：即使检测到隐性关切存在，类别推断准确率仅 42%——模型倾向于过度诊断"恐惧/焦虑"而忽略"实际障碍"和"用药障碍"；
(c) 引导生成过于直接：模型生成的引导问题往往过于直白（如"你是不是在担心什么？"），缺乏临床医生使用的间接、循序渐进的引导技巧。

隐性关切推理示例
患者画像：58岁男性，糖尿病患者，血糖控制不佳。隐性关切：经济困难导致无法负担胰岛素费用。

  • Claude-3.5 推断："患者可能存在用药依从性问题" → ✅ 接近（用药障碍类）
  • Claude-3.5 引导："目前的药物方案执行起来有没有什么困难？" → ✅ 较好的间接引导
  • 人类专家引导："胰岛素用得还顺利吗？有些患者会因为各种原因调整用量..." → ✅ 最佳（暗示性引导，不造成尴尬）

💡 结论与启发

MedConceal 揭示了 LLM 在医学对话中的一个关键盲区：理解言外之意。当前模型在检测"有未说出的东西"上表现尚可，但在精确推断其内容和生成恰当引导上还远落后于人类临床医生。这对医疗 AI 助手的设计启示是：不仅要训练"回答问题"的能力，更要训练"提出正确问题"的能力。

🌐 Multimodal — 多模态大模型

EXAONE 4.5 Technical Report: LG's First Open-Weight Vision-Language Model
📅 2026-04-10
🏛 LG AI Research
MultimodalVLM开源
📌 背景与动机

EXAONE 系列是 LG AI Research 持续开发的基础模型，此前的 EXAONE 3.0 和 4.0 均为纯文本模型。EXAONE 4.5 是该系列首个视觉语言模型（VLM），也是 LG 发布的首个开源权重 VLM，旨在为工业智能应用提供多模态理解能力。其架构将专用视觉编码器集成到现有 EXAONE 4.0 框架中，实现了原生多模态预训练。

🔬 核心方法详解
① 架构设计：视觉编码器集成
EXAONE 4.5 采用"视觉编码器 + 连接器 + LLM"的三段式架构：
视觉编码器：基于 SigLIP-SO400M，输入分辨率 384×384，输出 729 个视觉 token（每 token 对应 14×14 patch）。
连接器：两层 MLP + 下采样，将 729 个视觉 token 压缩为 256 个，并投影到 LLM 的 embedding 空间。
LLM 骨架：EXAONE 4.0（7.8B 参数），不冻结——视觉 token 和文本 token 在同一 attention 层中处理。
关键设计选择：不使用动态分辨率（固定 384×384），通过图像分块（tiling）处理高分辨率图像——最多 9 块 tiles。
② 三阶段训练流程
Stage 1 — 视觉-语言对齐预训练：在 50M 图文对上训练连接器（冻结视觉编码器和 LLM），学习视觉→语言的映射。
Stage 2 — 多模态联合预训练：解冻全部参数，在混合数据（图文对 + 纯文本 + OCR + 文档理解 + 图表理解）上联合预训练。纯文本数据占比 40%，防止语言能力退化。
Stage 3 — 后训练：包含 SFT（多模态指令微调）、DPO（视觉偏好对齐）和 RLHF 三个子阶段。
③ 工业场景优化
针对 LG 的工业应用需求（如家电维修、制造质检、零售分析），EXAONE 4.5 在后训练中加入了大量工业场景数据：
(a) 文档/表格理解：技术手册、检测报告、工单等结构化文档的信息提取；
(b) 物体检测+描述：工业产品的缺陷识别和分类；
(c) 多图对比：同一产品不同状态（正常/异常）的对比分析。
这些工业化需求驱动了 tile-based 高分辨率处理（需要看清小缺陷）和结构化输出能力。
④ 开源策略与许可
EXAONE 4.5 采用 CC BY-NC-ND 4.0 许可证——允许学术使用和非商业研究，但商业部署需要额外许可。发布三个版本：

ECHO: Efficient Chest X-ray Report Generation with One-step Block Diffusion
📅 2026-04-10
🏛 Midea AIRC, Beijing Jiaotong University, Dalian University of Technology
Multimodal医学影像扩散模型
📌 背景与动机

胸片报告生成（CXR-RG）是缓解放射科医生工作负担的关键应用。然而，传统自回归 VLM 的逐 token 解码导致推理延迟过高（通常 >2秒/报告），难以满足高通量临床需求。扩散语言模型（dLLM）可以并行生成所有 token，但直接应用 dLLM 生成医学报告面临两个挑战：(1) 多步去噪仍较慢；(2) 医学文本的幻觉问题在扩散生成中更严重。本文提出 ECHO，一个单步块扩散 VLM，兼顾效率和临床准确性。

🔬 核心方法详解
① 从自回归到扩散：AR-to-Diffusion 转化
ECHO 的训练分两步：先训练一个自回归基线模型 ECHO-AR，然后将其转化为扩散模型 ECHO。
转化方法：利用 ECHO-AR 生成的 (图像, 报告) 对作为教师数据，训练一个 Block Diffusion 模型来学习"给定图像，一步生成完整报告"的映射。
Block Diffusion 的核心：将报告 token 序列分为 \(B\) 个块（每块 32 token），每个块内部做离散扩散（掩码→去噪），块之间通过因果注意力连接：

② 直接条件蒸馏（DCD）
关键创新：通过蒸馏将多步扩散压缩为单步生成。DCD 包含三个组件：
Step-wise Token Reweighting (SW)：在蒸馏过程中，对不同时间步的 token 分配不同权重——早期时间步（噪声大）给低权重，晚期时间步（接近完成）给高权重；
Cross-Entropy Loss on EOS (CE)：对结束标记 \ 单独施加交叉熵损失，确保模型学会在正确位置终止生成；
Reverse KL Divergence (RKL)：用反向 KL 散度对齐学生（单步）和教师（多步）的输出分布：
$$\mathcal{L}_{\text{RKL}} = \sum_{i} p_\text{student}(x_i) \log \frac{p_\text{student}(x_i)}{p_\text{teacher}(x_i)}$$ 反向 KL 鼓励学生模型的输出分布更集中（mode-seeking），减少幻觉。
③ 幻觉缓解策略
医学报告中的幻觉（生成不存在的病变描述）是核心安全问题。ECHO 采用两种策略：
Report-Aware Distillation (RAD)：在蒸馏数据中加入"负样本"——将真实报告与不匹配的图像配对，训练模型识别不一致。
Fused Block KV Cache：在推理时，将视觉 KV cache 与文本 KV cache 融合，确保每个生成块都"看到"完整的视觉信息——防止纯扩散生成中因并行解码导致的视觉信息遗漏。
④ 推理效率分析
ECHO 的单步推理流程：输入胸片 → 视觉编码（一次）→ 扩散单步去噪（一次，并行生成所有 token）→ 输出报告。
时间对比：自回归 VLM 需要 ~200 次 forward pass（报告 ~200 token），ECHO 仅需 1 次 forward pass + KV cache 融合。
实际加速：在 A100 GPU 上，ECHO 的推理速度是自回归基线的 16.5x，延迟从 2.3 秒降至 0.14 秒/报告。

🔍 消融与讨论

DCD 消融显示：(1) SW（Step-wise Reweighting）单独使用降低 PPL 但提升有限；(2) CE（EOS 损失）是关键组件——没有它，模型无法正确终止生成，产生大量重复文本；(3) SW+CE 的组合效果最好（CIDEr 从 3.61 提升至 4.20），而加入 RKL 反而略有下降（可能因为 RKL 的 mode-seeking 特性限制了文本多样性）。RAD 数据规模从 5K→50K 持续提升 RateScore（+4.8），说明负样本学习对减少幻觉至关重要。局限性：单步生成在 ROUGE-L 上比多步略低（53.3 vs 55.0），表明极致压缩仍有少量质量损失；仅在英文胸片报告上验证。

💡 结论与启发

ECHO 证明了"高临床准确性 + 极致解码效率"可以同时实现。通过 AR→Diffusion 转化 + DCD 蒸馏，ECHO 在 0.14 秒内生成质量与自回归模型相当的胸片报告，实现 16.5 倍加速。对医学 AI 部署的启示是：扩散语言模型在医学报告生成中有独特优势——并行解码天然适合高通量场景。
