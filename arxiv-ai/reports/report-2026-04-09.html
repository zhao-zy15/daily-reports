The Stepwise Informativeness Assumption: Why are Entropy Dynamics and Reasoning Correlated in LLMs?

👤 Mar Gonzàlez I Català, Haitz Sáez de Ocáriz Borde, George D. Montañez, Pietro Liò 🏛 University of Cambridge, Harvey Mudd College 📅 2026-04-11 · arXiv:2604.06192 LLM Reasoning Information Theory
📌 背景与动机

近年来大量实证研究发现，LLM 推理过程中的内部熵动态与外部正确性之间存在鲁棒的相关性——低熵轨迹更可能导向正确答案。然而，内部熵完全由模型预测分布定义，而正确性由外部真实答案分布定义，两者理论上无先验关联。现有方法要么将此相关性视为经验事实直接利用，要么诉诸训练导致对齐的模糊直觉，却从未严格回答：为什么模型内部的不确定性降低能预测推理成功？

🔬 核心方法详解
① 逐步信息增益框架（Stepwise Information Gain）

定义推理链中每个 token \(C_k\) 的信息增益为：

$$\Delta_k(q,a,c_{1:k}) = h(a|q,c_{其中 \(h\) 是关于真实答案的点惊讶度。\(\Delta_k > 0\) 意味着该步骤使正确答案概率更高（信息性步骤），\(\Delta_k < 0\) 则反之。累积增益 \(G_k = \sum_{t=1}^k \Delta_t\) 可将条件答案熵分解为初始不确定性减去到目前为止积累的信息量。关键引理证明 \(\mathbb{E}[\Delta_k] = I(A; C_k | Q, C_{
② 逐步信息性假设（SIA）

在一个联合耦合 \(p \in \Pi\) 下（保持真实答案分布和模型推理分布的边际），假设推理前缀关于真实答案是信息性的：

$$I_p(A; C_{1:k} | Q) \geq \epsilon_k > 0 \quad \forall k \geq 1$$

SIA 不要求每个 token 都是信息性的——它约束的是前缀整体所包含的答案相关信息。SIA 成立时，条件答案熵可被解释为推理的进度变量：它沿成功推理链单调递减。Theorem 1 进一步通过 Fano 不等式证明：熵约束了可达准确率的下界 \(P_e^{(k)} \geq \frac{H_p(A|Q,C_{1:k}) - \log 2}{\log(|\mathcal{A}|-1)}\)。

③ SIA 从 MLE 训练中涌现

论文证明了一个 SIA 传递定理（Theorem 2）：如果人类推理轨迹的数据生成分布 \(r\) 满足 \(I_r(A; C_{\leq k}|Q) > 0\)，且模型通过 MLE 训练使得 \(\text{KL}(r \| p_\theta) \leq \delta_k\)，则模型分布也满足 \(I_{p_\theta}(A; C_{\leq k}|Q) \geq \epsilon_k/2\)。证明依赖于条件互信息在 KL 散度下的连续性（经由 Fannes-Audenaert 不等式→Pinsker 不等式的链式推导）。这解释了为什么 SFT 和 RL 后训练能诱导内部熵与正确性的对齐。

④ 可观测签名与失败模式

SIA 预测三个可观测签名：(a) 早期信息锁定——正确轨迹在推理早期就积累大量答案信息；(b) 早期可分性——用条件熵即可在推理早期区分正确/错误轨迹（AUC 显著高于随机）；(c) 饱和——对齐模型的条件熵收敛至接近零的平台。SIA 不成立时（如基座模型），这些签名消失或减弱，内部熵下降可能反映对错误假说的确信（即幻觉的信息论形式化）。

💡 深度案例
SIA 对齐系数实验：

对每条推理轨迹计算 \(\rho_{\text{SIA}} = \text{corr}_k(H_\theta(A|q,c_{\leq k}), -\log p_\theta(a^*|q,c_{\leq k}))\)：

  • Base Gemma-2-2B: \(\rho = -0.530\)（负相关！熵下降时远离正确答案）
  • SFT Qwen-2.5-3B-it: \(\rho = 0.744\)（强正对齐）
  • RL DeepSeek-R1-Distilled: \(\rho = 0.795\)（接近完美对齐）
  • SFT+RL OLMo-3-7B-Think-SFT: \(\rho = 0.964\)（几乎完美）

Shuffle-prefix 消融：随机打乱前缀 token 顺序后，Qwen-2.5-3B 的 \(\rho\) 从 0.682 暴跌至 -0.132，证实信息积累依赖结构化序列，而非简单的 token 计数。

🧪 消融与讨论

Shuffle-prefix 消融明确证实熵-正确性对齐依赖于 token 的结构化排列。Monte-Carlo 估计鲁棒性消融（MC=32, stride=4）显示结果定性不变。SIA 的关键局限：(1) 对创意写作等无明确终端变量的任务不直接适用；(2) 强 SIA 条件下 base 模型仍可能出现负对齐（对错误假说的确信），此时熵诊断方法理论上不成立。论文未覆盖是否可以通过干预熵动态来改变推理结果。

🎯 结论与启发

本文首次为"LLM 内部熵为何能预测推理正确性"提供了信息论解释：通过 SIA 假设，条件答案熵成为推理的进度变量。MLE 训练在人类推理轨迹上自然诱导 SIA，SFT 和 RL 进一步强化。这为所有基于熵的推理诊断方法（早停、探索、质量预测）提供了理论基础，也指出了失败模式——当训练未能诱导 SIA 时，熵信号可能具有误导性。

The Illusion of Superposition? A Principled Analysis of Latent Thinking in Language Models

👤 Michael Rizvi-Martel, Guillaume Rabusseau, Marius Mosbach 🏛 Mila, Université de Montréal, McGill University 📅 2026-04-07 · arXiv:2604.06374 LLM Latent Reasoning Interpretability
📌 背景与动机

连续思维链（Latent CoT）被认为比离散 CoT 更具表达力，其核心假说是叠加（superposition）：模型可以在单一表示中同时维护多个候选解，在提交答案前并行探索多条推理路径。然而，这一吸引人的假说缺乏实证验证。现有 Soft Thinking 和 Coconut 方法声称利用叠加，但模型实际上是否真的在用叠加推理？这直接关系到潜在推理方法的设计方向。

🔬 核心方法详解
① Soft Thinking 叠加崩溃分析

Soft Thinking 在每步构造软 token：\(\tilde{e}_t = \sum_{k \in \mathcal{V}} p_{t,k} \cdot e_k\)，其中 \(p_t\) 是 top-\(k_{\text{soft}}\) token 的 softmax 权重。使用 Logit Lens 在 QwQ-32B（64层）的 5 个均匀采样层分析软 token 与离散 token 的熵剖面。结果显示：Shannon 熵在各层几乎完全相同（图2a），两者在中间层的 KL 散度降至 \(\sim 10^{-4}\)。进一步的 token 级干预实验（将单个软 token 替换为其 argmax 离散 token）显示余弦相似度 \(0.996 \pm 0.025\)。叠加在前向传播的最初几层即崩溃为单一解释。

② Coconut 微调模型的快捷学习

在 ProsQA 图遍历任务上微调 GPT-2(124M)。关键发现：移除所有潜在 token 后，模型仍达到 96.6% 准确率（完整 Coconut 为 99.0%）。通过实体级信念探测（将 LM head 投影到词汇空间，追踪 correct-next/wrong-neighbor/target/other 四类实体的概率演化），发现 Coconut 模型的 target 实体从第 0 步就主导分布（无逐步推理签名），而离散 CoT 模型则展现正确的逐步信念转移。这表明微调模型学到了快捷解——在单次前向传播中直接获取答案，然后通过潜在 token 复制。

③ 从头训练模型中叠加的涌现

用 GPT-2 风格小模型（8头, 768维, 2层）在简化 ProsQA 上从头训练 Coconut。结果与微调截然不同：从头训练的模型确实展现叠加推理签名——中间推理步骤中 correct-next 实体概率主导但 wrong-neighbor 保持显著概率（图4），且移除潜在 token 后准确率从 94.5% 暴跌至 13.8%。信念在推理步骤间逐步演化，符合并行探索假说。

④ 崩溃的双因素解释

因素1：预训练的最后层 token 承诺——对 Qwen2-1.5B 注入均匀权重叠加输入，预训练模型在最终层熵急剧下降至接近零，而随机初始化模型保持高熵。预训练的 next-token prediction 目标创建了在最后层分离 token 身份的表示。因素2：容量效应——即使从头训练，过大的模型（8/12层）也倾向于学习快捷解；只有容量受限的浅层模型（2/4层）才发展出叠加推理。

💡 深度案例
ProsQA 示例："Every dax is a wug. Every dax is a zug. Every wug is a blicket. Rex is a dax. Is Rex a blicket or a gorple?"
  • 离散 CoT： 逐步推理 dax→wug→blicket，每步 correct-next 概率最高
  • 微调 Coconut： 从第 0 步起 target="blicket" 概率即占 80%+（无逐步推理）
  • 从头 Coconut (2L)： 步骤1 correct-next="wug" ~60%, wrong-neighbor="zug" ~25%；步骤2 target="blicket" 概率上升——真正的叠加信念演化

移除潜在 token 后：微调模型 96.6%→96.6%（几乎无影响），从头模型 94.5%→13.8%（灾难性下降）。

🧪 消融与讨论

容量消融（2/4/8/12 层）清晰展示：叠加与模型大小呈负相关——过大的模型直接学快捷解。Soft Thinking 的混合权重熵仅 \(0.18 \pm 0.34\) nats，输入已近乎离散。Top-3 token 可视化显示高 KL 步骤的叠加发生在语义无关的标点/格式 token 之间（如 "(" 和 "{"），而非有意义的推理路径探索。局限：未测试 RL 训练的连续 CoT（如 Butt et al. 2025）和循环层方法。

🎯 结论与启发

本文系统性地揭示了潜在推理中叠加假说的幻觉：现有的 Soft Thinking 和 Coconut 微调方法并未真正利用叠加推理，只有容量受限的从头训练模型才发展出叠加。根本原因是预训练的 next-token prediction 目标在最后层创建了 token 身份分离的压力。这对潜在推理方法的未来设计有重要启示：叠加可能需要在更高抽象层面（推理策略级而非 token 级）才有意义，且需要专门的训练机制来对抗预训练偏置。

ART: Attention Replacement Technique to Improve Factuality in LLMs

👤 Ziqin Luo, Yihao Quan, Xiaofeng Zhang, Xiaosong Yuan, Chen Shen 🏛 Shanghai Jiao Tong University, Fudan University, Alibaba Cloud 📅 2026-04-07 · arXiv:2604.06393 LLM Hallucination 🏢 Alibaba Cloud
📌 背景与动机

LLM 幻觉问题在 QA 任务中尤为严重。已有方法多需外部知识库或重训模型。本文发现 LLM 浅层存在大量均匀注意力头（对序列均匀分配注意力），导致模型无法聚焦关键信息，是幻觉来源之一。

🔬 核心方法详解
① 注意力模式分类与 m-index

定义 m-index 量化注意力头与均匀注意力的偏离：$$m_h^l = \frac{1}{N(N+1)/2} \sum_{i \geq j} \max\!\left(\frac{A_h^l[i,j]}{U[i,j]}, \frac{U[i,j]}{A_h^l[i,j]}\right)$$m-index 小→uniform 头，大→local 头。按此排序将头分为三类。

② 模式影响验证

消融实验表明：mask 单个 local 头即严重损害生成；mask 少量 uniform 头几乎无影响甚至可提升。uniform 头具有高冗余性。

③ ART 替换机制

浅层（前2层）中将 \(k = \lfloor 0.1 \times N_h \rfloor\) 个 uniform 头替换为 local 注意力：$$\text{ART-MHA}^l(X) = \sum_{i \in \mathcal{I}} A_{\text{local}}^l f_i^l(X) + \sum_{i \notin \mathcal{I}} A_i^l f_i^l(X)$$两种变体：ART-max（取 m-index 最大单头）和 ART-mean（所有 local 头均值）。

④ 零训练即插即用

仅在推理时干预浅层 MHA，不需微调或训练数据，不影响推理效率。

💡 深度案例
ART-inverse 反向实验：
  • Qwen2.5-7B：TruthfulQA 0.0%, LogiQA 2.0%, GSM8K 7.5%——将 local 替换为 uniform 使模型几乎完全失效
  • 正向 ART 则稳定提升 +1.0~3.4pp

🎯 结论与启发

ART 揭示了 LLM 浅层 uniform 头冗余、local 头关键的规律，通过简单注意力替换即可零训练提升事实性。在多模型多任务上均有效。

👤 Jiang Zhou, Yunhao Wang, Xing Wu, Tinghao Yu, Feng Zhang 🏛 Tencent 📅 2026-04-08 · arXiv:2604.06829 LLM Pretraining 🏢 Tencent
📌 背景与动机

合成数据改写是增强 LLM 预训练知识获取的重要技术，但现有方法仅在单文档级别操作。WRAP++ 将范围扩展到跨文档知识发现与放大。

🔬 核心方法详解
① 跨文档知识发现

③ 预训练整合

合成数据与原始 Web 数据混合。先纯数据预训练再混入跨文档合成数据效果最佳。

💡 深度案例
跨文档发现：文档A描述电池化学成分，文档B讨论EV续航与能量密度关系 → WRAP++ 综合推理出新电池对续航的具体提升幅度。

Does a Global Perspective Help Prune Sparse MoEs Elegantly? (GRAPE)

👤 Zeliang Zhang, Nikhil Ghosh, Jiani Liu, Bin Yu, Xiaodong Liu 🏛 University of Rochester, Flatiron Institute, UC Berkeley, Microsoft Research 📅 2026-04-08 · arXiv:2604.06542 LLM MoE Pruning
📌 背景与动机

稀疏 MoE 模型内存开销巨大。现有剪枝方法在局部层级独立评估每层专家，忽略跨层冗余差异。GRAPE 提出全局视角进行跨层预算分配。

🔬 核心方法详解
① 全局专家重要性评估

使用校准数据计算每个专家的激活频率与输出范数贡献。

② 全局预算分配

按重要性分数全局排序，移除分数最低的专家。“重要层”保留更多专家，“冗余层”更激进剪枝。

③ 轻量路由器微调

剪枝后仅微调路由器 gate 参数，无需全模型重训。

💡 深度案例
Mixtral-8x7B 50%剪枝分布：浅层保留6/8专家(25%剪枝)，中间层4/8(50%)，深层3/8(62%)。

🧪 消融与讨论
