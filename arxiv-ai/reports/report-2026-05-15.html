LLM-1 · Why Retrieval-Augmented Generation Fails: A Graph Perspective

📅 2026-05-13 🏛 Michigan State University, MIT arXiv:2605.14192
LLM RAG 可解释性 归因图

① 背景与动机

RAG 已成为减少 LLM 幻觉的事实标准，但在检索到正确证据的情况下仍频繁产生错误答案。现有解释主要依赖单层注意力或 Logit-lens 分析，只能给出"静态、单层"的内部状态切片，无法刻画证据信息在 32 层间是怎样被传播、转换、聚合的。作者主张把 RAG 推理视为一个跨层的有向计算图问题——只有在图结构上做差异分析，才能解释"为什么相关证据被检索到，模型却仍答错"。

② 核心方法详解

① 归因图构造（Attribution Graph）

对每个 query，把每一层每个 token 的残差流激活看作一个节点 $v_{t,\ell}$；用 transcoder 替换 MLP（attention 与 LayerNorm 保持不变并固定其前向值）使整个网络对激活线性，从而能把 logit 关于上游 token 的归因解为 token-pair 间的加性贡献：

$$w(v_{s,k}\!\to\!v_{t,\ell})=\sum_{u\in\mathcal U_{s,k}}\sum_{u'\in\mathcal U_{t,\ell}}\frac{\partial \mathrm{logit}_y}{\partial a_{u'}}\cdot J_{u',u}\cdot a_u$$

其中 $\mathcal U_{s,k}$ 是 token $(s,k)$ 关联的激活单元集合，$J$ 是线性化雅克比。token 对的边权 $w$ 即归因强度。

② 区域分层与诊断信号

把 token 划分为 4 个区域 $R\in\{Q,\, Ex,\, In,\, A\}$（Question / External-Context / Intermediate / Answer），并把 32 层切成低层 $L_{\text{low}}=[0,7]$ 与高层 $L_{\text{high}}=[8,31]$。在每层每个区域对的归因量 $\sum_{(u,v)\in R_a\times R_b}w_{uv}$ 上对正确/错误样本聚类，作者发现：

  • 正确预测在低层强 $Q\!\to\!Q$（自我巩固问题语义）、低 $*\!\to\!Ex$（晚一些再融入证据）；
  • 错误预测过早把流量打到 $Ex$ 区域，问题表征还没稳就被外部证据"污染"，作者称之为"Early External Drift"。
③ 基于图的 unfaithful 检测器

为每张归因图 $G$ 训练一个 2 层 graph transformer，节点特征拼接 one-hot 区域类型 + 入度 / 出度 / PageRank，边特征为归因权重；图级表示 $z(G)$ 后接 MLP 得到 $p(y=1\mid G)=\mathrm{softmax}(\mathrm{MLP}_{\text{out}}(z(G)))$，在 1000 张图（500 正确 / 500 错误）平衡数据上以 AdamW（lr $10^{-4}$）训练，作为 RAG 失败的"事中"检测器。

④ 区域感知注意力干预（推断时）

把诊断结果反作用回推断：在低层放大 $Q\!\to\!Q$ 路由、抑制 $*\!\to\!Ex$；在高层维持以问题为引导的 answer 精化。形式化为一组分区缩放因子：

$$\alpha_{Q\to Q}^{(\ell)}=\alpha_{QQ}=1.5,\;\;\alpha_{*\to Ex}^{(\ell)}=\alpha_{ctx}=0.5,\;\ell\in L_{\text{low}}$$

$$\alpha_{Q\to In}^{(\ell)}=\alpha_{QIn}=1.5,\;\ell\in L_{\text{high}}$$

在 softmax 前对相应 token-对的注意力 logits 乘以 $\alpha$，无须任何额外训练即可改写信息路由。

③ 深度案例：Mix-MuSiQue 上的"早期外部漂移"

问题：Mix-MuSiQue 多跳 QA，retrieved context 同时包含 1 篇支持证据 + 多篇 distractor。 原始模型行为（Before Control）：
  1. 第 0–4 层：模型尚未稳固问题表征，但 $\sum_{*\to Ex}w$ 已显著高于正确样本，distractor token 提前注入残差流；
  2. 第 5–10 层：question token 失去主导地位，answer 节点被外部 token 主导路由；
  3. 第 25–31 层：模型从 distractor 段落中抽取错误实体作为最终输出，输出"看起来流畅"但事实错误；
  4. baseline 在该 667 题集上得到 56.5% 的准确率。
启用 Region-Aware 控制后：低层 $Q\!\to\!Q$ × 1.5、$*\!\to\!Ex$ × 0.5，高层 $Q\!\to\!In$ × 1.5。模型在第 0–7 层重新巩固 query 表征，distractor 段落的早期写入被显著压缩； After Control 准确率 = 61.6%，绝对提升 5.1 pt（相对 9%），且不需要任何参数更新或额外推断成本。

⑤ 消融与讨论

消融分析：(1) 仅用低层控制（$Q\!\to\!Q$ + $*\!\to\!Ex$）已能从 56.5% 提至 ~60%，说明早期 Drift 是主要病因；(2) 移除高层 $Q\!\to\!In$ 控制会损失最后 ~1.5pt，说明高层仍需问题信号"低强度但持续"地引导。局限：(a) 归因图构造依赖 transcoder 替换 MLP，对未训练 transcoder 的封闭模型不可直接应用；(b) 干预系数 $\alpha$ 当前为全局常数，未考虑 query 难度自适应；(c) 评测仅限 LLaMA-3-8B 与 Gemma-2-9B 等开源模型，闭源 GPT-4 类模型无法实施。

⑥ 结论与启发

这篇工作把"RAG 为什么会错"从黑盒哲学问题转变为一个可观测的图结构问题：错误本质上是一种"早期外部漂移 + 答案过早收敛"的路由病理，而修复也不必微调，只需在低层放大问题注意力、抑制证据流入即可获得近 10% 的相对提升。对工业 RAG 系统的启发是：把归因图视作可监控的运行时信号——它既能做"事前调试 + 事后审计"，也能做"事中干预"，比 logit confidence 自我评估更可靠（+11.53 pt 检测准确率）。这有可能成为下一代 RAG/Agent 系统的标配可观测层。

LLM-2 · EndPrompt: Efficient Long-Context Extension via Terminal Anchoring

📅 2026-05-14 🏛 Nankai University, Baidu, Shanghai Jiao Tong University arXiv:2605.14589
LLM 长上下文 RoPE 位置编码

① 背景与动机

把 LLM 的上下文从 8K 扩到 64K，传统做法是真在 64K 长度上微调——内存与算力随长度二次增长，多数中小团队无法复现。退而求其次的"chunk + 跳过位置索引"方法（Positional Skip-Embedding 等）虽然便宜，但把连续语义切成几段拼接，会破坏文本的连贯性，让模型学到一种"虚假的远距离相关"。本文要解决的问题是：能不能只用短训练序列（≤8K），在不破坏语义连续性的前提下，让 LLM 学到 64K 内的所有相对位置？

② 核心方法详解

① 终端锚定（Terminal Anchoring）的两段式构造

给定一段长度为 $a$ 的短上下文 $x=(x_0,\ldots,x_{a-1})$ 与一段长度为 $b$ 的"终端 prompt" $e=(e_0,\ldots,e_{b-1})$，把它们物理拼接为 $y=(x_0,\ldots,x_{a-1},e_0,\ldots,e_{b-1})$，物理长度仅 $a+b\!\ll\!L$。但位置索引被打到目标长度 $L$ 的两端：

$$p_\ell=\begin{cases}\ell, & 0\le\ell

这样模型同时看到两类相对距离：(a) 段内 token 间的近距离（0 到 $a-1$），(b) 跨段 token 间的长距离（接近 $L$）。

② 与 RoPE / Position Interpolation 的耦合

在 RoPE 下 token $\ell$ 的旋转角是 $\theta_\ell=p_\ell\cdot\theta_{\text{base}}$，两段索引方案让 query-key 内积自然出现长距离 $\Delta p\approx L\!-\!b$ 的旋转项。配合 Position Interpolation 把所有 $p_\ell$ 缩放到原 8K 窗口内：$\hat p_\ell=p_\ell\cdot\frac{8K}{L}$，使旋转角始终落在预训练分布内，避免 OOD 角度。

③ Bernstein 不等式给出的"光滑性"保证

论文用 Bernstein 不等式刻画 PI 后注意力函数 $A(\Delta)$ 关于相对距离 $\Delta$ 的 Lipschitz 光滑性：观察到的 $\Delta\in\{0..a\}\cup\{L-b..L-1\}$ 在 PI 缩放下变成稠密的 $[0,8K]$ 子区间，再叠加 Transformer 跨头共享权重的归纳偏置，未观察到的中间距离的注意力外推被严格控制在一个紧致带宽内。理论上证明：稀疏的两端监督就足以约束整个 $A(\cdot)$ 函数空间，无需稠密长序列采样。

④ 训练流程

每 step 仅前向 $a+b$ 个 token（典型 $a=8K, b=512$，物理长度 8.5K），但模型实际上"接触"了 64K 的相对位置编码空间。loss 用标准 next-token CE：$\mathcal L=-\sum_\ell\log p_\theta(y_\ell\mid y_{<\ell},p_\ell)$。在 1B token 语料上微调即可；显存约束等价于 8K 训练，相比真 64K 训练节省一个数量级的算力。

③ 深度案例：为什么 chunk-based 方法失败而 EndPrompt 不会

对比实验场景：扩展 LLaMA-2-7B 上下文 8K→64K，在 RULER 的 NIAH_M3（多针穿插检索）任务上看 attention 行为。
  1. Positional Skip-Embedding（chunk-based 基线）：把同一段连贯文本切成 8 块，每块独立打位置索引并训练。结果：模型学到的注意力对块边界产生异常聚集，对真实长距离语义无效（NIAH_M3 仅 51.92%，与未训练时几乎无差）；
  2. Full-length FT（在 64K 训练）：NIAH_M3 = 56.20%，但训练显存约 EndPrompt 的 8 倍；
  3. EndPrompt（本文）：物理长度 8K + 512 token 终端 prompt，位置索引打到 [0, 8K] ∪ [63.5K, 64K]。NIAH_M3 = 62.92%，比 Full FT 还高 6.7pt，并且短文段语义连贯没被切碎。
关键差异：chunk-based 方法把"短-长"距离都打进了同一段连续文本里，导致模型学到的远距离 attention 实际上对应着"段间漂移"的虚假信号；EndPrompt 让长距离仅通过终端 prompt 这一个明确的"锚点"出现，模型清楚地分离了"局部连续语义"与"远距离锚定"两类信号。

⑤ 消融与讨论

消融实验在 RULER 上做了三种切片：(1) 底座模型：LLaMA-3-8B 76.03 → Mistral-7B-v0.3 68.39 → LLaMA-2-7B 45.82，说明方法对预训练质量敏感；(2) 扩展长度：ET-32K 76.45 / ET-96K 75.61 / ET-128K 72.82，扩到 96K 仍稳，128K 略掉点；(3) 训练 token 量：ET-2.0B 75.61 / ET-1.0B 76.03 / ET-0.5B 74.72，0.5B token 已经达到 98% 性能。局限：(a) 终端 prompt 长度 $b$ 是超参，未自适应；(b) 当 $L$ 远超 8K（>16×）时 PI 引入的旋转角变形会变大，需要更精细的尺度调度；(c) 评测仍以英文为主，多语种长文档表现未充分验证。

⑥ 结论与启发

EndPrompt 提出的"稀疏两端位置监督"挑战了"长上下文必须用长序列训练"的主流假设：在 RoPE + PI 框架下，只要 (a) 保留语义连续的短段、(b) 在末端加一个 anchor，模型就能从 8K 训练样本中学出 64K 范围的相对位置依赖。对工程团队的直接启示是：把 64K 微调降级为 8K + 512 token anchor 的微调，显存与时间双降一个数量级。这一思想还可推广到 GUI Agent / 代码长文件场景，作"局部连续 + 远距离锚定"的位置编码协议设计。

LLM-3 · DISCORD: Distribution Corrected Offline Data Distillation for Reasoning

📅 2026-05-13 🏛 George Mason University arXiv:2605.14071
LLM 蒸馏 数学推理 分布纠偏

① 背景与动机

把强模型的推理能力蒸馏到小模型有两条路：(a) 离线蒸馏（teacher 离线生成 trace，学生做 SFT/KL）—— 样本高效但存在训练-推理分布漂移：训练时学生总是看 teacher 生成的前缀，推理时却必须自回归自己生成的前缀，长 reasoning trace 上误差累积；(b) 在线蒸馏（GKD 等让学生采样自身 rollout）—— 分布匹配但需 ~16× 算力，且训练前期学生质量低拖慢学习。本文要在不引入任何 online sampling 的前提下，纠正离线蒸馏的分布漂移。

② 核心方法详解

① 核心思想：student-support 感知的密度比纠偏

对每条 teacher trace $\{x, y\sim\pi_T(\cdot|x;c)\}$，引入 student/teacher 概率密度比作为重要性权重 $\frac{\bar\pi_\theta(y|x)}{\bar\pi_T(y|x;c)}$（$\bar\pi$ 表示 stop-gradient）。直觉上：teacher 写但学生不会写的 token 权重低，teacher 写学生大概率会自己生成的 token 权重高——把学习火力压在"学生 inference 时容易踩到"的样本上。

② Sigmoid 稳定化 + Token-Level 修正

密度比无界且方差大，作者两步稳定：先做 sigmoid 压缩到 $(0,1)$，再做 token 级而非 sequence 级权重以避免长序列偏置：

$$r_t=\sigma\!\left(\tau\cdot\log\frac{\bar\pi_\theta(y_t\mid x,y_{

$\tau=1$ 时 $r_t$ 退化为 $\frac{\bar\pi_\theta}{\bar\pi_\theta+\bar\pi_T}$，正好等价于"$y_t$ 来自学生分布的后验概率"（uniform prior 下的贝叶斯分类器），这与对比解码（Contrastive Decoding）的训练时对偶。

这个修正项可作为 SFT、KL、Symmetric-KL 等任意离线 backbone 的插件式 reweighting，不改变其他训练流程。

④ 五阶段 reasoning 提示模板

为了引出 teacher 的高质量推理，作者还设计了一段结构化指令 $c$，把 teacher 的输出强制分成五个阶段：① problem understanding → ② strategy selection → ③ solution → ④ verification → ⑤ final answer。仅保留 teacher 最终答案正确的轨迹用于训练。这样 teacher trace 自身的语义结构更稳定，密度比噪声更小。

③ 深度案例：14B→1.5B 蒸馏中的分布纠偏

场景：用 Qwen2.5-14B-Instruct 作 teacher，把 MATH 推理蒸馏到 Qwen2.5-1.5B-Instruct（学生模型）。
  1. Vanilla SFT：学生在 MATH500 上仅 41.0%，因为 teacher 在某些 token 处用了"Wait, let me reconsider…"等只有大模型自然分布会输出的反思短语，学生从未学会自己说出这类 prefix；
  2. SFT + DISCORD：在 token "Wait" 处 $\bar\pi_\theta(\text{"Wait"})\approx 0.001$，$\bar\pi_T(\text{"Wait"})\approx 0.20$，密度比对数 $\log(0.001/0.20)\approx -5.3$，sigmoid 后 $r_t\approx 0.005$——几乎不学这个 token；而对学生本来就会说的"$x = $"等通用 token，$r_t\approx 0.5$，正常学习。结果 MATH500 从 41.0% → 45.4%（+4.4 pt），GSM8K 从 67.32% → 69.75%；
  3. Symmetric-KL + DISCORD：MATH500 进一步从 44.20% → 50.20%（+6.0 pt），证明纠偏可与不同 backbone 叠加。
关键观察：DISCORD 让学生学会"用学生的方式去说 teacher 的话"，而不是模仿 teacher 自己也说不出来的稀有 prefix；这与对比解码"放大目标模型相对于弱模型的优势"是同一原理的训练时镜像。

LLM-4 · Mistletoe: Stealthy Acceleration-Collapse Attacks on Speculative Decoding

📅 2026-05-13 🏛 Harbin Institute of Technology Shenzhen, Tsinghua University, Huawei arXiv:2605.14005
LLM 推测解码 对抗安全 推断系统

① 背景与动机

推测解码（Speculative Decoding）依赖一个"小 drafter"批量草拟 token，再由 target 大模型并行 verify，加速比由"平均接受长度 $\tau$"决定。EAGLE-3 等先进系统在 GSM8K/HumanEval 上常稳定提供 5–7× 加速。但这种加速比本身可被攻击吗？ 已有对抗攻击聚焦于改变模型最终输出，从未触及"加速机制"这一新攻击面。本文提出 Mistletoe：通过附加 16 token 左右的对抗后缀，使 drafter 草稿的 acceptance 率坍塌 (5.95→2.79)、加速比从 5×+ 退到 1.83×，但用户看到的最终回答语义几乎不变——典型的"DoS-on-acceleration"。

② 核心方法详解

① 威胁模型与约束优化目标

固定 target $M_\theta$ 与 drafter $D_\phi$，攻击者把 clean prompt $x$ 拼上短 suffix $\delta\in\mathcal V^m$ 得到 $x_\delta=x\oplus\delta$。Mistletoe 将攻击形式化为 constrained discrete optimization：

$$\max_{\delta\in\mathcal V^m}\;L_{\text{rej}}(x,\delta)\quad\text{s.t.}\;L_{\text{sem}}(x,\delta)\le\epsilon$$

$L_{\text{rej}}$ 提升 drafter 草稿在 target 上的"surprisal"（拒绝概率），$L_{\text{sem}}$ 用 KL 散度约束 target 输出分布漂移：$L_{\text{sem}}=\sum_t\mathrm{KL}(\pi_\theta(\cdot|x)\,\|\,\pi_\theta(\cdot|x_\delta))$，确保用户最终拿到的 token 序列保持可读。

② Target-side Draft-Token Surprisal

核心拒绝 loss 用 target 在 drafter 提议位置上的 surprisal：

直觉：$\rho_\phi$ 草拟的 token 在 $\pi_\theta$ 下越"出乎意料"，越会被 verify 拒。

③ Null-Space 投影解耦两个目标

"压低草稿被接受率"与"保持 target 语义"是冲突方向，作者建立局部线性化：用 $J_{\text{sem}}=\nabla_\delta L_{\text{sem}}$ 作为语义约束的雅可比，构造正交投影器

$$P_{\mathcal N}=I-J_{\text{sem}}^\top\big(J_{\text{sem}}J_{\text{sem}}^\top+\xi I\big)^{-1}J_{\text{sem}}$$

把拒绝梯度 $g_{\text{rej}}=\nabla_\delta L_{\text{rej}}$ 投影到 $\mathcal N$（语义约束的零空间）：$g_{\text{rej}}^{\mathcal N}=P_{\mathcal N}g_{\text{rej}}$。在标量 KL 约束下，这退化为高效的 rank-one 操作，开销可忽略。

最终更新方向 $g_{\text{final}}=-g_{\text{sem}}+\lambda g_{\text{rej}}^{\mathcal N}$，先把 KL 漂移拉回可行域，再沿"不破坏语义"的方向施加拒绝压力。

④ 离散搜索 + KL-bound veto

把连续梯度 $g_{\text{final}}$ 映射到 token 离散空间（top-k 候选 + greedy coordinate descent），每步对每个 suffix 位置选 $\arg\max$；最后加一道 KL-bound veto：若新 suffix 使 $L_{\text{sem}}>\epsilon$，直接丢弃。这保证可行性始终成立。Mistletoe 算法收敛后产出一段 16 token 的 ASCII 噪声 suffix（如"? : \ e ..."），看似乱码，实则精确瞄准了 drafter 与 target 的 mismatch surface。

③ 深度案例：EAGLE-3 上的"加速崩塌"

场景：Vicuna-13B + EAGLE-3 推测解码，输入 GSM8K 数学题。
  1. Clean prompt "Sarah has 5 apples..."：drafter 草拟 7 个 token，target 一次性 accept 6 个。$\tau=5.95$，端到端加速比 5.47×；
  2. + Mistletoe suffix（16 token，仅在 prompt 末尾追加）："Sarah has 5 apples... <adv>? : \ e # ! @ ^ ...</adv>"。drafter 在 suffix 影响下产出和 target 高概率分布偏离的 token；target verify 时几乎全部拒绝；
  3. 结果：每轮平均 accept 数从 5.95 → 2.79（−53%），加速比从 5.47× → 1.83×（−66%）；
  4. 但用户拿到的最终答案"Sarah has 8 apples now."与 clean 输出完全相同，PPL 几乎不变（可读性正常）；
  5. 实战意义：攻击者向云推断 API 注入 16 个看似乱码的 token，让该用户的算力消耗翻倍，但服务端 alarm 不会触发（因为输出未异常）。

RL-1 · CIPO: Learning from Failures with Correction-Oriented Policy Optimization

📅 2026-05-14 🏛 Chinese Academy of Sciences (ISCAS), UCAS, Xiaohongshu arXiv:2605.14539
RL RLVR GRPO 推理

① 背景与动机

RLVR（基于可验证奖励的强化学习）已是主流 LLM 推理 post-training 范式（GRPO/RLOO/DAPO 等），但奖励是二值的、稀疏的：失败轨迹得 0 分、成功得 1 分，所有失败都被一视同仁地"压制"。这忽略了一个事实：失败轨迹有 logical flaws、intermediate inconsistencies、minor final-step miscalculations 等异质错误模式，其中"差一点对"的轨迹其实承载了最多可挖掘的学习信号。本文要回答：能不能把失败轨迹从"被惩罚的对象"转化为"指导改进的方向性信号"？

② 核心方法详解

① 双数据流：Base Stream + Correction Stream

每个训练步同时维护两条 rollout 流：

  • Base Stream：标准 on-policy rollout，给 query $x$，从 $\pi_\theta(\cdot|x)$ 采样 $m$ 条轨迹 $y_i$；
  • Correction Stream：把失败轨迹 $y_{\text{fail}}$ 与原 query 拼起来作为新 prompt $x_{\text{rep}}=\text{Concat}(x, y)$，再让模型从 $\pi_\theta(\cdot|x_{\text{rep}})$ 生成 $n$ 条 refinement rollout $y'_i$。如果 $y'$ 答对了，那条 $(y_{\text{fail}}\!\to\!y')$ 链接就成了"如何从这种错误纠正回正确解"的可学习梯度路径。

组合 GRPO 目标：

$$J_{\text{CIPO}}(\theta)=J_{\text{GRPO}}^{\text{base}}(\theta)+\lambda\cdot J_{\text{GRPO}}^{\text{correction}}(\theta)$$

两个目标的 advantage 在各自 group 内独立标准化，避免相互干扰。

② Adaptive Replay Ratio

不加节制地学失败会导致 distribution shift。CIPO 维护一个动态 replay ratio $\rho_t\in[\rho_{\min},\rho_{\max}]$ 控制 base/correction 比例：当模型在最近的"recycled successful samples"上表现下降，说明已学的能力在退化，则增大 $\rho_t$（多回放成功轨迹）；反之放更多注意力在 correction 上。这是一个简单的"保命"反馈环。

③ Risk-Averse Reward Shaping

对 correction stream 引入风险厌恶奖励：如果 refinement $y'$ 仍然失败，相比 base stream 的失败给更大的负奖励，因为这意味着"模型连自己的错误都纠正不了"——这是 capability regression 的强信号。形式化为：$r_{\text{correction}}(y')=r_{\text{verify}}(y')-\beta\cdot\mathbb 1[y'\text{ fails}]$，$\beta>0$ 显式惩罚二次失败。

④ Difficulty-Aware Trajectory Preference

第三个机制：选择哪些 $y_{\text{fail}}$ 进入 correction stream。CIPO 优先挑选"接近边界"的失败——具体做法是计算同一 query 下 $m$ 条 base rollout 中 success ratio 在 $(0, 1)$ 内（既有成功也有失败）的 query，从这些"中等难度" query 的失败轨迹中采样进入 correction，避免在 0 success（太难）或全 success（太易）上浪费 correction 算力。

③ 深度案例：AIME24 上的"差一点对"轨迹纠错

场景：Qwen3-4B 在 AIME24 一道几何题上的训练片段。
  1. Vanilla GRPO 视角：8 条 rollout，5 错 3 对。5 条失败轨迹都被同样压制，但其中 1 条仅在最后一步把 $\sqrt{7}/2$ 算成 $\sqrt{14}/4$，另 1 条却在第二步就用错定理。GRPO 给出相同的负梯度，浪费了"差一点对"轨迹的精细信息；
  2. CIPO 视角：取那条"差一点对"的失败 $y_{\text{fail}}$，构造 prompt $x_{\text{rep}}=\text{原题} + \text{该失败轨迹}$，让模型生成 8 条 refinement。其中 6 条成功识别了"$\sqrt{7}/2$ 的最后一步错误"并给出正确答案，2 条仍失败；
  3. 这 8 条 correction rollout 的 group-relative advantage 给出明确梯度：把"识别最后一步算错"的能力作为正信号强化；2 条二次失败的轨迹被 risk-averse shaping 进一步加权惩罚；
  4. 结果：训 500 步后，AIME24 pass@1 从 23.54% → 47.50%（GRPO 同条件下 42.08%），AIME25 从 21.04% → 44.90%（GRPO 35.73%），AIME 系列绝对涨了 22-24pt，比 GRPO 多 5-9pt。

CIPO 在 pass@K 上同样领先（AIME24 pass@32 从 60.00% → ~70%+），说明它不是只把概率密度集中，而是真正扩大了模型的解题面，提升了 intrinsic reasoning 能力。三个机制的消融（论文 Appendix）都重要：移除 adaptive replay 训练后期会出现性能回退；移除 risk-averse shaping 学到的纠正能力会"过拟合"导致 base rollouts 退步；移除 difficulty-aware 选择会浪费大量 correction rollout 在已经全对/全错的 query 上。局限：(a) correction stream 把每步 rollout 量翻一倍，训练成本增加 ~2×；(b) 当前未与多 turn agent 场景结合（仅 single-turn QA）；(c) 只在 4B/8B 规模验证，更大模型上 risk-averse $\beta$ 是否需要重调未知。

⑥ 结论与启发

CIPO 的核心哲学是"失败不是负样本，而是 prompt"——把失败轨迹拼回 prompt 后再 rollout，使得模型从"消除错误"被动学习转为"主动纠错"主动学习。这对 RLVR 范式有两个重要启发：(1) 失败轨迹中蕴含的"how to correct"信息远比"this is wrong"信号丰富；(2) Self-correction 能力可以在不引入 verifier/critic LLM 的前提下、纯 on-policy 学到。对工业 RLHF/RLVR 团队的直接启发是：在已有 GRPO 训练管线上插入一个 correction stream（双 rollout），即可在 AIME 难题集上获得 5-9pt 的额外收益，性价比高。该工作已被中科院软件所 + 小红书联合署名，应是面向 ICLR/ICML 的强投稿。

RL-2 · Semantic-Reward RL Enables Low-Resource Language Expansion without Alignment Tax

📅 2026-05-14 🏛 Minzu University of China, Ant Group, Shanghai Jiao Tong University arXiv:2605.14366
RL GRPO 语义奖励 低资源语种

① 背景与动机

把 LLM 扩展到藏语、蒙语等低资源语种几乎总要付出"对齐税"——SFT 在窄分布数据上做 token-level 似然最大化，把基础能力（中文阅读理解 CMRC、英文常识等）严重侵蚀。论文先量化了这一现象：用 strong SFT 把 Qwen3-4B 适配到藏文翻译后，CMRC 性能从 base 的 ~85% 跌到 41.82% F1，下游中文能力近乎崩塌。问题：能否在不破坏原有多语种能力的前提下学到低资源语种？作者主张把 alignment 重新表述为语义空间一致性，让模型用自由的表达方式实现"等价含义"，自然减少对原参数空间的破坏。

② 核心方法详解

① 语义空间对齐的问题表述

放弃 token-level 似然 $\max\sum\log\pi(y_t\mid y_{"输出语义等价于参考即正确，无须形式逐字一致"。形式化为

$$\max_\theta\;\mathbb E_{x,y\sim\pi_\theta(\cdot|x)}[\,s(y, y^*)\,]\quad\text{s.t.}\;\mathrm{KL}(\pi_\theta\,\|\,\pi_{\text{ref}})\le\beta$$

$s(\cdot,\cdot)$ 是 sentence embedding 的余弦相似度，KL 约束保护预训练表征。

② Cold-Start + RL 的两阶段范式

Stage 1（Cold-Start SFT）：在仅 5K 平行语对上做 2 epoch 轻量 SFT，目的不是好分数，而是让模型能产出"目标 script"的非退化输出，作为 RL 的初始 policy $\pi_{\text{init}}$； Stage 2（GRPO 语义 RL）：从 $\pi_{\text{init}}$ 出发，在剩余 90K 数据上用 GRPO 训练 1 epoch。每个 prompt $x$ 采 $K$ 条 rollout $y^{(k)}$，按 group-relative advantage 更新 policy。

③ 语义奖励：阈值-缩放 + 语种一致硬约束

主奖励来自双语 sentence embedding $f(\cdot)$（基于 CINO，藏文增强的 XLM-R）：

$$s(y,y^*)=\cos(f(y), f(y^*))$$

但直接用余弦不稳定（早期 rollout 噪声大），引入 threshold-and-rescale shaping：

$$R_{\text{sim}}(y,y^*)=\begin{cases}0, & s\le\tau \\ \dfrac{s-\tau}{1-\tau}, & s>\tau\end{cases}$$

其中 $\tau$ 取 cold-start 后的最低充分性水平（实验 ~0.4-0.5），保证 RL 只 refine 已经能写出像样输出的样本。

语种一致性硬约束（防 reward hacking 用其他语言"骗"高 embedding 相似度）：

$$R_{\text{lang}}(y)=\begin{cases}1, & \text{output script consistent} \\ 0, & \text{otherwise}\end{cases}$$

用 Unicode 范围 + 正则做规则检查。最终奖励：

$$R(y, y^*)=\lambda_{\text{sim}}\cdot R_{\text{sim}}+\lambda_{\text{lang}}\cdot R_{\text{lang}},\;\lambda_{\text{sim}}=1.5,\;\lambda_{\text{lang}}=1.0$$

④ GRPO + LoRA 训练

使用 GRPO（value-free PPO 变体）作为约束 RL 的具体实现，与基础 PPO 的 trust-region 限制等价但无须 critic。底座 Qwen3-4B + LoRA（r=64, α=128, dropout=0.05）应用到 self-attention 和 MLP 的所有线性投影。RL 的 KL 系数控制每步 policy drift，从根上保证对预训练知识的微扰。

③ 深度案例：Strong SFT vs. Semantic-Reward RL 的"对齐税"对比

设置：Qwen3-4B + LoRA，藏-中机器翻译。Strong SFT 与 Semantic RL 用同样的训练数据（5K + 90K），同样的 LoRA 配置。
  1. Strong SFT 结果：BLEU-4 = 0.6006，语义相似度 = 0.8282——表面看翻译质量很高；但 CMRC（中文阅读理解）F1 从 base 的 ~85% 暴跌到 62.99，AVG 41.82。模型为了"匹配藏文翻译参考的具体词形"重写了原有中文表征；
  2. Semantic RL 结果：BLEU-4 = 0.4519（低于 SFT），但语义相似度 = 0.7164；关键的是 CMRC 几乎不掉（保持 ~80%+）；
  3. LLM-as-judge 偏好：Strong SFT 胜率 59.2%，Semantic RL 仍获 33.5% 胜率——尽管 BLEU 低 15pt，开放式翻译质量并未显著落后；
  4. 解读：Semantic RL 用"语义等价"作奖励让模型保留了表达自由度，因此不必把"中文表征空间"重写成"翻译参考语料的子集"，从根本上回避了 alignment tax。

RL-3 · SU-01: Achieving Gold-Medal-Level Olympiad Reasoning via Simple and Unified Scaling

📅 2026-05-13 🏛 Shanghai AI Laboratory, CUHK, Tsinghua University arXiv:2605.13301
RL RLVR 数学推理 奥数金牌

① 背景与动机

奥赛（IMO 数学、IPhO 物理）是 LLM long-horizon 推理的最佳压力测试：需要解题树搜索、严格假设管控、中间步骤自验、最终能通过严格评分。AlphaProof / AlphaGeometry 2 / Gemini DeepThink 通过专属符号引擎或巨量搜索拿到 IMO 银/金，但配方复杂且开源不友好。本文提出一个"简单统一可复现"的 recipe，用 30B-A3B（MoE-A3B 激活 3B 参数）开源底座 + 340K 短轨迹 SFT + 200 RL steps，在 IMO 2025 / USAMO 2026 / IPhO 2024-2025 上达到金牌水平，最终模型 SU-01 在 IMO-ProofBench 上 70.2% 接近 Gemini 3.1 Pro Thinking。

② 核心方法详解

① Reverse-Perplexity Curriculum SFT（Stage 1）

340K 长度 ≤8K token 的精选 reasoning trajectories，覆盖 Math/STEM/Code/Instruction Following。关键创新是反向困惑度课程：每个 epoch 把 trajectories 按当前 policy $\pi_\theta$ 的 perplexity $\mathrm{PPL}_\theta(y\mid x)$ 排序，先训 PPL 最高（最不熟）的样本，再训 PPL 低（已熟）的样本：

$$\text{order}(y_i)=\mathrm{rank}\!\big(\!-\mathrm{PPL}_\theta^{(t)}(y_i\mid x_i)\big)$$

直觉：先用陌生但严格证明轨迹"重塑"reasoning style，再用熟悉样本"巩固"原有能力，避免单纯训陌生样本带来的灾难遗忘。

② Coarse RL：Verifiable Reward + GSPO（Stage 2-1）

第一阶段 RL 走 RLVR 路线：使用可自动验证的数学题（最终答案匹配）。policy update 使用 GSPO（Group Sequence Policy Optimization），目标是放大 SFT 阶段已经习得的 search/verify 行为，提升对硬题的覆盖与求解率。reward 仅按答案对错给二值 0/1。

③ Refined RL：Generative Reward + Self-Refinement + Experience Replay（Stage 2-2）

第二阶段 RL 把目标从"答案对"升级到"证明严格"，引入三个机制：

  • Proof-level Generative Reward Model：用 DeepSeek-Math V2 给完整证明评分，按 ✓ CoT Validity / ✓ Proof Quality / ✓ Anti-Hacking 三个维度返回连续奖励（避免 verifiable reward 对推导过程不敏感的问题）；
  • Self-Refinement Prompts：训练 critique-and-repair 行为——模型先输出 initial solution，再被引导生成 bug report 与修订版，整个 solve→verify→refine 链路同时被优化；
  • Experience Replay：对在线 rollout 正确率 <25% 的硬题，把仅有的 1 条成功 trajectory 存入 experience buffer，下次该 query 出现时把它作为 hard-positive 重放进 batch；正确率回升到 50% 时退役。这是稀有成功轨迹的"保护机制"。
④ Test-Time Scaling：Self-Verify + Self-Refine 迭代（Stage 3）

推断时启动迭代闭环：solver 给 candidate proof → verifier 输出结构化 critique（critical errors / major justification gaps / minor justification）→ solver 按 critique 修订。该 loop 是 model-agnostic 的，可以叠加在已训好的 SU-01 之上把性能再向前推 12-15 pt。例如 IMO-ProofBench 直接生成 57.6% → TTS 后 70.2%。

③ 深度案例：USAMO 2026 上超过人类金牌

USAMO 2026 实测：直接生成的 SU-01 已经跨过铜牌线；启用 TTS（自验证-自修正循环 5 轮）后达到 35 分——
  1. 这一分数超过 USAMO 2026 金牌线 10 分；
  2. 匹配 USAMO 2026 全部 340 名参赛选手中最高的人类总分（按官方报告）；
  3. 对比：未做 TTS 的同 size 基线（如 P1-30B-A3B）仅得 23.1，Qwen3.6-35B-A3B 得 31.4；
  4. 关键得分点：SU-01 在 Problem 4（数论几何混合）上展示了 60K+ token 的完整 proof trajectory，含 7 次 self-correction，最终给出严谨且完整的论证。
意义：开源 30B-A3B（激活 3B 参数）模型用 200 RL steps 即可在 USAMO 上跨过金牌线，这意味着奥赛级 reasoning 不再是闭源大厂专利——团队具备工程复现条件。

Agent-1 · Self-Pruned KV Attention: Learning When to Write by Predicting Future Utility

📅 2026-05-13 🏛 Meta FAIR, CentraleSupélec arXiv:2605.14037
Agent 长上下文 KV Cache Memory

① 背景与动机

在现代 test-time compute 与 agentic 推断范式下，LLM 一次会话可能产生 100K+ token 的轨迹（多轮工具调用、长 reasoning、retrieval-augmented 上下文）。瓶颈不再是 compute，而是 KV cache 的内存占用与读带宽——每个新 token 都要读完整 KV，长序列导致 GPU memory traffic 爆炸。已有方案（GQA / MLA 共享 KV、QUEST / DeepSeek Sparse Attention 读时稀疏）要么写时仍存全量，要么需特定查询结构。本文 SP-KV 是另一条路：让模型自己在写入时就预测每个 KV 是否值得长期保留，硬剪枝、动态压缩比，对 agent 长 trajectory 友好。

② 核心方法详解

① Key-wise Utility 预测

在 layer $l$、key-head $k$、token 位置 $s$ 上，用一个 2-layer MLP $f^{l,k}_\theta$ 从隐藏状态 $h^l_s$ 预测 utility 分数：

$$u^{l,k}_s=\sigma\big(f^{l,k}_\theta(h^l_s)\big)\in(0,1)$$

推断时通过阈值 $\tau$ 二值化得到门控 $z_s=\mathbb 1[u_s\ge\tau]$：$z_s=0$ 表示这个 KV 不进入持久 cache（除非在 sliding window 内）。

② Sliding-Window + Gated Global Attention

始终保留最近 $w=128$ 个 token 的局部因果窗口（保护短程依赖），更早的 token 仅当 $z_s=1$ 时才进入全局 attention：

$$g_{t,s}=\begin{cases}0, & z_s=1\;\lor\;\mathbb 1_{\text{win}}(t,s) \\ -\infty, & \text{otherwise}\end{cases},\;B_{t,s}=M_{\text{causal}}(t,s)+g_{t,s}$$

$$o_t=\sum_s\mathrm{softmax}\!\Big(\frac{\langle q_t, k_s\rangle}{\sqrt d}+B_{t,s}\Big)v_s$$

③ Phase 1：Soft Gated Training（保留可微性）

训练时 thresholding 不可微。SP-KV 把 $\log u_s$ 直接加进 attention mask bias：$\tilde g_s=\log u_s$。$u_s\to 0$ 等价于 $-\infty$ mask（被屏蔽），$u_s\to 1$ 等价于 mask 0（完全开放）。所有 utility gate 在初始化时 logit 拉到大正数（默认全开放），用 vanilla next-token prediction loss 一统训练，无任何稀疏化辅助 loss——稀疏化作为 NTP loss 自然产物涌现。

④ Phase 2：Threshold-Aware Hard Gating + 块跳跃 kernel

在 cosine schedule 的最后 25%，冻结 utility predictor 权重，把 gate 用阈值 $\tau$ 二值化。为避免突变带来的优化不稳，500 步线性 anneal：

$$\tilde u=(1-\alpha)u+\alpha\cdot\mathbb 1[u\ge\tau],\;\alpha:0\to 1$$

训练-推断对齐后，作者修改 FlashAttention-3 加入块跳跃优化：以 64-token 粒度预计算稀疏 mask，对全零 KV 块直接 skip TMA load 与 MMA 计算。结果：1.6B-8.1B 模型上 Phase 2 ($\tau=0.7$) wall-clock throughput 达到 full-attention baseline 的 82-90%，但 KV cache 大小压缩 3-10×。

③ 深度案例：Agent 长 trajectory 上的动态压缩

场景：Llama-style 8.1B 模型处理一条 32K token 的 agent trajectory（含 10 次工具调用、多次 retrieval 注入）。
  1. Full Attention baseline：32K × 32 layers × 8 KV heads × 128 dim × 2 (K+V) ≈ 2.1 GB KV cache，每生成 1 个新 token 都要读完整 cache，bandwidth bound；
  2. SP-KV $\tau=0.7$：模型自己学到——retrieved 段落和工具结果中只有关键摘要 token（如 "result: 42", "error: timeout"）被打高 utility，中间填充 token（"the next thing to do is..."）被打低 utility 自然剪掉；最终 keep 比例在 $\rho\approx 0.10\sim 0.30$ 之间动态变化（越长越压缩），cache 压缩约 5-7×；
  3. 性能：在 NIAH（多针检索）、RULER 长上下文 QA 上对比 full-attention baseline 几乎无下降（<1pt 差异），但 decoding throughput 提升 ~2×；
  4. 意外发现：SP-KV 学到的 head-level utility pattern 揭示了"有些 attention head 天生就是 local，有些天生就是 global"——把高 average utility 的 head 设为 global、其他设为 local，得到的混合架构在同 KV 预算下超过传统 interleaved global-local layout。

消融分析：(1) 移除 sliding window：模型完全靠 utility predictor 的能力，短程依赖被破坏，PPL 大幅劣化；(2) 不做 Phase 2 hard gating：训练-推断 mismatch 严重，推断时强制阈值会带来 1-2pt 性能损失；(3) 预测粒度 token-level vs head-level vs layer-level：token-level（per-key）效果最好但最贵，head-level 是性能/复杂度最佳折衷；(4) 从随机初始化训练：可行但比 continued-pretraining 收敛慢约 1.5×。局限：(a) 当前方案仅对 standard MHA / GQA 适用，对 MLA 等结构需要重做；(b) Phase 2 的块跳跃 kernel 是 Hopper GPU 专属优化，老 GPU 性能优势会缩水；(c) utility predictor 增加了少量参数（每层 2 层 MLP），约 +1% 总参数。

⑥ 结论与启发

SP-KV 把 KV cache 优化从"读时稀疏"推进到"写时剪枝"——这是 long-context agent 系统的根本不同：写时压缩才能彻底解放 GPU 内存。论文的另一个隐藏价值是用 utility 分布作为"架构探针"，自动发现哪些 attention head 该 global / 哪些该 local，从而设计出更优的 hybrid attention 拓扑。Meta FAIR 的这项工作直接对标 agentic test-time-compute 趋势：在 Claude Code 类长 agent trajectory 中，5× KV cache 压缩意味着同硬件下能 hold 5× 更长的会话或 5× 并发。开源工业团队部署长 agent 推断时应将 SP-KV 列入考虑清单，特别是 longer reasoning loops（IMO solver、SWE-bench 修复任务）这种 trajectory 自然分层的场景。

Agent-2 · EvoLib: Test-Time Learning with an Evolving Library

📅 2026-05-14 🏛 Microsoft Research arXiv:2605.14477
Agent Test-Time Learning 技能库 自监督

① 背景与动机

现实部署中 LLM 几乎都是黑盒（无法 fine-tune），又往往缺乏 ground-truth feedback（无 reward 函数 / 测试用例）。已有 test-time learning (TTL) 方法多依赖梯度更新（不适用于闭源 API）或外部监督（不适用于现实未知答案任务）；test-time scaling (TTS) 如 Best-of-N、Self-Consistency 又把每个问题独立处理，跨问题知识无积累。本文 EvoLib 旨在两难之间走一条新路：用自监督的"技能库"演化让 LLM 在跨任务流上不断成长，参数不动、无需外部监督。

② 核心方法详解

① 问题表述与库结构

Agent 状态为 $(\Phi, \mathcal K)$：$\Phi$ 是 frozen LLM，$\mathcal K$ 是知识抽象库（带权集合）。任务流 $\{x_1,x_2,\ldots,x_T\}$，每步：

$$\{z_{t-1}\}\sim\mathrm{Sample}(\mathcal K_{t-1}, x_t)$$

$$\tilde y_t\sim P_\Phi(\cdot\mid x_t,\{z_{t-1}\})$$

$$\{z_t\}=\mathrm{Extract}(x_t,\tilde y_t),\;\mathcal K_t=\mathrm{Update}\big(\mathcal K_{t-1},(x_t,\tilde y_t,\{z_{t-1}\},\{z_t\})\big)$$

抽象有两类：modular skills（编程任务=函数；推理任务=带解的子问题；agentic 任务=带 workflow 的子任务）和 reflective insights（错误模式 / 启发式策略）。

② 自监督的 Sample-Solve-Extract 循环

不依赖外部 reward。每个 task 上，Agent 先从库里检索相关 abstraction 拼到 prompt，让 $\Phi$ 生成解 $\tilde y_t$；从 $\tilde y_t$ 中蒸馏新 abstraction（使用一个固定的 extraction prompt：identify reusable function、sub-task、insight）。无 ground-truth 即可执行——通过模型自评估和多样化采样代替外部 verifier。

③ Information Gain 权重 + Future IG 演化机制

核心创新：每个 abstraction $z$ 在库里持有一个权重 $w(z)$，它由两部分构成：

$$w(z)=\mathrm{IG}(z)+\beta\cdot\mathrm{FutureIG}(z)$$

  • $\mathrm{IG}(z)$（Immediate Information Gain）：在 $z$ 被 sample 进 prompt 时观察到的解题质量提升（用模型 self-evaluation 估计，如 verifier 自洽性、log-likelihood 提升）；
  • $\mathrm{FutureIG}(z)$（Future Information Gain）：$z$ 触发新 abstraction 生成的程度——一个 skill 越能被改写、组合、扩展，它的 long-term value 越高。

这是论文最巧妙的一步：不仅奖励"立刻有用"的 skill，还奖励"能孵化更多可复用 skill"的 skill。

④ Consolidation：把碎片合并为通用抽象

定期对库进行 consolidation：找语义相似/功能重复的 abstractions，让 LLM 自己 merge 成更通用版本。例如 task 1 抽出"detect cycles in directed graph using DFS"，task 5 抽出"check graph connectivity with BFS"，consolidation 后合成"graph traversal utilities (DFS/BFS)"。这一过程让 instance-specific skill 演化为 general、reusable 的 abstraction。

③ 深度案例：BigCodeBench Hard 上的技能演化

场景：GPT-4o 解决 148 道 BigCodeBench Hard 编程题，按顺序流式呈现。
  1. Task 1（CSV 数据清洗）：模型解决，extract 出 skill handle_missing_with_median(df)，IG 权重 ~0.3；
  2. Task 12（Time-series fillna）：sample 该 skill 进 prompt，得 hint "use median imputation"，解题质量提升，IG 权重提升至 0.5；同时 extract 出 resample_and_impute(df, freq)；
  3. Task 30（数据 ETL pipeline）：库内 4 个相关 skills 被 sample，模型组合使用并解出题；trigger consolidation：4 个 skill 被合并为更通用的 data_clean_pipeline(df, config)，原 4 个 skill 权重转入新 abstraction；
  4. Task 80（pandas 复杂 ETL + 时序）：直接 sample data_clean_pipeline，开箱即用；
  5. 结果：BigCodeBench Hard pass rate 从 base 29.7% → EvoLib 40.8%（+11.1pt），超过 Best-of-N (37.2%) 与 RSA (31.7%)；GPT-4o + EvoLib 甚至超过 leaderboard 上更强的 Gemini-Exp-1206 (40.5%)。

Agent-3 · MAP: A Map-then-Act Paradigm for Long-Horizon Interactive Agent Reasoning

📅 2026-05-13 🏛 USTC, Meituan, CASIA arXiv:2605.13037
Agent 长周期 认知地图 ARC-AGI-3

① 背景与动机

ReAct/CoT 类 agent 范式遵循"act-during-think"：模型只在执行时才被动获得环境感知。这造成Delayed Environmental Perception（延迟环境感知）：agent 必须通过 trial-and-error 推断空间布局、动作语法和约束，掉进低效失败循环——作者称之为 Epistemic Bottleneck。这是一个范式问题，不能靠扩大 reasoning 能力解决（ARC-AGI-3 上 Claude 4.6 也几乎为零分）。从 Pearl do-calculus 角度看，根因是 $P(o_{t+1}\mid do(a_t))$ 干预分布无法仅从观测分布 $P(a_t\mid o_t)$ 复原。本文借鉴人类 affordance perception 与 cognitive map 理论，提出"map-then-act"：先勘探建图、再带图执行。

② 核心方法详解

① 任务形式化：把策略拆为 mapping + acting

标准 act-during-think：

$$\pi_\theta(e\mid u)=\prod_{t=1}^n\pi_\theta(a_t\mid u, a_1, o_1, \ldots, o_{t-1})$$

受限于观测分布 $P(a_t\mid o_t)$。MAP 改为：

$$M\sim\pi_\theta(M\mid u, \tau_{\text{exp}}),\quad \pi_\theta(e\mid u, M)=\prod_t\pi_\theta(a_t\mid u, M, h_t)$$

$\tau_{\text{exp}}$ 是 $do(\text{explore})$ 干预产生的探索轨迹，$M$ 是从中蒸馏出的 cognitive map（包含空间布局、对象-动作 affordances）。这把 agent 决策从"相关性 pattern matching"提升到"因果性 knowledge-driven reasoning"。

② Stage 1：Cross-Task Global Exploration（跨任务全局先验 $K_g$）

每个环境只执行一次。Agent 先以 Focus Analyzer 角色从 $\mathcal D_{\text{env}}$ 与少量手工轨迹 $F_{\text{manual}}$ 中提炼 Focus Points（如"测试 action syntax 严格性"），再以 Explorer 角色在训练任务集上执行探索。用状态新颖度内在奖励 $r(o_t)=1/N(o_t)$ 鼓励到达稀有状态。轨迹经 Reflection / Knowledge Distillation 蒸馏为持久化全局先验 $K_g$（含 action syntax、interaction rules、error patterns）。

③ Stage 2：Task-Specific Cognitive Mapping

给定具体任务指令 $u$，agent 在 $K_g$ 引导下用 RPP（Role-Purpose-Priority）prompt 协议执行轻量探索，构造任务特定 cognitive map $M_t$。$M_t$ 包含三类条目：(a) Object-Action Affordances（"turn on lamp → lamp on"）；(b) Spatial Layouts（kitchen contains [stove, fridge, sink]）；(c) Game Rules（ARC-AGI-3 的 latent rules）。增量更新：知识增益 $\Delta|K_t|=K_t-K_{t-1}$ 作为停止准则。

④ Stage 3：Knowledge-Augmented Execution + Cognitive Fine-tuning

执行阶段策略基于双层知识：

$$a_t\sim\pi_\theta(a_t\mid u, M_t, K_g, h_t)$$

对应内化阶段：作者构造 MAP-2K 数据集（2000 条 map-then-act 轨迹），用 ms-swift 在 8 张 H800 上 SFT Qwen3-4B-Thinking → MAP-4B（lr=1e-5, 3 epochs）。关键发现：训练 map-then-act 轨迹比训练同样数量的 expert execution traces 效果更好——"理解环境比模仿专家行动更基础"。

③ 深度案例：ALFWorld "用台灯检查碗"任务

任务：Examine the bowl with the desklamp.
  1. ReAct 路径：观察当前房间没有 bowl 也没有 desklamp →探索每个房间→在 desk 上找到 bowl→在 dresser 上找到 desklamp →尝试 take desklamp（失败：lamp 是固定家具）→尝试 use desklamp on bowl（失败：需要 proximity）...典型 trial-and-error 死循环；
  2. MAP 路径（带 cognitive map）：mapping 阶段已提取关键 affordance：desklamp: turn_on/off, NOT takeable; needs proximity to use，bowl: takeable, on desk。
    execution 阶段：
    • Think: According to the map, the bowl is on desk, the desklamp is on dresser. I decide to get the desk lamp first.
    • Action: Move to dresser. Take desklamp → "Nothing happens"（map 中已注：not takeable）；
    • Think: The lamp can't be taken. So I'll change my approach: I'll bring the bowl over first.
    • Action: Move to desk. Take bowl. Go to dresser. Examine bowl with desklamp. ✓
  3. 不带 map：Qwen3-4B 完成率 31.8%；带 MAP（同模型）→ 61.1% (+29.3pt 翻倍)；GPT-4o-mini 21.9%→46.2% (+24.3pt)。

⑥ 结论与启发

MAP 把 LLM agent 的 paradigm 从"行动中理解"转为"理解后行动"，这是第一性原理层面的范式跃迁。其因果论证（Pearl do-calculus 区分观测分布与干预分布）严谨而有说服力。对工业 agent 团队（包括美团这种业务多场景 agent 部署）的启发是：对每个新业务环境先做一轮"沙盒探索 + 先验蒸馏"，将得到的 $K_g$ + 任务级 cognitive map 嵌入 agent 系统提示，能在小模型上取得近 GPT-4o 级的成功率。这种范式特别适合：(a) 浏览器 agent / GUI agent 的元素探索阶段；(b) 代码 agent 进入新 codebase 时的"先 grok 再 PR"流程；(c) 任何 ARC-AGI-3 类 zero-knowledge 环境。USTC + 美团署名，应是 NeurIPS/ICLR 强投稿。

Medical-1 · RealICU: Do LLM Agents Understand Long-Context ICU Data? A Hindsight-Annotated Benchmark

📅 2026-05-13 🏛 Technical University of Munich, LMU Munich, Imperial College London arXiv:2605.13542
Medical ICU Benchmark 长上下文

① 背景与动机

ICU 是医院最信息密集的场景之一：化验、生命体征、药物、护理记录、影像报告每小时刷新。AI co-pilot 价值巨大，但现有 ICU benchmark 的根本缺陷是把"医生记录的实际操作"当作 ground truth——这忽略了一个临床现实：医生在床旁的决策是基于"当时不完整的信息"做出的，真正最优的操作往往要回过头来看才清楚。把 LLM 强行学医生的实际操作就是在"奖励模仿不完美的人类行为"。本文提出 RealICU：一个由资深 ICU 医生带"事后视角"（hindsight） 重新标注的 benchmark，并伴随一个新 agent 系统 ICU-Evo，作为长程 ICU 决策支持的全新测试床。

② 核心方法详解

① 四项临床任务（与 30 位医生协同设计）

作者咨询 30+ 位 board-certified ICU 临床医生，归纳出 ICU co-pilot 的四类核心能力：

  • Patient Status：评估病情整体趋势（改善 / 稳定 / 恶化），细分到 hemodynamics / oxygenation / inflammation / organ function；
  • Acute Problems：识别当前主要急性问题（如 INFECTION、SEPTIC SHOCK、LACTIC ACIDOSIS）；
  • Action Recommendation：给出推荐操作清单（NOREPINEPHRINE 0.15μg/kg/min、LR 500mL 等）；
  • Red Flags：警示不应做的操作（aggressive diuresis 在脓毒症时会恶化休克；β-blocker 会抑制代偿）；评估指标 HRR@5（Hazardous Recommendation Ratio）越低越好。
② Hindsight-Annotated 数据：Gold + Scale

RealICU-Gold：基于 MIMIC-IV 选择代表性 ICU 入住，资深医生看完整个住院全程后，对每个 6 小时窗口做四项任务的标注。这与"床旁实时决策"形成对照：医生标注时拥有 hindsight 信息，因此标签代表"理想最优操作"而非"实际操作"。

RealICU-Scale：手工标注昂贵稀疏。引入 Oracle（一个在同样 hindsight 条件下工作的 LLM 评估器），将其与医生共识 F1 验证为 ≥0.895，再用它密集标注 11,862 个窗口，得到 Scale 子集用于大规模评估。这是"用 LLM 协助 LLM evaluator"的可信构造方法。

③ ICU-Evo：结构化记忆 agent 系统

作为参考 agent，作者提出 ICU-Evo，含三个子 agent + 一个预测器：

  • Working Memory $M^{\text{work}}_t$ 与 Trend Memory $M^{\text{trend}}_t$：滚动维护短期 / 长期生理趋势；
  • Assessment Agent $A_{\text{assess}}$：每 $k_a$ 个窗口生成 trajectory summary $z_t$ 与 critical events $e_t$：

$$(z_t, e_t)=A_{\text{assess}}\!\big(M^{\text{work}}_{t-k_a:t},\,M^{\text{trend}}_{t-k_a:t}\big)$$

  • Insight Agent $A_{\text{insight}}$：每 $k_i$ 个窗口提出关于患者特异性 pattern 的假设 $h_t$（如"该患者对升压药反应弱于群体平均"），用支持证据 $e_s$ 与反证 $e_r$ 评分接受/拒绝：

$$M^{\text{insight}}_t=A_{\text{insight}}\!\big(M^{\text{insight}}_{t-1}, M^{\text{event}}_{t-k_i:t}\big),\;\text{accept }h_t\text{ if }s(h_t)>r(h_t)$$

  • Predictor $f^{(k)}$：在四项任务上 prompted LLM，基于完整 memory state 与静态患者上下文输出预测，与 agent 解耦，便于公平比较 backbone。
④ 评估协议

同 backbone 下三种 baseline 与 ICU-Evo 比较：(a) full-context（喂入全部历史 token）、(b) local-window（仅当前 6h 窗口）、(c) RAG（PubMedBERT 嵌入检索 top-5 窗口）。指标：Acc/F1（Patient Status）、Hit@5/Recall@5（Acute Problems / Action Rec）、HRR@5↓（Red Flags 越低越好）。

③ 深度案例：脓毒症患者的"床旁 vs 事后视角"标签差异

真实场景：MIMIC-IV 中一位多脏器功能不全的脓毒症患者，住 ICU 第 18-24 小时窗口。
  1. 床旁实际操作（被既往 benchmark 当 ground truth）：医生当时给了 etomidate 做插管诱导。原因：信息不全，未注意到该患者已经有肾上腺皮质功能不全风险；
  2. Hindsight 标签（RealICU 的）：从全程回看，etomidate 在脓毒症患者中加重肾上腺抑制，造成后续 6 小时 vasopressor 难以撤除。所以这是 Red Flag action；正确推荐应是 ketamine 或 propofol；
  3. 差异意义：在传统 benchmark 上，模型答 etomidate 会被打 ✓（模仿了真实医生）；但在 RealICU 上会被记入 HRR@5 增加分母（推荐了危险操作）。这从根本上改变了 LLM ICU 应用的安全评估范式；
  4. 当前 frontier 模型表现：Gemini-3.1-pro + ICU-Evo 仍有 HRR@5 = 0.300（30% 推荐里含至少一个 Red Flag），GPT-5.4 + ICU-Evo 更高至 0.473——当前 LLM 仍在 ICU 高风险场景持续推荐危险操作。

Medical-2 · COTCAgent: Preventive Consultation via Probabilistic Chain-of-Thought Completion

📅 2026-05-14 🏛 The University of Hong Kong, UESTC, The University of Sydney arXiv:2605.15016
Medical EHR CoT 差异诊断

① 背景与动机

纵向 EHR（labs、vitals、症状）是临床诊断的关键时序证据，但 LLM 在这上面有两个根本缺陷：(1) 缺乏精细统计推理——把"血肌酐 1 月内从 80 到 130 上升"这种需要回归才能确认的趋势直接幻觉成"急剧升高"，扭曲诊断；(2) 非均匀时间序列 + 标签稀缺——纯 LLM 难以捕获长程时间依赖。本文 COTCAgent 通过"统计代码 + 知识库匹配 + 概率 CoT 补全"三层架构，把 LLM 的幻觉风险隔离在"自然语言 paraphrase"这一最小角色，让真正的数值推理走在确定性管道里。

② 核心方法详解

① TSA 模块（Temporal-Statistics Adapter）：自然语言 → 统计代码

把临床查询 $Q$ 映射为可执行统计：

$$M:Q\to\Phi\to\Lambda\to C$$

$\Phi$ 是解析的语义结构，$\Lambda$ 是选择的统计模型，$C$ 是生成的可执行脚本。常用的两种模型：

趋势查询 → 混合效应模型：

$$y_{ij}=\beta_0+\beta_1 t_{ij}+u_i+\epsilon_{ij},\;u_i\sim\mathcal N(0,\sigma_u^2),\;\epsilon_{ij}\sim\mathcal N(0,\sigma_\epsilon^2)$$

急转查询 → 贝叶斯变点检测：

$$P(\tau\mid y)\propto\Big[\prod_{t=1}^\tau f_1(y_t)\Big]\Big[\prod_{t=\tau+1}^T f_2(y_t)\Big]P(\tau)$$

每个 TSA 输出是结构化 tuple (span, estimand, value, qual)，estimand 来自有限 schema（slope / change-point posterior / cohort z-score 等）；拟合失败自动降级并打 UNSTABLE/SPARSE 标记。这把 "LLM 说 p=0.03 却没似然函数"的常见幻觉模式从源头堵死。

② COTC 模块：IDF-Weighted Gibbs 能量打分

知识库覆盖 9,948 个内科疾病（心血管、呼吸、代谢、感染、神经五大类）的 Symptom-Trend-Disease 三元组。给定症状/趋势集合 $S_p$，对每个候选疾病 $d_i$ 计算 Gibbs 能量：

$$E(d_i\mid S_p)=-\sum_{s_j\in S_{d_i}\cap S_p}\log w^{\text{IDF}}_j+\log\phi(s_j, d_i)$$

其中 IDF 权重

$$w^{\text{IDF}}_j=\log\frac{|D|+1}{|\{d_i\in D:s_j\in S_{d_i}\}|+1}$$

常见症状（如发热、乏力）权重低；罕见特异指标（如 Koplik 斑）权重高，对应医学直觉中的"病理特异性"。$\phi(s_j, d_i)\in[0.5, 1]$ 是临床医生预设的 specificity 系数。最终 softmax 归一化得

$$\tilde P(d_i\mid S_p)=\frac{\exp(-E(d_i\mid S_p)/T)}{\sum_k\exp(-E(d_k\mid S_p)/T)}$$

过滤掉 $R_i<T$（Risk threshold）的疾病，保留有意义的候选。

③ Probabilistic CoT Completion：有上限的多轮澄清

当 $\max_i\tilde P(d_i\mid S_p)$ 仍低于阈值 $\theta$（候选未足够区分）时，触发 completion 循环（Algorithm 1）：

  1. TopGaps(KB, $S_p$, t)：找出当前最有诊断价值的 $k$ 个 unknown 症状/化验，用 entropy 引导选择"最能压缩候选集合"的查询；
  2. 用模板 + LLM constrained paraphrase 渲染为简洁问题 $q$；
  3. 解析用户回答 $a$ 为离散 evidence，加入 $S_p$，重新计算能量与概率；
  4. 当 $\max_i\tilde P\ge\theta$、entropy 低于阈值或达到 round cap $R_{\max}$ 时停止，返回排名 + CoT 日志 $\mathcal L$。

这是受限 Bayesian information gathering——避免 LLM 自由生成无意义问题，把对话生成限制在 KB 已知 gap 上。

④ 三个随机源严格隔离

架构不变量：(i) 可执行统计消耗原始时序数据，输出有 likelihood 的回归结果；(ii) 离散匹配仅消耗已 tokenize 的 predicate（如 severe-trend）；(iii) 自然语言只用于"paraphrase 预定的 gap 问题"和"解析用户回答"。当 $M$ 抛错，COTC 收到的是显式 uncertainty flag 而非幻觉数字；当 KB miss，Algorithm 1 走 low-coverage 分支而非编造连边。失败可读、回路清晰是该设计的核心承诺。

③ 深度案例：糖尿病肾病前驱期的"非问诊式"识别

场景：一位 58 岁男性，最近两次体检数据进入系统：血肌酐 80→130 μmol/L、尿素氮 5.2→8.1、尿蛋白 +、空腹血糖 10.8。
  1. TSA 阶段：执行混合效应回归，输出 (blood_creatinine, slope=+12.5/month, p<0.01, qual=CONFIRMED)；变点检测在尿蛋白时间序列上 detect change at week 8；
  2. COTC 阶段：在 KB 中匹配 "rising_creatinine, persistent_proteinuria, hyperglycemia" → 候选 top-3：糖尿病肾病 (Risk 95%)、原发性肾小球肾炎 (Risk 72%)、高血压肾病 (Risk 55%)。最大概率达到 $\theta=0.9$，无需 completion；
  3. 对比：单纯让 LLM "看一眼数字"判断，由于无统计回归可能错把"80→130"当作正常波动；让 LLM 通过 RAG 检索也容易因为忽视 IDF 权重把"持续性蛋白尿"这种特异指标按 hot keyword 频率混入排序；
  4. 性能：在自建 9948 疾病数据集上，COTCAgent + Baichuan-M2 达到 Top-1 acc 90.47%、Top-2 acc 92.18%、F1 86.74%，超过 KARE + GPT-4o (Top-1 83.76%)、DirPred + GPT-4o (88.27%)、Gemini 2.5 Pro 单 pass (86.08%) 等所有 baseline。

Multimodal-1 · Video2GUI: Synthesizing 12M GUI Interaction Trajectories from Internet Videos

📅 2026-05-14 🏛 Peking University, Xiaomi LLM-Core, Renmin University of China arXiv:2605.14747
Multimodal GUI Agent 数据合成 VLM

① 背景与动机

GUI agent（浏览器、桌面、移动）是 multimodal LLM 重要落地方向，但泛化受限于训练数据规模与多样性。已有数据集（MIND2WEB 137 个网站、AndroidControl 833 个应用、AITW 357 个 mobile 环境）规模有限且依赖人工标注，难覆盖真实世界 1500+ 应用场景。互联网上有海量教学视频（YouTube 上 5 亿+ 视频含大量 software tutorial），但要把它们转化为有 grounding 的 GUI 操作轨迹有两大难关：(1) 海量视频里如何高效筛选高质量 GUI tutorial？(2) 如何把视频中的鼠标点击 / 键盘输入精确映射到屏幕坐标？本文 Video2GUI 给出端到端管线，最终发布 WildGUI：12M 轨迹 / 124M 图像 / 1500+ 应用，迄今最大开源 GUI 预训练数据集。

② 核心方法详解

① 阶段 1：粗到细的视频筛选（500M → 4.16M）

Meta Info Filtering：先用 video metadata（标题、描述、关键词）做廉价文本筛选。从 500M YouTube 视频元数据出发，作者用 DeepSeek-V3 标注 10K 样本得到 GUI-relevant 训练集，fine-tune Qwen2.5-7B 分类头做大规模推断；得到 ~20M 候选。

Video Quality Scoring：对内容做细粒度评估，用 Qwen2.5-Omni（多模态评分模型）从三维度打分：(a) Topic Relevance（是否真在教 GUI 操作）、(b) Instruction Clarity（讲解是否清晰连贯）、(c) Screen Recording Quality（视觉清晰、完整、稳定）。Gemini 3 Pro 在 200 小时视频上做种子标注，fine-tune 出 omnimodal scorer 应用到全部 20M 视频，最终保留 4.16M 高质量视频 / 30 万小时内容。

② 阶段 2：Trajectory Extraction（视频 → 结构化轨迹）

从原始视频 $V$ 抽取指令-轨迹对 $\{(u, e)^{(i)}\}$，每条 $e=\{(a_t, o_t, r_t)\}$ 含 action 类型、 时间戳、参数、推理 rationale。流程：

  • 用 ASR + caption 模型抽取 narration 与时间戳；
  • 用 VLM 在视频帧上识别 high-level task instruction（"how to add a chart in Excel"）；
  • 切分为 atomic action（click / type / drag）+ timestamps + 视觉 rationale；
  • 每条 action 关联 3 个候选关键帧（action 前 / 中 / 后）。
③ 阶段 3：Action Space Grounding（视频 → 屏幕坐标）

这是最难的一步。VideoAgentTrek 等已有方法依赖低层视觉 cue（如鼠标光标位置），噪声大且对无光标 mobile 视频失效。Video2GUI 的方案是：对每个 action，用 VLM 在 3 个候选帧中尝试 spatial grounding（输入"用户在点击 'File' 菜单"，输出 bbox）。选第一个能产出有效 grounding 的帧作为最终监督信号。手工核验 200 个采样 action 显示该策略 ≥95% 准确率。这是把 VLM 的 grounding 能力用作 ground-truth-free 标注器，规避了"鼠标可见性"假设。

④ 阶段 4：两阶段 Agent Training Strategy

Stage 1：在 WildGUI 12M 轨迹上做 continual pre-training（CPT），让模型获得 GUI 元素识别 + action grounding 的通用能力；

Stage 2：在 high-quality 任务特定 supervised post-training（SFT）数据集上做精调，学习严格的 action 输出格式与多步 planning。CPT (Stage 1) 是 Video2GUI 数据的核心受益场景。

③ 深度案例：Mimo-VL-7B + WildGUI 在 GUI Grounding 上的飞跃

场景：开源 Mimo-VL-7B（Xiaomi 自研）在 ScreenSpot-Pro 与 OSWorld-G 两大 GUI grounding bench 上的表现。
  1. 无 WildGUI 预训练：Mimo-VL-7B 在 ScreenSpot-Pro Avg 41.2%，在 OSWorld-G Avg 54.7%；
  2. + WildGUI Stage 1 CPT：ScreenSpot-Pro Avg 56.9% (+15.7pt 绝对提升)；OSWorld-G Avg 67.6% (+12.9pt)；
  3. 这一 +15.7pt 跃迁已超过参数更大的 Qwen3-VL-32B (54.9%)、UI-TARS-72B (38.1%)，仅次于闭源 Seed1.5-VL (60.9%)；
  4. 更激进对比：Qwen2.5-VL-7B 原始 ScreenSpot-Pro 仅 26.8%（一个相对早期 VLM），+ WildGUI 后跃升到 41.9%（+15.1pt），OSWorld-G 从 27.3% 飙到 53.7% (+26.4pt)。这证明 Video2GUI 数据对"GUI 弱底座"的提升甚至更显著；
  5. Online benchmark：在 AndroidWorld 真实环境，Mimo-VL-7B 基础 16.4% → Stage 2 only 23.3% → Stage 1+2 完整 Pipeline 31.9% (近 2× 基础)；OSWorld 也提升至 12.3%。证明 offline 视频蒸馏数据能泛化到 online open-ended 环境。

⑤ 消融与讨论

⑥ 结论与启发

Video2GUI 是 GUI agent 领域的"数据规模法则"工程实证：用极少手工标注 + 三个 LLM/VLM 模型（DeepSeek-V3 标分类、Gemini 3 Pro 标质量、VLM 做 grounding）就能从 5 亿原始视频蒸出 12M 高质量轨迹。其哲学价值在于"互联网视频本身就是 GUI agent 训练语料"——只要有合适的筛选与 grounding 管线。对赵政昀博士在 IPO ECM 投行场景下的启发是：当前 ECM 工作中也有大量 SOP 视频（招股书审核流程、Excel 模型搭建录屏），同样的 video → trajectory 蒸馏管线可生成定制 ECM agent 的训练数据。这是 PKU + 小米 LLM-Core 的高价值合作产出，论文同时发布数据集与代码（项目页 weiminxiong.github.io/Video2GUI），对开源 GUI agent 社区是直接利好。
