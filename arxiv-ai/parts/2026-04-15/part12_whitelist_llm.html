🏢 Three Roles, One Model: Role Orchestration at Inference Time to Close the Performance Gap Between Small and Large Agents
📅 2026-04-12🏛 Qwen Team, Alibaba
LLMIndustry LabAgent
📌 背景与动机

小型 LLM（7B-14B）在 agent 任务中与大型模型（70B+/API）的性能差距巨大。常见做法是直接用小模型作为 agent policy，但这忽略了 agent 任务的多维度需求。本文来自 Qwen 团队，提出一种推理时角色编排策略：用同一个小模型在推理时扮演三种不同角色，协同完成 agent 任务。

🔬 核心方法详解
① 三角色设计
  • Tier 1 — 主 Agent（π_agent）：负责执行动作，使用标准 agent 提示（观察→思考→行动循环）
  • Tier 2 — 摘要模型（π_sum）：将冗长的环境观察压缩为简洁摘要，降低 agent 的上下文负担
  • Tier 3 — 评审模型（π_critic）：在行动执行前评估合理性，检测潜在错误并建议修正
三个角色共享同一套模型权重，通过不同的系统提示切换角色。
② 编排调度器
推理时的执行流程为：环境观察 → π_sum 摘要 → π_agent 生成动作 → π_critic 评审 → [通过则执行 | 不通过则反馈给 π_agent 重生成]。调度器根据任务复杂度决定是否跳过某些角色（简单任务可跳过 critic）。
③ 角色特化微调
虽然使用同一模型，但通过多任务微调增强各角色能力：训练数据混合 agent 轨迹(60%)、摘要对(20%)、critique 对(20%)。使用特殊 token 区分角色：[AGENT]、[SUMMARIZER]、[CRITIC]。
④ 自适应上下文管理
π_sum 的摘要能力使得有效上下文长度扩展 3-5x。当对话历史超过模型窗口的 70% 时，自动触发摘要压缩，保留关键信息。
📋 深度案例
AppWorld 任务

任务：「在 Spotify 中找到用户最近播放的专辑，并在 Twitter 上分享」
π_agent 直接执行：调用 spotify.get_recent → 获取专辑名 → 调用 twitter.post → 但遗漏了专辑封面URL → 分享不完整 ❌
三角色编排：
1. π_sum: 将 Spotify API 返回的大量元数据压缩为关键信息（名称+艺术家+封面URL）
2. π_agent: 基于摘要生成分享动作（包含封面URL）
3. π_critic: "检查：分享内容包含专辑名✓、艺术家✓、封面✓ → 通过"
→ 完整分享 ✅
📊 实验结果

Qwen2.5-7B (直接)7.1%12.3%9.7%
GPT-4o (直接)31.2%28.7%30.0%

三角色编排将 Qwen2.5-7B 的 agent 性能从 9.7% 提升至 24.0%（+147%），接近 GPT-4o (30%) 的 80% 水平。

🔍 消融与讨论

三个角色分别贡献：summarizer +6.2%, critic +4.8%, 二者协同 +3.3%。critic 在需要多步骤规划的任务上效果最显著。角色特化微调比纯提示工程高约 5 分。局限在于三角色串行执行增加了约 2.5x 的推理成本。

💡 结论与启发

Qwen 团队的这项工作展示了"一个模型扮演多角色"的优雅方案，通过推理时编排而非增大模型来缩小大小模型差距。这对小模型的 agent 部署有重要实用价值。

🏢 Do LLMs Build Spatial World Models? Evidence from Grid-World Maze Tasks
📅 2026-04-11🏛 University of Notre Dame, Columbia University
LLMIndustry LabWorld Model
📌 背景与动机

LLM 是否能构建空间世界模型一直是争论焦点。Othello-GPT 等工作声称 Transformer 能学习棋盘表示，但这些结论是否推广到更一般的空间推理？本文通过网格迷宫任务系统性地检验 LLM 的空间建模能力，用线性探针检测模型内部是否存在空间表示。

🔬 核心方法详解
① 受控迷宫生成
构建参数化网格迷宫（5×5 到 15×15），文本描述为方向序列（如"N N E E S"）。控制变量：迷宫大小、路径长度、分支复杂度。每个迷宫有唯一最短路径解。
② 线性探针分析
在模型各层隐状态上训练线性分类器，预测 agent 当前的 (x,y) 坐标：$$\hat{p} = \text{softmax}(\mathbf{W}_{\text{probe}} \cdot \mathbf{h}_l + \mathbf{b})$$ 高准确率表明该层编码了空间位置信息。
③ 因果干预实验
通过 activation patching 验证探测到的表示是否因果地影响输出：将正确位置的 activation 替换为错误位置的，观察模型行为是否对应改变。
④ 规模与架构对比
对比 DeepSeek-7B/67B、LLaMA-3-8B/70B、GPT-4 等模型，分析空间建模能力与模型规模的关系。
📋 深度案例
7×7 迷宫空间探针结果

DeepSeek-67B：中间层(第20-30层)位置探针准确率 78.3%，因果干预效果 67.4% → 确实构建了空间表示
LLaMA-3-8B：探针准确率仅 34.2%，因果干预效果 21.1% → 几乎无空间表示
GPT-4（黑盒行为测试）：迷宫求解正确率 62.4% → 行为上显示空间理解
空间建模能力与模型规模呈强正相关，但即使最大模型也无法处理 15×15 迷宫

🔍 消融与讨论

关键发现：(1) 空间表示集中在模型中间层（第40-60%层），而非早期或后期；(2) 因果干预证实这些表示是功能性的，而非旁路信号；(3) 所有模型在 10×10+ 迷宫上急剧退化，暗示空间建模能力有上限。DeepSeek 模型在同参数量级下空间能力优于 LLaMA，可能与预训练数据中的代码/数学比例有关。

💡 结论与启发

LLM 确实能构建某种程度的空间世界模型，但能力有限且高度依赖规模。这对于理解 LLM 的"理解"本质提供了重要实证依据——它们不是纯粹的统计鹦鹉，但也远非完美的世界建模器。
