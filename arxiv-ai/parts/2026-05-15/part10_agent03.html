Agent-3 · MAP: A Map-then-Act Paradigm for Long-Horizon Interactive Agent Reasoning

📅 2026-05-13 🏛 USTC, Meituan, CASIA arXiv:2605.13037
Agent 长周期 认知地图 ARC-AGI-3

① 背景与动机

ReAct/CoT 类 agent 范式遵循"act-during-think"：模型只在执行时才被动获得环境感知。这造成Delayed Environmental Perception（延迟环境感知）：agent 必须通过 trial-and-error 推断空间布局、动作语法和约束，掉进低效失败循环——作者称之为 Epistemic Bottleneck。这是一个范式问题，不能靠扩大 reasoning 能力解决（ARC-AGI-3 上 Claude 4.6 也几乎为零分）。从 Pearl do-calculus 角度看，根因是 $P(o_{t+1}\mid do(a_t))$ 干预分布无法仅从观测分布 $P(a_t\mid o_t)$ 复原。本文借鉴人类 affordance perception 与 cognitive map 理论，提出"map-then-act"：先勘探建图、再带图执行。

② 核心方法详解

① 任务形式化：把策略拆为 mapping + acting

标准 act-during-think：

$$\pi_\theta(e\mid u)=\prod_{t=1}^n\pi_\theta(a_t\mid u, a_1, o_1, \ldots, o_{t-1})$$

受限于观测分布 $P(a_t\mid o_t)$。MAP 改为：

$$M\sim\pi_\theta(M\mid u, \tau_{\text{exp}}),\quad \pi_\theta(e\mid u, M)=\prod_t\pi_\theta(a_t\mid u, M, h_t)$$

$\tau_{\text{exp}}$ 是 $do(\text{explore})$ 干预产生的探索轨迹，$M$ 是从中蒸馏出的 cognitive map（包含空间布局、对象-动作 affordances）。这把 agent 决策从"相关性 pattern matching"提升到"因果性 knowledge-driven reasoning"。

② Stage 1：Cross-Task Global Exploration（跨任务全局先验 $K_g$）

每个环境只执行一次。Agent 先以 Focus Analyzer 角色从 $\mathcal D_{\text{env}}$ 与少量手工轨迹 $F_{\text{manual}}$ 中提炼 Focus Points（如"测试 action syntax 严格性"），再以 Explorer 角色在训练任务集上执行探索。用状态新颖度内在奖励 $r(o_t)=1/N(o_t)$ 鼓励到达稀有状态。轨迹经 Reflection / Knowledge Distillation 蒸馏为持久化全局先验 $K_g$（含 action syntax、interaction rules、error patterns）。

③ Stage 2：Task-Specific Cognitive Mapping

给定具体任务指令 $u$，agent 在 $K_g$ 引导下用 RPP（Role-Purpose-Priority）prompt 协议执行轻量探索，构造任务特定 cognitive map $M_t$。$M_t$ 包含三类条目：(a) Object-Action Affordances（"turn on lamp → lamp on"）；(b) Spatial Layouts（kitchen contains [stove, fridge, sink]）；(c) Game Rules（ARC-AGI-3 的 latent rules）。增量更新：知识增益 $\Delta|K_t|=K_t-K_{t-1}$ 作为停止准则。

④ Stage 3：Knowledge-Augmented Execution + Cognitive Fine-tuning

执行阶段策略基于双层知识：

$$a_t\sim\pi_\theta(a_t\mid u, M_t, K_g, h_t)$$

对应内化阶段：作者构造 MAP-2K 数据集（2000 条 map-then-act 轨迹），用 ms-swift 在 8 张 H800 上 SFT Qwen3-4B-Thinking → MAP-4B（lr=1e-5, 3 epochs）。关键发现：训练 map-then-act 轨迹比训练同样数量的 expert execution traces 效果更好——"理解环境比模仿专家行动更基础"。

③ 深度案例：ALFWorld "用台灯检查碗"任务

任务：Examine the bowl with the desklamp.
  1. ReAct 路径：观察当前房间没有 bowl 也没有 desklamp →探索每个房间→在 desk 上找到 bowl→在 dresser 上找到 desklamp →尝试 take desklamp（失败：lamp 是固定家具）→尝试 use desklamp on bowl（失败：需要 proximity）...典型 trial-and-error 死循环；
  2. MAP 路径（带 cognitive map）：mapping 阶段已提取关键 affordance：desklamp: turn_on/off, NOT takeable; needs proximity to use，bowl: takeable, on desk。
    execution 阶段：
    • Think: According to the map, the bowl is on desk, the desklamp is on dresser. I decide to get the desk lamp first.
    • Action: Move to dresser. Take desklamp → "Nothing happens"（map 中已注：not takeable）；
    • Think: The lamp can't be taken. So I'll change my approach: I'll bring the bowl over first.
    • Action: Move to desk. Take bowl. Go to dresser. Examine bowl with desklamp. ✓
  3. 不带 map：Qwen3-4B 完成率 31.8%；带 MAP（同模型）→ 61.1% (+29.3pt 翻倍)；GPT-4o-mini 21.9%→46.2% (+24.3pt)。

⑤ 消融与讨论

消融分析：(1) CoMAP（map-and-act 同时进行）vs MAP（先 map 后 act）：CoMAP 比 ReAct 平均 +5pt，但 MAP 比 CoMAP 再 +9-30pt——证明"先后分离"是关键而非仅仅"加 map"；(2) 训练 expert execution（ACT-4B） vs 训练 MAP 轨迹（MAP-4B）：相同 token 预算下，MAP-4B 在 OOD 任务上显著强于 ACT-4B，验证"环境理解 > 行动模仿"；(3) 移除 $K_g$（仅有 task-specific map）：跨环境泛化能力丧失，新环境性能降回 ReAct 基线水平。局限：(a) Stage 1 需要 manual trajectory 启动（非完全 zero-shot）；(b) cognitive map 当前是文本结构化，对极复杂动态环境（如 Minecraft 大世界）可能不够紧凑；(c) mapping 阶段消耗的 token 预算与 acting 相当，预算敏感场景需 trade-off。

⑥ 结论与启发

MAP 把 LLM agent 的 paradigm 从"行动中理解"转为"理解后行动"，这是第一性原理层面的范式跃迁。其因果论证（Pearl do-calculus 区分观测分布与干预分布）严谨而有说服力。对工业 agent 团队（包括美团这种业务多场景 agent 部署）的启发是：对每个新业务环境先做一轮"沙盒探索 + 先验蒸馏"，将得到的 $K_g$ + 任务级 cognitive map 嵌入 agent 系统提示，能在小模型上取得近 GPT-4o 级的成功率。这种范式特别适合：(a) 浏览器 agent / GUI agent 的元素探索阶段；(b) 代码 agent 进入新 codebase 时的"先 grok 再 PR"流程；(c) 任何 ARC-AGI-3 类 zero-knowledge 环境。USTC + 美团署名，应是 NeurIPS/ICLR 强投稿。
