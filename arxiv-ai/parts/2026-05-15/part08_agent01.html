Agent-1 · Self-Pruned KV Attention: Learning When to Write by Predicting Future Utility

📅 2026-05-13 🏛 Meta FAIR, CentraleSupélec arXiv:2605.14037
Agent 长上下文 KV Cache Memory

① 背景与动机

在现代 test-time compute 与 agentic 推断范式下，LLM 一次会话可能产生 100K+ token 的轨迹（多轮工具调用、长 reasoning、retrieval-augmented 上下文）。瓶颈不再是 compute，而是 KV cache 的内存占用与读带宽——每个新 token 都要读完整 KV，长序列导致 GPU memory traffic 爆炸。已有方案（GQA / MLA 共享 KV、QUEST / DeepSeek Sparse Attention 读时稀疏）要么写时仍存全量，要么需特定查询结构。本文 SP-KV 是另一条路：让模型自己在写入时就预测每个 KV 是否值得长期保留，硬剪枝、动态压缩比，对 agent 长 trajectory 友好。

② 核心方法详解

① Key-wise Utility 预测

在 layer $l$、key-head $k$、token 位置 $s$ 上，用一个 2-layer MLP $f^{l,k}_\theta$ 从隐藏状态 $h^l_s$ 预测 utility 分数：

$$u^{l,k}_s=\sigma\big(f^{l,k}_\theta(h^l_s)\big)\in(0,1)$$

推断时通过阈值 $\tau$ 二值化得到门控 $z_s=\mathbb 1[u_s\ge\tau]$：$z_s=0$ 表示这个 KV 不进入持久 cache（除非在 sliding window 内）。

② Sliding-Window + Gated Global Attention

始终保留最近 $w=128$ 个 token 的局部因果窗口（保护短程依赖），更早的 token 仅当 $z_s=1$ 时才进入全局 attention：

$$g_{t,s}=\begin{cases}0, & z_s=1\;\lor\;\mathbb 1_{\text{win}}(t,s) \\ -\infty, & \text{otherwise}\end{cases},\;B_{t,s}=M_{\text{causal}}(t,s)+g_{t,s}$$

$$o_t=\sum_s\mathrm{softmax}\!\Big(\frac{\langle q_t, k_s\rangle}{\sqrt d}+B_{t,s}\Big)v_s$$

③ Phase 1：Soft Gated Training（保留可微性）

训练时 thresholding 不可微。SP-KV 把 $\log u_s$ 直接加进 attention mask bias：$\tilde g_s=\log u_s$。$u_s\to 0$ 等价于 $-\infty$ mask（被屏蔽），$u_s\to 1$ 等价于 mask 0（完全开放）。所有 utility gate 在初始化时 logit 拉到大正数（默认全开放），用 vanilla next-token prediction loss 一统训练，无任何稀疏化辅助 loss——稀疏化作为 NTP loss 自然产物涌现。

④ Phase 2：Threshold-Aware Hard Gating + 块跳跃 kernel

在 cosine schedule 的最后 25%，冻结 utility predictor 权重，把 gate 用阈值 $\tau$ 二值化。为避免突变带来的优化不稳，500 步线性 anneal：

$$\tilde u=(1-\alpha)u+\alpha\cdot\mathbb 1[u\ge\tau],\;\alpha:0\to 1$$

训练-推断对齐后，作者修改 FlashAttention-3 加入块跳跃优化：以 64-token 粒度预计算稀疏 mask，对全零 KV 块直接 skip TMA load 与 MMA 计算。结果：1.6B-8.1B 模型上 Phase 2 ($\tau=0.7$) wall-clock throughput 达到 full-attention baseline 的 82-90%，但 KV cache 大小压缩 3-10×。

③ 深度案例：Agent 长 trajectory 上的动态压缩

场景：Llama-style 8.1B 模型处理一条 32K token 的 agent trajectory（含 10 次工具调用、多次 retrieval 注入）。
  1. Full Attention baseline：32K × 32 layers × 8 KV heads × 128 dim × 2 (K+V) ≈ 2.1 GB KV cache，每生成 1 个新 token 都要读完整 cache，bandwidth bound；
  2. SP-KV $\tau=0.7$：模型自己学到——retrieved 段落和工具结果中只有关键摘要 token（如 "result: 42", "error: timeout"）被打高 utility，中间填充 token（"the next thing to do is..."）被打低 utility 自然剪掉；最终 keep 比例在 $\rho\approx 0.10\sim 0.30$ 之间动态变化（越长越压缩），cache 压缩约 5-7×；
  3. 性能：在 NIAH（多针检索）、RULER 长上下文 QA 上对比 full-attention baseline 几乎无下降（<1pt 差异），但 decoding throughput 提升 ~2×；
  4. 意外发现：SP-KV 学到的 head-level utility pattern 揭示了"有些 attention head 天生就是 local，有些天生就是 global"——把高 average utility 的 head 设为 global、其他设为 local，得到的混合架构在同 KV 预算下超过传统 interleaved global-local layout。

⑤ 消融与讨论

消融分析：(1) 移除 sliding window：模型完全靠 utility predictor 的能力，短程依赖被破坏，PPL 大幅劣化；(2) 不做 Phase 2 hard gating：训练-推断 mismatch 严重，推断时强制阈值会带来 1-2pt 性能损失；(3) 预测粒度 token-level vs head-level vs layer-level：token-level（per-key）效果最好但最贵，head-level 是性能/复杂度最佳折衷；(4) 从随机初始化训练：可行但比 continued-pretraining 收敛慢约 1.5×。局限：(a) 当前方案仅对 standard MHA / GQA 适用，对 MLA 等结构需要重做；(b) Phase 2 的块跳跃 kernel 是 Hopper GPU 专属优化，老 GPU 性能优势会缩水；(c) utility predictor 增加了少量参数（每层 2 层 MLP），约 +1% 总参数。

⑥ 结论与启发

SP-KV 把 KV cache 优化从"读时稀疏"推进到"写时剪枝"——这是 long-context agent 系统的根本不同：写时压缩才能彻底解放 GPU 内存。论文的另一个隐藏价值是用 utility 分布作为"架构探针"，自动发现哪些 attention head 该 global / 哪些该 local，从而设计出更优的 hybrid attention 拓扑。Meta FAIR 的这项工作直接对标 agentic test-time-compute 趋势：在 Claude Code 类长 agent trajectory 中，5× KV cache 压缩意味着同硬件下能 hold 5× 更长的会话或 5× 并发。开源工业团队部署长 agent 推断时应将 SP-KV 列入考虑清单，特别是 longer reasoning loops（IMO solver、SWE-bench 修复任务）这种 trajectory 自然分层的场景。
