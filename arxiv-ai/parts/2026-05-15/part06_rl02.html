RL-2 · Semantic-Reward RL Enables Low-Resource Language Expansion without Alignment Tax

📅 2026-05-14 🏛 Minzu University of China, Ant Group, Shanghai Jiao Tong University arXiv:2605.14366
RL GRPO 语义奖励 低资源语种

① 背景与动机

把 LLM 扩展到藏语、蒙语等低资源语种几乎总要付出"对齐税"——SFT 在窄分布数据上做 token-level 似然最大化，把基础能力（中文阅读理解 CMRC、英文常识等）严重侵蚀。论文先量化了这一现象：用 strong SFT 把 Qwen3-4B 适配到藏文翻译后，CMRC 性能从 base 的 ~85% 跌到 41.82% F1，下游中文能力近乎崩塌。问题：能否在不破坏原有多语种能力的前提下学到低资源语种？作者主张把 alignment 重新表述为语义空间一致性，让模型用自由的表达方式实现"等价含义"，自然减少对原参数空间的破坏。

② 核心方法详解

① 语义空间对齐的问题表述

放弃 token-level 似然 $\max\sum\log\pi(y_t\mid y_{"输出语义等价于参考即正确，无须形式逐字一致"。形式化为

$$\max_\theta\;\mathbb E_{x,y\sim\pi_\theta(\cdot|x)}[\,s(y, y^*)\,]\quad\text{s.t.}\;\mathrm{KL}(\pi_\theta\,\|\,\pi_{\text{ref}})\le\beta$$

$s(\cdot,\cdot)$ 是 sentence embedding 的余弦相似度，KL 约束保护预训练表征。

② Cold-Start + RL 的两阶段范式

Stage 1（Cold-Start SFT）：在仅 5K 平行语对上做 2 epoch 轻量 SFT，目的不是好分数，而是让模型能产出"目标 script"的非退化输出，作为 RL 的初始 policy $\pi_{\text{init}}$； Stage 2（GRPO 语义 RL）：从 $\pi_{\text{init}}$ 出发，在剩余 90K 数据上用 GRPO 训练 1 epoch。每个 prompt $x$ 采 $K$ 条 rollout $y^{(k)}$，按 group-relative advantage 更新 policy。

③ 语义奖励：阈值-缩放 + 语种一致硬约束

主奖励来自双语 sentence embedding $f(\cdot)$（基于 CINO，藏文增强的 XLM-R）：

$$s(y,y^*)=\cos(f(y), f(y^*))$$

但直接用余弦不稳定（早期 rollout 噪声大），引入 threshold-and-rescale shaping：

$$R_{\text{sim}}(y,y^*)=\begin{cases}0, & s\le\tau \\ \dfrac{s-\tau}{1-\tau}, & s>\tau\end{cases}$$

其中 $\tau$ 取 cold-start 后的最低充分性水平（实验 ~0.4-0.5），保证 RL 只 refine 已经能写出像样输出的样本。

语种一致性硬约束（防 reward hacking 用其他语言"骗"高 embedding 相似度）：

$$R_{\text{lang}}(y)=\begin{cases}1, & \text{output script consistent} \\ 0, & \text{otherwise}\end{cases}$$

用 Unicode 范围 + 正则做规则检查。最终奖励：

$$R(y, y^*)=\lambda_{\text{sim}}\cdot R_{\text{sim}}+\lambda_{\text{lang}}\cdot R_{\text{lang}},\;\lambda_{\text{sim}}=1.5,\;\lambda_{\text{lang}}=1.0$$

④ GRPO + LoRA 训练

使用 GRPO（value-free PPO 变体）作为约束 RL 的具体实现，与基础 PPO 的 trust-region 限制等价但无须 critic。底座 Qwen3-4B + LoRA（r=64, α=128, dropout=0.05）应用到 self-attention 和 MLP 的所有线性投影。RL 的 KL 系数控制每步 policy drift，从根上保证对预训练知识的微扰。

③ 深度案例：Strong SFT vs. Semantic-Reward RL 的"对齐税"对比

设置：Qwen3-4B + LoRA，藏-中机器翻译。Strong SFT 与 Semantic RL 用同样的训练数据（5K + 90K），同样的 LoRA 配置。
  1. Strong SFT 结果：BLEU-4 = 0.6006，语义相似度 = 0.8282——表面看翻译质量很高；但 CMRC（中文阅读理解）F1 从 base 的 ~85% 暴跌到 62.99，AVG 41.82。模型为了"匹配藏文翻译参考的具体词形"重写了原有中文表征；
  2. Semantic RL 结果：BLEU-4 = 0.4519（低于 SFT），但语义相似度 = 0.7164；关键的是 CMRC 几乎不掉（保持 ~80%+）；
  3. LLM-as-judge 偏好：Strong SFT 胜率 59.2%，Semantic RL 仍获 33.5% 胜率——尽管 BLEU 低 15pt，开放式翻译质量并未显著落后；
  4. 解读：Semantic RL 用"语义等价"作奖励让模型保留了表达自由度，因此不必把"中文表征空间"重写成"翻译参考语料的子集"，从根本上回避了 alignment tax。

⑤ 消融与讨论

消融分析：(1) 移除 $R_{\text{lang}}$：模型出现 reward hacking——用中英文回答提高 cosine 相似度，藏语 script 比例从 99% 跌到 60%；(2) $\tau=0$ 不做 shaping：训练前期低质量 rollout 占主导，policy 收敛缓慢；(3) 用 BLEU 作 reward 而非 semantic embedding：BLEU-4 涨更多但 alignment tax 急剧反弹（CMRC 跌至 ~70%），证明"形式 reward 必然带来形式 SFT 的损失"。few-shot 迁移实验显示 RL 训出的 model 在小样本上的可迁移性优于 SFT，说明语义 reward 学到了"更可迁移的表征"。局限：(a) 仅在藏语-汉语单一组合上验证，对其他低资源语种（如蒙语、维语）效果未给出；(b) 双语 sentence embedding 自身的偏差（CINO 对藏语已有先验）会被 RL 放大；(c) BLEU 一定低于 SFT，对工程上"硬指标导向"的产品场景需权衡。

⑥ 结论与启发

这是一个对低资源语种适配领域具有方法论价值的工作：把"语种适配"从"分布拟合"重新定义为"语义对齐"，并用 GRPO + 嵌入 reward 把这一定义落地。其核心 takeaway 是 "alignment tax 的根因是 SFT 的 token-level 形式压力，而不是新语种本身"——用语义层奖励 + RL 的小步约束更新就能绕过这一陷阱。对蚂蚁集团这样要部署多语种产品的工业团队，这个范式可直接复用：先 5K SFT 冷启 → 90K GRPO 语义 RL，CMRC 等关键中文能力几乎无损，藏文/蒙文翻译质量已基本可用。这是中文社区在 alignment / multilingual NLP 上的一个实在贡献。
