LLM-Rosetta: A Hub-and-Spoke Intermediate Representation for Cross-Provider LLM API Translation

👤 Peng Ding 🏛 University of Chicago 📅 2026-04-10 | Industry Lab API Interoperability 🏢 OpenAI
📌 背景与动机

LLM 提供商各自使用专有 API 格式，应用开发者需要为每对提供商编写适配器——\(O(N^2)\) 的维护成本。LLM-Rosetta 采用轮辐式中间表示（Hub-and-Spoke IR）：只需为每个提供商编写一个到 IR 的映射，就能实现任意两个提供商间的 API 互通，将复杂度降为 \(O(N)\)。

🔬 核心方法详解
① 统一中间表示设计

IR 将 LLM API 的核心概念抽象为 5 个原语：Message（消息）、Model（模型参数）、Tool（工具调用）、Stream（流式输出）和 Response（响应格式）。每个原语有标准化的 schema，覆盖 OpenAI、Anthropic、Google、Cohere 等主流提供商 95%+ 的 API 功能。

② 语义保持的翻译规则

对每个提供商定义双向映射规则 \(f_{\text{provider} \to \text{IR}}\) 和 \(f_{\text{IR} \to \text{provider}}\)。翻译规则处理语义差异：如 OpenAI 的 function_calling vs. Anthropic 的 tool_use，以及参数命名差异（max_tokens vs. max_output_tokens）。不可翻译的特性标记为可选降级。

③ 运行时适配层

提供一个轻量级的运行时中间件：应用发送 IR 格式请求 → 中间件翻译为目标提供商格式 → 转发请求 → 翻译响应回 IR → 返回应用。延迟开销 < 5ms，不影响流式输出。

💡 深度案例
案例：从 OpenAI 无缝切换到 Anthropic

原始 OpenAI API 调用：

  • model: "gpt-4o", messages: [{role: "user", content: "..."}], tools: [{type: "function", function: {...}}]

IR 中间表示：

  • model: IR_MODEL("gpt-4o"), messages: [IR_MSG(user, "...")], tools: [IR_TOOL(function, ...)]

翻译为 Anthropic API：

  • model: "claude-sonnet-4-20250514", messages: [{role: "user", content: [{type: "text", text: "..."}]}], tools: [{name: ..., input_schema: {...}}]

应用代码无需任何修改，仅需更改配置中的目标提供商。

🔍 消融与讨论

功能覆盖差距主要来自提供商特有功能（如 Anthropic 的 thinking 模式、Google 的 Grounding API），这些无法在其他提供商上实现。IR 设计中将这些标记为"optional extensions"。与直接双边适配器相比，IR 方案在 10 个提供商的场景下将维护成本从 90 个适配器降到 10 个。局限性：快速变化的 API 需要频繁更新 IR schema。

🎯 结论与启发

LLM-Rosetta 用经典的编译器中间表示思想解决了 LLM 生态碎片化问题。\(O(N)\) 的维护成本和 <5ms 的延迟开销使其具有实际工程价值。这一工作也暴露了 LLM API 标准化的迫切需求。

SPASM: Stable Persona-driven Agent Simulation for Multi-turn Dialogue Generation

👤 Han Luo, Guy Laban 🏛 University of Leeds, Southwest Jiaotong University, Ben-Gurion University 📅 2026-04-10 | Industry Lab Dialogue Persona 🏢 DeepSeek
📌 背景与动机

LLM 在多轮对话中越来越多地扮演特定角色（如辅导、咨询），但 LLM-LLM 对话会累积身份相关失效——角色漂移、目标遗忘、语气不一致。这在生成合成训练数据时尤为严重。SPASM 提出一个稳定的人格驱动对话生成框架，确保跨长对话的角色一致性。

🔬 核心方法详解
① 结构化人格表示

将人格（persona）分解为 4 个维度：身份（名字、背景）、目标（对话要达成的任务）、性格（语气、表达习惯）和知识（领域专业知识）。每个维度用结构化模板表示，并在每轮对话开始时作为系统 prompt 的一部分注入。

② 一致性检查器（Consistency Checker）

在每轮生成后，运行一个轻量级的一致性检查模块，检测 4 个维度的漂移：

$$C_{\text{drift}}(t) = \sum_{d \in \{id, goal, style, know\}} w_d \cdot \text{sim}(r_t^d, p^d)$$

其中 \(r_t^d\) 是第 \(t\) 轮回复在维度 \(d\) 上的表现，\(p^d\) 是原始人格设定。当 \(C_{\text{drift}} < \tau\) 时触发重新生成。

③ 长程记忆锚定

在长对话（>10 轮）中，仅靠系统 prompt 不足以维持人格一致性。SPASM 引入记忆锚点机制：每 5 轮提取一次对话中的关键人格表现（如"在第 3 轮使用了幽默的语气"），并将这些锚点追加到后续轮次的 prompt 中，作为额外的人格约束。

💡 深度案例
案例：心理咨询师角色的 20 轮对话

人格设定：温和的心理咨询师 "Dr. Chen"，专注于认知行为疗法，目标是帮助来访者识别消极思维模式。

  • 无 SPASM（基线）：在第 12 轮后，"Dr. Chen" 开始给出直接建议（偏离 CBT 引导式风格），第 16 轮角色称呼从 "Dr. Chen" 变成了"AI 助手"
  • SPASM：一致性检查器在第 12 轮检测到目标漂移（sim=0.62 < 阈值 0.70），触发重新生成；记忆锚点提醒模型"之前一直使用开放式提问"，成功维持 20 轮角色一致
📊 实验结果

🔍 消融与讨论

(1) 去除一致性检查器：20 轮一致性从 0.82 降至 0.68，角色漂移率从 8.5% 升至 19.2%；(2) 去除记忆锚定：10 轮内影响小（-0.03），但 20 轮下降明显（-0.11），说明长程记忆对长对话至关重要；(3) 不同 LLM 后端：DeepSeek-V3 和 Qwen2.5-72B 的角色一致性差异 < 0.05，说明 SPASM 框架具有模型无关性。

🎯 结论与启发

SPASM 为 LLM 多轮角色扮演提供了系统化的解决方案：结构化人格表示 + 实时一致性检查 + 长程记忆锚定。将 20 轮对话的角色漂移率从 28.7% 降至 8.5%。这对合成对话数据生成和实际的角色驱动应用都有重要价值。

TaxPraBen: A Scalable Benchmark for Structured Evaluation of LLMs in Chinese Real-World Tax Practice

👤 Gang Hu, Yating Chen, Haiyan Ding, Wang Gao, Jiajia Huang, Min Peng 🏛 Yunnan University, Wuhan University, Jianghan University, Nanjing Audit University 📅 2026-04-10 | Industry Lab Benchmark Tax/Finance 🏢 Alibaba
📌 背景与动机

LLM 在通用领域表现出色，但在中国税务这一高度专业化、知识密集且法律严格监管的领域存在明显差距。现有税务基准多聚焦于孤立的 NLP 任务，忽略了真实实践能力。TaxPraBen 是首个面向中国真实税务实践的结构化基准，基于 Bloom 认知分类法将任务分为知识记忆、知识理解和知识应用三层。

🔬 核心方法详解
① Bloom 认知分类法驱动的任务设计

三层 7 个子任务：知识记忆（TaxRead 阅读理解、TaxMCQ 多选题）、知识理解（TaxEnt 实体识别、TaxRel 关系抽取、TaxSum 摘要生成）、知识应用（TaxCalc 税额计算、TaxInspect 税务稽查、TaxPlan 税务筹划）。任务难度从记忆到应用逐层递增。

② 数据来源与质量控制

数据源涵盖中国税法法规、国家税务总局公告、真实税务案例和注册税务师考试题库。由 5 名持证税务师进行标注和审核，确保法律准确性。数据集包含 4,200+ 条测试样本，覆盖增值税、企业所得税、个人所得税、房产税等 12 个税种。

③ 多维度评估体系

针对不同任务类型设计评估指标：结构化预测任务（TaxCalc、TaxMCQ）使用精确匹配（EM），生成任务（TaxSum、TaxPlan）使用 BERTScore 语义相似度，混合任务（TaxInspect）综合 EM 和 BERTScore。所有指标取 5 次运行平均值以消除随机性。

💡 深度案例
案例：TaxCalc 税额计算任务

问题："某企业 2024 年应纳税所得额为 280 万元，其中包含技术转让所得 550 万元。计算该企业应缴纳的企业所得税。"

  • GPT-4o：直接按 25% 计算 → 70 万（✗，忽略了技术转让优惠政策）
  • ERNIE-3.5：技术转让所得 500 万以内免税，超出部分减半征收。应税 = (280-550+550) 需拆分…最终答案 33.75 万（✓）
  • 难点：需要同时掌握企业所得税法第27条和技术转让所得优惠政策的具体免税额度（500万元）和减半征收规则

🔍 消融与讨论

关键发现：(1) 知识应用远难于记忆：最好模型在应用任务上仅 41.5%，比记忆低 31.3pp，反映税务实践需要将法律知识与经济活动情境结合；(2) 中文模型优势显著：ERNIE-3.5 和 Qwen 在涉及中文税法术语的任务上比英文模型高 10-15pp；(3) few-shot 提升有限：1-shot 相比 0-shot 平均仅提升 1.8pp，说明税务推理不易通过简单示例学习；(4) 税务数据微调（在 2000 条税务QA上 SFT）仅提升 3.2pp，暗示数据覆盖不足是瓶颈。

🎯 结论与启发

TaxPraBen 填补了中国税务领域 LLM 评估的空白。核心启示：当前 LLM 在专业垂直领域仍有巨大提升空间，特别是涉及多法规交叉推理的知识应用场景。中文语言模型在中文法律领域的内在优势也得到了量化验证。
