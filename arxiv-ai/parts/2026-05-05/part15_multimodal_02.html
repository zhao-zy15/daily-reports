SAPO: Segment-Aligned Policy Optimization for Multi-Modal Reasoning
📅 2026-05-05 🏛 (机构信息未标注)
多模态推理 RL段对齐策略优化熵自适应分段
🔥 背景与动机

当前 LLM/MLLM RL 的两个极端：(a) PPO 等 token-level 方法用 GAE 给每个 token 算 advantage，但 long-CoT 中稀疏终局奖励要回传几千 token，方差大且同一推理步内多个 token 信号不一致；(b) GRPO 等 sequence-level 方法用整轨迹相对优势，无法区分中间步骤贡献——当模型大部分步骤正确仅最后一步格式错误时，所有正确步骤同样被惩罚。本文提出 SAPO：把强化学习的优化粒度从 token / sequence 提升到推理步骤（segment），并配合自适应熵分段机制，对多模态推理尤其重要——视觉证据通常在某个明确的推理步集中出现。

⚙️ 核心方法详解
① 步骤级 MDP 抽象。 标准 token-MDP 中 \(s_t\) = prompt + 已生成 token，\(a_t\) = 下一个 token。SAPO 重新定义： $$s_k = (x, \text{seg}_{
② 段级 value function 与 advantage。 引入步骤级 value： $$V^\pi(s_k) = \mathbb{E}_\pi\Big[\sum_{j=k}^K \gamma^{j-k} r(s_j, a_j) \mid s_k \Big]$$ 段级 advantage：\(\hat A_k = R_k - V_\phi(s_k)\)，其中 \(V_\phi\) 是辅助 value 网络在 segment 边界处估值。这样 credit assignment 与推理步骤一一对应，避免 token 级噪声。
③ 段对齐重要性采样。 标准 PPO 重要性比率是 token 级 \(r_t = \pi_\theta(a_t)/\pi_{\theta_{\text{old}}}(a_t)\)。SAPO 用段级几何均值： $$\rho_k = \exp\!\left(\tfrac{1}{|a_k|}\sum_{t\in a_k}\log\frac{\pi_\theta(y_t|s_t)}{\pi_{\theta_{\text{old}}}(y_t|s_t)}\right)$$ 这样防止长 segment 上比率连乘爆炸，是稳定 RL 训练的关键技巧。Clipped 目标： $$\mathcal L_{\text{SAPO}} = -\mathbb{E}\Big[\min\big(\rho_k \hat A_k, \mathrm{clip}(\rho_k, 1-\epsilon, 1+\epsilon) \hat A_k\big)\Big]$$
④ 熵自适应分段。 不依赖 prompt 格式（如 \n\n 或 ###Step），SAPO 用 token 级熵识别逻辑边界：高熵 token 标志模型面对多种竞争续写——这正是推理步骤切换点。具体规则：当连续 \(\eta\) 个 token 熵超过阈值 \(\theta_H\) 时切段。作者用 \(\eta=30\)，相当于自动识别 ~30 token 长的 logical chunk。这种模型内部信号驱动的分段比 prompt-driven 更鲁棒。
深度案例：Geo3K 几何题"求外接圆半径"
问题：图片中等边三角形边长 6，求其外接圆半径。

典型推理 trajectory（SAPO 自动分段）：
  1. 段 1（理解）：识别图中是等边三角形 + 边长 6 → 段熵中等，但首 token "Looking" 高熵触发段边界。
  2. 段 2（公式回忆）：等边三角形外接圆半径公式 \(R = \tfrac{a}{\sqrt{3}}\) → "Recall" 高熵触发，明确知识检索段。
  3. 段 3（代入计算）：\(R = 6/\sqrt{3} = 2\sqrt{3}\) → 计算段内熵低，单一路径。
  4. 段 4（最终答案）："The answer is \(2\sqrt{3}\)" 终局奖励 r=+1。
SAPO credit 分配：段 1 (Adv=+0.05)、段 2 (Adv=+0.45 知识对)、段 3 (Adv=+0.30 计算对)、段 4 (Adv=+0.20 答案对)。
对比 GRPO：把整轨迹平均 +1 reward 等分到所有 token，区分不出"知识检索"和"计算"哪一步贡献大。
对比 PPO：每个 token 独立 GAE 估值，公式中常数 \(\sqrt{3}\) 这样的 token 反而被赋负 advantage（学困难），引入梯度噪声。

🔬 消融与讨论

(1) 段长 \(\eta\)：\(\eta=30\) 最优，过短退化为 token、过长退化为 sequence；(2) 熵阈值 \(\theta_H\)：取分布的 90th percentile 自适应；(3) 不用熵分段而用 prompt-guided 段（\n\n）：性能下降 1.8 pp，因 prompt 不一致；(4) 文本数学任务（GSM8K, MATH）也提升明显。局限：(1) 仍需训练辅助 value 网络，比 GRPO 显存高 ~30%；(2) 视觉模态外的多模态（音频、视频）未验证；(3) 极长 CoT（>10k tokens）下段数过多可能拖慢 backward。

💡 结论与启发

SAPO 是 RL credit assignment 粒度演进的自然下一步：从 token （PPO）→ trajectory（GRPO）→ step（SAPO）。其特别价值在多模态推理：视觉信息整合通常发生在某个明确的 segment（"识别图中物体" → "代入数值"），段级 credit 让 RL 信号与多模态融合的天然结构对齐。这种思路也启发未来在 agentic RL（每个 tool call 是一个 segment）、code RL（每个函数定义/调用是一段）等场景的应用。"RL 优化粒度应与任务结构粒度一致"是 2026 年 RL 后训练的重要方法论命题。
