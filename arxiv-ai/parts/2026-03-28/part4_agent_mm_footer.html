CRAFT: Grounded Multi-Agent Coordination Under Partial Information

👤 Abhijnan Nath, Hannah VanderHoeven, Nikhil Krishnaswamy
📅 2026-03-26
🏛 Colorado State University (SIGNAL Lab)
Agent · 多智能体 · 协调
当多个 LLM Agent 只能看到局部信息时，它们能协调到什么程度？CRAFT 构建了一个积木协调基准：两个"总监"Agent 各有独特视角，需要通过自然语言指导一个"建造者"搭建积木。测试 15 个模型后发现一个悖论：前沿模型的个体沟通质量更高，但任务完成率反而更低。原因是前沿模型过度利用私有视角进行精细推理，导致过多"修正指令"，消耗回合预算却不推进任务。

🔬 核心方法详解

① 基准设计：部分可观测协调

环境：积木搭建世界。两个总监 Agent（D1, D2）各从不同角度看到部分棋盘状态，一个建造者执行操作。总监只能通过自然语言消息沟通，建造者根据综合指令执行。游戏引擎强制物理规则（不能移除底层积木等）。

② 三维 LLM 自动评分

使用 GPT-4o-mini 作为评分器，诊断通信失败：SG（空间基础）：私有推理是否正确识别缺失块；MM（心智建模）：消息是否增加新信息、利用独特视角、解决冲突；PS（语用充分性）：集体输出是否足够让理性建造者识别正确操作。

③ 因果中介分析

发现"独特视角利用"(MM5) 与任务进度的相关性被移除缺口完全中介：
- 独特视角单独解释进度 \(R^2=0.330\)
- 移除缺口单独解释 \(R^2=0.609\)
- 两者结合 \(R^2=0.633\)（增益仅 0.024）
- 控制移除缺口后偏相关不显著（\(r=-0.247, p=0.374\)）
结论：精细推理→过多修正指令→回合浪费。

④ 修正螺旋失效模式

典型案例（Qwen-32B）：第 10-14 回合，D1 和 D2 持续指示从"底层"移除橙色积木（实际在第 1 层），游戏引擎拒绝执行。Agent 缺乏跨回合失败追踪机制，重复相同错误指令消耗多个回合。前沿模型的移除缺口是开源模型的 2-5 倍。

  1. 能力悖论：GPT-4.1-Mini 的 SG/MM 评分最高（0.937, 0.787），但进度仅 0.312；7B 开源模型 Mistral 进度反而是其两倍

  3. 个体能力 ≠ 集体智慧：个体语用能力是集体任务成功的必要非充分条件——协调能力才是瓶颈

Natural-Language Agent Harnesses (NLAH)

👤 Linyue Pan, Lexiao Zou, Shuo Guo, Jingchen Ni, Hai-Tao Zheng
📅 2026-03-26
🏛 清华大学深圳国际研究生院, 哈尔滨工业大学（深圳）
Agent · 框架工程 · 编排
Agent 的性能越来越依赖"harness 工程"（编排逻辑），但这些逻辑通常被埋藏在控制器代码中。NLAH 将 harness 的控制逻辑外部化为可编辑的结构化自然语言文档，由"智能 Harness 运行时 (IHR)"直接解释执行。在 OSWorld 基准上，将代码 harness 迁移到 NLAH 后性能从 30.4% 飙升至 47.2%，LLM 调用从 1.2k 降至 34，运行时间从 361 分钟降至 141 分钟。

🔬 核心方法详解

① NLAH 六大组件规范

一个可执行的 NLAH 必须包含：契约（I/O、验证关卡、权限、停止规则）、角色（如解决者/验证者，职责不重叠）、阶段结构（显式工作流拓扑：计划→执行→验证）、适配器（确定性操作钩子）、状态语义（跨步骤持久化的产物和重开机制）、失败分类（命名失败模式驱动恢复策略）。

② 智能 Harness 运行时 (IHR)

共享运行时环境，将运行时章程（通用规则）与任务族的 harness 逻辑分离。循环内 LLM 直接解释 NLAH 文档并执行。关键设计：约 90% 的 Prompt Tokens 和 LLM 调用发生在委托的子 Agent 中而非父线程。

③ 文件支持的状态模块

解决长周期自主性中的稳定性问题。强制三个属性：外部化（状态写入产物文件）、路径可寻址（通过路径重新打开对象）、压缩稳定（在截断或重启中存活）。在 OSWorld 上带来 +5.5% 提升。

④ 模块化消融与自进化

逐模块消融显示：自进化模块在 SWE-bench 上 +4.8%（通过收紧接受关卡改进解决循环）。但验证器 (-0.8%) 和多候选搜索 (-2.4%) 反而有害——结构增加不等于性能提升，开销敏感场景需谨慎。

  1. 代码→文本：性能飙升：OSWorld 上同一任务从代码 harness 迁移到 NLAH 后，性能从 30.4% 升至 47.2%，LLM 调用降低 35×
  2. 行为模式转变：代码 harness 倾向于基于屏幕的 GUI 修复循环，NLAH 更倾向于文件级验证和确定性修复（直接编辑 .pptx 包或配置文件），更高效
  3. 并非所有结构都有益：验证器和多候选搜索在 OSWorld 上分别导致 -8.4% 和 -5.6%，过度结构化在开销敏感场景中反成负担

🔑 启示：Agent 的控制逻辑用自然语言表达反而比代码更好——LLM 更擅长解释文本指令而非执行代码框架。

Humans vs Vision-Language Models: A Unified Measure of Narrative Coherence

👤 Nikolai Ilinykh, Hyewon Jang, Shalom Lappin, Asad Sayeed, Sharid Loáiciga
📅 2026-03-26
🏛 University of Gothenburg, Queen Mary University of London, King's College London
Multimodal · VLM · 叙事评估
当 VLM 根据图片序列讲故事时，它们的叙事连贯性与人类相比如何？本文提出 5 维叙事连贯性评分（NCS），测试 5 个顶级 VLM 后发现：人类在所有文本导向指标上均占优（NCSarith 0.50 vs 最佳模型 0.46），但在多模态角色定位上人类反而不占优——模型更受视觉内容字面约束。所有模型对人类文本赋予更高困惑度，表明人类叙事更具多样性和"惊喜感"。

🔬 核心方法详解

① 五维叙事连贯性指标

指代连贯 R：平均链大小/链数量，衡量实体引用集中度。话语关系类型 D：独特关系类型/总关系数量，衡量逻辑多样性。主题切换 T：相邻片段主题改变频率。角色持久度 ChP：角色在故事中的连续性（结合连续性和分布）。多模态角色定位 MCG：文本角色引用与视觉序列角色出现的一致性。

② 综合叙事连贯性分数 (NCS)

两种聚合方式：算术平均 \(NCS^{arith}\) 和几何平均 \(NCS^{geom}\)。几何平均对各维度不均衡更敏感——Llama 4 Scout 的 \(NCS^{geom}\) 仅 0.06（vs 人类 0.36），暴露其极不均衡的连贯性表现。

③ 实验设计与规模

使用 Visual Writing Prompts (VWP) 语料库，功效分析后采样 60 个视觉故事序列。5 个 VLM：InternVL3-78B、Qwen3-VL-235B、Claude-4.5 Sonnet、GPT-4o、Llama-4-Scout。两种提示条件（短提示/长提示），人类数据来自 VWP 原始标注和 AMT 新收集。

④ 困惑度与多样性分析

所有评估模型对人类文本赋予更高困惑度——人类叙事比模型生成更不可预测（更多样化）。长提示条件下人类仍然保持最高 NCS，提示工程未能使模型收敛于人类的连贯性特征。

Vision Hopfield Memory Networks (V-HMN)

👤 Jianfeng Wang, Amine M'Charrak, Luk Koska, Xiangtao Wang et al. (10 authors)
📅 2026-03-26
🏛 University of Vienna, TU Wien (推断自 FWF 资助)
Multimodal · 视觉 · 记忆网络
受大脑 Hopfield 联想记忆启发，V-HMN 提出了一种不依赖卷积或自注意力的视觉骨干网络，使用分层记忆检索作为核心构建模块。在 CIFAR-10/100、SVHN、Fashion-MNIST 上全面超越 ViT、Swin、MLP-Mixer、Vim（93.94% vs 92.97%），在仅 10% 标注数据下优势更加显著（80.22% vs 76.14%），ImageNet-1k 达到 80.3%。

🔬 核心方法详解

① Hopfield 记忆检索机制

给定表示 \(z \in \mathbb{R}^D\) 和记忆库 \(M \in \mathbb{R}^{K \times D}\)，\(\ell_2\) 归一化后计算检索权重和原型：

$$\alpha = \text{softmax}(\sqrt{D} \, \hat{z}\hat{M}^\top), \quad m = \sum_{j=1}^{K}\alpha_j M_j$$

缩放因子 \(\sqrt{D}\) 恢复余弦相似度方差至单位方差，防止 softmax 过平。记忆库使用类平衡环形缓冲区，训练时动态更新，推理时冻结。

② 局部+全局双记忆路径

局部记忆：对每个 token 的 \(k \times k\) 空间邻域展开、展平、投影，通过 Hopfield 检索对局部模式去噪补全（类似卷积但基于原型先验）。全局记忆：对所有 token 均值池化形成查询向量，检索场景级原型并广播回所有 token 作为上下文先验。

③ 迭代细化（预测编码）

通过预测编码启发的更新规则进行误差校正：$$z^{(t+1)} = z^{(t)} + \beta(m - z^{(t)})$$

其中 \(\beta\) 是可学习参数。\(t=0\) 到 \(t=1\) 提升 CIFAR-10 准确率 0.38%，在噪声/遮挡鲁棒性测试中平均准确率从 71.65% 升至 74.35%，\(t=2\) 后收益递减。

④ 数据效率优势

V-HMN 的记忆机制提供了显式的原型先验，在标注数据稀缺时优势突出。10% 标注数据下 CIFAR-10：V-HMN 80.22% vs ViT 72.73% vs MLP-Mixer 76.14%（+4–7.5%），因为记忆库中的原型补偿了标注不足。

  1. 10% 数据效率：CIFAR-10 仅用 10% 标注，V-HMN 80.22% vs ViT 72.73%——记忆机制提供了强大的归纳偏置
  2. 鲁棒性：在高斯噪声、遮挡和对比度退化下，迭代细化将平均准确率从 71.65% 提升至 74.35%

📝 今日总结与反思

  • 置信度校准新范式：Paper 1&2 揭示了 LLM 内部的"知道-说出"脱节问题。CAA 转向和 M-ratio 分别从因果干预和测量理论角度提供了解决方案。
  • RL 训练效率革命：ARRoL（剪枝 rollout）和 HIVE（选择 prompt）从不同维度砍掉 RLVR 的计算浪费，均实现了"少练但练得更好"。OPD 的三种失效模式诊断则为蒸馏方法提供了 debug 指南。
  • 安全与能力不矛盾：SafeMath 证明安全干预可以同时提升数学准确率，因为安全和准确的注意力分配存在协同效应。
  • Agent 的"聪明陷阱"：CRAFT 揭示前沿模型在多 Agent 协调中反而更差，NLAH 显示自然语言框架优于代码框架——Agent 设计需要反直觉思维。
  • 记忆驱动的视觉骨干：V-HMN 展示了 Hopfield 记忆作为 Transformer 替代方案的潜力，尤其在数据稀缺场景下。
