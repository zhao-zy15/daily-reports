Prune as You Generate: Online Rollout Pruning for Faster and Better RLVR (ARRoL)

👤 Haobo Xu, Sirui Chen, Ruizhong Qiu, Yuchen Yan, Chen Luo et al. (8 authors)
📅 2026-03-25
🏛 UIUC, Amazon
RL · RLVR · 训练加速
GRPO/DAPO 等 RLVR 方法的计算瓶颈在于为每个 prompt 采样大量 rollout，但多数样本要么全对要么全错（低方差=无学习信号）。ARRoL 引入轻量级"质量头"在生成过程中预测成功概率，在检测点 \(L_{\text{detect}}=512\) 处剪枝低价值 rollout，实现 1.7× 端到端加速和 +2.3–2.99 准确率提升，并支持推理时加权投票。

🔬 核心方法详解

① 质量预测头

2 层 MLP，类似 next-token 头。使用 RL 过程中自然获得的（推理路径, 最终奖励）对在线训练，梯度与骨干断开避免开销。采用在线分箱概率估计器校准原始分数为后验概率 \(q_i \in [0,1]\)：

$$q(s') = P(Y=1|b(s')) = \frac{\pi P(b(s')|Y=1)}{\pi P(b(s')|Y=1) + (1-\pi)P(b(s')|Y=0)}$$

其中 \(\pi=P(Y=1)\) 是先验概率，类条件似然基于历史缓冲区估计。

② 概率剪枝机制

目标：将保留路径的正样本比率控制在 \(\rho=0.5\)，最大化组内奖励方差。每条路径根据 \(q_i\) 和 \(\rho\) 分配生存概率 \(p_i\)（\((\rho - q_i)\) 的单调函数），通过二分搜索偏置 \(\delta\) 使期望保留率匹配 \(\kappa\)。理论保证：剪枝后真实正比率与 \(\rho\) 的偏差满足：

$$|\hat{p}_{-\hat{\jmath}} - \rho| \le \min_{j \in [G]}|\mu^*_{-j} - \rho| + 2\epsilon + \sqrt{\frac{\log(2/\delta)}{2(G-1)}}$$

③ 系统级集成

剪枝在 vLLM 推理后端内部执行。路径达到 \(L_{\text{detect}}\) 即被评估，被剪枝路径释放的 GPU 容量立即重新分配。前端接收剪枝掩码后对幸存路径重批处理计算 log-prob。效率分解：生成加速 1.46×，log-prob 1.84×，模型更新 2.08×。

④ 测试时加权投票

训练好的质量头复用为正确性预测器，将质量分数线性重缩放为 [0,1] 作为投票权重，替代简单多数投票。在 AMC'23 上 60.0%（DeepConf 57.5%），AIME'25 上 13.3%（DeepConf 6.7%）。

  1. AIME'24 单项飙升：Qwen-3-8B 在 AIME'24 上提升 +10.00（56.67→66.67），难题上剪枝的学习信号聚焦效果最为显著

HIVE: Online-Verified Prompt Selection for Efficient RL Training of Large Reasoning Model

👤 Jiahao Wu, Ning Lu, Shengcai Liu, Kun Wang, Yanting Yang, Qing Li, Ke Tang
📅 2026-03-26
🏛 Unknown (投稿 ICML)
RL · 训练效率 · 提示选择
RL 训练中大量 rollout 浪费在"太简单"或"太难"的 prompt 上——HIVE 追踪随训练移动的"学习边缘"，通过两阶段筛选（历史感知 + 在线验证）将 rollout 数量减少 70%（3.4×），同时精度提升 +1.9%，rollout 时间加速 3.8×，总训练时间加速 2.3×，在线验证开销仅占总迭代时间的 0.4%。

🔬 核心方法详解

① 历史感知选择（阶段一）

利用零成本历史数据初步筛选。零方差计数：\(z_i = \max\{k: \prod_{j=n-k+1}^{n}\mathbb{I}_{i,j}=1\}\)，连续产生相同奖励的 prompt 被降权。历史响应熵：\(H_i = \frac{1}{G}\sum_{r=1}^{G} U^{(r)}(x_i)\)，高熵=高多样性=高学习价值。综合选择概率：\(P_{\text{select}} = \lambda \cdot P_{\text{Rew}} + (1-\lambda)\cdot P_{\text{Ent}}\)。

② 在线验证选择（阶段二）

解决"数据陈旧性"——历史指标无法反映当前模型状态。使用提示熵作为低成本代理：\(V(x) = \frac{1}{L_p-1}\sum_{l=2}^{L_p}\mathcal{H}(p_\theta(\cdot|x_{

③ 移动边缘策略

高梯度效用样本集中在中间难度 × 高熵的交集处，该"边缘"随训练动态移动。HIVE 通过在线中位数阈值实时追踪边缘，防止过度采样已掌握或仍无法解决的 prompt。理论保证：排名一致性定理证明 \(\text{sign}(V(x)-V(x')) = \text{sign}(\hat{U}(x)-\hat{U}(x'))\)（提示熵排序与真实响应熵排序一致）。

④ 开销极低的设计

阶段二每步额外延迟仅 0.82 秒，占总迭代时间（~147s）的 0.4%。核心洞见：提示熵（仅看 prompt 自身）足以近似完整 rollout 的信息价值，无需采样任何响应。

Revisiting On-Policy Distillation: Empirical Failure Modes and Simple Fixes

👤 Yuqian Fu, Haohuan Huang, Kaiwen Jiang, Yuanheng Zhu, Dongbin Zhao
📅 2026-03-26
🏛 CASIA (中科院自动化所), UCAS
RL · 知识蒸馏 · 策略优化
On-policy distillation (OPD) 在长期任务中脆弱的根因是什么？本文识别出三种失效模式：(1) 采样 token 信号严重不平衡，(2) 教师在学生前缀上的信号不可靠，(3) 分词器/特殊 token 不匹配。提出 Teacher Top-K Local Support Matching——在教师的高概率 token 集上做分布级比较替代单 token 比较，将平均分从 36.4 提升至 41.5，并在多任务设置（数学+Agent）中达到 ALFWorld 97.7%。

🔬 核心方法详解

① 三种失效模式诊断

模式 1：大多数采样 token 的奖励为负值（教师概率低于学生），正向优势极少，训练信号不平衡。模式 2：学生生成的前缀对教师来说是分布外的，教师会给表面合理但方向错误的 token 高概率。模式 3：不同分词器将相同内容切分不同（如 <think> vs <th+ink+>），单 token 比较将分词差异误判为语义错误。

② Teacher Top-K 局部支持匹配

不再基于单个采样 token 更新，而是在教师 Top-K token 集 \(S(c_{i,t})\) 上计算截断反向 KL：

$$\mathcal{L}_{\text{LSM}} = \mathbb{E}_{x,\{o_i\}\sim\pi_{\theta,\text{infer}}}\left[\frac{1}{\sum_i|o_i|}\sum_i\sum_t\sum_{v\in S(c_{i,t})}\hat{\pi}_\theta(v|c_{i,t})\log\frac{\hat{\pi}_\theta(v|c_{i,t})}{\hat{q}(v|c_{i,t})}\right]$$

其中 \(\hat{\pi}_\theta\) 和 \(\hat{q}\) 在局部支持集上重新归一化。

③ Top-p 推理采样 + 特殊 token 掩码

使用 top-p 核采样生成推理轨迹，避免极低概率 token，保持轨迹在教师分布的典型延续范围内。对容易产生假阴性的特殊 token 进行掩码处理。消融实验：仅 Teacher Top-K 无 top-p 时 AIME24 avg@32 从 20.4 降至 17.7，结合后升至 23.6。

④ 方差分析与理论基础

序列级 OPD 梯度估计器方差增长 \(O(T^4)\)，token 级近似降至 \(O(T^2)\)，但引入了即时奖励偏差。局部支持匹配在保持 \(O(T^2)\) 方差的同时，通过分布级比较消除了单 token 的偏差源。

💡 深度案例分析
  1. 数学+Agent 多任务：在数学和 ALFWorld 智能体联合训练中，本方法在 ALFWorld 达到 97.7%，同时数学平均分 41.7（基线 36.6），证明修复对多任务泛化有效
  2. 重归一化至关重要：消融显示不在支持集上重归一化会导致优化迅速崩溃
  3. 对噪声鲁棒：即使不使用特殊 token 掩码，局部支持匹配也达 41.0（vs 基线的 36.4），说明方法本身对分词器噪声具有鲁棒性

Maximum Entropy Behavior Exploration for Sim2Real Zero-Shot Reinforcement Learning (FB-MEBE)

👤 Jiajun Hu, Núria Armengol Urpí, Jin Cheng, Stelian Coros
📅 2026-03-26
🏛 ETH Zurich, Max Planck ETH Center
RL · Sim2Real · 探索
零样本 RL 的核心挑战是：不知道下游任务时，如何在无奖励数据集中覆盖足够广的行为空间？FB-MEBE 在 Forward-Backward 框架上引入最大熵行为探索——最大化"已实现行为分布"的熵，同时用正则化评论家抑制不自然动作。在 Unitree Go2 四足机器人上实现了零微调直接部署，成功跟踪速度、调整姿态。

🔬 核心方法详解

① Forward-Backward 表示学习

将后继度量分解为前向和后向表示：\(M^{\pi_z}(ds'|s,a) \approx F(s,a,z)^\top B(s')\rho(ds')\)，其中 \(z\) 是任务嵌入。策略定义为：\(\pi_z(s) = \arg\max_a F(s,a,z)^\top z\)。这种分解使得策略可以在测试时通过切换 \(z\) 直接适配新任务。

② 最大熵行为探索

目标：最大化已实现行为分布 \(p^t_{z^a_r}\) 的熵 \(J_{\text{MEBE}} = -H[p^t_{z^a_r}]\)。实践中通过密度模型 \(q_\Psi\) 估计访问密度，按逆密度采样探索行为：\(z_r^E \propto (q_\psi(z^a_r)+\epsilon)^{-\beta}\)，\(\beta=2.0\)。80% 使用 MEBE 策略，20% 均匀随机嵌入。

③ 正则化评论家

为确保策略自然且物理可行，修改 Actor 损失加入正则化项：

$$\mathcal{L}_{\text{FB-Critic}} = -\mathbb{E}\left[F(s,a,z)^\top z + \lambda_{\text{reg}} Q_{\text{reg}}(s,a)\right]$$

\(Q_{\text{reg}}\) 预测行为正则化奖励（关节加速度惩罚 + 动作率惩罚 + 脚滑惩罚）的长期回报，\(\lambda_{\text{reg}}=20\)。这抑制了拖地等不自然行为，使策略可直接部署。

④ Sim2Real 零样本部署

在 IsaacLab（Unitree Go2, 12 自由度）中训练，直接部署到真实机器人。通过操纵杆接口发送指令，复合奖励函数结合速度 \(r_v\)、基础方向 \(r_g\) 和高度 \(r_h\)。机器人成功跟踪平移速度、调整俯仰/横滚角、在保持 15° 俯仰角同时向前行走。

💡 深度案例分析
  1. 零样本硬件部署：无需微调直接在 Unitree Go2 上运行——复合指令如"在 0.32m 基础高度跟踪偏航角速度"均能稳定执行
  2. 探索覆盖优势：在高速任务中，其他基线因回放缓冲区收缩到低速区域而性能下降，FB-MEBE 保持强劲
  3. 自然性保证：正则化评论家显著抑制脚滑（vs 纯 FB），熵增加的代价是略高脚滑，但反映了更广行为覆盖

🔑 启示：最大熵探索 + 行为正则化 = 多样而自然的技能库，使零样本 Sim2Real 部署成为现实。
