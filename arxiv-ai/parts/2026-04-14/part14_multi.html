EXAONE 4.5 Technical Report: LG's First Open-Weight Vision-Language Model
📅 2026-04-10
🏛 LG AI Research
MultimodalVLM开源
📌 背景与动机

EXAONE 系列是 LG AI Research 持续开发的基础模型，此前的 EXAONE 3.0 和 4.0 均为纯文本模型。EXAONE 4.5 是该系列首个视觉语言模型（VLM），也是 LG 发布的首个开源权重 VLM，旨在为工业智能应用提供多模态理解能力。其架构将专用视觉编码器集成到现有 EXAONE 4.0 框架中，实现了原生多模态预训练。

🔬 核心方法详解
① 架构设计：视觉编码器集成
EXAONE 4.5 采用"视觉编码器 + 连接器 + LLM"的三段式架构：
视觉编码器：基于 SigLIP-SO400M，输入分辨率 384×384，输出 729 个视觉 token（每 token 对应 14×14 patch）。
连接器：两层 MLP + 下采样，将 729 个视觉 token 压缩为 256 个，并投影到 LLM 的 embedding 空间。
LLM 骨架：EXAONE 4.0（7.8B 参数），不冻结——视觉 token 和文本 token 在同一 attention 层中处理。
关键设计选择：不使用动态分辨率（固定 384×384），通过图像分块（tiling）处理高分辨率图像——最多 9 块 tiles。
② 三阶段训练流程
Stage 1 — 视觉-语言对齐预训练：在 50M 图文对上训练连接器（冻结视觉编码器和 LLM），学习视觉→语言的映射。
Stage 2 — 多模态联合预训练：解冻全部参数，在混合数据（图文对 + 纯文本 + OCR + 文档理解 + 图表理解）上联合预训练。纯文本数据占比 40%，防止语言能力退化。
Stage 3 — 后训练：包含 SFT（多模态指令微调）、DPO（视觉偏好对齐）和 RLHF 三个子阶段。
③ 工业场景优化
针对 LG 的工业应用需求（如家电维修、制造质检、零售分析），EXAONE 4.5 在后训练中加入了大量工业场景数据：
(a) 文档/表格理解：技术手册、检测报告、工单等结构化文档的信息提取；
(b) 物体检测+描述：工业产品的缺陷识别和分类；
(c) 多图对比：同一产品不同状态（正常/异常）的对比分析。
这些工业化需求驱动了 tile-based 高分辨率处理（需要看清小缺陷）和结构化输出能力。
④ 开源策略与许可
EXAONE 4.5 采用 CC BY-NC-ND 4.0 许可证——允许学术使用和非商业研究，但商业部署需要额外许可。发布三个版本：
EXAONE-4.5-VL-7.8B：完整版，视觉+语言全能力。
EXAONE-4.5-VL-2.4B：小模型版，适用于边缘设备。
EXAONE-4.5-VL-Instruct：指令微调版，直接可用于对话。

工业场景应用：家电维修辅助
输入：一张洗衣机内部的照片 + 用户描述"洗衣机发出异常噪音"

🔍 消融与讨论

技术报告详细分析了：(1) 视觉编码器选择——SigLIP 优于 CLIP（MMBench +2.1）和 EVA-02（+3.5），主要因为 SigLIP 的 sigmoid loss 对细粒度特征更友好；(2) Tile 数量从 1→9 可提升 OCRBench +8.3 和 DocVQA +11.2，但对通用 VQA 影响较小（+1.5）；(3) 纯文本数据在 Stage 2 中的占比从 20%→40% 可将 MMLU 从 68.5 提升至 71.2，但过高（60%）反而降低视觉性能。局限性：CC BY-NC-ND 许可证限制了商业社区的使用；7.8B 模型在需要深度推理的 MMMU 上仍显著落后于闭源大模型。

💡 结论与启发

EXAONE 4.5 的核心贡献在于证明了垂直领域（工业智能）的VLM开发路径：在通用视觉能力的基础上，通过工业场景数据的后训练，可以在文档/OCR/缺陷检测等工业任务上达到甚至超越通用VLM。对工业AI实践者的启示是：定制化后训练比更大的模型规模更有效。

ECHO: Efficient Chest X-ray Report Generation with One-step Block Diffusion
📅 2026-04-10
🏛 Midea AIRC, Beijing Jiaotong University, Dalian University of Technology
Multimodal医学影像扩散模型
📌 背景与动机

胸片报告生成（CXR-RG）是缓解放射科医生工作负担的关键应用。然而，传统自回归 VLM 的逐 token 解码导致推理延迟过高（通常 >2秒/报告），难以满足高通量临床需求。扩散语言模型（dLLM）可以并行生成所有 token，但直接应用 dLLM 生成医学报告面临两个挑战：(1) 多步去噪仍较慢；(2) 医学文本的幻觉问题在扩散生成中更严重。本文提出 ECHO，一个单步块扩散 VLM，兼顾效率和临床准确性。

🔬 核心方法详解
① 从自回归到扩散：AR-to-Diffusion 转化
ECHO 的训练分两步：先训练一个自回归基线模型 ECHO-AR，然后将其转化为扩散模型 ECHO。
转化方法：利用 ECHO-AR 生成的 (图像, 报告) 对作为教师数据，训练一个 Block Diffusion 模型来学习"给定图像，一步生成完整报告"的映射。
Block Diffusion 的核心：将报告 token 序列分为 \(B\) 个块（每块 32 token），每个块内部做离散扩散（掩码→去噪），块之间通过因果注意力连接：
$$p(\mathbf{x}_1, ..., \mathbf{x}_B | \mathbf{v}) = \prod_{b=1}^{B} p_\theta(\mathbf{x}_b | \mathbf{x}_{
② 直接条件蒸馏（DCD）
关键创新：通过蒸馏将多步扩散压缩为单步生成。DCD 包含三个组件：
Step-wise Token Reweighting (SW)：在蒸馏过程中，对不同时间步的 token 分配不同权重——早期时间步（噪声大）给低权重，晚期时间步（接近完成）给高权重；
Cross-Entropy Loss on EOS (CE)：对结束标记 \ 单独施加交叉熵损失，确保模型学会在正确位置终止生成；
Reverse KL Divergence (RKL)：用反向 KL 散度对齐学生（单步）和教师（多步）的输出分布：
$$\mathcal{L}_{\text{RKL}} = \sum_{i} p_\text{student}(x_i) \log \frac{p_\text{student}(x_i)}{p_\text{teacher}(x_i)}$$ 反向 KL 鼓励学生模型的输出分布更集中（mode-seeking），减少幻觉。
③ 幻觉缓解策略
医学报告中的幻觉（生成不存在的病变描述）是核心安全问题。ECHO 采用两种策略：
Report-Aware Distillation (RAD)：在蒸馏数据中加入"负样本"——将真实报告与不匹配的图像配对，训练模型识别不一致。
Fused Block KV Cache：在推理时，将视觉 KV cache 与文本 KV cache 融合，确保每个生成块都"看到"完整的视觉信息——防止纯扩散生成中因并行解码导致的视觉信息遗漏。
④ 推理效率分析
ECHO 的单步推理流程：输入胸片 → 视觉编码（一次）→ 扩散单步去噪（一次，并行生成所有 token）→ 输出报告。
时间对比：自回归 VLM 需要 ~200 次 forward pass（报告 ~200 token），ECHO 仅需 1 次 forward pass + KV cache 融合。
实际加速：在 A100 GPU 上，ECHO 的推理速度是自回归基线的 16.5x，延迟从 2.3 秒降至 0.14 秒/报告。

🔍 消融与讨论

DCD 消融显示：(1) SW（Step-wise Reweighting）单独使用降低 PPL 但提升有限；(2) CE（EOS 损失）是关键组件——没有它，模型无法正确终止生成，产生大量重复文本；(3) SW+CE 的组合效果最好（CIDEr 从 3.61 提升至 4.20），而加入 RKL 反而略有下降（可能因为 RKL 的 mode-seeking 特性限制了文本多样性）。RAD 数据规模从 5K→50K 持续提升 RateScore（+4.8），说明负样本学习对减少幻觉至关重要。局限性：单步生成在 ROUGE-L 上比多步略低（53.3 vs 55.0），表明极致压缩仍有少量质量损失；仅在英文胸片报告上验证。

💡 结论与启发

ECHO 证明了"高临床准确性 + 极致解码效率"可以同时实现。通过 AR→Diffusion 转化 + DCD 蒸馏，ECHO 在 0.14 秒内生成质量与自回归模型相当的胸片报告，实现 16.5 倍加速。对医学 AI 部署的启示是：扩散语言模型在医学报告生成中有独特优势——并行解码天然适合高通量场景。
