Marco DeepResearch: Unlocking Efficient Deep Research Agents via Verification-Centric Design

👤 Bin Zhu, Qianghuai Jia, Tian Lan, Junyang Ren, Feng Gu, Feihu Jiang, Longyue Wang, Zhao Xu, Weihua Luo 📅 2026-03-30 · arXiv:2603.28376 🏛 Alibaba International Digital Commerce Agent Deep Research Verification

📌 背景与动机

深度研究智能体需要通过多轮搜索、推理和综合来回答复杂查询，但现有方法在训练和推理中缺乏显式验证机制，导致错误在长链推理中不断累积。Marco DeepResearch 在数据合成、轨迹构建和测试时扩展三个层面引入验证机制，以 8B 参数规模达到了超越 30B 模型的性能。

🔬 核心方法详解

① 验证式数据合成
两条互补管线：图论对抗性合成——"答案优先"逆向路径搜索 + 三角色（Generator/Attacker/Analyzer）迭代确保答案唯一性，每轮添加区分性约束直到收敛；智能体网络探索合成——先构建证据图再生成问题，含事实一致性检查、闭卷过滤（排除无需检索即可解决的问题）和独立搜索验证。
② 验证驱动的轨迹构建
多智能体验证框架：主智能体（任务分解/聚合）+ 搜索子智能体（解决子任务）+ 验证子智能体（使用 Web 工具独立验证中间结果和最终答案）。验证失败时记录修正行为。另有验证-反思重推演：收集最终答案错误的轨迹，验证智能体诊断失败原因后重新推演。多智能体轨迹最终转换为单智能体 ReAct 格式用于训练。
③ 验证引导的测试时扩展
Discard All 策略：当触发退化信号（如达到最大步数）时，清除所有中间推理历史，仅保留原始查询重新开始——探索新路径。验证引导扩展：每个候选答案独立验证（规则检查+智能体评判），只要未达预算上限 \(T_{\max}\)，继续探索更多候选并联合验证。
④ 长上下文 RL 训练
将上下文窗口从 64K 扩展到 128K 进行 RL 训练。在 SFT 基础上加入 RL（可验证奖励），平均提升 +2.6 分。测试时验证引导扩展再贡献 +12.1 分的巨大增益。消融显示合成数据中的对抗性唯一性验证贡献 +2.3 分，多智能体轨迹构建贡献 +2.0 分。

💡 深度案例

案例：BrowseComp 复杂网络查询
  1. 查询："Find the earliest patent number filed by Tesla Motors that specifically mentions 'battery thermal management system' in the abstract"
  2. 首次尝试：搜索子智能体找到 3 个候选专利号
  3. 验证子智能体：独立访问 USPTO 数据库，确认候选 1 摘要不含目标短语 → 排除
  4. 修正：扩大搜索范围，发现更早的专利 → 验证通过 → 最终答案正确
  5. 对比：无验证机制的基线错误地返回了候选 1（摘要匹配度最高但不含确切短语）

🔍 消融与讨论

最大亮点：测试时验证扩展贡献 +12.1 分（BC-ZH 上甚至 +17.8），远超 RL 训练和数据改进的增量。8B 模型在多数基准上超越同规模竞品，在 BC-ZH 和 xBench 上甚至超过 30B 模型。验证是核心杠杆——它在数据、训练和推理三层面系统性地减少了错误累积。局限性：验证子智能体增加了推理延迟和 API 成本；Discard All 策略在简单查询上浪费计算。

Greedy Is a Strong Default: Agents as Iterative Optimizers

👤 Yitao Li 📅 2026-03-28 · arXiv:2603.27415 🏛 Independent Researcher Agent Optimization AutoML

📌 背景与动机

经典优化算法（爬山法、模拟退火）使用随机提议生成器探索解空间。如果用具备推理能力的 LLM 替换随机提议器，最简单的贪婪策略（只接受改进）是否就足够了？还是需要模拟退火等复杂接受规则来跨越局部最优？本文通过在四种不同任务上的系统实验证明：贪婪就是最强默认选择。

🔬 核心方法详解

① 提议-评估-接受框架
三步循环：提议——LLM 智能体读取评估诊断信息，通过推理提出候选工件 \(q_{\mathcal{L}}(\theta'|\theta)\)；评估——在数据集上计算候选指标 \(M(\theta')\)；接受/拒绝——贪婪法仅当 \(M(\theta') > M(\theta)\) 时接受。严格的信息屏障（训练/验证/测试分离）确保泛化。
② 为什么贪婪优于模拟退火
LLM 的强先验使得有效搜索空间 \(\Theta_{q_{\mathcal{L}}} \ll \Theta\) 远小于全空间。在这个缩小的空间中，目标函数 \(M\) 可能是单峰的。模拟退火的接受概率 \(P = \exp\left(\frac{M(\theta')-M(\theta)}{T}\right)\) 在回归幅度 \(\epsilon\) 很小时 \(P \approx 1\)，退化为无方向的随机游走。贪婪则过滤噪声、单调收敛。
③ 角色分工：调查员-审查员-协调员
调查员：根据训练日志的错误分析、过拟合检测等诊断信息提出修改建议；审查员：基于验证集分数执行贪婪/SA 规则；协调员：管理迭代循环和可选的并行调查员。诊断推理（读取训练日志）是核心价值——OPRO-lite（仅基于历史分数）显著逊色。
④ 跨任务验证
四种不同结构的任务：离散规则发现（Breast Cancer YAML 规则）、混合超参流水线（STL-10 图像分类）、LoRA 微调优化（SST-2 情感分类）和纯连续超参（XGBoost）。贪婪在所有任务上匹配或超过 SA、并行调查员和多模型变体，同时评估次数减少 2-3 倍。

💡 深度案例

案例：XGBoost 超参优化（12 维连续空间）
  1. Round 0：种子配置 AUC-ROC = 0.9297
  2. Round 1：智能体读取训练日志，发现类别不平衡，调整 scale_pos_weight → AUC = 0.9312（最大一步提升）
  3. Round 2-5：微调 learning_rate 和 max_depth → AUC = 0.9317
  4. 对比：CMA-ES 需 50 次评估达 0.9317，贪婪智能体仅需 13 次（效率 3×）
  5. SA 对比：SA 在 Round 1 后接受了一个 AUC=0.9308 的倒退步，延缓了 2 轮收敛

🔍 消融与讨论

关键发现：0% 灾难性失败率（Random Search 为 60%），因为 LLM 的领域知识避免了明显的参数陷阱（如不兼容的优化器-调度器组合）。OPRO-lite 消融证明了诊断推理（读取训练日志）是核心价值来源。在纯连续空间中 LLM 有竞争力但受限于"常规配置"偏好。建议：贪婪应作为 LLM-agent 优化的默认基线，只有在证明搜索空间存在多个深度局部最优时才考虑复杂策略。

MediHive: A Decentralized Agent Collective for Medical Reasoning

👤 Xiaoyang Wang, Christopher C. Yang 📅 2026-03-28 · arXiv:2603.27150 🏛 NSF & DoD 资助 Agent Medical Multi-Agent

📌 背景与动机

医疗推理需要整合多学科知识、处理不确定性和矛盾证据。现有多智能体系统多采用中央协调架构（Central Orchestrator），面临单点故障、角色泄露和可扩展性瓶颈。MediHive 提出去中心化智能体集体，通过共享内存池和迭代融合实现自组织协作。

🔬 核心方法详解

① 自演化角色分配
每个智能体 \(A_i\) 分析查询 \(Q\) 后生成角色提案 \(R_{i,0}\) 及理由，发布到共享内存池 \(\mathcal{M}\)。阅读所有同行提案后，通过自我反思更新为最终角色 \(R_i\)，优化清晰度、差异化（低语义重叠）和对齐度。无中央协调器，角色自然分化。
② 条件辩论与分歧检测
智能体独立分析后生成初步答案 \(Ans_{i,1}\) 和置信度 \(c_{i,1}\)。自主检查同意水平——若低于阈值 \(\tau_{\text{agree}}=0.8\)，激活 \(T_{\text{debate}}\) 轮辩论。辩论中发表结构化论证（反驳/辩护/新提议），不改变正式答案，仅丰富证据库和论证强度。
③ 迭代共享融合
第 2 轮（综合整合）：智能体读取全部历史（含辩论日志），批判同行论点并更新立场。后续轮次（增量细化）：仅读取上一轮同行输出 \(\{(Reasoning_j, Ans_j, c_j)\}\)。终止条件：连续两轮同意水平达 \(\tau_{\text{agree}}\) 或达到最大轮次 \(K\)。
④ 自适应聚合与报告
强共识时：确认超级多数答案。未达共识时：置信度加权投票 \(S(a) = \sum_{i=1}^N c_{i,k} \cdot \mathbf{1}(Ans_{i,k}=a)\)，选择 \(a^* = \arg\max_a S(a)\)。仅在推理完成后，专门的"报告员"汇总输出——报告员不参与决策。

💡 深度案例

案例：MedQA 多学科诊断题
  1. 查询：62 岁男性，胸痛+呼吸困难+心电图 ST 段压低。选项含心梗、肺栓塞、主动脉夹层、气胸
  2. Agent A（心内科角色）：倾向心梗（c=0.7），基于 ST 段压低
  3. Agent B（急诊角色）：倾向肺栓塞（c=0.6），指出 D-dimer 升高
  4. 辩论：Agent A 反驳"ST 段压低+胸痛更符合 NSTEMI"；Agent B 提出"Wells 评分支持 PE"
  5. 融合后：Agent B 更新为心梗（c=0.75），考虑到 ST 段压低的特异性更高 → 正确答案

🔍 消融与讨论

消融实验：移除 CoT 导致 MedQA 降至 78.0%（-6.3%），移除自演化角色降至 81.5%（-2.8%），移除置信度加权降至 82.4%（-1.9%）。智能体数量消融：5 个最优（3 个缺多样性，7 个增加噪音）。去中心化的核心优势是避免了角色泄露（单 LLM 模拟多角色时的常见问题）。局限性：推理延迟随辩论轮次线性增长；在 PubMedQA 上未超越 MedAgents，可能因为 PubMedQA 的二选一格式不需要深度辩论。
