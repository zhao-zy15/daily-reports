Agent-1 · Look Before You Leap: Autonomous Exploration for LLM Agents

📅 2026-05-19 🏛 University of Science and Technology of China, Meituan arXiv:2605.16143
Agent Exploration Explore-then-Act GRPO

① 背景与动机

LLM agent 在不熟悉的环境中频繁失败，根因是 premature exploitation：模型会基于先验立刻行动，而不肯先收集环境信息。现有 ReAct 风格的 loop 都是 goal-conditioned $\pi(a_t \mid H_t, g)$ —— 没有任何机制为 agent 分配"先观察、再行动"的预算。论文提出两个深刻问题：(1) 当前 LLM agent 究竟有多差的 autonomous exploration 能力？(2) 任务驱动 RL（task-only GRPO）会不会反而损害 exploration 能力？答案是：损害（与 LLM-4 / NUDGERL 完全一致——RL 让模型更窄）。本文把 exploration 作为一种可量化、可单独训练的能力提取出来，并给出 Explore-then-Act 推理范式。

② 核心方法详解

① Exploration Checkpoint Coverage (ECC)：第一个可验证的 exploration 指标

对每个环境实例预定义有限 checkpoint 集合 $\mathcal C = \{c_1, \ldots, c_M\}$，每个 checkpoint 对应"一个有能力的探索者应该发现的事实"——可达地点、关键物体、有效交互目标、功能状态、affordance、约束等。给定 exploration trajectory $\tau_{\mathrm{EXP}}$：

$$\mathrm{ECC}(\tau_{\mathrm{EXP}}) = \frac{1}{M}\sum_{i=1}^M \mathbb 1[c_i \in \tau_{\mathrm{EXP}}]$$

这是不依赖任务成功的 process-level metric，等价于 agent 学界长期缺失的"地图覆盖率"指标。论文为 ALFWorld / SciWorld / TextCraft 三个环境分别构造了 checkpoint set（Appendix E）。

② Autonomous Exploration 形式化

普通 task agent：$a_t \sim \pi_{\mathrm{ACT}}(\cdot \mid H_t, g)$。Autonomous exploration：把 $g$ 移除，agent 只有交互预算 $N$ 步：

$$a_t \sim \pi_{\mathrm{EXP}}(\cdot \mid H_t),\quad \tau_{\mathrm{EXP}} = (o_1, a_1, \ldots, a_N, o_{N+1})$$

其目标是建立环境的内在模型——transition $T(o_{t+1} \mid o_t, a_t)$、状态空间、动作语义、隐藏约束。Exploration 完成后再合成 grounded knowledge summary $\mathcal K = \mathrm{SUMMARIZE}(\tau_{\mathrm{EXP}})$。

③ Interleaved GRPO：交错的 task + exploration 训练

对 exploration 步用 ECC 作 verifiable reward，对 task 步用 task success reward。两种 step 在训练中交错出现：

$$\hat A_i^{\mathrm{exp}} = \frac{\mathrm{ECC}(\tau_i^{\mathrm{exp}}) - \mu^{\mathrm{exp}}}{\sigma^{\mathrm{exp}} + \epsilon},\quad \hat A_i^{\mathrm{task}} = \frac{R_i^{\mathrm{task}} - \mu^{\mathrm{task}}}{\sigma^{\mathrm{task}} + \epsilon}$$

每种 reward 走自己的 GRPO clipped surrogate，混合训练让两种能力同时存在于一个 policy，不必额外切换 expert。论文称为 GRPO (Interleaved)。

④ Explore-then-Act 推理范式

推理时分两阶段：先 $N$ 步 goal-free exploration 产生 $\tau_{\mathrm{EXP}}$，再合成 $\mathcal K$，最后 goal-conditioned acting：

$$a_t \sim \pi_{\mathrm{ACT}}(\cdot \mid H_t, g, \mathcal K)$$

$\mathcal K$ 作为结构化自然语言注入 prompt——包含状态布局、object affordance、action 前置条件、已发现的约束、failure cases。这是把"环境知识"从 implicit policy memory 显式化到 prompt 上下文，可解释、可检查、可调试。

③ 深度案例：Qwen3-4B 在 ALFWorld 上从 31.8% 跃升到 89.7%

诊断阶段（无任务、100 步预算）：
  • Qwen2.5-7B 三环境平均 ECC = 22.2%——大量重复动作、过早终止；
  • Qwen3-4B 平均 ECC = 28.5%；Qwen3-4B + Task-Only GRPO 后反而降到 18.8%——任务训练显著损伤 exploration；
  • Claude-Opus-4.5 平均 ECC = 89.5%（接近完美），GPT-4.1 = 49.3%——开闭源差距巨大。
训练 + Explore-then-Act 后：以 Qwen3-4B 在 ALFWorld 为例，Task-Only baseline 直接执行成功率 72.6%，Interleaved GRPO + E-t-A 达到 89.7%。Qwen2.5-7B 同样从 78.4% → 90.9%。
失败行为对比（Table 3）：Task-Only GRPO 训练后的 agent —— 重复动作率 63.4%、loop 率 16.0%、信息搜索率 1.0%、错误恢复率 0.0%。Exploration-Aware 训练后 —— 重复动作率 24.9%、loop 率 7.7%、信息搜索率 7.5%、错误恢复率 20.1%。这是质变：错误恢复能力从无到有。
关键：探索能力本身就是抗失败的护城河——会 recover 的 agent 就是会 explore 的 agent。

⑤ 消融与讨论

消融分析：(1) Task-Exploration ratio sensitivity（Appendix D）：50:50 配比是甜点，全 exploration 反而损害 task 完成；(2) 不同 exploration model + 同一 acting model：固定 Qwen3-4B 当 actor，仅替换 explorer（如用 Claude 探索 → Qwen 行动），$\mathcal K$ 质量直接决定下游任务表现；(3) 关闭 $\mathcal K$ summary：直接把 $\tau_{\mathrm{EXP}}$ 拼入 prompt 反而性能下降——必须经过 SUMMARIZE 把 noisy trajectory 压成结构化知识；(4) 错误恢复率 0.0%→20.1% 是定性转变。局限：(a) ECC 依赖人工预定义的 checkpoint，对 open-world 任务难以自动构造；(b) 训练数据需要 environment-specific exploration 信号，泛化到新环境仍需要建 checkpoint；(c) E-t-A 范式的 budget $N$ 是手调超参，未优化自适应分配。

⑥ 结论与启发

这篇工作的核心贡献是把"探索"作为 LLM agent 的独立可训练能力提取出来——它有自己的 verifiable reward (ECC)、自己的训练目标、自己的推理范式 (E-t-A)。三个深刻发现共同形成了一个连贯认知：(1) 任务驱动 RL 会损伤 base 模型的 exploration 能力（与 LLM-4 / RL-2 完全同构）；(2) 错误恢复是 exploration 的副产品；(3) Claude-Opus-4.5 是唯一在该指标上接近"人类可信代理"水准的大模型。对正在做的 GPU IPO 和 LLM agent 投资分析有直接参考价值——下一代 agent 的核心壁垒不只是 reasoning step length，而是能在陌生环境里"先看再动"的元能力。这也指出了 RLVR/LPO 研究的一个重要扩展方向：不要只优化 task reward，要把 process-level reward（如 ECC 这类 environment-aware 信号）纳入混合训练目标。
