Medical-1 · SEMA-RAG: Self-Evolving Multi-Agent RAG for Medical Reasoning

📅 2026-05-19 🏛 The Chinese University of Hong Kong, Wuhan University of Technology arXiv:2605.17101
Medical RAG Multi-Agent 临床推理

① 背景与动机

医学 QA 是 RAG 部署的"高难度典型"：临床问答需要先把 patient narrative 解释成可搜索的临床问题、再多轮检索 + 综合 + 仲裁，最后给出可追溯结论。但主流 RAG 框架仍把检索当 static one-shot：单轮 BM25/Embedding → 直接生成。这与临床推理的多阶段过程严重不匹配，导致两类失败：(1) Question-to-query 失真——问题里隐藏的关键约束（如 "hospital day 7"、"renal impairment"、"pregnant"）丢失，retrieval 漂向 generic symptom evidence；(2) Sufficiency 缺乏检查——单轮 retrieval 拿回的证据是否足以做决策没有反馈。SEMA-RAG 把 RAG 从"一次性查询"升级为三角色 + 自演化迭代：I-Agent (interpreter) → E-Agent (explorer) → A-Agent (arbiter)。

② 核心方法详解

① I-Agent：临床 Schema 化的问题解释器

I-Agent 不只是 paraphrase 问题 $Q$，而是把它投影到结构化 clinical schema：

$$Q' = \langle o_{\mathrm{int}},\, o_{\mathrm{ent}},\, o_{\mathrm{cons}},\, q_{\mathrm{init}}\rangle = \mathrm{Agent}_I(P_{\mathrm{mt}I},\, Q)$$

四个槽位：clinical intent（diagnosis / treatment / dosage）、medical entities（疾病、药物）、clinical constraints（孕妇、肾损伤、成人等）、initial retrieval query。然后 linearize 成 retrieval-ready string：

$$\hat q_{\mathrm{init}} = \mathrm{Linearize}(Q') = \mathrm{Concat}(q_{\mathrm{init}} \oplus o_{\mathrm{int}} \oplus o_{\mathrm{ent}} \oplus o_{\mathrm{cons}})$$

关键设计：linearize 是无参数的拼接函数，但显式把 implicit constraints 暴露给 dense retriever，避免 retrieval 漂移。

② E-Agent：Sufficiency-Driven 自演化迭代检索

E-Agent 维护当前 query 集 $\mathcal Q_t$ 与累积证据集 $\mathcal C_t$，每轮做 Top-k 检索：

$$\mathcal D_t = \bigcup_{q \in \mathcal Q_t} \mathrm{TopK}(q),\quad \mathcal C_t = \mathcal C_{t-1} \cup \{D_i \in \mathcal D_t : \mathrm{ID}(D_i) \notin \mathrm{IDs}(\mathcal C_{t-1})\}$$

然后做 sufficiency 判断：

$$[s_t, g_t, \mathcal Q_{t+1}] = \mathrm{Agent}_E\big(P_{\mathrm{mt}E},\,[Q', \mathcal Q_t, \mathcal C_t]\big)$$

$s_t \in \{0, 1\}$ 是 sufficiency flag；$g_t$ 是 gap description；如果 $s_t = 0$ 则生成 $m$ 个 follow-up query 针对 gap 继续检索。终止条件：$s_t = 1$ 或达到 $T_{\max}$ 或 stagnation（$\mathcal Q_{t+1} = \emptyset$）。这是 RAG 第一次在 medical 域内引入"可追踪的证据收敛轨迹"$\tau = \{[\mathcal Q_1, \mathcal C_1], \ldots, [\mathcal Q_T, \mathcal C_T]\}$。

③ A-Agent：证据仲裁与可追溯报告

A-Agent 从收敛证据集 $\mathcal C^\ast$ 中：(a) 删除冗余/不相关；(b) 识别 consistencies 和 conflicts；(c) 把 supporting / refuting clue 组织成结构化报告 $R$，并保留每个证据的 source ID：

$$\mathcal S^\ast = \{(\mathrm{ID}(D_i), D_i) \mid D_i \in \mathcal C^\ast\},\quad R = \mathrm{Agent}_A(P_{\mathrm{mt}\,\mathrm{adj}},\,[Q, \mathcal C^\ast, \mathcal S^\ast])$$

最终答案选择 grounded 在 $R$ 上——这一步把"reasoning trace"和"answer"显式分离，让审计员可以独立检查证据链而非只看最终答案。

④ 关键超参 $T_{\max}$ 与 $m$

论文把"探索深度 $T_{\max}$"与"探索广度 $m$"作正交超参分析。Figure 3 显示 $T_{\max}=2$（仅 2 轮）已捕获大部分收益，$T_{\max}=3$ 略有提升，$T_{\max}>3$ 收益饱和并引入 noise；Table 3 显示 $m=1 \to 86.72$, $m=2 \to 89.00$, $m=3 \to 89.95$。最优配置 ($T_{\max}=2, m=3$) 仅 2 轮、每轮 3 query，显著少于 i-MedRAG 的 3 轮固定迭代但效果反超——证明 sufficiency-driven 比 fixed-iteration 更高效。

③ 深度案例：MedQA-US 上 deepseek-v3.1 的差异化

问题：MedQA-US 一道典型多选题——患者 hospital day 7 出现 late-onset 感染症状，问最可能的致病菌（candidate 包括 S. aureus / Pseudomonas / E. coli 等）。
朴素 RAG 失败模式：
  • CoT only（无检索）：依赖 LLM 内部知识，可能遗漏住院日 7 这一时间约束。
  • + MedCPT（单轮检索）：retrieve 到的多是 "S. aureus general" 资料，没有 anchored on "hospital day 7 = late onset"。
  • + MedRAG / i-MedRAG：能多轮，但 question-to-query 把 "hospital day 7" 作为辅助信息丢失，3 轮检索仍无法 surface late-onset evidence。
SEMA-RAG 工作流程：
  1. I-Agent 把问题 schema 化：intent=diagnosis, entities={late-onset infection, S. aureus}, constraints={"hospital day 7", inpatient}, q_init="late-onset hospital-acquired infection day 7 most likely pathogen"。
  2. E-Agent 第 1 轮检索得到 S. aureus 与其他菌的"general profile"——sufficiency=0, gap="time-onset specific differential"。
  3. E-Agent 第 2 轮生成 3 个 gap-targeted 子查询，retrieve 到 late-onset/early-onset 时间窗 epidemiology 资料——sufficiency=1。
  4. A-Agent 把证据组织成报告，识别"late-onset (>5 days) 强烈支持 S. aureus"为 supporting clue，最终答案 S. aureus 与 source ID 一并输出。
结果：deepseek-v3.1 + SEMA-RAG 在 MedQA-US 拿到 89.95%（同模型 i-MedRAG 仅 74.78%，CoT-only 77.53%）。

⑤ 消融与讨论

消融分析（Table 2 - role-wise removal on MedQA-US / PubMedQA*, deepseek-v3.1）：(1) w/o I-Agent: 85.47 / 54.20（vs full 89.95 / 59.20）——question schema 化贡献 ~4.5 pt，对 PubMedQA* 影响更大；(2) w/o E-Agent: 83.58 / 50.80——最大降幅，self-evolving 是核心机制；(3) w/o A-Agent: 86.49 / 53.60——adjudication 也是必需的；(4) $T_{\max}$ 调参：$T_{\max}=2$ 已得到 89.95，$T_{\max}=3$ 略升 90.10，$T_{\max}=9$ 反而退到 87.67——deeper exploration 引入 noise；(5) $m$ 调参：$m=1$→86.72, $m=3$→89.95，monotonic 但 saturate。局限：(a) 三 agent 串行调用使每个问题约 3-5× CoT 推理成本；(b) sufficiency 判断依赖 LLM 自身 calibration，对小模型可能误判；(c) 仅在英文 medical QA 上验证，中文场景（如中文病历摘要）需要重新调 schema。

⑥ 结论与启发

SEMA-RAG 把 medical RAG 从"检索 + 拼接 + 生成"升级到"解释 + 探索 + 仲裁"的临床推理 mimicry。它最深刻的意义不在于具体的 +8 pt 提升，而在于把"多阶段推理"作为架构原语而非 prompt trick——三个 agent 各自的输入输出严格定义，可以独立调试和升级。这与本日 Agent-1 (Look Before You Leap) 的"先探索再行动" / Agent-2 (ICRL) 的"内化 critique" / Agent-3 (EvoMemBench) 的 "execution memory" 共同指向一个方向：下一代 LLM agent 不是 single-LLM-with-prompt，而是多角色协作的微型工作流。对中文医学知识库 RAG（包括公众号文章 RAG / 内部研报 RAG）的产品化建议：可以借鉴这一三 agent 框架，但 schema 需要按目标领域重新设计（金融 schema 应该是"投资意图、行业标的、约束条件、关键时间窗"四元组）。
