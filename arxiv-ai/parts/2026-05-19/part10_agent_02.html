Agent-2 · ICRL: Learning to Internalize Self-Critique with Reinforcement Learning

📅 2026-05-19 🏛 HKUST (Guangzhou), Nanjing University, Microsoft Research arXiv:2605.15224
Agent Self-Critique GRPO 分布校准

① 背景与动机

"Self-Refine"和"Critic-then-Revise"已经成为 LLM agent 的标准 inference-time 增强：模型先尝试 → critic 给反馈 → 模型再修正。但有两个尚未解决的根本问题：(1) Internalization 失败：删掉 critic 后，模型在同一问题上又会失败——说明它学到的是"有 critic 时正确"，没把能力真正内化；(2) Critic 静态化：现有方法的 critic 通常是 frozen 的（甚至来自外部 teacher 模型），随 solver 进化会越来越无关。ICRL 同时解决这两点：从同一 backbone 联合训练 solver 和 critic，并通过 token-level 分布校准把 critique-guided 修正选择性地迁移回 critique-free policy，避免 solver 退化为"对 critic 上瘾"。

② 核心方法详解

① Joint Solver + Critic from Shared Backbone

从同一个 base 模型 $\pi_\theta$ 派生两种 role 行为：solver 角色 $\pi_\theta^S(\cdot \mid q)$ 和 critic 角色 $\pi_\theta^C(\cdot \mid q, \tau_i)$。critic 的 reward 不是 critique 自身的"好坏"，而是下游 solver 接受 critique 后的 performance gain：

$$R^C(c) = R\big(\tau_{i+1} \sim \pi_\theta^S(\cdot \mid q, c)\big) - R(\tau_i)$$

这把 critique 的"actionability"作为可优化目标——critic 必须给出对当前 solver真正能用的反馈，不仅是"对的话"。

② Critique-Conditioned Distribution Calibration

修正轨迹 $\tau_{i+1}$ 来自 critique-conditioned 分布 $\pi^S(\cdot \mid q, c)$，但要更新的是 critique-free policy $\pi^S(\cdot \mid q)$——直接 plug-in 会得到有偏估计。ICRL 引入 token-level reweight ratio：

$$w_t = \frac{\pi_{\theta_{\mathrm{rollout}}}^S(y_t \mid q, y_{

$w_t$ 度量"这个 token 在没有 critique 时是否仍然合理"：

  • $w_t \approx 1$：token 不依赖 critique，可以直接迁移；
  • $w_t \ll 1$：token 强烈依赖 critique，降权避免 reinforce critique-dependence；
  • $w_t > 1$：critique-free 反而会更自然地生成的 token，升权。

这是一种 importance sampling 校正——但用在 token 粒度而非 trajectory 粒度，因此可以同一条 trajectory 内"取其精华去其糟粕"。

③ Role-Wise Group Advantage Estimation

solver 和 critic 在不同 prompt 下采样，奖励量纲不同，混在同一个 group 做 standardize 会把信号污染。ICRL 把 advantage 按 role 分组：

$$\hat A_i^S = \frac{R_i^S - \mu^S}{\sigma^S + \epsilon},\quad \hat A_i^C = \frac{R_i^C - \mu^C}{\sigma^C + \epsilon}$$

各自 GRPO clipped surrogate 加权后联合更新共享 backbone。这是处理 multi-role 联合训练的通用模式。

④ 完整 ICRL 目标

对修正轨迹中 critique-guided 部分应用 token-level $w_t$ 校准的 GRPO surrogate：

$$\mathcal L_{\mathrm{ICRL}} = -\mathbb E\!\left[\sum_t w_t \cdot \min\big(r_t \hat A^S, \mathrm{clip}(r_t)\hat A^S\big)\right] - \mathbb E\!\left[\min\big(r^C \hat A^C, \mathrm{clip}(r^C)\hat A^C\big)\right]$$

无需外部 teacher、无需手工标注 critique 数据，纯 self-improvement 闭环。

③ 深度案例：Qwen3-8B 在 AIME24 上从 50% → 65.1%

设定：base model = Qwen3-8B，benchmark = AIME24（30 道高难度奥数）。
各方法表现：
  • Qwen3-8B base: 40.0%
  • SFT: 40.0%（SFT 对竞赛数学增量很小）
  • GRPO（标准 RLVR）: 50.0%（+10pt 来自答题 reward）
  • Critique-GRPO（外加 frozen critic）: 60.0%（+10pt 来自 critique）
  • ICRL（本文）: 65.1%（+15pt over GRPO、+5pt over Critique-GRPO）
测试时 multi-round refinement（ALFWorld 上）：GRPO baseline 第 1 轮 ~80% 第 3 轮约 ~83%（gradual gains 主要来自重复采样的多次机会，而非真的修正）；Critic-GRPO 第 1 轮 ~85% 第 3 轮 ~91%；ICRL 第 1 轮就 ~92%，第 3 轮达 98%——同时 first-round 更强、refinement 增益更陡，证明它的 critic 真正在产生 actionable feedback。
关键：8B 学到的 critic 能力已经达到可与 32B critic 相比的水平，且 token 数少很多——critic 能力可以从 backbone 共享中"涌现"，不必单独喂大模型。

⑤ 消融与讨论

消融分析：(1) 关闭 token-level reweight（直接把 critique-conditioned 轨迹当成 critique-free 轨迹训练）：solver 完全 critique-dependent，移除 critic 后立即退化到 base 水准；(2) 用 frozen critic：消除 critic 共训，性能与 Critique-GRPO 接近，无法持续 self-improve；(3) Role-wise advantage 关闭（混合 group）：训练严重不稳，loss spike 频繁；(4) 训练动态：critic 与 solver 交替进步——solver 解决能力强后，critic 也学会针对更微妙的错误做反馈。局限：(a) 在 Bamboogle / MuSiQue 等数据集上 ICRL 并非全面最优，多跳 RAG 任务 critic 学习更难；(b) AMC23 上 Critique-GRPO（95.0%）反超 ICRL（93.1%），说明在 distributionally easy 的任务上分布校准会带来轻微保守偏差；(c) Joint training 显存占用高（同时维护 solver + critic 两套 rollout），训练成本约 1.5× GRPO。

⑥ 结论与启发

ICRL 是 self-improvement RL 的关键拐点：它正面回答"为什么 self-refine 训练经常退化为 'critic 依赖'"——根因是 critique-conditioned trajectory 直接 plug 进 GRPO 会改变有效目标分布。token-level reweight ratio 是一个简单但极有效的工程修正，可以推广到任何 mixed-prompt-prefix 联合训练场景（包括 NUDGERL 的 strategy context 训练）。对政昀 RLVR/LPO 研究的启发：(a) 在 importance ratio 上做 token-level 校正是处理 distributional shift 的最直接方式，比 SFT mixing 优雅得多；(b) critic 与 solver 共训 + reward 绑定下游 utility 是构造 self-improvement 闭环的标准模板；(c) ICRL 的 8B critic ≈ 32B critic 这一观察具有深刻系统含义——critic 能力不是"参数量越多越好"，而是"reward shaping 越准越好"。
