① 背景与动机

Diffusion Language Models（DLMs，如 SDAR、LLaDA）通过迭代去噪生成文本，提供 parallelism 与 quality 之间的折中。每步通过 confidence threshold 控制揭示多少 token，但 quality 随每步 unmask token 数 $r$ 单调下降——baseline $r=2$ 在 MATH500 上比 $r=1$ 损失 7.0 pt 准确率（4B 模型）。能不能"更激进地多揭示，但用一个轻量模块预测下一步 logit 的修正"？作者的关键经验观察：相邻两步去噪的 logit 差异 $\delta_t = \ell_{t-1} - \ell_t$ 的 RMS 仅为完整 logits 的 ~20%（$k=1$），远比从头预测简单。

② 核心方法详解

① Logit Residual 的存在性（实证 + 理论）

定义 per-step logit residual：

$$\boldsymbol{\delta}_t = \ell_{t-1} - \ell_t \in \mathbb{R}^{L \times V}$$

实证（GSM8K + SDAR 1.7B/4B/8B）：RMS$(\boldsymbol{\delta}_t)$ 在 $k=1$ 时为 RMS$(\ell_t)$ 的 ~20%，$k=16$ 时仍 < 100%。理论：partially masked states $\boldsymbol{x}_T, \ldots, \boldsymbol{x}_0$ 是 Markov chain，相邻状态仅由 unmask 集合 $\mathcal{R}_t$ 改变，所以 $f(\boldsymbol{x}_{t-1})$ 与 $f(\boldsymbol{x}_t)$ 的差异由该 chain transition 上界。

② MRP 模块结构

MRP 是 2-3 层 transformer 模块 $g_\phi$，输入是 backbone 当前的 hidden states $\boldsymbol{h}$ 和当前 partially masked sequence $\boldsymbol{x}_t$，输出 hidden residual $\boldsymbol{\Delta}$ 与 logit residual $\boldsymbol{\delta}$：

$$(\boldsymbol{\Delta}, \boldsymbol{\delta}) = g_\phi(\boldsymbol{x}_t, \boldsymbol{h})$$

训练时 backbone 冻结，仅训练 $g_\phi$，使用 KL divergence 对齐 teacher logits：

$$\mathcal{L} = \text{KL}\big( \text{softmax}(\ell_{t-1})_{\text{masked}} \big\| \text{softmax}(\ell_t + \boldsymbol{\delta})_{\text{masked}} \big)$$

③ Direct Decoding 模式（牺牲一点 quality 换更高速）

每次 backbone forward 后，运行 $K$ 步 MRP 迭代：每步 $\boldsymbol{h} \leftarrow \boldsymbol{h} + \boldsymbol{\Delta}, \ell \leftarrow \ell + \boldsymbol{\delta}$，按阈值 $\tau$ 揭示新位置。一次 backbone forward 等价于 $K+1$ 步去噪，引入小近似误差。

④ Speculative Decoding 模式（无损加速）

把 MRP 解出的 $K$ 步 token 当作 draft，让 backbone 一次 forward 做验证（采用 Self Speculative Decoding 验证框架）。Verified pass 部分提交，错误部分回退。K=3 时实现 1.42× 无损加速。

③ 深度案例

SDAR-1.7B 解一道 GSM8K 数学题：
  1. Base (r=1)：每步揭示 1 token，accuracy 78.7%，TPS 267.4。
  2. Base (r=2)：贪心多揭示 2 token，TPS 提升到 1.86×（497），但 accuracy 暴跌到 70.8%（-7.9 pt），不可接受。
  3. Direct K=1：用 MRP 预测 1 步残差，accuracy 77.5%（-1.2 pt），TPS 1.50×（401）。Quality 损失温和。
  4. Spec K=3：MRP 解 3 步，backbone 验证。accuracy 77.8%（与 base 平齐 -0.9 pt），TPS 1.20×。在 8B 上 Spec K=3 反而比 base accuracy 略高（90.7% vs 91.0%）—— verification 严格 lossless。
  5. 实操关键：对代码（HumanEval）任务 direct 模式 accuracy 跌得多（48.2% vs 53.7%），因为 code token 分布转移更陡——这是 MRP 的 known limitation。speculative 模式则把这个问题完全消除。

⑤ 消融与讨论

残差 vs 直接蒸馏：同一架构、同一数据下，用残差目标训 vs 直接蒸馏目标训，在 $K=4$ 时残差版本平均提升 +65 pt accuracy——证明利用相邻 logit 相似性的归纳偏置是有效的，而非简单地"训了一个 draft model"。MRP 深度 sweep：2-3 层是 sweet spot，超过 4 层 accuracy 平台、throughput 下降——验证残差信号确实低复杂度。局限：code generation 在 direct 模式损失较多（code token 分布更陡）；MRP 训练数据需要来自 backbone 的 hidden states，跨 backbone 迁移性弱。

⑥ 结论与启发

MRP 对扩散 LLM 提出了"speculative decoding for diffusion"的范式，类似于 autoregressive Medusa/EAGLE：把昂贵的 backbone 一次前向当作 oracle，用轻量 head 解多步。更深层的启示：大模型推理中，"邻近 distribution 高度相似"是普遍现象（既适用于 autoregressive 邻近 token，也适用于 diffusion 邻近 step），只要能 cheap 地刻画残差，就能换 throughput。对于希望部署扩散 LLM（如 SDAR、Mercury Coder、Inception）的厂商，MRP 是近期可立刻接入 SGLang/vLLM 的 1.4× 免费加速。
