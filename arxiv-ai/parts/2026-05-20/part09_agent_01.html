① 背景与动机

LLM Agent 在长程交互任务（ALFWorld 房间任务、WebShop 网购）中面对部分可观测环境。Agent 必须从不完整观察持续推断 latent state，小推断误差会累积成 belief drift：Agent 自信地认为"钥匙在 box 里"，但打开后发现 box 是空的；如果策略不修正这个 belief，agent 会反复检查错误位置，trajectory 走得越来越偏。终端 reward 太稀疏，无法定位是 action 选错还是 state estimation 错——两者在 POMDP 下紧耦合，standard GRPO 只在 episode 级别归一化 reward，无法分离这两类错误，sample efficiency 极低。

② 核心方法详解

① 显式 Belief State 建模

在标准 LLM agent 输出（thought + action）之外，要求模型同时输出结构化 belief state $b_t$：

$$\pi_\theta(a_t, b_t | h_t) = \pi_\theta(\text{thought, action, belief} | \text{history})$$

belief $b_t$ 是显式可解析的 JSON（如 ALFWorld 中的 "key_location"、"target_object"、"progress"）。这把 implicit hidden state 提升为可监督的显式向量。

② Belief-Consistency 自监督 Reward

核心创新：把 belief 与后续 observation 的不一致性转化为 dense reward 信号。形式上：

$$r_t^{\text{belief}} = -\mathbb{1}[b_t \text{ 与 } o_{t+1:t+k} \text{ 不一致}] \cdot \lambda_b$$

比如 belief 说"钥匙在 box1"，下一步 observation 显示 box1 empty，则 belief 与 observation inconsistent，扣 $\lambda_b$ reward。这是不需要外部 verifier 的自监督信号，dense 程度比 terminal reward 高 10-50×。

③ Belief-Aware Grouping for Advantage

标准 GRPO 按 prompt 分组归一化 advantage，但在 POMDP 中相同 prompt 下 agent 的 belief 可能完全不同（trajectory 走到不同 subspace）。ReBel 改为按 belief state $b_t$ 聚类分组，在 belief 相似的 trajectory 之间做 advantage 归一化：

$$\hat{A}_{i,t} = \frac{r_{i,t} - \mu_{b(i,t)}}{\sigma_{b(i,t)}}, \quad b(i,t) = \text{cluster}(b_{i,t})$$

这显著降低 variance——agent 不会因为某个 belief subset 普遍 reward 低而被错误打压。

④ SFT Warm-Start on Belief-Annotated Data

RL 前先用 SFT 在 belief-annotated 数据上预训练，让模型先学会输出格式良好的 belief JSON。这是工程必要步骤——cold start RL 上模型不会输出合理 belief，自监督 reward 无法启动。论文 Appendix E.2 量化 isolated contribution。

③ 深度案例

ALFWorld：在卧室中找到并清洁苹果，放到厨房柜台。
  1. Step 1：Agent 输出 thought "查看桌子"，action look at desk，belief {apple_location: "desk", phase: "search"}。Observation：桌上没苹果。
  2. Belief check：predicted belief 说苹果在桌上，但 obs 不一致 → 扣 $\lambda_b = 0.1$ reward。这个 dense 信号让 agent 立即学到 "随便猜 belief 会被惩罚"。
  3. Step 2：Agent 修正 belief：{apple_location: "unknown, need to search drawer", phase: "search"}。打开抽屉发现苹果。Belief 与 obs 一致，无扣分。
  4. Belief-aware grouping：在 group 内 8 条 rollout，按 belief cluster 分组——cluster A "认为苹果在 desk" (4 条) 全部失败，cluster B "认为 apple_location unknown 主动 search" (4 条) 全部成功。在 cluster B 内部归一化 advantage 反而能鼓励早期就 belief honest 的策略。

⑤ 消融与讨论

(i) 去掉 belief-consistency reward：performance 跌回 GRPO 水平，证明 dense self-supervision 是核心。(ii) 去掉 belief-aware grouping：variance 飙升 40%，训练不稳定。(iii) Cold start without SFT warm：model 无法输出 valid belief JSON，自监督信号失效，RL training 完全 fail。局限：belief 结构需要 task-specific 设计（ALFWorld 用什么 schema，WebShop 用什么 schema 不同）；belief consistency check 需要 verifier（论文用规则 + LLM judge 混合）；对极长 horizon（如 1000+ step）belief 自身可能 drift 也需要重新评估。

⑥ 结论与启发

ReBel 的核心洞见——"奖励信念而非动作"——为 partially observable LLM agent 提供了新的 supervision 范式。它把 POMDP 中模糊的 latent state 提升为可显式监督的 belief vector，并把"belief 与后续 observation 的不一致性"作为天然 dense reward。这与 EvoMemBench / Lifelong Memory（Agent-2）的思想形成互补：ReBel 关注当前 episode 内部 belief 的演化，Lifelong Memory 关注跨 episode 的记忆机制。对于工业 agent 系统（如 Operator、Devin、Cursor Agent），ReBel 提示了一个工程方向：让 agent 显式输出结构化中间状态（不是 chain-of-thought 自由文本，而是 typed belief），用 state consistency 作 process reward，可显著提升长程任务成功率。
