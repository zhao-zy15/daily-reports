RL-4 · Drop the Act: Probe-Filtered RL for Faithful Chain-of-Thought Reasoning (ProFIL)

📅 2026-05-12 🏛 Intuit 👤 Swapnil Parekh arXiv:2605.11467
RLCoT FaithfulnessProbe

1️⃣ 背景与动机

Reasoning 模型常常先内部承诺答案，再生成大段"看似在思考"的 token 来事后合理化——作者称之为"reasoning theater"（推理表演）。后果：浪费推理 token、污染可解释性、误导 RL 优化。已有工作（Lanham 2023、Anthropic CoT faithfulness）试图惩罚 theater，但常导致 RL-obfuscation 失败模式——模型学会绕开监督信号产生更隐蔽的 theater。本文动机：用冻结 base 模型上的 attention probe 检测 commitment，并把检测信号嵌入 GRPO，使监督信号抗 RL gaming。

2️⃣ 核心方法详解

① Performativity（表演度）与 Faithful Fraction 量化

对每条 rollout 的每步 reasoning，定义"是否在 commit 之后"：若该步发生在模型内部第一次产生正确答案 token 之后，则标记为 performative。整条链的 performativity ratio：

$$ \rho \;=\; \frac{\#\text{post-commit steps}}{\#\text{total steps}} $$

Faithful fraction = 链中 \(\rho<\delta\)（如 \(\delta=0.1\)）的比例，即真正"思考完才答"的比例。

② Frozen-Base Multi-Head Attention Probe

在冻结的 base 模型激活上训练一个 multi-head attention probe：输入是每步 reasoning 的 hidden state 序列，输出是该步是否 post-commitment。监督信号来自 verifier（无需人工标注）。关键设计：probe 训练于冻结 base 模型，因此 RL 更新无法通过修改激活分布来骗过 probe（破解 RL-gaming）。

③ Probe Filter：Advantage Zeroing

GRPO rollout 时，每条 rollout 用 probe 打分得到 performativity score。若超过阈值 \(\delta\)：

$$ A_i' = \begin{cases} 0, & \text{if probe}(x_i)>\delta \\ A_i, & \text{otherwise}\end{cases} $$

即把表演成分高的 rollout 在 advantage estimation 中清零——不奖励、不惩罚。论文证明这比"直接惩罚"更不容易触发 RL-gaming。

④ 与长度惩罚的区分（长度压缩不是真正机制）

论文专设 5.3 节做等长惩罚对照：匹配 chain length 减少幅度的 GRPO + length-penalty 基线，结果其 faithful-fraction 远不及 ProFIL。这证明 ProFIL 的增益来自语义 commitment 检测而非简单压缩 token。

5️⃣ 消融与讨论

消融：(a) probe 不冻结 base、随 RL 一起训 → 出现 RL-gaming，performativity 表面降低但 frozen audit 揭示真实 theater 上升；(b) 用 length-penalty 匹配 chain 长度 → faithful-fraction 大幅落后 ProFIL，证明"压缩长度 ≠ 减少表演"；(c) 阈值 \(\delta\) 灵敏性：附录 A.3 给出 ROC 曲线，\(\delta=0.5\) 附近大区间稳定。局限：(i) 需要为每个 base 模型重新训 probe；(ii) commitment 的"真值"由 verifier 推断（"first correct token position"），对开放式生成定义不明；(iii) 当前 4 个 domain 都是 verifiable reasoning，对创造写作类任务的扩展开放问题。

6️⃣ 结论与启发

"Reasoning theater"是 reasoning model 时代的一个普遍现象，过去多依赖 LLM judge 评估，难以 scale。本文证明用模型自身的激活，配合一个小 probe，就能稳定检测 commitment——而且通过"冻结 base"巧妙规避了 RL-gaming。对未来 reasoning RL pipeline 是一个"反 hallucination 思考"的 systematic 框架，意义远超技术工程。Intuit 单作者完成，工程性非常强。
