1️⃣ 背景与动机

Looped Transformer / Universal Transformer 通过权重共享 + 循环细化潜在表征带来语言建模和推理增益，但有三大顽疾：(i) 显存随循环深度 \(T\) 线性增长（BPTT 全展开）；(ii) 训练高度不稳定；(iii) 推理深度被锁死，无法对难题"多想几步"。CoT 通过显式 token 写出"思考过程"绕开这些问题，但被迫把 thinking 路由到离散 token 通道——即使其本质应当是潜在表征的 refinement。Attractor Models 提出："让循环成为模型可以学会内化的计算"。

2️⃣ 核心方法详解

① Backbone + Attractor 双模块架构

(a) Backbone（非循环 Transformer）一次前向给出初始嵌入提议 \(\hat z_0\)；(b) Attractor module \(g_\phi\) 求解不动点（fixed point）：

$$ z^* \;=\; g_\phi(z^*,\; \hat z_0) $$

解出 \(z^*\) 后再解码为 token 概率分布。这是 DEQ（Deep Equilibrium）思路与 Looped Transformer 的合流。

② 隐式微分（Implicit Differentiation）—— 训练显存与深度无关

训练时不展开循环，只对不动点方程求隐式雅可比：

$$ \frac{\partial z^*}{\partial \theta} \;=\; (I - J_g)^{-1}\,\frac{\partial g_\phi}{\partial \theta} \bigg|_{z=z^*} $$

用 Neumann 级数 / 线性 CG 求解。训练显存仅取决于 \(z^*\) 大小，与有效循环深度无关——这是 looped Transformer 此前最大痛点的根本性突破。

③ Equilibrium Internalization（均衡内化）— 论文亮点现象

论文发现一个全新现象：fixed-point 训练会把 backbone 的初始 embedding \(\hat z_0\) 拉近不动点 \(z^*\)。因此推理时即使跳过 attractor solver（只用 backbone 一次前向），性能下降轻微。这意味着模型学会了"内化"循环——把"应当反复细化的事"在单次前向中就做掉。

④ 自适应推理深度（Adaptive Inference）

因 attractor 是 contraction 算子，推理时按收敛误差 \(\|z^{(k+1)}-z^{(k)}\|<\epsilon\) 自适应停止。难题多迭代、易题少迭代，且若开启 equilibrium internalization 则可直接 0 步迭代——给了 latency-quality 的多种 deployment 选项。

3️⃣ 深度案例

Sudoku-Extreme：27M Attractor vs 万亿前沿模型
  • 训练设定：仅用 ~1000 个 Sudoku-Extreme example，模型仅 27M 参数。
  • 结果：Sudoku-Extreme 91.4% / Maze-Hard 93.1%（论文摘要明示）。
  • 对比基线：Claude / GPT-o3 等 frontier 模型在这些 hard reasoning 任务上"完全失败"（论文原文："fail completely"）；专门的 recursive reasoner（如 TRM）在同等参数量级"collapse at larger sizes"。
  • 启示：可学习的不动点细化是 small-data + small-model + hard-reasoning 三难场景的解。

5️⃣ 消融与讨论

消融：(a) 关闭 equilibrium internalization → 推理时必须保留 solver，latency 翻倍但精度无明显损失；(b) 用全展开 BPTT 训练 → 显存爆炸 OOM 且训练发散；(c) attractor 模块过浅 → contraction 不成立、无法收敛。局限：CG 内迭代引入 ~20-40% 推理 latency；attractor 存在性 / 唯一性需 contraction 正则；当前实验主要在中小模型，超大规模（70B+）可扩展性待验证。

6️⃣ 结论与启发

把 DEQ 思想正式引入语言推理 backbone 是 2025-2026 年迭代细化派路线的一个里程碑。"Equilibrium Internalization"是本文最有趣的发现——它暗示循环 / 反复 reasoning 可以被压缩进单次前向计算，这对推理 LLM 的部署有直接价值。对 frontier 推理任务（数独、迷宫等）的"小模型暴打大模型"结果尤其惊艳。
