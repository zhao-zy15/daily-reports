Medical-2 · MedMemoryBench: Benchmarking Agent Memory in Personalized Healthcare

📅 2026-05-12 🏛 Zhejiang University, Ant Group, Alibaba Group 👤 Yihao Wang, Haoran Xu, Renjie Gu, Yixuan Ye, ... Jinjie Gu (13 authors) arXiv:2605.11814
MedicalAgent MemoryHealthcare

1️⃣ 背景与动机

蚂蚁集团（Ant Group）运营着一个服务数千万 active 用户的健康管理 agent。在生产环境中观察到：(i) 现有 agent memory benchmark（LoCoMo、LongMemEval、MemoryAgentBench 等）都是 open-domain 对话，无法捕捉医疗高 stakes 复杂度；(ii) 真实部署中"missed retrieval"和"over-retrieval"频繁发生；(iii) 静态 evaluation 协议无法反映"动态记忆累积"。本文出于工业一手痛点，构建首个面向个性化健康 agent 记忆的压力测试 benchmark。

2️⃣ 核心方法详解

① Human-Agent 协作数据合成 pipeline

合成 ~2,000 sessions / 16,000 interaction turns 的长时医疗轨迹。基于临床有依据的合成患者原型（patient archetypes）：人口学 + 既往史 + 用药 + 检验结果。LLM 生成对话流，专家审核所有关键医疗事实 ground truth。

② "Evaluate-While-Constructing"（流式评估）协议

不同于传统"先构建完整记忆再评估"，本文提出流式协议：每 \(\Delta = 10\) 个 session 触发一次 checkpoint 评估：

$$ \text{StreamEval}(p,\Delta) = \bigcup_{k=1}^{\lfloor N/\Delta \rfloor} \mathrm{Evaluate}(M_{k\Delta},\;Q_{k\Delta}) $$

且 query 的所有 source knowledge points 必须在 checkpoint 之前出现，防止 future information leakage：

$$ \forall q\in Q_t:\;\max\{\text{session\_id}(kp) : kp \in \mathrm{src}(q)\} \leq t $$
③ 六大查询类型（Query Types）

论文设计了 6 类 query 测试不同 memory 能力：(EEM) Event Extraction & Memorization、(TLA) Temporal Logic & Aggregation、(SUA) State Update Awareness、(MQ) Medical Q&A、(IG) Information Grounding、(MCD) Multi-Cycle Diagnosis。每类专门考察记忆 system 的一种 failure mode。

④ Memory Saturation 现象的形式化

论文形式化并系统研究了"memory saturation"——持续信息流入会主动降级 retrieval / reasoning robustness。在 Efficient（纯医疗轨迹）和 Mixed（注入 auxiliary 噪声 session）两种 setting 下，发现所有 11 个主流记忆方法都在 Mixed setting 下显著掉点，揭示噪声 resilience 是关键短板。

3️⃣ 深度案例

SUA（State Update Awareness）问题示例
  • Session 1: 患者报告每天服用 metformin 500mg。
  • Session 5: 医生调整剂量到 1000mg。
  • Session 15: Agent 被问"患者当前的 metformin 剂量是多少？"
  • 错误答案: 500mg（基于 Session 1，未更新）。
  • 正确答案: 1000mg。
  • 评测结果：MemOS Efficient setting SUA accuracy 仅 66%；Mixed setting 进一步降到 58%——saturation 把它打掉。

5️⃣ 消融与讨论

关键观察：(a) 复杂医疗推理（MCD multi-cycle diagnosis）所有方法都低于 35%——这是 frontier；(b) Mixed 与 Efficient 差距越大说明噪声 robustness 越差，A-Mem 差距 15+ 点暴露其在生产环境的脆弱性；(c) 6 类 query 中 IG 和 MCD 是最大 bottleneck。局限：(i) 数据是合成而非真实临床（隐私限制）；(ii) "patient archetype"无法覆盖所有 corner case；(iii) 评估依赖 LLM judge，复杂临床判断的可信度需要更多验证。

6️⃣ 结论与启发

MedMemoryBench 把"工业生产 agent 在跑什么坑"做成了学术 benchmark——这是工业 AI 团队对学界最大的贡献之一。"Memory Saturation"作为现象首次被形式化，让 agent memory 社区从"对话记得多少"转向"压力下还记得几成"。论文已开源数据 + benchmark (github.com/AQ-MedAI/MedMemoryBench)，对所有医疗 agent 研究者直接可用。对清华俞声实验室与 DR.EHR 研究的启发：EHR retrieval 同样应进入"streaming evaluation"和"saturation aware"的范式。
