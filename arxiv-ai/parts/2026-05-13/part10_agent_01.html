Agent-1 · OLIVIA: Online Learning via Inference-time Action Adaptation for ReAct Agents

📅 2026-05-11 🏛 UC San Diego, UIUC, Adobe Research 👤 Sheldon Yu, Junda Wu, Xintong Li, ... Jiawei Han, Jingbo Shang, Julian McAuley arXiv:2605.11169
AgentReActOnline Learning

1️⃣ 背景与动机

部署后的 LLM agent 在重复任务流（tool calls、API 编排、企业工作流）中会遇到相似的 decision state，但静态 prompt 不学习——同样的错误反复犯，浪费 budget。已有 inference-time adaptation 方法（reflection、CLIN 记忆增强）通过修改 prompt context 间接调整行为，但它们：(i) 没有显式 decision layer对候选 action 打分；(ii) 不能表达 uncertainty；(iii) 难以从 step-level feedback 做 online update。本文提出 OLIVIA：把 ReAct agent 的最后一层 action selection 视为 contextual linear bandit，做轻量 online RL。

2️⃣ 核心方法详解

① ReAct Action Selection 形式化为 Contextual Linear Bandit

每个 step 的 hidden state 作为 context \(x_t \in \mathbb{R}^d\)（来自冻结的 LLM），候选 action 集合 \(\mathcal{A}_t\)。Reward 由 verifier / 环境给出。论文证明在 LLM 输出层下，action 价值的线性近似是合理的：

$$ \hat r(x_t,a) \;=\; \langle \theta, \phi(x_t,a)\rangle $$
② Hidden-State Decision Layer + LLM-aligned Initialization

构建一个轻量的可训练打分层（线性 head），初始化用 LLM 自身在该任务的输出概率（softmax logits）作为初始 \(\theta_0\)。这保证 cold-start 时不丢弃 LLM 的先验知识，online update 只是在其上做residual 校正。

③ UCB 探索（LinUCB）

每步选择 action 时不是贪心 argmax，而是用 LinUCB 加置信带：

$$ a_t^* \;=\; \arg\max_{a}\Bigl[\hat r(x_t,a) \;+\; \alpha\sqrt{\phi(x_t,a)^\top A_t^{-1} \phi(x_t,a)}\Bigr] $$

其中 \(A_t = \sum_{s\le t}\phi_s\phi_s^\top + \lambda I\)。这给 agent 显式的 exploration mechanism——遇到不确定 action 会尝试，而不是反复犯错。

④ Step-Level Feedback + Online Update

每步执行后从 environment 拿到 reward，做 ridge regression closed-form 在线更新 \(\theta_{t+1}\)：

$$ \theta_{t+1} = (X_t^\top X_t + \lambda I)^{-1} X_t^\top r_t $$

这是 \(O(d^3)\) 的轻量更新（\(d\) 是 hidden dim，可降维），完全不动 backbone LLM 参数。整个 agent 像一个"装了 RL controller 的 frozen 大脑"。

3️⃣ 深度案例

ToolBench：从 ReAct 0.520 提升到 OLIVIA 0.585（+12.5% relative）
  • 设置：Qwen3-4B 作为 backbone，episode stream 持续到来；初始 100 episode 用作 warm-up，后续在线调整。
  • Static baseline (ReAct)：F1 = 0.520（每 episode 独立、无学习）。
  • CLIN（episodic memory）：F1 = 0.553，渐进改进但慢。
  • OLIVIA：F1 = 0.585，收敛更快、更高，纯靠 step-level reward 自适应。
  • 合成数据集 sanity check：用线性奖励的 \(\theta^*\) ground-truth bandit 验证；OLIVIA cumulative regret 和参数估计误差 \(\|\hat\theta_t - \theta^*\|_2^2\) 都收敛得最快——证明不是巧合而是真的恢复了 reward 结构。

5️⃣ 消融与讨论

消融：(a) 关闭 LLM-aligned init（随机 \(\theta_0\)）→ 收敛慢；(b) 关闭 UCB 用 \(\epsilon\)-greedy → 收敛慢且 final F1 略低；(c) exploration coefficient \(\alpha\)：在合理区间稳定，过小欠探索、过大过度探索都不好；(d) reward 设计（dense vs sparse step reward）：dense 收敛更快。局限：(i) 假设 action 价值的线性近似合理——对极端非线性任务可能失效；(ii) 需要 step-level reward signal，对 long-horizon sparse-reward agent 适用度有限；(iii) 不更新 backbone，长期改进上限受限于 backbone 表达能力。

6️⃣ 结论与启发

OLIVIA 给出了 "frozen LLM + 显式可学 decision layer"这条 deployment-friendly 的路径——既不动 backbone（避免 GPU 训练），又有真正的online learning + uncertainty quantification。这与"prompt tuning"和"全参数微调"形成正交补充，对企业级 agent 部署（每天数百万次调用）极为现实。论文的另一价值是在 4 个 benchmark + 2 个 backbone 上做了完整对照，统计严谨。
