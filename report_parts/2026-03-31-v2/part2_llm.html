Model Capability Dominates: Inference-Time Optimization Lessons from AIMO 3

👤 Natapong Nitarach 📅 2026-03-29 · arXiv:2603.27844 🏛 Independent Researcher (Kaggle) LLM Inference-Time Math Reasoning

📌 背景与动机

在数学推理竞赛中，多数投票（majority voting）是提升 LLM 准确率的标准手段，但相关误差（correlated errors）限制了投票的有效样本量。一个自然的改进思路是为不同投票者分配结构差异化的推理策略（如倒推法、分类求解法等），以去相关误差。然而，这种"多样化提示词混合器"（Diverse Prompt Mixer）是否真的有效？本文通过 AIMO 3 竞赛的 50 道 IMO 级别数学题、23+ 实验、3 个模型的系统性实验，给出了否定的答案。

🔬 核心方法详解

① 系统架构：单 H100 上的并行推理管线
使用 gpt-oss-120b（116.8B 总参/5.1B 激活参数的 MoE 模型），通过 vLLM 引擎以 FP8 量化部署在单张 H100 80GB 上。每个问题启动 \(N=8\) 个并行推理线程，共享系统提示但使用不同随机种子（\(T=1.0\)）。每个线程可通过 tool_call 调用 Python/SymPy 沙箱执行多轮工具辅助推理。预算分配采用均匀策略：\(\text{budget}_i = \text{remaining\_time} / \text{remaining\_problems}\)。
② 多样化提示词混合器设计
设计了四种互补的推理策略：Original（分步探索+验证）、Small Cases First（枚举 \(n=1,2,3,\dots\) 发现模式后证明）、Work Backwards（从约束倒推缩小搜索空间）、Classify Then Solve（识别题型后匹配技术）。测试了从 Conservative（5+1+1+1）到 Equal（2+2+2+2）等多种配比。
③ 早停与熵加权投票
当 8 个线程中有 4 个达成相同非平凡答案（\(>1\)）时触发早停。最终答案通过熵加权多数决选择：\(w = 1 + 1/(\text{entropy} + 0.1)\)，低熵（高置信度）线程权重更大。
④ 误差相关性分析：为什么多样化无效
在 \(T=1.0\) 下测量成对错误相关性 \(\rho\)，发现平均值为 \(-0.258\)（接近零甚至略负），表明高温采样已经充分去相关。引入弱策略在降低 \(\rho\) 的同时更大幅度地降低了单次准确率 \(\hat{p}\)，得不偿失。模型能力差距（\(\hat{p}=0.69\) vs \(0.46\)）造成的 17 分差距远远大于所有推理时优化的 ±2 分波动。

GeoBlock: Inferring Block Granularity from Dependency Geometry in Diffusion Language Models

👤 Lipeng Wan, Junjie Ma, Jianhui Gu, Zeyang Liu, Xuyang Lu, Xuguang Lan 📅 2026-03-31 · arXiv:2603.26675 🏛 西安交通大学 LLM Diffusion LM Efficient Inference

📌 背景与动机

块扩散（Block Diffusion）通过并行 refine 一组 token 实现高效文本生成，但块大小的选择至关重要。现有方法依赖固定规则或置信度启发式，忽略了决定哪些 token 可以安全并行更新的依赖几何结构。直觉上，具有强因果顺序的区域需要顺序更新，而语义内聚的区域可以并行处理。

🔬 核心方法详解

① 依赖几何视角：三分区分解
在解码前沿位置 \(y\)，将索引划分为三个集合：\(H=\{1,\dots,x\}\)（已处理历史）、\(C=\{x+1,\dots,y\}\)（候选块）、\(F=\{y+1,\dots\}\)（未来）。核心思想是分析注意力矩阵 \(A\) 在 \((H,C,F)\) 分区上的子块，特别关注内部耦合 \(A_{C,C}\)、过去锚定 \(A_{C,H}\) 和未来泄漏 \(A_{C,F}\)。
② 依赖闭合得分
定义依赖强度 \(S_{U \to V} = \sum_{i \in U} \sum_{j \in V} A_{ij}\)，构建闭合得分： $$\text{Score}(x) = \frac{S_{C \to C} + \alpha \cdot S_{C \to H}}{S_{C \to C} + \alpha \cdot S_{C \to H} + S_{C \to F}}$$ 其中 \(\alpha \in [0,1]\) 为锚定系数。得分越高表示候选块越自包含——内部凝聚力强、依赖可在当前前缀下解决、对未来泄漏最小。
③ 多层显著注意力融合
对选定的语义成熟层 \(\mathcal{L}\)（如第 16/21/26 层），提取 ROI 注意力，按总注意力质量选择 top-\(k\) 显著头，加权融合：\(A \leftarrow A + w_\ell \cdot \bar{A}^{(\ell)}\)。实验表明中高层（语义成熟层）效果最佳。
④ 右移边界选择
找到最大得分 \(S_{\max}\)，设阈值 \(\tau = S_{\max} - \delta\)，选择满足 \(\text{Score}(x) \geq \tau\) 的最大边界 \(x^\star\)。右移容忍度 \(\delta\) 控制块扩展的激进程度——较大的 \(\delta\) 产生更大的块但可能降低准确率。

Adaptive Block-Scaled Data Types (IF4)

👤 Jack Cook, Hyemin S. Lee, Kathryn Le, Junxian Guo, Giovanni Traverso, Anantha P. Chandrakasan, Song Han 📅 2026-03-30 · arXiv:2603.28765 🏛 MIT, NVIDIA LLM Quantization 4-bit

📌 背景与动机

NVFP4 已成为 4-bit LLM 量化的主流格式，但其固有局限——高接近最大值区域的量化误差大、缩放因子符号位未利用——限制了精度。现有改进方案（如 4/6 缩放法）虽能降低误差，但以牺牲 42.9% 动态范围为代价。能否在保持完整动态范围的同时进一步降低量化误差？

🔬 核心方法详解

① IF4 格式设计：FP4/INT4 自适应选择
核心创新：利用 NVFP4 缩放因子（E4M3）的未使用符号位作为模式指示器。每组 16 个值可在 FP4（E2M1，最大值 6）和 INT4（最大值 7）两种编码间自适应选择。FP4 适合有异常值的稀疏分布（如原始权重），INT4 适合均匀分布（如 Hadamard 变换后的数据）。
② 量化流程
全局缩放因子：\(\alpha^{\text{FP32}} = \frac{\max(|\mathbf{X}|)}{M^{\text{FP4}} \times M^{\text{FP8}}}\)。块级缩放因子：\(\Delta_i^{\text{FP8}} = \frac{\max(|\mathbf{X}_{16i \dots 16(i+1)}|)}{\alpha \cdot M^{\text{FP4}}}\)。选择机制：计算两种模式的均方量化误差，取较小者： $$\bar{\mathbf{X}}_i = \begin{cases} \bar{\mathbf{X}}_i^{(\text{INT})}, & \text{if } E^{(\text{INT})} < E^{(\text{FP})} \\ \bar{\mathbf{X}}_i^{(\text{FP})}, & \text{otherwise} \end{cases}$$
③ 6/7 缩放法解决动态范围对齐
INT4 最大值为 7 而 FP4 为 6，为保持统一动态范围 [0,6]，INT4 路径量化前乘以 \(7/6\)、反量化后乘以 \(6/7\)。这确保了所有组共享相同的动态范围（100%），无需像 4/6 法那样牺牲范围。
④ 硬件实现
在 28nm CMOS 上设计了 IF4 MAC 单元，延迟仅比纯 NVFP4 MAC 高 4.7%。由于 AI 负载通常显存受限而非计算受限，这种微小的逻辑开销对整体吞吐量影响可忽略。

💡 深度案例

案例：Hadamard 变换后的量化选择
  1. 原始数据：一组 16 个权重值呈稀疏分布（1 个异常值 5.8，其余 <0.5）→ FP4 模式最优
  2. Hadamard 变换后：同组数据变为均匀分布（值集中在 1.2-2.8）→ INT4 模式最优
  3. IF4 自适应：原始组选 FP4（MSE = 0.004），变换组选 INT4（MSE = 0.003），综合 MSE 比固定 NVFP4 降低 31%

TAPS: Task Aware Proposal Distributions for Speculative Sampling

👤 Mohamad Zbib, Mohamad Bazzi, Ammar Mohanna, Hasan Abed Al Kader Hammoud, Bernard Ghanem 📅 2026-03-27 · arXiv:2603.27027 🏛 KAUST LLM Speculative Decoding Efficient Inference

📌 背景与动机

投机采样（Speculative Sampling）使用小型 draft 模型快速生成候选 token，再由大型 target 模型批量验证，以加速 LLM 推理。然而，标准 draft 模型的提议分布（proposal distribution）是任务无关的——无论是代码生成还是创意写作，都使用同一个 draft 模型。这导致在特定任务上接受率低、加速效果受限。

🔬 核心方法详解

① 任务感知提议分布框架
TAPS 的核心思想是让 draft 模型的提议分布适应当前任务的特性。定义投机采样的接受率为 \(\text{AR} = \mathbb{E}_{x \sim q}\left[\min\left(1, \frac{p(x)}{q(x)}\right)\right]\)，其中 \(p\) 为 target 分布，\(q\) 为 proposal 分布。TAPS 通过最小化 \(q\) 与 \(p\) 在目标任务上的 KL 散度来提升接受率。
② 轻量级任务适配器
在 draft 模型顶部添加可训练的 LoRA 适配器（rank 8-16），在少量任务样本上微调以拟合 target 模型在该任务上的输出分布。微调目标：\(\min_\phi \mathbb{E}_{x \sim \mathcal{D}_{\text{task}}} \left[ D_{\text{KL}}(p_{\text{target}}(\cdot|x) \| q_\phi(\cdot|x)) \right]\)。适配器仅需数百到数千个样本即可收敛。
③ 动态路由机制
维护多个任务特化的适配器（代码、数学、对话等），通过轻量级分类器（基于输入 embedding 的线性探针）在推理时动态选择最匹配的适配器。路由延迟 <0.1ms，可忽略。
④ 无损保证
TAPS 保留了投机采样的无损性质（lossless property）：最终输出分布始终精确等于 target 模型分布。适配器仅改变 draft 的提议效率，不影响输出质量。理论证明：任务感知的 \(q_\phi\) 严格优于或等于原始 \(q\)，即 \(\text{AR}(q_\phi) \geq \text{AR}(q)\)。

💡 深度案例

  4. 关键：代码任务的 token 分布高度集中（关键字、缩进、分号），适配器能快速学习这种分布特征

Arithmetic OOD Failure Unfolds in Stages in Minimal GPTs

👤 Seine A. Shintani 📅 2026-03-27 · arXiv:2603.26828 🏛 Chubu University, Nagoya University LLM Mechanistic Interpretability Arithmetic

📌 背景与动机

最小 GPT 在穷举 2 位数加法（10000 对）上训练后，虽然覆盖了所有局部数字转换规则 \((d_a, d_b, c_{in}) \to (d_{out}, c_{out})\)，但泛化到 3 位数加法时仍然失败。这种失败不是简单的"遗漏规则"，而是一个分阶段展开的过程——每个阶段有独立的失败机制和修复方式。

🔬 核心方法详解

① 阶段一：布局屏障
即使相同数值仅改变位数布局（2 位→3 位字符串），性能从 92% 骤降至 <0.1%。原因：模型学习了绝对位置编码，位置偏移导致角色混乱。修复：仅混合布局暴露（在训练中看到 3 位格式的低范围加法）能有效削弱此屏障，结构化位置偏差无效。
② 阶段二：进位语义屏障
修复布局后，百位位置表现为"进位标志"（carry flag）而非真正的百位数值。证据：Logit 边际从 +288（偏好标志数字）转为 -110（偏好真实数字）后性能才恢复。注意力模式也发生转变：对低位的关注从 0.64-0.70 降至 0.35-0.44，表明模型开始正确利用位置值信息。
③ 阶段三：条件重组屏障
模型能正确计算高位状态，但无法将正确的低位尾数绑定到该状态上。关键发现：tailhigh（在正确高位状态下看到低位数据）的 Exact Match 为 0.541，显著优于仅高位（0.310）或仅低位（0.396）。重组指标 \(P(\text{exact} \mid \text{high2\_correct})\) 从 0.446 提升至 0.736。
④ 阶段四：进位条件的十位符号误差
最终残差呈高度结构化：无进位时 (\(c_2=0\)) 十位欠射 (\(\Delta=-2.3\))，有进位时 (\(c_2=1\)) 十位过射 (\(\Delta=+1.3\))。tenspolarity 干预将 Exact Match 从 0.664 提升至 0.822，显著收缩偏差。
