Medical LLMs

Can LLM Agents Generate Real-World Evidence? Evaluating Observational Studies in Medical Databases
⚕️ Medical RWE Dubai Li, Yuxiang He, Yan Hu, Yu Tian, Jingsong Li arXiv:2603.22767
背景与动机

观察性研究（observational studies）是产生真实世界证据（Real-World Evidence, RWE）的重要途径，但在真实医疗数据库上执行这些研究是一个高度开放性的任务——需要在队列构建（cohort construction）、统计分析（analysis）、报告撰写（reporting）之间做出连贯的决策链。现有对 LLM Agent 的评估往往只考察孤立步骤或单一问题的回答，忽略了生成完整"证据包"的系统性质量。核心问题是：LLM Agent 能否端到端地在真实医疗数据库上执行观察性研究，并产出可信的证据报告？

核心方法详解
① RWE-bench 基准构建

RWE-bench 基于 MIMIC-IV 数据库构建，参考已发表的同行评审观察性研究，构建了 162 项研究任务。每项任务包含：

  • 研究方案（Protocol）：定义纳排标准、暴露/结局变量、分析方法（如 Cox 回归、Kaplan-Meier 曲线）
  • 参考标准：来自原始论文的关键统计结果（HR、CI、p-value）
  • 评估维度：不仅检查最终数值，还评估证据包的内部一致性（如队列大小与报告是否匹配）

任务涵盖心血管、重症监护、感染性疾病等多个临床领域。

② 树状证据包生成范式

Agent 需要生成一个树状结构的证据包 \(\mathcal{E}\)，层次为：

$$\mathcal{E} = \{\text{Protocol} \rightarrow \text{Cohort} \rightarrow \text{Analysis} \rightarrow \text{Results} \rightarrow \text{Report}\}$$

每一层的输出是下一层的输入，且必须保持内部一致性：

  1. Protocol → SQL 查询：将纳排标准转化为 MIMIC-IV 上的 SQL 查询
  2. SQL → 队列数据：执行查询获取患者队列
  3. 队列 → 统计分析：应用指定的统计方法（生存分析、倾向性评分匹配等）
  4. 分析 → 报告：将结果整理为结构化医学报告
③ 自动化队列评估方法

为快速定位 agent 的错误来源，提出了自动化队列验证：

  • Schema Validator：检查 SQL 查询是否符合 MIMIC-IV 的表结构和命名规范
  • Cohort Size Check：队列大小应在参考论文的 \(\pm 20\%\) 范围内
  • Covariate Balance Check：倾向性评分匹配后各协变量的标准化均值差 \(|SMD| < 0.1\)
  • Result Plausibility：Hazard Ratio 的方向（保护/风险因素）应与参考一致

通过逐层验证，可以精确定位是"SQL 错误"还是"统计方法选择错误"还是"结果解读错误"。

④ 故障模式分类学

对 6 个 LLM × 3 个 Agent 框架的失败案例进行了系统化分类：

  1. Schema Hallucination（28%）：Agent 引用了 MIMIC-IV 中不存在的表或字段
  2. Cohort Leakage（19%）：纳排标准执行不严格，将不符合条件的患者纳入
  3. Method Mismatch（22%）：使用了不适合研究设计的统计方法
  4. Numerical Drift（15%）：中间计算步骤的累积误差导致最终结果偏差
  5. Report Inconsistency（16%）：报告中的数值与实际计算结果不匹配
深度案例
💡 实例：Agent 执行"ICU 患者早期活动对死亡率影响"的观察性研究
  • 任务：复现一项研究——ICU 住院 >48h 的患者中，入院 48h 内开始物理治疗是否降低 28 天死亡率
  • 参考结果：HR = 0.72 (95% CI: 0.58-0.89)，早期活动为保护因素
  • GPT-4 + ReAct 框架：成功生成 SQL 查询、构建队列（n=3,847 vs 参考 n=4,012），执行 Cox 回归得 HR = 0.68 (0.54-0.86)——方向正确但偏差略大（cohort 差异导致）

关键发现：(1) Agent 框架的选择影响巨大——同一 LLM 在不同框架下性能差异可达 30%+；(2) SQL 正确率是主要瓶颈，只有 ~65% 的查询能正确映射到 MIMIC-IV schema；(3) 即使 SQL 正确，统计方法选择的正确率仅 ~50%，说明 LLM 对研究方法论的理解有限。局限性：RWE-bench 仅基于 MIMIC-IV（ICU 数据），对门诊数据、多中心数据的泛化性有待验证；39.9% 的最高成功率说明距离临床实用仍有很大差距；自动化评估本身可能存在偏差（如对"合理但不同"的分析方法过于严格）。

CLiGNet: Clinical Label-Interaction Graph Network for Medical Specialty Classification
⚕️ Medical Clinical NLP Pronob Kumar Barman, Pronoy Kumar Barman arXiv:2603.22752
背景与动机

将临床文本自动分类到医学专科（如心内科、骨科、精神科）对于诊疗路由和编码至关重要。然而，广泛使用的 MTSamples 基准存在严重的数据泄漏问题——先前研究在训练-测试划分之前应用了 SMOTE 过采样，导致测试集中混入了合成的训练样本，虚高了准确率。此外，40 个医学专科之间存在极端的类别不均衡（最大/最小类比 181:1），以及标签之间的语义关联（如"心脏外科"和"心内科"高度相关）。

核心方法详解
① 无泄漏基准建立

首先修正数据泄漏问题：严格执行"先划分、后过采样"：

  1. 将 4,966 条记录按 80/20 分层随机划分为训练/测试集
  2. 仅在训练集上应用 SMOTE（如需要），测试集保持原始分布
  3. 对比修正前后：先前报告的准确率 ~95% 降至修正后的 ~25%（Macro F1），揭示了任务的真实难度
② BioClinicalBERT + 标签图网络架构

CLiGNet 包含两个平行分支：

  • 文本分支：使用 BioClinicalBERT 编码临床文本，得到文档表示 \(\mathbf{d} \in \mathbb{R}^{768}\)
  • 标签图分支：构建一个 40 节点的医学专科图，节点特征为标签名称的 embedding，边权重基于两种信号：

$$w_{ij} = \alpha \cdot \cos(\mathbf{e}_i, \mathbf{e}_j) + (1-\alpha) \cdot \mathbb{1}[\text{ICD-10 chapter}(i) = \text{ICD-10 chapter}(j)]$$

两层 GCN 传播标签间的关联信息：\(\mathbf{H}^{(l+1)} = \sigma(\hat{\mathbf{A}} \mathbf{H}^{(l)} \mathbf{W}^{(l)})\)，得到标签表示 \(\mathbf{L} \in \mathbb{R}^{40 \times 768}\)。

③ 逐标签注意力门控融合

将文档表示与标签图表示通过注意力门控融合：

$$\mathbf{g}_j = \sigma\left(\mathbf{W}_g [\mathbf{d}; \mathbf{l}_j] + \mathbf{b}_g\right)$$

$$\hat{y}_j = \sigma\left(\mathbf{W}_o (\mathbf{g}_j \odot \mathbf{d} + (1-\mathbf{g}_j) \odot \mathbf{l}_j)\right)$$

其中 \(\mathbf{l}_j\) 是第 \(j\) 个标签的 GCN 输出表示，\(\mathbf{g}_j\) 是门控权重。这允许模型对不同标签动态调整"文本特征"和"标签关系特征"的贡献比例。

④ 焦点损失 + Platt Scaling 校准

使用焦点二元交叉熵损失处理极端类别不均衡：

$$\mathcal{L}_{\text{focal}} = -\frac{1}{C}\sum_{j=1}^{40} \left[(1-\hat{y}_j)^\gamma y_j \log \hat{y}_j + \hat{y}_j^\gamma (1-y_j) \log(1-\hat{y}_j)\right]$$

其中 \(\gamma = 2\) 使模型更关注困难/稀有类别。训练后使用 Platt Scaling 进行概率校准，通过在验证集上学习标量参数 \(a, b\) 将 logits 映射为校准概率：\(P_{\text{cal}} = \sigma(a \cdot z + b)\)。校准后期望校准误差（ECE）仅 0.007。

深度案例
💡 实例：心脏手术报告的分类

一份包含"冠状动脉搭桥术后...主动脉瓣置换..."的手术记录：

  • 真实标签：Cardiovascular / Cardiothoracic Surgery
  • BioClinicalBERT alone：预测 "General Surgery"（混淆类别）
  • CLiGNet：标签图中 "Cardiothoracic Surgery" 和 "Cardiovascular" 的强连接提升了正确类别的得分，最终正确预测
  • 关键：标签图的 GCN 传播使得模型能利用"如果可能是心血管相关，那么也更可能是心胸外科"这种先验关系

IR & Retrieval

From Questions to Trust Reports: A LLM-IR Framework for the TREC 2025 DRAGUN Track
🔍 IR Trust & Credibility Ignacy Alwasiak, Kene Nnolim, Jaclyn Thi, Samy Ateia, Markus Bink et al. arXiv:2603.23125
背景与动机

在线新闻的可信度评估是一个日益紧迫的问题。TREC 2025 DRAGUN Track 针对此需求设立了两个任务：Task 1 生成用于评估新闻可信度的"关键问题"（critical questions），Task 2 基于检索到的证据生成结构化的可信度报告（trust report）。这要求系统同时具备问题生成、大规模检索、重排序和报告综合生成能力。核心挑战是：如何将 LLM 的生成能力与传统 IR 的检索能力有效结合？

核心方法详解
① LLM 驱动的关键问题生成（Task 1）

给定一篇新闻文章 \(d\)，系统生成一组关键问题 \(\{q_1, \ldots, q_n\}\)，帮助用户评估其可信度。流程包括：

  1. 初始生成：使用 LLM 生成 20+ 候选问题，覆盖来源可靠性、事实准确性、偏见、逻辑一致性等维度
  2. 语义过滤：计算问题间的 pairwise cosine similarity，移除冗余问题（\(\text{sim} > 0.85\)）
  3. 多样性聚类：对剩余问题使用 K-Means 聚类（\(K=8\)），每个 cluster 选择最具代表性的问题

最终输出 8-10 个覆盖多维度的关键问题。

② 多策略查询扩展与检索

为从 MS MARCO V2.1 语料库中检索支持可信度判断的证据，使用多种查询扩展策略：

  • Chain-of-Thought (CoT) 扩展：LLM 对每个关键问题进行推理，生成扩展查询 \(q' = \text{LLM}(\text{CoT\_prompt}, q)\)
  • Entity-based 扩展：从文章中提取关键实体，与问题组合形成复合查询
  • Paraphrase 扩展：生成问题的多种表述变体，增加召回覆盖

使用 BM25 + dense retrieval (E5-large) 的 hybrid 检索，初始返回 top-1000 候选文档。

③ monoT5 重排序 + 领域可信度过滤

对检索到的候选文档进行两阶段精排：

  1. monoT5 相关性重排序：使用 fine-tuned monoT5-3B 对 (query, document) pair 打分，选择 top-50
  2. LLM 相关性判断：对 top-50 进一步使用 LLM 评估相关性（0-3 分），阈值过滤保留 ≥2 分的文档
  3. 领域可信度评估：利用预构建的域名可信度数据集，对来源进行可信度加权：\(\text{final\_score}_i = \text{relevance}_i \times \text{domain\_trust}_i\)
④ 检索增强可信度报告生成（Task 2）

将筛选后的证据文档 + 关键问题 + 原始文章输入 LLM，生成结构化可信度报告：

  • 摘要：一句话概括文章可信度判断
  • 逐问题分析：针对每个关键问题，引用具体证据文档给出分析
  • 可信度评分：1-5 分的综合评分及理由
  • 引用：每个分析点标注来源文档 ID，确保可追溯性

报告生成使用 few-shot prompting + 结构化输出约束（JSON schema 引导）。

SIDReasoner: Reasoning over Semantic IDs Enhances Generative Recommendation
🔍 IR Recommendation Yingzhi He, Yan Sun, Junfei Tan, Yuxin Chen, Xiaoyu Kong et al. arXiv:2603.23183
背景与动机

生成式推荐将序列推荐表述为自回归 token 生成——每个物品用一组离散 token（Semantic IDs, SIDs）表示，LLM 在语言 token 和物品 token 的统一空间中做自回归预测。SID 使得 LLM 可以高效解码大规模物品库。然而，LLM 推理能力的爆发式增长虽然带来了新机遇，但对 SID 的有效推理仍未被探索。核心挑战是：物品 token 对 LLM 来说本质上无意义（不像语言 token 有语义），且面向推荐的推理轨迹极难获取标注。

核心方法详解
① SID-语言对应强化（SID-Language Alignment）

为让 LLM 理解物品 token 的语义，SIDReasoner 第一阶段在丰富的以 SID 为中心的语料上进行多任务训练：

  • SID → Description：给定 SID 序列，生成物品描述文本
  • Description → SID：给定物品描述，预测其 SID 序列
  • SID-SID Relation：给定两个物品的 SID，判断其语义关系（互补/替代/无关）

训练语料由更强的教师模型合成。多任务目标为：

$$\mathcal{L}_{\text{align}} = \sum_{k} \lambda_k \mathcal{L}_k = \lambda_1 \mathcal{L}_{\text{S2D}} + \lambda_2 \mathcal{L}_{\text{D2S}} + \lambda_3 \mathcal{L}_{\text{S2S}}$$

经过对齐训练后，LLM 的 SID embedding 空间与语言 embedding 空间对齐。

② 结果驱动的强化优化（Outcome-Driven RL）

第二阶段使用 RL 优化推荐推理能力。关键设计是不需要显式的推理标注——仅使用推荐结果作为奖励信号：

$$R(\tau, y^*) = \begin{cases} 1 & \text{if predicted SID matches } y^* \\ \text{partial}(\hat{y}, y^*) & \text{if first } k \text{ tokens match} \\ 0 & \text{otherwise} \end{cases}$$

其中 \(\text{partial}\) 提供部分匹配奖励（前缀匹配越长，奖励越高），缓解稀疏奖励问题。使用 GRPO 作为策略优化算法，鼓励模型在生成 SID 之前产生推理 token（如"用户之前买了耳机和手机壳 → 可能对数据线感兴趣 → SID:[34,12,7]"）。

③ 推理增强的解码策略

推理时，SIDReasoner 采用"先推理后预测"的模式：

  1. 模型先生成一段自由形式的推理文本（reasoning trace），分析用户历史行为模式
  2. 基于推理文本，预测目标物品的 SID token 序列
  3. 使用 constrained beam search 确保生成的 SID 对应真实物品（通过 SID trie 约束）

推理文本长度通过 length penalty \(\alpha_{\text{len}}\) 控制，实验中最优长度为 30-50 tokens。

④ 跨域泛化能力

SIDReasoner 的一个意外发现是其跨域泛化能力。在域 A（如电子产品）上训练的推理能力可以迁移到域 B（如书籍）：

$$\text{Transfer Performance} = (1 - \delta) \cdot \text{In-domain} + \delta \cdot \text{Random}$$

实验中 \(\delta \approx 0.15\)，即跨域性能仅损失约 15%——远好于无推理的基线（损失 30-40%）。这表明推理能力捕获了领域无关的用户行为模式（如"互补购买"、"品牌忠诚"等抽象规律）。
