Self-Improvement of Large Language Models: A Technical Overview and Future Outlook

👤 Haoyan Yang, Mario Xerri, Solha Park, Huajian Zhang, Yiyang Feng, Sai Akhil Kogilathota, Jiawei Zhou 📅 2026-03-26 | arXiv:2603.25681 🏛 Independent Research Group LLM Survey

本文提出了一个统一的系统级框架来组织 LLM 自我改进技术。将自改进系统概念化为一个闭环生命周期，包含四个紧密耦合的过程：数据获取、数据选择、模型优化和推理优化，以及一个自主评估层。当模型能力接近人类水平时，人类反馈难以提供足够信号，自我改进成为关键方向。

核心方法详解

① 数据获取（Data Acquisition）

模型自主收集或生成训练数据。包括自指令生成（Self-Instruct）、自博弈数据生成（Self-Play）、基于推理链的自数据合成（Self-CoT Synthesis）。核心思路是让模型利用自身能力产生高质量数据，摆脱对人类标注的依赖。

② 数据选择（Data Selection）

并非所有自生成数据都有效。该阶段通过模型自身的评估能力筛选高质量样本。关键技术包括：基于奖励模型的过滤、基于一致性的验证（多次采样取共识）、以及基于难度梯度的课程选择。数学上可表示为：

\(\mathcal{D}_{\text{selected}} = \{(x, y) \in \mathcal{D}_{\text{gen}} \mid R_\theta(x, y) \geq \tau\}\)

其中 \(R_\theta\) 是模型自身作为评估器打出的分数，\(\tau\) 为质量阈值。

③ 模型优化（Model Optimization）

利用筛选后的数据更新模型参数。涵盖监督微调（SFT on self-generated data）、强化学习（RLHF / RLVR / DPO）、以及直接偏好优化。闭环结构使模型能迭代地 "generate → evaluate → train → generate"。

④ 推理优化与自主评估（Inference Refinement & Autonomous Evaluation）

在推理阶段进一步优化输出质量。关键技术：自一致性解码（Self-Consistency）、思维链自修正（Self-Refinement）、树搜索与验证器引导（Best-of-N with Verifier）。自主评估层贯穿整个生命周期，持续监控各阶段的质量。评估方式包括 LLM-as-Judge、基于规则的验证、以及循环一致性检查。

💡 Deep Dive: 自改进的"天花板"问题
  1. 模式坍缩风险：模型在自数据上反复训练可能导致多样性丧失。论文指出需要外部锚点（如规则验证器）来维持分布健康
  2. 评估可靠性：LLM-as-Judge 存在自偏好偏差（self-preference bias），即模型倾向于给自己生成的内容更高分
  3. 收敛保证：目前缺乏理论上的收敛保证。实践中发现 2-3 轮迭代后增益递减
  4. 未来方向：论文提出"社会化自改进"愿景——多个模型形成生态系统，通过互评、竞争和协作实现群体智能涌现

Closing the Confidence-Faithfulness Gap in Large Language Models

👤 Miranda Muqing Miao, Lyle Ungar 📅 2026-03-26 | arXiv:2603.25052 🏛 University of Pennsylvania LLM Calibration

通过机制可解释性分析发现，LLM 的校准信号和语言化置信度信号虽然都是线性编码的，但彼此正交（余弦相似度 < 0.04）。更关键的是，当模型同时推理问题和输出置信度时，推理过程会破坏置信度方向，称为"推理污染效应"。提出两阶段自适应引导流水线，将校准误差降低 4-7×。

核心方法详解

① 线性探针解耦分析

在模型残差流上训练岭回归探针，分别预测：(a) 金标校准信号（二元正确性标签）和 (b) 语言化置信度分数。关键发现：两种信号各自在残差流中占据独立线性方向，余弦相似度在所有层保持 < 0.04。

这意味着模型"知道"自己何时可能出错（校准方向可预测正确性），但生成过程无法提取这一信号到输出文本中。

② 推理污染效应的发现

三种提示条件对比实验：

  • 纯正确性提示：仅回答问题
  • 纯置信度提示：仅输出置信度，不回答
  • 联合提示：同时回答并输出置信度

发现联合提示下，置信度方向与准确性方向的余弦从 \(+0.26\) 反转为 \(-0.63\)，表明联合推理主动反转了内部知识与外部表达的关系。

③ 对比激活添加（CAA）引导

构建对比激活向量：

$$\mathbf{v}^{(\ell)} = \frac{1}{|\mathcal{Q}|}\sum_{q\in\mathcal{Q}} \left(\frac{1}{|\mathcal{H}_q|}\sum_{k\in\mathcal{H}_q}\mathbf{h}_{q,k}^{(\ell)} - \frac{1}{|\mathcal{L}_q|}\sum_{k\in\mathcal{L}_q}\mathbf{h}_{q,k}^{(\ell)}\right)$$

在生成时注入残差流：\(\tilde{\mathbf{h}}_t^{(\ell)} = \mathbf{h}_t^{(\ell)} + \alpha\hat{\mathbf{v}}^{(\ell)}\)。该向量在 MATH 上计算，可零样本迁移到 MMLU、TriviaQA、TruthfulQA。

④ 两阶段自适应校准流水线

阶段1：用金标校准探针在推理前读取模型内部准确性估计 \(c^*_q\) 作为目标置信度。阶段2：构建传递函数 \(\alpha \mapsto \text{mean confidence}\)，通过单调插值找到引导强度 \(\alpha^*_q = f^{-1}(c^*_q)\)。对过度自信的问题向下引导，对不自信的问题向上引导。

结果：Llama-3.1-8B-Inst 的 ECE 从 14.9 降至 3.7；Mistral-7B-Inst 的 ECE 从 35.1 降至 3.3。

💡 Deep Dive: 推理污染的机制
  1. 正交编码：模型在内部空间里为"我是否正确"和"我应该说多自信"分配了独立的线性子空间
  2. 联合推理的破坏：当同时要求推理+置信度时，推理过程的注意力机制会覆盖（overwrite）置信度编码方向，导致 Cohen's d 从 1.88 降至接近 0
  3. 实际启示：不应让模型在同一次生成中同时给出答案和置信度估计。分两步效果更好

Do LLMs Know What They Know? Measuring Metacognitive Efficiency with Signal Detection Theory

👤 Jon-Paul Cacioli 📅 2026-03-26 | arXiv:2603.25112 🏛 Independent Researcher, Melbourne LLM Metacognition

引入Type-2 信号检测理论框架评估 LLM 的元认知效率。核心指标 M-ratio = meta-d′/d′ 将"模型知道多少"与"模型对自己知道什么的了解程度"分离。关键发现：AUROC2 和 M-ratio 产生完全倒序的模型排名——Mistral 拥有最高 d′ 但最低 M-ratio。

核心方法详解

① M-ratio：元认知效率度量

定义核心指标：

$$M\text{-ratio} = \frac{\text{meta-}d'}{d'}$$

其中 \(d'\) 衡量 Type-1 敏感性（模型区分正确/错误答案的能力），\(\text{meta-}d'\) 衡量元认知信号质量。\(M=1\) 表示最优元认知；\(M<1\) 表示信心信号丢失了决策证据中的信息。

② 归一化对数概率（NLP）作为信心度量

使用归一化对数概率作为连续信心变量：

$$\text{NLP} = \frac{1}{L}\sum_{i=1}^{L}\log p(t_i \mid t_{

将 NLP 值分箱为 \(2 \times K\) 个有序类别（\(K=4\)），构建计数数组，通过极大似然估计拟合 SDT 参数。所有置信区间基于 10,000 次 bootstrap 重采样。

③ 温度解离实验

每个模型在 7 个温度 \(T \in \{0.1, 0.3, 0.5, 0.7, 1.0, 1.5, 2.0\}\) 下回答每个问题，总计 224,000 次试验。发现 Mistral 和 Gemma-2 中 \(d'\) 随温度变化，但 meta-d′ 保持稳定，证明温度改变的是信心策略（Type-2 标准），而非元认知信号质量。

④ 排名倒序现象

AUROC2（传统信心评估指标）和 M-ratio 产生完全倒序：Mistral 在 AUROC2 上第一但 M-ratio 最后。原因是 AUROC2 混淆了"模型多准确"和"模型多了解自己"两种不同能力。对于需要选择性预测的应用（如拒绝回答），应选 M-ratio 高的模型而非 AUROC2 高的。

  1. 各模型弱点不同：Llama-3-Inst 在 Science & Tech 最差（M=0.788），Mistral 在 Arts & Literature 最差（M=0.677）
  2. 指令微调的效果：指令微调将信心策略与元认知信号解耦，使温度操纵只影响策略不影响信号。基础模型不具备这一特性
  3. 实用建议：对于依赖信心的系统（如知识库检索、医疗诊断），应优先选择 M-ratio 高的模型，而非仅看准确率或传统校准指标

S2D2: Fast Decoding for Diffusion LLMs via Training-Free Self-Speculation

👤 Ligong Han, Hao Wang, Han Gao, Kai Xu, Akash Srivastava 📅 2026-03-26 | arXiv:2603.25702 🏛 Red Hat AI Innovation, MIT-IBM Watson AI Lab, Iowa State University LLM Inference

针对分块扩散语言模型（Block Diffusion LLMs）的解码瓶颈，提出S2D2——一种无需训练的自推测解码框架。核心洞察：当块大小减为 1 时，扩散模型退化为自回归模型，因此同一模型可同时充当草稿器和验证器。在 SDAR 上实现 4.7× 加速，准确率提升 4.5 个百分点。

核心方法详解

① 扩散-自回归双角色

分块扩散模型使用块大小 \(B\) 并行生成多个 token。S2D2 的核心发现：同一预训练模型在 \(B=1\) 时等价于自回归模型。因此：

  • 草稿器：标准扩散模式（\(B>1\)），并行提议多个 token
  • 验证器：\(B=1\) 自回归模式，作为序列级批评者

无需额外训练任何模型，实现即插即用。

② 推测验证流程

每个去噪步骤中：(1) 草稿器提议 token \(\hat{x}\) 和概率 \(p\)；(2) 验证器计算验证概率 \(q\)；(3) 以概率 \(\min(1, q_i/p_i)\) 接受。关键公式——局部残差能量：

$$E_i(\hat{x}_i) = -\log q_i + \log p_i$$

低残差能量的 token 更可能被接受，高能量不匹配通过残差重采样修正。

③ 自验证掩码（2LL Trick）

对于位置对齐的扩散 LLM（如 LLaDA, SDAR），构建特殊验证器掩码：

$$\mathbf{M}_{\mathrm{ver}} = \begin{bmatrix}\mathbf{A}_L & \mathbf{0}_L \\ \mathbf{A}_L^{<} & \mathbf{I}_L\end{bmatrix}$$

该掩码允许在单次前向传播中同时获得所有草稿 token 的验证概率，避免逐 token 验证的开销。

④ 轻量级路由策略

验证增加一次额外前向传播，因此需要判断何时值得验证：

  • 最小跨度：当连续被掩码跨度 \(|C_t| \geq \tau_{\text{span}}\) 时触发
  • 分数阈值：估算预期接受前缀长度 \(\hat{K} = \sum_{k=1}^L \prod_{i=1}^k \alpha_i\)，当超过阈值时触发
  • 滞后机制：防止推测/扩散模式间振荡

  1. 零成本自验证：不同于传统推测解码需要单独的小模型，S2D2 重用同一模型的不同块大小模式，零额外参数
  2. 与 EDLM 的对比：EDLM 使用全局多样本重加权，S2D2 使用在线局部修正。S2D2 不需要多次完整解码，计算效率更高

Separate Before You Compress: The WWHO Tokenization Architecture

👤 Kusal Darshana 📅 2026-03-26 | arXiv:2603.25309 🏛 Remeinium Research LLM Tokenization

标准 BPE 分词器对元音附标文字（如僧伽罗语、印地语）造成严重的"token 税"。本文提出三层架构 WWHO 和算法 SGPE，将语言学规则与统计压缩解耦。僧伽罗语减少 61.7% token，上下文窗口扩展 4.38×，保证零语言破坏。

核心方法详解

① Layer 1: Router (Where) — 脚本路由

基于 Unicode 范围的 \(O(N)\) 线性扫描，识别脚本边界。将文本段路由到对应处理器（拉丁文 → BPE，元音附标文字 → LinguisTrie+SGPE）。特殊处理 ZWJ/ZWNJ 控制字符，防止错误分割。

② Layer 2: LinguisTrie (What) — 音节原子化

利用有限状态自动机（DFA）将 Unicode 字符分组为原子音节。形式化为正则语言：

$$S_s = \mathbf{C}(\mathbf{H}\mathbf{Z}?\mathbf{C})^*(\mathbf{P} \mid \mathbf{H})?\mathbf{M}? \mid \mathbf{V}\mathbf{M}?$$

保证零破坏：任何有效音节永不跨多个 token 拆分。在 1.22 亿字符的往返测试中一致性 100%。

③ Layer 3: SGPE (How Often) — 音节感知压缩

SGPE (Syllable-aware Grapheme Pair Encoding) 在原子音节上进行统计合并，而非字节上。与 BPE 的三个关键区别：

  • 音节初始化：初始词汇为完整原子音节，非单字节
  • 边界感知合并：只在音节对之间合并，不跨越音节边界
  • 低频修剪：低频异常标记为 [UNK]，避免词汇表膨胀
④ 统一元词汇表

将 BPE 和 SGPE 的词汇空间映射到连续数组：\(ID_{\text{SGPE}} = ID_{\text{SGPE\_local}} + V_{\text{BPE}}\)，避免 ID 冲突。支持无缝多语言混合分词。上下文窗口容量乘数：\(\text{Capacity} = \frac{\text{Tokens}_{\text{baseline}}}{\text{Tokens}_{\text{SGPE}}}\)。

  1. 成本不平等：同样一句话，僧伽罗语用户使用 GPT 的 API 费用是英语用户的 4-5 倍，因为 token 数多 4-5 倍
