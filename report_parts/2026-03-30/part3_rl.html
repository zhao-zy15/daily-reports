Reaching Beyond the Mode: RL for Distributional Reasoning in Language Models

👤 Isha Puri, Mehul Damani, Idan Shenfeld, Marzyeh Ghassemi, Jacob Andreas, Yoon Kim 📅 2026-03-25 | arXiv:2603.24844 🏛 Massachusetts Institute of Technology (MIT) RL Distributional

标准 RL 后训练将答案分布坍缩为单一模式。本文提出多答案强化学习框架，训练模型在单次前向传播中显式生成多个候选答案，将推理时搜索内部化。在编码任务上 Top-1 准确率从 0.29 提升至 0.49（+69%），token 用量减少 44%。

核心方法详解

① 多答案 RLVR 目标

扩展标准 RLVR：策略 \(\pi_\theta\) 在单次 CoT 中生成 \(K\) 个不同候选答案 \(A = \{a_1, \dots, a_K\}\)，使用集合级奖励：

$$R_{\text{RLVR}}^{\text{multi}}(A, \mathcal{Y}^*) = \sum_{i=1}^K \mathbf{1}[a_i \in \mathcal{Y}^*]$$

该目标统一涵盖标准 RLVR（K=1）、Pass@K、以及完整集合恢复等场景。

② 多答案 RLCR：校准奖励

模型同时生成答案集 \(A\) 和置信度集 \(Q = \{q_1, \dots, q_K\}\)。引入多 Brier 分数：

$$R_{\text{Brier}}^{\text{multi}} = \frac{1}{K}\sum_{i=1}^K (q_i - \mathbf{1}[a_i \in \mathcal{Y}^*])^2$$

完整目标联合激励覆盖率和校准：\(R_{\text{RLCR}}^{\text{multi}} = R_{\text{RLVR}}^{\text{multi}} - R_{\text{Brier}}^{\text{multi}}\)。模型输出可解释为定义了合理答案的分布。

③ 格式奖励与多样性约束

为防止答案重复（模式坍缩），引入格式奖励：要求 \(K\) 个答案互不相同。训练时使用 Qwen3-8B，最大响应长度 1536 token。关键发现：多答案训练使独特答案数量接近独立采样 30 次的水平，但 token 开销仅为后者的 56%。

④ 内部化推理时搜索

传统方法通过 Best-of-K 外部采样提高覆盖率，但产生大量冗余 token。多答案 RL 将搜索过程内部化到模型的单次生成中。效果：在 MBPP 上，多答案 RLVR 的 Top-1 准确率 0.49 vs 单答案 0.29，且总 token 用量更少。模型学会了在思维链中系统探索不同解法路径。

💡 Deep Dive: 打破模式坍缩
  1. 核心问题：标准 RL 只奖励单一正确答案，导致模型只学会生成最常见答案，忽略其他有效解
  2. 医学应用：症状可能对应多个诊断。多答案模型显著提高了鉴别诊断的覆盖率
  3. 代码生成：同一需求可有多种正确实现。模型学会了在 CoT 中"先生成方案A，再生成方案B"
  4. 扩展性：K 从 2 增到 5，正确答案数单调递增且训练稳定

HIVE: Online-Verified Prompt Selection for Efficient RL Training of Large Reasoning Model

👤 Jiahao Wu, Ning Lu, Shengcai Liu, Kun Wang, Yanting Yang, Qing Li, Ke Tang 📅 2026-03-26 | arXiv:2603.25184 🏛 SUSTech, ICML 2026 RL Efficiency

在 GRPO 等算法中，大量 prompt 提供零梯度（过易或过难），造成计算浪费。本文提出 HIVE，一个双阶段数据高效 RL 框架：历史信息粗筛 + prompt 熵在线验证。实现 3.8× rollout 加速、2.2× 总训练时间加速，精度持平或更优。

核心方法详解

① "学习边缘"的发现

通过分析 GRPO 训练中每个 prompt 的梯度贡献，发现最强学习信号集中在中等难度 × 高不确定性的交集（"学习边缘"），且该边缘随训练动态移动。过易 prompt（全对→零方差→零梯度）和过难 prompt（全错→零方差→零梯度）均浪费计算。

② 阶段1：历史信息选择（零成本）

基于奖励轨迹和响应熵的粗筛：

选择概率：\(P_{\text{select}} = \lambda \cdot P_{\text{Rew}}(x_i) + (1-\lambda) \cdot P_{\text{Ent}}(x_i)\)

其中 \(P_{\text{Rew}}(x_i) = p_e^{z_i}\)（\(z_i\) 为连续零方差次数，频繁零方差的 prompt 被指数衰减）。\(P_{\text{Ent}}\) 为归一化历史响应熵，优先选高熵样本。

③ 阶段2：在线 Prompt 熵验证

为解决历史指标陈旧性问题，利用 Prompt 熵作为实时代理：

$$V(x) = \frac{1}{L_p - 1}\sum_{l=2}^{L_p} \mathcal{H}(p_\theta(\cdot \mid x_{

仅需一次前向传播（计算 prompt 本身的 token 级熵），开销 < 0.4%。论文证明 Prompt 熵与响应熵具有排序一致性。选择标准：\(V(x_i) \geq \text{median}(\{V(x)\})\)，始终选前 50% 信息量最大的样本。

④ 理论保证与整合

核心定理：在高概率下，若两个 prompt 的 prompt 熵差异足够大，其排序与真实响应熵排序一致。两阶段层级过滤：历史先验 → 在线熵验证 → rollout。每一步淘汰约 50% 低效用样本，最终仅对约 25% 的原始 prompt 执行完整 rollout。

FB-MEBE: Maximum Entropy Behavior Exploration for Sim2Real Zero-Shot RL

👤 Jiajun Hu†, Núria Armengol Urpí†, Jin Cheng, Stelian Coros 📅 2026-03-26 | arXiv:2603.25464 🏛 ETH Zurich, Max Planck ETH Center RL Sim2Real

在线零样本 RL（无奖励函数预训练策略库）面临数据多样性不足和步态不自然导致无法直接部署到硬件的问题。本文提出 FB-MEBE，通过最大化已实现行为分布的熵引导探索，结合正则化批评者约束步态自然性。在 Unitree Go2 四足机器人上实现零样本 Sim2Real 部署。

核心方法详解

① Forward-Backward (FB) 算法基础

学习前向表示 \(F\) 和后向表示 \(B\)，使后继测度低秩分解：

$$M^{\pi_z}(ds' \mid s, a) \approx F(s, a, z)^\top B(s')\rho(ds')$$

任务嵌入 \(z\) 指定目标，策略 \(\pi_z(s) = \arg\max_a F(s,a,z)^\top z\)。测试时给定任意奖励函数，可直接恢复最优策略。

② 最大熵行为探索（MEBE）

传统无指导探索（均匀采样随机奖励嵌入）导致行为多样性低。MEBE 通过最小密度采样引导探索：

$$z_r^E \propto (q_\psi(z_r^a) + \epsilon)^{-\beta}$$

使用 Normalizing Flows 拟合密度模型 \(q_\psi\)，在已实现行为分布的低密度区域采样探索策略，最大化行为熵 \(H[p^t_{z_r^a}]\)。

③ 正则化批评者

结合步态正则化奖励防止不自然动作：

$$\mathcal{L}_{\text{actor}} = -\mathbb{E}[Q_{\text{FB}}(s,a,z) + \lambda_{\text{reg}} Q_{\text{reg}}(s,a)]$$

正则化奖励 \(r_{\text{reg}} = -2.5 \times 10^{-7} r_{\text{joint\_acc}} - 0.1 r_{\text{action\_rate}} - 0.1 r_{\text{feet\_slide}}\)，惩罚关节加速度、动作变化率和足部滑移。

④ 低维行为空间投影

探索在状态空间的低维投影上进行（如基座速度 \([v_x, v_y]\) 或重力向量 \([g_x, g_y, g_z]\)），聚焦相关行为空间。在 IsaacLab 中使用 Unitree Go2（12 DoF），200Hz 模拟频率。17 个速度跟踪任务 + 17 个姿态任务。

Revisiting On-Policy Distillation: Empirical Failure Modes and Simple Fixes

👤 Yuqian Fu, Haohuan Huang, Kaiwen Jiang, Yuanheng Zhu, Dongbin Zhao 📅 2026-03-26 | arXiv:2603.25562 🏛 CASIA (中国科学院自动化研究所), UCAS RL Distillation

在线策略蒸馏（OPD）在 LLM 后训练中广泛使用，但采样 token 变体存在三种失败模式。本文提出教师 Top-K 局部支持匹配，通过截断反向 KL + Top-p 滚动 + 特殊 token 掩码实现更稳定的优化。多任务训练中数学平均分从 36.6 提升至 41.7。

核心方法详解

① 三种失败模式诊断

采样 token OPD 的致命缺陷：

  • 信号极度不平衡：大多数采样 token 获负奖励（惩罚），正学习信号集中在极少数 token，优化对填充词过度敏感
  • 教师信号不可靠：当 rollout 偏离教师常见前缀时，教师对错误 token 赋高概率（如重复循环、自我重置推理）
  • 分词器不匹配：单 token 比较无法处理学生和教师分词策略差异，语义正确但分词不同的输出被错误惩罚
② 教师 Top-K 局部支持匹配

在每个前缀处，取教师概率最高的 \(K\) 个 token 作为支持集 \(S(c_{i,t}) = \text{TopK}_q(c_{i,t})\)。在支持集内重新归一化：

$$\hat{\pi}_\theta(v \mid c_{i,t}) = \frac{\pi_\theta(v \mid c_{i,t})}{\sum_{u \in S} \pi_\theta(u \mid c_{i,t})}$$

损失函数为所有位置的截断反向 KL 平均：

$$\mathcal{L}_{\text{LSM}} = \mathbb{E}\left[\frac{1}{\sum_i |o_i|}\sum_i\sum_t\sum_{v \in S} \hat{\pi}_\theta(v) \log\frac{\hat{\pi}_\theta(v)}{\hat{q}(v)}\right]$$

③ 三重稳定化措施
  • 支持集归一化：必须的——移除会导致快速崩溃，因为支持集内外质量不可直接比较
  • Top-p 滚动采样：以 Top-p=0.9 采样生成轨迹，避免极低概率 token 带来的噪声
  • 特殊 token 掩码：掩码有问题的特殊 token（如 EOS），减少分词不匹配的假阴性
④ 估计器理论分析

令牌级 OPD 相对于序列级反向 KL 是有偏的，但最坏情况方差界更紧。序列级估计器的方差随序列长度 \(T\) 指数增长，而令牌级方差仅线性增长。因此在长视距 LLM 后训练中，令牌级更实用——但需要用局部支持匹配修补其信号质量问题。

💡 Deep Dive: 为什么单 Token 信号不够
  1. 信号稀疏性：在一个 1000 token 的 rollout 中，可能只有 5-10 个 token 贡献正信号，其余 990+ 个都在惩罚
  2. 前缀偏移：学生模型生成的前缀可能是教师从未见过的，此时教师的 logits 不可靠
  3. 局部支持匹配的直觉：不问"教师更喜欢哪个具体 token"，而问"在教师关心的 K 个 token 中，学生的分布匹配得如何"
