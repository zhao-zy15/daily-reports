Improving Lean4 Autoformalization via Cycle Consistency Fine-tuning

Arsen Shebzukhov · arXiv:2603.24372 · cs.CL · Paper Link

背景与动机

自动形式化（Autoformalization）是将自然语言数学文本自动翻译为 Lean4 等形式证明语言的任务，可以大幅加速 AI 辅助的数学研究与证明验证。然而，这一任务面临的核心挑战是：(1) 高质量的 NL→Lean4 平行数据极度稀缺；(2) 传统监督微调（SFT）依赖于精确的参考翻译，但形式化往往有多种等价表达方式；(3) 课程学习的效果在该任务上存疑。

核心方法详解

① 循环一致性奖励 (Cycle Consistency Reward)

核心创新：不依赖精确的参考翻译，而是利用"循环翻译"的一致性作为奖励信号。对于自然语言输入 \(x\)，模型先将其翻译为 Lean4 代码 \(\hat{y}\)，再将 \(\hat{y}\) 反向翻译回自然语言 \(x'\)。循环一致性奖励定义为：

$$R_{\text{cycle}}(x, \hat{y}) = \cos(\text{Emb}(x), \text{Emb}(x'))$$

其中 \(\text{Emb}(\cdot)\) 是预训练句子嵌入模型的编码函数。奖励值域 \([-1, 1]\)，越接近 1 表示翻译越忠实地保留了原始数学含义。

② GRPO 强化学习训练 (Group Relative Policy Optimization)

使用 GRPO 而非 PPO 来优化策略，避免训练独立的 value function。对于每个输入 \(x\)，采样一组候选翻译 \(\{\hat{y}_1, ..., \hat{y}_G\}\)，计算各自的循环一致性奖励后进行组内归一化：

$$\hat{A}_i = \frac{R_{\text{cycle}}(x, \hat{y}_i) - \mu_G}{\sigma_G}$$

其中 \(\mu_G, \sigma_G\) 是组内奖励的均值和标准差。策略梯度更新为：

$$\nabla_\theta \mathcal{L} = \mathbb{E}\left[\sum_{i=1}^{G} \hat{A}_i \cdot \nabla_\theta \log \pi_\theta(\hat{y}_i | x)\right]$$

GRPO 的关键优势是无需 critic 网络，计算开销仅比 SFT 高约 40%。

③ LoRA 高效微调

基于 Qwen3.5-2B 使用 LoRA 进行参数高效微调，仅更新不到 1% 的参数。LoRA 矩阵分解为：

$$W' = W + \alpha \cdot B \cdot A, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times d}, r \ll d$$

其中 rank \(r = 16\)，缩放因子 \(\alpha = 32\)。在 FineLeanCorpus 数据集上训练，结合循环一致性奖励和 GRPO，使得 2B 参数的小模型也能取得显著的形式化提升。

④ 课程学习对比实验

对比了三种训练方案：(a) SFT + 课程学习（难度 1→10 递进）、(b) SFT 无课程（随机顺序）、(c) GRPO + 循环一致性奖励。结果显示课程学习在该任务上无可测量的收益，而 RL 方案在所有测试集上均显著优于 SFT。这可能因为形式化任务中"难度"不是单调的——简单的数学概念可能对应复杂的 Lean4 语法。

深度案例

📋 案例: 数学定理形式化

输入（自然语言）："For any real number x, if x² + 2x + 1 = 0, then x = -1."

SFT 输出：theorem ex1 (x : ℝ) : x^2 + 2*x + 1 = 0 → x = -1（缺少 := by 部分和 sorry，Lean4 报语法错误）

RL (GRPO) 输出：theorem ex1 (x : ℝ) (h : x^2 + 2*x + 1 = 0) : x = -1 := by nlinarith [sq_nonneg (x+1)]（Lean4 编译通过 ✓）

循环一致性：RL 输出的 \(R_{\text{cycle}} = 0.94\)，SFT 输出的 \(R_{\text{cycle}} = 0.71\)。

消融与讨论

关键消融结果：(1) 将嵌入模型从 all-MiniLM-L6 换为 text-embedding-3-large 使循环一致性提升 0.03，但训练速度降低 2×；(2) GRPO 组大小 \(G\) 从 4 增至 16 时一致性提升 0.04，但 GPU 内存需求翻倍；(3) RL 仅使交叉熵增加 0.011 nats，表明形式化质量的提升几乎不以牺牲流畅性为代价。局限性：当前方法依赖现成的句子嵌入模型质量，对高度专业化的数学领域（如范畴论）的嵌入质量可能不够。

ITPO: Implicit Turn-Wise Policy Optimization for Proactive User-LLM Interaction

Haoyu Wang, Yuxin Chen, Liang Luo et al. · arXiv:2603.23550 · cs.LG · Paper Link · Code

背景与动机

多轮人机协作（如自适应辅导、对话推荐、专业咨询）是交互式 AI 服务的核心。但用强化学习优化这类多轮交互面临两大难题：(1) 可验证的中间奖励极度稀疏——通常只有交互结束后才有明确反馈（如学生是否学会）；(2) 用户响应高度随机，同一策略在不同用户身上效果可能截然不同。现有方法要么依赖 token 级奖励（不稳定），要么完全使用结果奖励（稀疏）。

核心方法详解

① 隐式过程奖励模型 (Implicit Process Reward Model)

ITPO 的核心是从稀疏的结果信号中推导出逐轮的过程奖励。给定一个 \(T\) 轮对话 \(\tau = (a_1, u_1, a_2, u_2, ..., a_T, u_T)\)，结果奖励为 \(R(\tau)\)。ITPO 学习一个隐式价值函数 \(V_\phi\)，使得第 \(t\) 轮的过程奖励可以由差分推导：

$$r_t = V_\phi(\tau_{\leq t}) - V_\phi(\tau_{\leq t-1})$$

其中 \(\tau_{\leq t}\) 是到第 \(t\) 轮为止的对话前缀。价值函数通过 TD 学习训练：

$$\mathcal{L}_V = \sum_{t=1}^{T} \left(V_\phi(\tau_{\leq t}) - \gamma V_\phi(\tau_{\leq t+1}) - R(\tau) \cdot \mathbb{1}[t=T]\right)^2$$
② 逐轮信号的归一化与鲁棒性

与 token 级奖励不同，逐轮信号在更粗粒度上操作，天然具有更低的方差。为进一步增强稳定性，ITPO 引入批次归一化：

$$\hat{r}_t = \frac{r_t - \mu_B}{\sigma_B + \epsilon}$$

其中 \(\mu_B, \sigma_B\) 是 mini-batch 内所有轮次奖励的统计量。这种归一化使得不同长度的对话可以在统一尺度下比较，避免长对话的过程奖励因为更多的累积步数而天然偏高。

③ 与 PPO/GRPO/RLOO 的集成

ITPO 推导出的逐轮奖励 \(\hat{r}_t\) 可以直接替换任何 RL 算法中的奖励信号。以 GRPO 为例，优势函数变为：

$$\hat{A}_t^{\text{GRPO}} = \frac{\hat{r}_t - \mu_G}{\sigma_G}$$

作者证明这种集成在 PPO（使用 \(\hat{r}_t\) 作为 per-step reward）、GRPO（组内归一化）和 RLOO（leave-one-out baseline）三种算法上都一致优于使用结果奖励的对应版本。关键原因：更密集的奖励信号加速了信用分配。

④ 语义对齐验证

为验证推导出的逐轮偏好是否合理，作者设计了人工标注实验：让 3 名标注员对 200 段多轮对话的每一轮标注质量分（1-5），然后计算 ITPO 推导的 \(\hat{r}_t\) 与人类标注之间的 Spearman 相关系数。结果显示 \(\rho = 0.71\)（p < 0.001），证明隐式推导的奖励与人类判断在语义上高度一致。

深度案例

📋 案例: 数学辅导多轮交互

场景：AI 辅导员教学生解一元二次方程 \(x^2 - 5x + 6 = 0\)。

Turn 1 (AI)："你能看出这个方程可以因式分解吗？试试看。" → ITPO \(\hat{r}_1 = +0.82\)（引导性好）

Turn 2 (学生)："我觉得是 (x-2)(x-3)？"

Turn 3 (AI, 无 ITPO)："正确！答案是 x=2 或 x=3。" → \(\hat{r}_3 = +0.15\)（直接给答案，缺乏深化）

Turn 3 (AI, 有 ITPO)："很好！你是怎么找到 2 和 3 的？你能用求根公式验证一下吗？" → \(\hat{r}_3 = +0.91\)（促进深层理解）

结果：有 ITPO 的策略使学生在后续测试中独立解题正确率提升 18.3%。

消融与讨论

消融结果：(1) 移除批次归一化后，PPO+ITPO 的训练方差增大 3.2×，最终性能下降 4.7 分；(2) 将隐式 reward 替换为 token 级 reward，性能下降 6.8 分且训练不稳定；(3) 对话轮数从 3 增至 10 时，ITPO 的收益更显著（Δ从 +5.2 增至 +12.1），说明多轮场景下逐轮信号的价值更大。局限性：当前 ITPO 假设用户行为是固定的（不随 AI 策略改变），未建模用户的自适应行为。

PrefPO: Pairwise Preference Prompt Optimization

Rahul Singhal, Pradyumna Tambwekar, Karime Maamari · arXiv:2603.19311 · cs.CL · Paper Link · Code

背景与动机

提示工程（Prompt Engineering）虽然有效但劳动密集，催生了自动化提示优化方法。然而现有方法（如 TextGrad、MIPRO、GEPA）存在三大问题：(1) 需要标注数据集（往往不可得）；(2) 生成的提示冗长且重复——平均比原始提示长 14.7×或含 34% 重复内容；(3) 存在"提示 hacking"现象，即优化器博弈评估标准而非真正提升质量。

核心方法详解

① 基于偏好的提示评估 (Preference-based Evaluation)

PrefPO 不使用绝对分数评估提示质量，而是使用成对偏好比较。给定两个候选提示 \(p_A, p_B\) 及其对应输出 \(o_A, o_B\)，LLM 判别器按照自然语言标准 \(c\) 表达偏好：

$$\text{Pref}(p_A, p_B | c) = \text{LLM}_{\text{judge}}(o_A, o_B, c) \in \{A, B, \text{tie}\}$$

这种设计受 RLHF 中 Bradley-Terry 偏好模型的启发：人类（和 LLM）更擅长做相对比较而非绝对评分。

② 迭代优化循环

PrefPO 在每次迭代中：(1) 生成多个候选提示变体；(2) 执行任务并收集输出；(3) 通过成对比较选出最佳变体；(4) 将偏好反馈给 LLM 优化器生成下一轮候选。形式化为：

$$p^{(t+1)} = \text{LLM}_{\text{opt}}(p^{(t)}, \text{feedback}^{(t)}, c)$$

其中反馈包含：赢的示例为什么更好、输的示例的具体弱点。这种"元反馈"使优化器不仅知道方向，还理解原因。

③ 提示卫生控制 (Prompt Hygiene)

为解决提示膨胀和重复问题，PrefPO 在优化过程中引入长度和多样性约束：

$$\mathcal{C}_{\text{hygiene}} = \lambda_L \cdot \max(0, |p| - L_{\max}) + \lambda_D \cdot \text{Dup}(p)$$

其中 \(L_{\max}\) 是最大允许长度，\(\text{Dup}(p)\) 是重复段检测率。实验表明 PrefPO 将膨胀和重复问题减少了 3-5×。

④ 无标签优化能力

PrefPO 的一大优势是不依赖标注数据。在无标签设置下，LLM 判别器完全基于自然语言标准 \(c\) 进行成对比较。实验显示无标签的 PrefPO 在 9 个 BBH 任务中的 6 个上紧追有标签版本（差距 < 2%），这意味着只需一个初始提示和一段文字描述期望即可开始优化。

深度案例

📋 案例: BBH-Logical Deduction 任务优化

初始提示："Please solve the following logical deduction problem."

TextGrad 优化后："You are an expert in logical reasoning. Please carefully analyze each premise step by step. Consider all possible orderings. Think about contrapositive implications. Use systematic elimination..."（膨胀至 278 tokens，重复率 41%）

PrefPO 优化后："Solve the logical deduction. List each constraint, then eliminate impossible orderings one by one."（仅 19 tokens）

BBH 准确率：TextGrad = 84.2%，PrefPO = 86.7%（更短的提示反而更好）。

提示 hacking 率：TextGrad = 86%，PrefPO = 37%。

消融结果：(1) 移除提示卫生约束后膨胀率从 3.1× 升至 11.2×，说明无约束的 LLM 优化器天然倾向于生成冗长提示；(2) 将判别器从 GPT-4 换为 GPT-3.5 仅导致 BBH 下降 1.8 分，成本降低 90%；(3) 迭代次数在 5-8 次时最优，超过 10 次出现过拟合。局限性：PrefPO 的效果高度依赖判别器的质量——如果判别器对某类任务有系统性偏差，优化方向也会偏移。
