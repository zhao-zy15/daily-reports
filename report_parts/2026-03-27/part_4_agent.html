LLM-Driven Reasoning for Constraint-Aware Feature Selection in Industrial Systems (MoFA)
📋 arXiv:2603.24979 👤 Yuhang Zhou, Zhuokai Zhao, Ke Li 等 17 位作者 📅 2026-03-26 🤖 Agent
背景与动机

大规模工业 ML 系统的特征选择通常依赖于统计启发式（如互信息、Lasso），但这些方法忽略了生产环境中的操作约束：延迟上限、特征组依赖、特征更新频率、合规要求等。标注数据的稀缺使传统方法更难应用。MoFA (Model Feature Agent) 提出让 LLM 作为"特征工程师"，利用语义理解和定量信息执行约束感知的顺序特征选择。

核心方法详解
①

结构化特征信息聚合

为 LLM 构建结构化 Prompt，整合四类信息：
(a) 特征定义：每个特征的语义描述（如"user_7d_click_count: 用户近7天点击次数"）
(b) 重要性评分：从已有模型提取的 SHAP 值或 feature importance
(c) 特征相关性矩阵：Pearson 相关或 mutual information 的 Top-k 关系
(d) 元数据约束：特征组（如"用户画像"组有 50ms 延迟，"实时行为"组有 200ms 延迟）
LLM 综合这些信息生成选择理由，格式为：$$\text{Select}(f_i) \leftarrow \text{Reasoning}(\text{Def}(f_i), \text{SHAP}(f_i), \text{Corr}(f_i, f_j), \text{Constraints})$$

②

顺序特征选择推理链

MoFA 不是一次性选择所有特征，而是模拟人类特征工程师的顺序决策过程。每轮选择一个特征，并更新上下文：
Step 1: 从最重要特征开始，LLM 选择 \(f_1\) 并解释理由
Step 2: 给定 \(f_1\)，LLM 考虑约束（如已选特征的延迟占用）并选择 \(f_2\)
...
Step N: 当新增特征的边际收益低于阈值或约束被触发时停止
停止条件：$$\text{Stop if } \Delta\text{SHAP}(f_{\text{new}} | f_{1:k}) < \epsilon \text{ or } \sum_{i=1}^{k+1} \text{Latency}(f_i) > L_\text{max}$$

③

高阶交互项发现

MoFA 的独特能力：LLM 可以基于语义推理发现统计方法难以捕捉的高阶交互。例如，LLM 推理出"user_purchase_frequency × item_price_range"的交互项可能对价值预测重要，因为"高频购买低价商品"和"低频购买高价商品"的用户价值模式截然不同。具体操作是让 LLM 在选择完基础特征后，提出候选交叉特征：$$f_\text{cross} = g(f_i, f_j), \quad g \in \{\times, +, \text{concat}, \text{bucket}\}$$ 然后用小样本验证增益。

Prompt Attack Detection with LLM-as-a-Judge and Mixture-of-Models
📋 arXiv:2603.25176 👤 Hieu Xuan Le, Benjamin Goh, Quy Anh Tang 📅 2026-03-26 🤖 Agent
背景与动机

Prompt 攻击（越狱 + 注入）是 LLM 系统的关键安全风险。生产环境中，防护措施必须在低延迟约束下运行，这造成了部署困境：轻量分类器泛化差，大模型判官太慢太贵。本文研究轻量级通用 LLM 能否在实时生产约束下可靠地充当安全判官，并实际部署在新加坡政府聊天机器人中。

核心方法详解
①

结构化推理 Prompt 设计

设计四阶段结构化推理流程，引导轻量 LLM 执行系统性安全分析：
(i) 意图分解（Intent Decomposition）：将用户输入分解为显式请求和隐式意图
(ii) 安全信号验证（Safety Signal Check）：检查 5 类安全信号（注入标记、角色扮演、编码混淆、上下文操纵、间接指令）
(iii) 伤害评估（Harm Assessment）：评估潜在伤害的严重程度和可能性
(iv) 自我反思（Self-Reflection）：模型重新审视自己的判断，检查是否过度敏感或遗漏
输出格式规范为 JSON：\(\{verdict, confidence, reasoning, signals\_found\}\)

②

轻量 LLM 判官选择与优化

评估了多个轻量模型（Gemini-2.0-Flash-Lite、GPT-4o-mini、Claude-3.5-Haiku 等）。关键发现：经过结构化 Prompt 引导后，Gemini-2.0-Flash-Lite 在延迟和准确率上达到最优权衡：$$\text{Score} = \alpha \cdot F1_\text{attack} + (1-\alpha) \cdot \frac{1}{\text{Latency}_\text{p95}}$$ 其中 \(\alpha = 0.7\) 。Flash-Lite 的 p95 延迟为 180ms，F1 为 0.91，满足生产要求（SLA <500ms）。

③

Mixture-of-Models 聚合实验

测试了多模型聚合策略以评估是否能进一步提升检测性能：
多数投票：3 个判官中 2 个以上判定为攻击则标记
加权投票：按各模型在验证集上的 F1 加权
级联策略：Fast-Judge 先筛，不确定的交给 Strong-Judge
实验发现聚合带来的收益适度（F1 从 0.91→0.93），但延迟和成本增加 2-3 倍。在生产成本约束下，单模型 Flash-Lite 仍是最优选择。
