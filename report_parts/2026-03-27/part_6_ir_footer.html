GraphER: An Efficient Graph-Based Enrichment and Reranking Method for Retrieval-Augmented Generation
📋 arXiv:2603.24925 👤 Ruizhong Miao, Yuying Wang, Rongguang Wang 等 📅 2026-03-26 🔍 IR
背景与动机

RAG 系统中的语义检索在处理分散证据时表现不佳——相关信息可能分布在多个文档中，单靠语义相似度难以捕捉结构关系。现有方案要么用 Agent 迭代检索（效率低），要么构建知识图谱（维护成本高、与向量存储不兼容）。GraphER 提出了一种零维护成本的图增强方案：离线索引时丰富数据对象，查询时执行图重排序。

核心方法详解
①

离线图结构索引增强

在索引构建阶段，对每个文档块 \(d_i\) 提取多种邻近关系，构建邻接图 \(G = (V, E)\)：
语义边：\(e_\text{sem}(d_i, d_j) = \cos(\mathbf{e}_i, \mathbf{e}_j) > \tau_\text{sem}\)
结构边：来自同一文档的不同块、相邻章节、引用关系等
实体边：共享命名实体（如同一公司名、人名）
每条边附带类型标签和权重。图与向量索引并行存储，不替换原有基础设施。图的构建是离线的、增量的。

②

查询时图传播重排序

给定 query \(q\)，先用标准检索器获得 Top-N 候选 \(\{d_1, ..., d_N\}\) 及其初始分数 \(s_i^{(0)}\)。然后在图 \(G\) 上执行 \(K\) 轮消息传播：$$s_i^{(k+1)} = (1-\alpha) \cdot s_i^{(0)} + \alpha \cdot \sum_{j \in \mathcal{N}(i)} w_{ij} \cdot s_j^{(k)}$$ 其中 \(\alpha\) 是图传播强度（类似 PersonalizedPageRank），\(w_{ij}\) 是边权重。关键：传播只在 Top-N 的子图上进行，计算开销为 \(O(NK |\overline{\mathcal{N}}|)\)，对于典型参数（N=100, K=3, \(|\overline{\mathcal{N}}|=5\)）仅需 1500 次乘法，延迟 <1ms。

③

边类型感知加权

不同类型的边对重排序的贡献不同。GraphER 学习边类型权重：$$w_{ij} = \sum_{t \in \text{types}} \beta_t \cdot \mathbb{1}[(i,j) \in E_t]$$ 其中 \(\beta_t\) 是可学习的类型权重。在验证集上通过 grid search 或轻量梯度优化确定。实验发现结构边（同一文档/章节）的贡献最大（\(\beta_\text{struct} \approx 0.45\)），因为它们直接捕捉了证据的局部聚集性。

AuthorityBench: Benchmarking LLM Authority Perception for Reliable Retrieval-Augmented Generation
📋 arXiv:2603.25092 👤 Zhihui Yao, Hengran Zhang, Keping Bi 📅 2026-03-26（投稿 ACL 2026） 🔍 IR
背景与动机

RAG 利用外部知识增强 LLM，但容易受到低权威来源的影响，传播错误信息。一个关键但未被充分研究的问题是：LLM 是否具备感知信息权威性的能力？权威性是超越语义理解的维度——同样内容的信息，来自 WHO 和来自匿名博客的可信度截然不同。AuthorityBench 是首个系统性评估此能力的基准。

核心方法详解
①

三层数据集构建

DomainAuth (10K 网页域)：收集 10,000 个网页域，以 PageRank 和 Majestic Trust Flow 作为权威性 ground truth。评估 LLM 对域名权威性的感知能力。
EntityAuth (22K 实体)：从 Wikidata 提取 22,000 个实体，以 Wikipedia 页面浏览量和引用数量作为流行度/权威性指标。
RAGAuth (120 查询)：构建 120 个 QA 查询，每个查询配有来自不同权威级别的文档（高权威如政府网站、中权威如新闻媒体、低权威如个人博客），评估权威性过滤对下游答案质量的影响。

②

三种评估范式

设计了三种提示范式评估 LLM 的权威性判断：
PointJudge：直接给单个域名/实体打分（1-10 分），\(\text{Score}_\text{point}(x) = f_\text{LLM}(x)\)
PairJudge：给定两个域名，判断哪个更权威，\(\text{Pref}(x_i, x_j) = f_\text{LLM}(x_i, x_j)\)
ListJudge：给定 5-10 个域名列表，排序它们的权威性
评估指标：与 ground truth 的 Kendall's τ 和 Spearman's ρ。

③

权威性过滤改善 RAG

在 RAGAuth 上验证权威性感知的实用价值。过滤流程：$$\text{Filtered\_docs} = \{d \in \text{Retrieved} : \text{AuthScore}(d) > \theta_\text{auth}\}$$ 然后仅将高权威文档输入 LLM 生成答案。关键发现：包含网页正文内容（full text）反而降低了权威性判断准确率——模型容易被文本风格"迷惑"。仅提供元数据（域名、URL 结构）时判断最准确，说明权威性与文本内容是正交特征。

ColBERT-Att: Late-Interaction Meets Attention for Enhanced Retrieval
📋 arXiv:2603.25248 👤 Raj Nath Patel, Sourav Dutta 📅 2026-03-26 🔍 IR
背景与动机

ColBERT 的"晚期交互"范式通过 MaxSim 操作实现高效检索：为每个 query token 找到最匹配的 document token，然后求和。然而，MaxSim 对所有 query token 等权重——"the"、"of" 与关键实体词获得相同贡献。ColBERT-Att 认为 query/document token 的注意力权重捕捉了它们的"重要性"，应该被显式利用。

核心方法详解
①

注意力权重提取

从 ColBERT 的 BERT 编码器中提取最后一层的自注意力权重。对 query 序列 \(q_1, ..., q_m\)，每个 token \(q_i\) 的注意力权重定义为 [CLS] token 对 \(q_i\) 的注意力值（跨所有注意力头平均）：$$\alpha_i^q = \frac{1}{H}\sum_{h=1}^H A_h^{\text{cls} \to i}$$ 对文档 token 类似定义 \(\alpha_j^d\)。高权重意味着该 token 对整体语义更重要。

②

注意力加权的 MaxSim

将标准 ColBERT 的 MaxSim 替换为注意力加权版本：
标准 ColBERT：$$S(q, d) = \sum_{i=1}^m \max_{j=1}^n \cos(q_i, d_j)$$ ColBERT-Att：$$S_\text{att}(q, d) = \sum_{i=1}^m \alpha_i^q \cdot \max_{j=1}^n \left[\alpha_j^d \cdot \cos(q_i, d_j)\right]$$ 这使得关键 query token（如实体名、动词）的匹配获得更大权重，停用词的贡献被自然压低。注意力权重在编码时已经计算好，推理时无额外成本。

③

训练与索引兼容性

ColBERT-Att 使用与 ColBERT 完全相同的训练流程（对比学习 + hard negatives），仅在 scoring 函数中加入注意力权重。这意味着：
(a) 现有 ColBERT 索引可直接复用（只需额外存储每个 token 的一个 float 注意力权重，存储开销 <1%）
(b) ANN 检索阶段仍使用标准 MaxSim（快速召回），注意力加权仅在重排序阶段应用
两阶段设计：$$\text{Top-1000} \xrightarrow{\text{MaxSim}} \text{Top-100} \xrightarrow{\text{Att-MaxSim}} \text{Top-10}$$

  • RLVR 效率优化成热点：ARROL 的在线剪枝思路（边生成边筛选低价值 rollout）和 On-Policy Distillation 的修复工作，都指向同一个方向——如何用更少的计算获得更强的学习信号。这对降低 RL 训练的 GPU 成本至关重要。

  • 形式化验证进入 Agent：SEVerA 将程序验证引入自进化 Agent，零违约率 + 性能提升的结果很有说服力。随着 Agent 自主性增加，形式化安全保证将越来越重要。
